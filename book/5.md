## Chapter 5: Security & Authentication

Every sysadmin eventually inherits the same short list of headaches: a password that needs to be typed without showing up on someone's shoulder-surfed screen, a credential that needs to be hashed instead of stored in the clear, a file transfer that needs a checksum to prove nothing was tampered with in transit, an API key that needs a home that isn't a sticky note. This chapter collects seventeen small, focused scripts that solve exactly those problems — nothing more, nothing decorative.

None of these scripts are meant to replace a real secrets manager (Vault, AWS Secrets Manager, 1Password CLI, etc.) in a production environment. They are meant to be the scripts you reach for on a single box, in a lab, in a CI runner, or while you're building the muscle memory for how these primitives actually work under the hood — salts, KDFs, symmetric ciphers, key fingerprints, certificate validity windows. Each script is intentionally short enough to read top to bottom in a couple of minutes, and each one leans on well-audited libraries (Python's standard library `hashlib`, `hmac`, `secrets`, `getpass`, plus the third-party `cryptography` package) rather than hand-rolled crypto.

The chapter is organized roughly in the order a credential moves through its life: you collect it from a human, you store it safely, you generate strong material to replace it, you encrypt whatever needs encrypting, you manage keys and certificates, and finally you verify that nothing was corrupted along the way.

### 5.1 Handle Password Input Securely

#### Use case
Any interactive script that needs a password, passphrase, or other secret typed in by a human — a login helper, a setup wizard, a CLI tool — needs to avoid echoing that secret to the terminal or leaving it visible in the scrollback.

#### Prerequisites
Uses only the Python standard library (`getpass`, `sys`). No external packages are required.

```bash
python3 --version   # 3.7+ recommended
# No pip installs needed — getpass ships with Python
```

Works on Linux, macOS, and Windows. On most IDE-integrated terminals (some notebook environments, certain CI log streams) `getpass` may fall back to a visible prompt because there is no real TTY — this is a limitation of the terminal, not the script.

#### Cautions
This script does not store, log, or transmit the password anywhere — that responsibility belongs to whatever calls `get_secure_password()`. Don't print the returned password to stdout in your own code, and don't pass it as a command-line argument elsewhere (arguments are visible in process lists like `ps aux`).

#### Script
```python
#!/usr/bin/env python3
"""
handle_password_input.py
------------------------------
Demonstrates and provides a reusable way to accept password input securely
(hidden from the terminal, not echoed to screen, not stored in shell history).

Usage:
    python handle_password_input.py

Example:
    python handle_password_input.py
    # Prompts: Enter password: (input is hidden as you type)
"""

import getpass
import sys


def get_secure_password(prompt: str = "Enter password: ") -> str:
    """Prompt the user for a password without echoing it to the terminal."""
    try:
        password = getpass.getpass(prompt)
    except (EOFError, KeyboardInterrupt):
        print("\nPassword input cancelled.")
        sys.exit(1)

    if not password:
        print("Error: password cannot be empty.")
        sys.exit(1)

    return password


def main():
    print("Secure password input demo.")
    print("Your input will NOT be shown on screen and is NOT saved anywhere.\n")

    password = get_secure_password("Enter your password: ")

    print(f"\nPassword received successfully ({len(password)} characters).")
    print("Note: this script does not store, log, or transmit the password anywhere.")


if __name__ == "__main__":
    main()
```

#### How it works and how to run
`getpass.getpass()` disables terminal echo before reading input and restores it afterward, so nothing appears on screen as you type — not even asterisks. `get_secure_password()` wraps this with two guardrails: it catches `Ctrl+C`/`EOF` so the script exits cleanly instead of crashing with a traceback, and it rejects an empty password.

```bash
python3 handle_password_input.py
```

You can also import the helper into another script instead of running it standalone:

```python
from handle_password_input import get_secure_password

db_password = get_secure_password("Database password: ")
```

#### Sample output
```
$ python3 handle_password_input.py
Secure password input demo.
Your input will NOT be shown on screen and is NOT saved anywhere.

Enter your password: 
Password received successfully (14 characters).
Note: this script does not store, log, or transmit the password anywhere.
```

#### Variations / flags
- Add an `argparse` `--prompt "Custom label: "` flag to change the prompt text.
- Add a `--confirm` flag that calls `get_secure_password` twice and checks the two entries match (see 5.3 for a full implementation of this pattern).
- Wrap the function in a `try/finally` that zeroes out the password variable in memory when done, for extra defense-in-depth in long-running processes.

#### Common pitfalls
- **Running inside an IDE's built-in terminal or a Jupyter cell**: some of these don't provide a real TTY, so `getpass` silently echoes input instead of hiding it. Test in an actual terminal before relying on this behavior.
- **Piping input**: `echo "mypassword" | python3 handle_password_input.py` will make `getpass` fall back to reading from stdin visibly (with a warning on some platforms) — this defeats the purpose and should be avoided outside of automated testing.
- **Logging frameworks**: if the calling script logs all local variables on exception (some debug/logging setups do this), the password can leak into log files even though this script itself never prints it. Be careful with broad exception logging around secret-handling code.

### 5.2 Use getpass

#### Use case
You're writing any script that asks a human for a username and password pair — a quick login helper, a database connection wizard, a one-off automation script — and you want the canonical, minimal example of `getpass` in action before building something more elaborate.

#### Prerequisites
Standard library only (`getpass`, `sys`).

```bash
python3 --version   # 3.7+
# No installation required
```

Cross-platform: Linux, macOS, Windows (Windows uses `msvcrt` under the hood; Unix systems use `termios`).

#### Cautions
This is a demo/teaching script — it prints the password *length*, never the password itself, so it's safe to run, but don't copy the `print(f"\nUsername entered: {username}")` pattern for anything containing a secret.

#### Script
```python
#!/usr/bin/env python3
"""
use_getpass.py
--------------------
Focused example of Python's built-in 'getpass' module for reading
sensitive input (like passwords) without echoing it to the terminal.

Usage:
    python use_getpass.py

Example:
    python use_getpass.py
    # Password: (typing is hidden)
"""

import getpass
import sys


def main():
    print("Demo: reading a password using the getpass module.\n")

    if not sys.stdin.isatty():
        print("Warning: input is not coming from an interactive terminal.")
        print("getpass will fall back to a visible prompt in some environments (e.g. some IDEs).\n")

    try:
        username = input("Username: ")
        password = getpass.getpass("Password: ")
    except (EOFError, KeyboardInterrupt):
        print("\nCancelled.")
        sys.exit(1)

    print(f"\nUsername entered: {username}")
    print(f"Password length: {len(password)} characters")
    print("(The password itself is never printed or stored.)")


if __name__ == "__main__":
    main()
```

#### How it works and how to run
`input()` is used for the (non-sensitive) username, and `getpass.getpass()` is used for the password. The script proactively checks `sys.stdin.isatty()` and warns you up front if it detects it isn't running in a real terminal, rather than letting you discover the problem after typing a password in plaintext.

```bash
python3 use_getpass.py
```

#### Sample output
```
$ python3 use_getpass.py
Demo: reading a password using the getpass module.

Username: admin
Password: 

Username entered: admin
Password length: 8 characters
(The password itself is never printed or stored.)
```

#### Variations / flags
- Swap `getpass.getpass()`'s default prompt or stream target using `getpass.getpass(prompt="PIN: ", stream=sys.stderr)` if you want the prompt to go to stderr instead of stdout (useful when stdout is being redirected to a file).
- Combine with `argparse` to accept `--username` as a flag while still prompting only for the password interactively.

#### Common pitfalls
- **Non-TTY environments**: as the script itself warns, some IDE consoles and CI systems don't provide a proper terminal, so `getpass` degrades to visible input — always test the real invocation path before trusting it in production.
- **Confusing `input()` and `getpass.getpass()`**: it's an easy copy-paste mistake to use `input()` for the password field by accident, which would echo it in plaintext. Double-check which function is on which line.
- **SSH sessions without a pseudo-terminal**: running via `ssh host 'python3 use_getpass.py'` without `-t` can break interactive prompting entirely; add `-t` to allocate a TTY.

### 5.3 Reprompt for Password Input

#### Use case
Building a signup flow, a "set your new password" utility, or a bootstrap script that provisions a new account, where you need to enforce minimum complexity rules and make sure the user didn't fat-finger a typo by having them type the password twice.

#### Prerequisites
Standard library only (`getpass`, `sys`, `argparse`).

```bash
python3 --version   # 3.7+ (uses PEP 604-style tuple return type hints; works fine on 3.7+ since it's just a plain tuple annotation)
# No installation required
```

Cross-platform: Linux, macOS, Windows.

#### Cautions
The built-in complexity rule (`is_valid`) only checks length, one uppercase letter, and one digit — it does not check against breached-password lists (e.g. Have I Been Pwned's range API) or block common passwords like `Password1`. Treat this as a minimum floor, not a complete policy, for anything user-facing in production.

#### Script
```python
#!/usr/bin/env python3
"""
reprompt_password.py
--------------------------
Prompts for a password and reprompts if it doesn't meet requirements,
or asks the user to confirm it by typing it twice (like a signup form).

Usage:
    python reprompt_password.py [--confirm] [--min-length 8] [--max-attempts 3]

Example:
    python reprompt_password.py
    python reprompt_password.py --confirm --min-length 10
"""

import argparse
import getpass
import sys


def is_valid(password: str, min_length: int) -> tuple[bool, str]:
    if len(password) < min_length:
        return False, f"Password must be at least {min_length} characters long."
    if not any(c.isupper() for c in password):
        return False, "Password must contain at least one uppercase letter."
    if not any(c.isdigit() for c in password):
        return False, "Password must contain at least one digit."
    return True, ""


def prompt_password(min_length: int, max_attempts: int, confirm: bool) -> str:
    for attempt in range(1, max_attempts + 1):
        password = getpass.getpass("Enter password: ")
        valid, reason = is_valid(password, min_length)

        if not valid:
            print(f"Invalid password: {reason}")
            print(f"Attempt {attempt}/{max_attempts}\n")
            continue

        if confirm:
            confirm_password = getpass.getpass("Confirm password: ")
            if password != confirm_password:
                print("Passwords do not match. Please try again.\n")
                continue

        return password

    print("\nMaximum attempts reached. Exiting.")
    sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Prompt for a password with validation and reprompting.")
    parser.add_argument("--confirm", action="store_true", help="Ask the user to re-type the password to confirm")
    parser.add_argument("--min-length", type=int, default=8, help="Minimum required password length (default: 8)")
    parser.add_argument("--max-attempts", type=int, default=3, help="Maximum number of attempts (default: 3)")
    args = parser.parse_args()

    password = prompt_password(args.min_length, args.max_attempts, args.confirm)
    print(f"\nPassword accepted ({len(password)} characters). Not stored or displayed.")


if __name__ == "__main__":
    main()
```

#### How it works and how to run
`is_valid()` checks three rules in sequence and returns the first failure reason it finds. `prompt_password()` loops up to `--max-attempts` times, re-prompting on any validation failure or (when `--confirm` is set) on a mismatch between the two entries, then exits with a non-zero status if the attempt budget runs out — which makes it safe to use as a gate in a larger provisioning script.

```bash
# Basic: min 8 chars, 1 uppercase, 1 digit, 3 attempts
python3 reprompt_password.py

# Signup-style: ask twice and require it to match
python3 reprompt_password.py --confirm

# Stricter policy, more attempts
python3 reprompt_password.py --confirm --min-length 12 --max-attempts 5
```

#### Sample output
```
$ python3 reprompt_password.py --confirm --min-length 10
Enter password: 
Invalid password: Password must be at least 10 characters long.
Attempt 1/3

Enter password: 
Confirm password: 
Passwords do not match. Please try again.

Enter password: 
Confirm password: 

Password accepted (12 characters). Not stored or displayed.
```

#### Variations / flags
- Add a `--forbid-common` flag that checks the candidate password against a small local wordlist of the top 10,000 breached passwords.
- Add a `--require-symbol` rule alongside the existing uppercase/digit checks.
- Return the validation reasons as a list instead of stopping at the first failure, so the user sees every unmet requirement at once instead of one per attempt.

#### Common pitfalls
- **Off-by-one on attempts**: `--max-attempts 1` means the user gets exactly one try — useful for scripted/non-interactive contexts, easy to misconfigure for a human-facing flow.
- **Confirm mismatch doesn't count against `is_valid` re-checks**: a mismatched confirmation consumes an attempt but doesn't re-validate the first password's complexity — both entries must independently satisfy `is_valid` in your own extensions if that matters to you.
- **Type hint syntax**: `tuple[bool, str]` as a bare annotation works on Python 3.9+ without `from __future__ import annotations`; on 3.7–3.8 it will raise a `TypeError` at import time unless that future import is added. If you're targeting older interpreters, add `from __future__ import annotations` at the top of the file.

### 5.4 Hash Passwords

#### Use case
You're building any system that stores user credentials — a login table in a database, a local config file for a small internal tool — and need to store a password so it can later be *verified* without ever keeping the plaintext around.

#### Prerequisites
Standard library only (`hashlib`, `getpass`, `base64`, `os`).

```bash
python3 --version   # 3.6+ (hashlib.pbkdf2_hmac has been available since 2.7.8 / 3.4)
# No installation required
```

Cross-platform: Linux, macOS, Windows.

#### Cautions
PBKDF2-HMAC-SHA256 at 390,000 iterations is a reasonable general-purpose choice, but for new systems consider `argon2-cf` or `bcrypt` (via the `argon2-cffi` or `bcrypt` packages), which are specifically designed to resist GPU/ASIC cracking better than PBKDF2. Never reduce the iteration count for "performance" without understanding the security trade-off, and never reuse a salt across users.

#### Script
```python
#!/usr/bin/env python3
"""
hash_passwords.py
------------------------
Securely hashes a password using PBKDF2-HMAC-SHA256 with a random salt.
Outputs a self-contained string (salt + hash) that can be stored safely
and later checked with verify_password_hash.py.

NEVER store plain-text passwords. Always store only the hash.

Usage:
    python hash_passwords.py

Example:
    python hash_passwords.py
    # Enter password: (hidden input)
    # Output: pbkdf2_sha256$390000$<salt_hex>$<hash_hex>
"""

import base64
import getpass
import hashlib
import os
import sys

ITERATIONS = 390000


def hash_password(password: str) -> str:
    salt = os.urandom(16)
    derived = hashlib.pbkdf2_hmac("sha256", password.encode("utf-8"), salt, ITERATIONS)

    salt_b64 = base64.b64encode(salt).decode("ascii")
    hash_b64 = base64.b64encode(derived).decode("ascii")

    return f"pbkdf2_sha256${ITERATIONS}${salt_b64}${hash_b64}"


def main():
    print("Secure password hashing (PBKDF2-HMAC-SHA256).\n")

    try:
        password = getpass.getpass("Enter password to hash: ")
    except (EOFError, KeyboardInterrupt):
        print("\nCancelled.")
        sys.exit(1)

    if not password:
        print("Error: password cannot be empty.")
        sys.exit(1)

    hashed = hash_password(password)

    print("\nPassword hashed successfully.")
    print(f"Stored hash string:\n{hashed}")
    print("\nSave this entire string (e.g. in a database). It contains the algorithm,")
    print("iteration count, salt, and hash — everything needed to verify later,")
    print("but NOT the original password.")
    print("\nUse verify_password_hash.py with this string to check a password later.")


if __name__ == "__main__":
    main()
```

#### How it works and how to run
A fresh 16-byte random salt is generated per password with `os.urandom()`, then `hashlib.pbkdf2_hmac()` stretches the password + salt through 390,000 rounds of HMAC-SHA256. The algorithm name, iteration count, salt, and derived hash are packed into one `$`-delimited string — a self-describing format (similar in spirit to Django's password hash format) so future code can re-derive the exact same computation for verification, and so the iteration count can be bumped later without breaking old hashes.

```bash
python3 hash_passwords.py
```

#### Sample output
```
$ python3 hash_passwords.py
Secure password hashing (PBKDF2-HMAC-SHA256).

Enter password to hash: 

Password hashed successfully.
Stored hash string:
pbkdf2_sha256$390000$V3JvbmdTYWx0RXhhbXBsZQ==$K3n2H8pQzR1vY6mLxT9sD4gW7cJ0eA5b

Save this entire string (e.g. in a database). It contains the algorithm,
iteration count, salt, and hash — everything needed to verify later,
but NOT the original password.

Use verify_password_hash.py with this string to check a password later.
```

#### Variations / flags
- Add a `--iterations` flag to override `ITERATIONS`, useful for benchmarking hash time on your target hardware (OWASP recommends tuning so hashing takes roughly 250ms–1s on your server).
- Add `--algo sha512` support by parameterizing the `hashlib.pbkdf2_hmac()` call.
- Pipe the output straight into a database insert: `python3 hash_passwords.py | tail -1 | psql ...` (be mindful of shell history / process visibility when scripting this).

#### Common pitfalls
- **Forgetting to store the whole string**: the salt and iteration count are *part of* the stored value — truncating it or storing only the hash portion makes verification impossible.
- **Comparing hashes with `==`**: this script only produces the hash; 5.5 (verification) correctly uses `hmac.compare_digest()` for a timing-safe comparison — don't swap that out for a plain `==` when you write your own verification logic.
- **Iteration count drift**: if you increase `ITERATIONS` later, old stored hashes still carry their original count in the string, so verification still works — just don't hardcode the iteration count anywhere else in your codebase.

### 5.5 Verify Password Hashes

#### Use case
A login flow needs to check a freshly typed password against the hash you stored with `hash_passwords.py` (5.4) — without ever decrypting or reversing the hash, and without leaking timing information that could help an attacker guess the password character-by-character.

#### Prerequisites
Standard library only (`base64`, `getpass`, `hashlib`, `hmac`, `argparse`).

```bash
python3 --version   # 3.6+
# No installation required
```

Cross-platform: Linux, macOS, Windows.

#### Cautions
This script exits with status code `1` on an invalid password — that's intentional so it can be used as a gate in shell scripts (`if python3 verify_password_hash.py "$hash"; then ...`), but don't rely on stdout text-matching in security-sensitive automation; check the exit code instead.

#### Script
```python
#!/usr/bin/env python3
"""
verify_password_hash.py
-----------------------------
Verifies a password against a hash string produced by hash_passwords.py
(format: pbkdf2_sha256$iterations$salt_b64$hash_b64).

Usage:
    python verify_password_hash.py "<stored_hash_string>"

Example:
    python verify_password_hash.py "pbkdf2_sha256$390000$abc123...$def456..."
    # Enter password: (hidden input)
    # Output: Password is VALID / Password is INVALID
"""

import argparse
import base64
import getpass
import hashlib
import hmac
import sys


def verify_password(password: str, stored_hash: str) -> bool:
    try:
        algorithm, iterations_str, salt_b64, hash_b64 = stored_hash.split("$")
    except ValueError:
        print("Error: stored hash string is not in the expected format.")
        print("Expected format: pbkdf2_sha256$iterations$salt_b64$hash_b64")
        sys.exit(1)

    if algorithm != "pbkdf2_sha256":
        print(f"Error: unsupported algorithm '{algorithm}'.")
        sys.exit(1)

    iterations = int(iterations_str)
    salt = base64.b64decode(salt_b64)
    expected_hash = base64.b64decode(hash_b64)

    derived = hashlib.pbkdf2_hmac("sha256", password.encode("utf-8"), salt, iterations)

    return hmac.compare_digest(derived, expected_hash)


def main():
    parser = argparse.ArgumentParser(description="Verify a password against a stored PBKDF2 hash.")
    parser.add_argument("stored_hash", help="The stored hash string (from hash_passwords.py)")
    args = parser.parse_args()

    try:
        password = getpass.getpass("Enter password to verify: ")
    except (EOFError, KeyboardInterrupt):
        print("\nCancelled.")
        sys.exit(1)

    is_valid = verify_password(password, args.stored_hash)

    if is_valid:
        print("\nResult: Password is VALID.")
    else:
        print("\nResult: Password is INVALID.")
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run
The stored hash string is split back into its four components (algorithm, iteration count, salt, hash), the salt and iteration count are used to re-run PBKDF2 on the freshly typed password, and the result is compared to the stored hash with `hmac.compare_digest()` — a constant-time comparison that avoids leaking how many leading bytes matched via response-time differences.

```bash
python3 verify_password_hash.py "pbkdf2_sha256$390000$V3JvbmdTYWx0RXhhbXBsZQ==$K3n2H8pQzR1vY6mLxT9sD4gW7cJ0eA5b"
```

#### Sample output
```
$ python3 verify_password_hash.py "pbkdf2_sha256$390000$V3JvbmdTYWx0RXhhbXBsZQ==$K3n2H8pQzR1vY6mLxT9sD4gW7cJ0eA5b"
Enter password to verify: 

Result: Password is VALID.
```

```
$ python3 verify_password_hash.py "pbkdf2_sha256$390000$V3JvbmdTYWx0RXhhbXBsZQ==$K3n2H8pQzR1vY6mLxT9sD4gW7cJ0eA5b"
Enter password to verify: 

Result: Password is INVALID.
```

#### Variations / flags
- Add a `--stdin` flag to read the stored hash from a file instead of a shell argument, avoiding the hash appearing in `ps` output or shell history.
- Wrap `verify_password` in a small Flask/FastAPI login endpoint, reading the stored hash from your user table instead of a CLI argument.
- Add rate-limiting/lockout logic around repeated failed calls to defend against online brute-force attempts.

#### Common pitfalls
- **Passing the hash as a bare shell argument**: on shared/multi-user systems, command-line arguments are visible to other users via `ps aux` while the process runs — prefer reading it from a file or environment variable in production use.
- **Algorithm mismatch**: if you hash with a different algorithm string later (e.g. switch to `pbkdf2_sha512`), this script's `if algorithm != "pbkdf2_sha256"` guard will correctly reject it — make sure any upgraded verifier supports all algorithm versions you've ever issued.
- **Using `==` instead of `hmac.compare_digest()`**: if you adapt this script, keep the constant-time comparison; a naive `==` on bytes short-circuits early and can theoretically be timed.

### 5.6 Generate Random Passwords

#### Use case
You need to provision a strong, random password for a new service account, a temporary admin login, or a batch of test accounts — and you want it generated by a cryptographically secure random source rather than a weak method like `random.choice()`.

#### Prerequisites
Standard library only (`secrets`, `string`, `argparse`).

```bash
python3 --version   # 3.6+ (secrets module was added in 3.6)
# No installation required
```

Cross-platform: Linux, macOS, Windows.

#### Cautions
This script prints the generated password(s) directly to the terminal — anyone who can see your screen or scrollback can read them. It deliberately does not save them anywhere; copy them into a password manager immediately and clear your terminal scrollback afterward if this matters in your environment.

#### Script
```python
#!/usr/bin/env python3
"""
generate_random_password.py
---------------------------------
Generates one or more cryptographically secure random passwords using
Python's 'secrets' module.

Usage:
    python generate_random_password.py [--length 16] [--count 1] [--no-symbols]

Example:
    python generate_random_password.py
    python generate_random_password.py --length 20 --count 5
    python generate_random_password.py --length 12 --no-symbols
"""

import argparse
import secrets
import string
import sys


def generate_password(length: int, use_symbols: bool) -> str:
    lowercase = string.ascii_lowercase
    uppercase = string.ascii_uppercase
    digits = string.digits
    symbols = "!@#$%^&*()-_=+[]{}?"

    alphabet = lowercase + uppercase + digits
    if use_symbols:
        alphabet += symbols

    if length < 4:
        print("Error: length must be at least 4 to guarantee character variety.")
        sys.exit(1)

    while True:
        password = "".join(secrets.choice(alphabet) for _ in range(length))

        has_lower = any(c in lowercase for c in password)
        has_upper = any(c in uppercase for c in password)
        has_digit = any(c in digits for c in password)
        has_symbol = any(c in symbols for c in password) if use_symbols else True

        if has_lower and has_upper and has_digit and has_symbol:
            return password


def main():
    parser = argparse.ArgumentParser(description="Generate cryptographically secure random passwords.")
    parser.add_argument("--length", type=int, default=16, help="Password length (default: 16)")
    parser.add_argument("--count", type=int, default=1, help="Number of passwords to generate (default: 1)")
    parser.add_argument("--no-symbols", action="store_true", help="Exclude special symbols from the password")
    args = parser.parse_args()

    print(f"Generating {args.count} password(s) of length {args.length}:\n")

    for i in range(1, args.count + 1):
        password = generate_password(args.length, use_symbols=not args.no_symbols)
        print(f"{i}. {password}")

    print("\nStore these somewhere safe (e.g. a password manager). They are not saved by this script.")


if __name__ == "__main__":
    main()
```

#### How it works and how to run
`secrets.choice()` (unlike `random.choice()`) draws from the OS's cryptographically secure random source, making the output suitable for real credentials. The function loops, regenerating the whole password, until it happens to contain at least one lowercase, one uppercase, one digit, and (if symbols are enabled) one symbol — a simple rejection-sampling approach that keeps the code easy to read at the cost of a few extra iterations on average.

```bash
# Default: one 16-character password with symbols
python3 generate_random_password.py

# Five 20-character passwords
python3 generate_random_password.py --length 20 --count 5

# Alphanumeric only, no symbols (e.g. for systems that reject special characters)
python3 generate_random_password.py --length 12 --no-symbols
```

#### Sample output
```
$ python3 generate_random_password.py --length 20 --count 3
Generating 3 password(s) of length 20:

1. qT8#mK2$pL9@vN4xR6wZ
2. Bf3!hJ7&dS1%gY5*cA0q
3. Wn6^tU4?rE8=oI2-yP0m

Store these somewhere safe (e.g. a password manager). They are not saved by this script.
```

#### Variations / flags
- Add a `--exclude-ambiguous` flag that strips visually confusable characters (`0`, `O`, `1`, `l`, `I`) for passwords that will be manually typed or read aloud.
- Add a `--pronounceable` mode that builds syllable-like patterns instead of pure randomness, trading a small amount of entropy for memorability.
- Pipe straight into `manage_api_keys.py` (5.12) or `secure_credential_storage.py` (5.16) to generate-and-store in one step.

#### Common pitfalls
- **`--length` below 4**: the script exits with an error rather than looping forever, since it's mathematically impossible to guarantee all required character classes in fewer than 4 characters.
- **Confusing `random` and `secrets`**: if you extend this script, never swap in `random.choice()` — it's not cryptographically secure and its output can be predicted by an attacker who observes enough samples.
- **Terminal scrollback / screen-recording tools**: since passwords print in plaintext, be mindful of screen-sharing sessions or terminal loggers that might capture the output.

### 5.7 Generate Secure Tokens

#### Use case
You need a random, unguessable string for something that isn't a human-typed password — an API key, a session identifier, a CSRF token, or a one-time password-reset link.

#### Prerequisites
Standard library only (`secrets`, `base64`, `argparse`).

```bash
python3 --version   # 3.6+
# No installation required
```

Cross-platform: Linux, macOS, Windows.

#### Cautions
The token is printed once and not stored anywhere by the script — if it's meant to authenticate something (like a password-reset link), you are responsible for storing a hash of it server-side and setting an expiry, rather than storing the raw token.

#### Script
```python
#!/usr/bin/env python3
"""
generate_secure_token.py
------------------------------
Generates cryptographically secure random tokens, useful for API keys,
session tokens, password-reset links, CSRF tokens, etc.

Usage:
    python generate_secure_token.py [--bytes 32] [--format hex|urlsafe|base64]

Example:
    python generate_secure_token.py
    python generate_secure_token.py --bytes 48 --format urlsafe
"""

import argparse
import secrets


def generate_token(num_bytes: int, fmt: str) -> str:
    if fmt == "hex":
        return secrets.token_hex(num_bytes)
    elif fmt == "urlsafe":
        return secrets.token_urlsafe(num_bytes)
    elif fmt == "base64":
        import base64
        return base64.b64encode(secrets.token_bytes(num_bytes)).decode("ascii")
    else:
        raise ValueError(f"Unknown format: {fmt}")


def main():
    parser = argparse.ArgumentParser(description="Generate a cryptographically secure random token.")
    parser.add_argument("--bytes", type=int, default=32, dest="num_bytes",
                         help="Number of random bytes to generate (default: 32)")
    parser.add_argument("--format", default="urlsafe", choices=["hex", "urlsafe", "base64"],
                         help="Output format (default: urlsafe)")
    args = parser.parse_args()

    token = generate_token(args.num_bytes, args.format)

    print(f"Generated a {args.num_bytes}-byte secure token ({args.format} format):\n")
    print(token)
    print("\nUse this for API keys, session IDs, or password-reset tokens.")
    print("This token is not stored anywhere — copy it now if you need it.")


if __name__ == "__main__":
    main()
```

#### How it works and how to run
`secrets.token_hex()`, `secrets.token_urlsafe()`, and `secrets.token_bytes()` all draw from the same OS-level CSPRNG as `secrets.choice()`. The `--bytes` value controls actual entropy (32 bytes = 256 bits by default); the `--format` flag only changes how those bytes are encoded for display/transport — hex is longest and most universally safe to embed anywhere, `urlsafe` (base64 with `-`/`_` instead of `+`/`/`, no padding) is ideal for URLs, and plain `base64` is compact but may need URL-encoding if placed in a query string.

```bash
# Default: 32 random bytes, URL-safe base64 encoding
python3 generate_secure_token.py

# Longer token, hex encoded (good for API keys stored as text)
python3 generate_secure_token.py --bytes 48 --format hex

# Standard base64
python3 generate_secure_token.py --bytes 32 --format base64
```

#### Sample output
```
$ python3 generate_secure_token.py --bytes 32 --format hex
Generated a 32-byte secure token (hex format):

f3a1c9e02b6d48a7159cfe3d0a812b6c4e9f7a2d1c0b3e8f5a6d9c2b1e4f7a8

Use this for API keys, session IDs, or password-reset tokens.
This token is not stored anywhere — copy it now if you need it.
```

#### Variations / flags
- Add a `--prefix` flag (e.g. `sk_live_`) to mimic common API key conventions like Stripe's, making tokens self-identifying in logs.
- Add a `--count` flag to batch-generate multiple tokens at once (mirroring 5.6's `--count`).
- Feed the output directly into `manage_api_keys.py` (5.12) with `python3 generate_secure_token.py --format hex | ...` for a one-line "generate and store" workflow.

#### Common pitfalls
- **Too few bytes**: dropping `--bytes` below ~16 significantly weakens the token's resistance to brute-force guessing — 32 bytes (256 bits) is a safe default for anything security-sensitive.
- **Base64 vs. URL-safe base64 confusion**: standard base64 output can contain `+`, `/`, and `=` characters that need escaping in URLs; use `--format urlsafe` for anything going into a link or query parameter.
- **Storing the raw token instead of a hash**: for anything like a password-reset token, store a hash of the token server-side (similar to 5.4) so a database leak doesn't hand out valid tokens directly.

### 5.8 Encrypt Sensitive Data

#### Use case
You have a piece of text (a database connection string, an internal note) or a small file (a config with embedded secrets, a backup of credentials) that needs to be encrypted at rest with a password, so it's safe to store on disk, email to yourself, or check into a private location.

#### Prerequisites
Requires the third-party `cryptography` package.

```bash
python3 -m pip install cryptography
# or, if you keep a requirements file:
echo "cryptography" >> requirements.txt
pip install -r requirements.txt
```

Cross-platform: Linux, macOS, Windows. The `cryptography` package ships prebuilt wheels for all three, so no compiler toolchain is normally needed.

#### Cautions
This is password-based encryption — the strength of the protection is only as good as the password. A short or guessable password can be brute-forced even though the underlying AES/Fernet cipher is strong. This script also has no built-in way to recover data if the password is lost: **there is no backdoor**, so losing the password means losing the data permanently.

#### Script
```python
#!/usr/bin/env python3
"""
encrypt_sensitive_data.py
-------------------------------
Encrypts a piece of text (or a file) using password-based Fernet
(AES) symmetric encryption, provided by the 'cryptography' library.

Requires: pip install cryptography

Usage:
    python encrypt_sensitive_data.py --text "my secret data" --password MyPass123
    python encrypt_sensitive_data.py --file secrets.txt --password MyPass123

Example:
    python encrypt_sensitive_data.py --text "database password: hunter2" --password Correct-Horse
"""

import argparse
import base64
import os
import sys

try:
    from cryptography.fernet import Fernet
    from cryptography.hazmat.primitives import hashes
    from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
except ImportError:
    print("Error: the 'cryptography' package is required.")
    print("Install it with: pip install -r requirements.txt")
    sys.exit(1)


def derive_key(password: str, salt: bytes) -> bytes:
    kdf = PBKDF2HMAC(algorithm=hashes.SHA256(), length=32, salt=salt, iterations=390000)
    return base64.urlsafe_b64encode(kdf.derive(password.encode()))


def encrypt_data(data: bytes, password: str) -> bytes:
    salt = os.urandom(16)
    key = derive_key(password, salt)
    fernet = Fernet(key)
    encrypted = fernet.encrypt(data)
    # Prepend the salt so decryption can regenerate the same key later.
    return base64.urlsafe_b64encode(salt) + b"." + encrypted


def main():
    parser = argparse.ArgumentParser(description="Encrypt text or a file using a password.")
    group = parser.add_mutually_exclusive_group(required=True)
    group.add_argument("--text", help="Text string to encrypt")
    group.add_argument("--file", help="Path to a file to encrypt")
    parser.add_argument("--password", required=True, help="Password used to derive the encryption key")
    parser.add_argument("--output", default=None, help="Output file path (default: <file>.enc, or prints for --text)")
    args = parser.parse_args()

    if args.text:
        encrypted = encrypt_data(args.text.encode("utf-8"), args.password)
        if args.output:
            with open(args.output, "wb") as f:
                f.write(encrypted)
            print(f"Encrypted text written to: {args.output}")
        else:
            print("Encrypted text (base64):\n")
            print(encrypted.decode("ascii"))
    else:
        if not os.path.isfile(args.file):
            print(f"Error: '{args.file}' does not exist.")
            sys.exit(1)

        with open(args.file, "rb") as f:
            data = f.read()

        encrypted = encrypt_data(data, args.password)
        output_path = args.output or (args.file + ".enc")

        with open(output_path, "wb") as f:
            f.write(encrypted)

        print(f"Encrypted: '{args.file}' -> '{output_path}'")

    print("\nKeep your password safe. It is required to decrypt this data and cannot be recovered.")


if __name__ == "__main__":
    main()
```

#### How it works and how to run
A fresh random 16-byte salt is generated for every encryption, PBKDF2-HMAC-SHA256 (390,000 iterations) stretches your password + salt into a 32-byte key, and that key is used with Fernet — a `cryptography`-library primitive that combines AES-128 in CBC mode with an HMAC for authenticity, so tampered ciphertext is rejected on decryption rather than silently producing garbage. The salt is base64-encoded and prepended to the ciphertext (separated by a `.`) so `decrypt_sensitive_data.py` (5.9) can regenerate the exact same key later using only the password.

```bash
# Encrypt a short string, print the result
python3 encrypt_sensitive_data.py --text "database password: hunter2" --password "Correct-Horse-Battery"

# Encrypt a file (writes secrets.txt.enc by default)
python3 encrypt_sensitive_data.py --file secrets.txt --password "Correct-Horse-Battery"

# Encrypt a file to a custom output path
python3 encrypt_sensitive_data.py --file secrets.txt --password "Correct-Horse-Battery" --output vault/secrets.enc
```

#### Sample output
```
$ python3 encrypt_sensitive_data.py --text "database password: hunter2" --password Correct-Horse
Encrypted text (base64):

VjNKdmRHZ3RTV0YwYQ==.gAAAAABmO7x9qK3nJp2rX8wY4vL6tM1sD0hR7fN9cB5aE2gH8jK...

Keep your password safe. It is required to decrypt this data and cannot be recovered.
```

```
$ python3 encrypt_sensitive_data.py --file secrets.txt --password Correct-Horse
Encrypted: 'secrets.txt' -> 'secrets.txt.enc'

Keep your password safe. It is required to decrypt this data and cannot be recovered.
```

#### Variations / flags
- Add a `--iterations` flag matched to `decrypt_sensitive_data.py` if you want to tune KDF cost for your hardware.
- Add support for encrypting entire directories by tarring them first (`tar czf - mydir | python3 encrypt_sensitive_data.py --file /dev/stdin ...` with minor script changes to support stdin).
- Chain with 5.6 (`generate_random_password.py`) to auto-generate a strong one-time encryption password instead of typing one manually.

#### Common pitfalls
- **Passing `--password` on the command line**: like other scripts here, this exposes the password in shell history and `ps` output. For anything sensitive, prefer prompting via `getpass` (see 5.1) if you adapt this script for interactive use, or read the password from an environment variable / secrets manager in automation.
- **Losing the password**: there is intentionally no recovery mechanism — treat the password itself as sensitive data that needs its own backup plan (e.g. a password manager).
- **Re-encrypting already-encrypted output accidentally**: running the script twice on the same `--output` path silently overwrites the previous ciphertext; double-check output paths in scripted/batch use.

### 5.9 Decrypt Sensitive Data

#### Use case
You previously encrypted a string or file with `encrypt_sensitive_data.py` (5.8) and now need to get the original data back using the same password — for example, restoring a backed-up secret or reading an encrypted config at deploy time.

#### Prerequisites
Requires the third-party `cryptography` package (same dependency as 5.8).

```bash
python3 -m pip install cryptography
```

Cross-platform: Linux, macOS, Windows.

#### Cautions
If the password is wrong or the ciphertext has been altered/corrupted, Fernet's built-in authentication will correctly refuse to decrypt rather than returning garbage data — treat any "decryption failed" error as a hard stop, not something to retry blindly with slight password variations in an automated loop (that would effectively be an offline brute-force attack against your own data).

#### Script
```python
#!/usr/bin/env python3
"""
decrypt_sensitive_data.py
-------------------------------
Decrypts data that was encrypted with encrypt_sensitive_data.py,
using the same password.

Requires: pip install cryptography

Usage:
    python decrypt_sensitive_data.py --text "<encrypted_base64_string>" --password MyPass123
    python decrypt_sensitive_data.py --file secrets.txt.enc --password MyPass123

Example:
    python decrypt_sensitive_data.py --file secrets.txt.enc --password Correct-Horse
"""

import argparse
import base64
import os
import sys

try:
    from cryptography.fernet import Fernet, InvalidToken
    from cryptography.hazmat.primitives import hashes
    from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
except ImportError:
    print("Error: the 'cryptography' package is required.")
    print("Install it with: pip install -r requirements.txt")
    sys.exit(1)


def derive_key(password: str, salt: bytes) -> bytes:
    kdf = PBKDF2HMAC(algorithm=hashes.SHA256(), length=32, salt=salt, iterations=390000)
    return base64.urlsafe_b64encode(kdf.derive(password.encode()))


def decrypt_data(payload: bytes, password: str) -> bytes:
    try:
        salt_b64, encrypted = payload.split(b".", 1)
        salt = base64.urlsafe_b64decode(salt_b64)
    except (ValueError, Exception):
        print("Error: encrypted data is not in the expected format.")
        sys.exit(1)

    key = derive_key(password, salt)
    fernet = Fernet(key)

    try:
        return fernet.decrypt(encrypted)
    except InvalidToken:
        print("Error: decryption failed. Wrong password or corrupted data.")
        sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Decrypt text or a file encrypted with encrypt_sensitive_data.py.")
    group = parser.add_mutually_exclusive_group(required=True)
    group.add_argument("--text", help="Encrypted text string (base64) to decrypt")
    group.add_argument("--file", help="Path to an encrypted file")
    parser.add_argument("--password", required=True, help="Password used to originally encrypt the data")
    parser.add_argument("--output", default=None, help="Output file path (default: prints text, or strips .enc for files)")
    args = parser.parse_args()

    if args.text:
        payload = args.text.encode("ascii")
        decrypted = decrypt_data(payload, args.password)

        if args.output:
            with open(args.output, "wb") as f:
                f.write(decrypted)
            print(f"Decrypted data written to: {args.output}")
        else:
            print("Decrypted text:\n")
            print(decrypted.decode("utf-8", errors="replace"))
    else:
        if not os.path.isfile(args.file):
            print(f"Error: '{args.file}' does not exist.")
            sys.exit(1)

        with open(args.file, "rb") as f:
            payload = f.read()

        decrypted = decrypt_data(payload, args.password)
        output_path = args.output or (args.file[:-4] if args.file.endswith(".enc") else args.file + ".decrypted")

        with open(output_path, "wb") as f:
            f.write(decrypted)

        print(f"Decrypted: '{args.file}' -> '{output_path}'")


if __name__ == "__main__":
    main()
```

#### How it works and how to run
The payload is split on the `.` separator into the base64-encoded salt and the Fernet ciphertext, exactly mirroring how 5.8 packed them. The salt is reused to re-derive the identical key via the same PBKDF2 parameters, and `fernet.decrypt()` both decrypts and verifies the HMAC in one step, raising `InvalidToken` on any mismatch (wrong password or tampered data) rather than returning corrupted plaintext.

```bash
# Decrypt a text payload and print it
python3 decrypt_sensitive_data.py --text "VjNKdmRHZ3RTV0YwYQ==.gAAAAABmO7x9..." --password "Correct-Horse-Battery"

# Decrypt a file (writes secrets.txt by default, stripping the .enc suffix)
python3 decrypt_sensitive_data.py --file secrets.txt.enc --password "Correct-Horse-Battery"

# Decrypt to a custom path
python3 decrypt_sensitive_data.py --file secrets.txt.enc --password "Correct-Horse-Battery" --output restored/secrets.txt
```

#### Sample output
```
$ python3 decrypt_sensitive_data.py --file secrets.txt.enc --password Correct-Horse
Decrypted: 'secrets.txt.enc' -> 'secrets.txt'
```

```
$ python3 decrypt_sensitive_data.py --file secrets.txt.enc --password wrong-password
Error: decryption failed. Wrong password or corrupted data.
```

#### Variations / flags
- Add a `--stdout` flag for file decryption so the plaintext streams to stdout instead of writing to disk, for piping into another command without leaving a decrypted copy behind.
- Add automatic secure deletion (overwrite-then-delete) of the `.enc` file after a successful decrypt, if your workflow requires it.
- Wrap this in a small shell function that decrypts, opens the result in `$EDITOR`, and re-encrypts on save, for a lightweight "encrypted notes" workflow.

#### Common pitfalls
- **Mismatched iteration counts**: if you changed `ITERATIONS` in a customized version of `encrypt_sensitive_data.py`, this script's hardcoded `390000` must match, or every decryption will fail with `InvalidToken` even given the correct password. Keep the encrypt/decrypt scripts' KDF parameters in sync (or encode the iteration count into the payload format, similar to 5.4's approach).
- **Text payload truncation**: copy-pasting the encrypted base64 string between terminals can accidentally drop characters (especially across line wraps) — verify the full string was copied if decryption unexpectedly fails.
- **Treating "corrupted data" as recoverable**: unlike a simple checksum mismatch, there is no partial recovery from a Fernet `InvalidToken` error — if the ciphertext file was truncated or altered, that data is gone unless you have a backup.

### 5.10 Generate SSH Keys

#### Use case
You're provisioning a new server, setting up a new deploy/automation account, or onboarding a new machine, and need a fresh SSH key pair to enable passwordless, public-key authentication instead of password logins.

#### Prerequisites
Requires the third-party `cryptography` package.

```bash
python3 -m pip install cryptography
```

Cross-platform: Linux and macOS support the `os.chmod(..., stat.S_IRUSR | stat.S_IWUSR)` permission call natively (equivalent to `chmod 600`). On Windows, `os.chmod` has much more limited effect on NTFS permissions — see the pitfall note below.

#### Cautions
The private key is written **unencrypted** to disk (`serialization.NoEncryption()`). Anyone who can read that file can use the key immediately. For anything beyond a quick lab/test key, either add a passphrase (see Variations below) or make sure the file lands somewhere with strict OS-level access controls, and never commit it to version control.

#### Script
```python
#!/usr/bin/env python3
"""
generate_ssh_keys.py
--------------------------
Generates a new SSH key pair (RSA or Ed25519) using the 'cryptography' library
and saves the private/public keys to disk in standard OpenSSH format.

Requires: pip install cryptography

Usage:
    python generate_ssh_keys.py <output_basename> [--type rsa|ed25519] [--bits 4096]

Example:
    python generate_ssh_keys.py my_key
    python generate_ssh_keys.py my_key --type ed25519
    # Produces: my_key (private) and my_key.pub (public)
"""

import argparse
import os
import stat
import sys

try:
    from cryptography.hazmat.primitives import serialization
    from cryptography.hazmat.primitives.asymmetric import rsa, ed25519
except ImportError:
    print("Error: the 'cryptography' package is required.")
    print("Install it with: pip install -r requirements.txt")
    sys.exit(1)


def generate_rsa_key(bits: int):
    return rsa.generate_private_key(public_exponent=65537, key_size=bits)


def generate_ed25519_key():
    return ed25519.Ed25519PrivateKey.generate()


def save_key_pair(private_key, output_basename: str) -> None:
    private_bytes = private_key.private_bytes(
        encoding=serialization.Encoding.PEM,
        format=serialization.PrivateFormat.OpenSSH,
        encryption_algorithm=serialization.NoEncryption(),
    )

    public_key = private_key.public_key()
    public_bytes = public_key.public_bytes(
        encoding=serialization.Encoding.OpenSSH,
        format=serialization.PublicFormat.OpenSSH,
    )

    private_path = output_basename
    public_path = output_basename + ".pub"

    if os.path.exists(private_path) or os.path.exists(public_path):
        print(f"Error: '{private_path}' or '{public_path}' already exists. Choose a different name.")
        sys.exit(1)

    with open(private_path, "wb") as f:
        f.write(private_bytes)
    os.chmod(private_path, stat.S_IRUSR | stat.S_IWUSR)  # chmod 600

    with open(public_path, "wb") as f:
        f.write(public_bytes + b"\n")

    print(f"Private key saved to: {private_path} (permissions set to 600)")
    print(f"Public key saved to:  {public_path}")


def main():
    parser = argparse.ArgumentParser(description="Generate an SSH key pair.")
    parser.add_argument("output_basename", help="Base filename for the key pair (e.g. 'my_key')")
    parser.add_argument("--type", choices=["rsa", "ed25519"], default="ed25519",
                         help="Key type (default: ed25519, recommended)")
    parser.add_argument("--bits", type=int, default=4096, help="Key size in bits for RSA only (default: 4096)")
    args = parser.parse_args()

    print(f"Generating a {args.type.upper()} SSH key pair...")

    if args.type == "rsa":
        private_key = generate_rsa_key(args.bits)
    else:
        private_key = generate_ed25519_key()

    save_key_pair(private_key, args.output_basename)

    print("\nDone. Add the .pub file's content to a server's ~/.ssh/authorized_keys")
    print("to allow login using the private key. Never share the private key file.")


if __name__ == "__main__":
    main()
```

#### How it works and how to run
Ed25519 (the default) generates a modern elliptic-curve key that's fast, small, and considered at least as secure as 4096-bit RSA with far less computation; RSA is offered for compatibility with older systems that don't yet support Ed25519. Both key types are serialized to the standard OpenSSH PEM format that `ssh`, `scp`, and `authorized_keys` files expect natively, and the private key file's permissions are set to `600` (owner read/write only) immediately after writing, matching what OpenSSH itself requires before it will use the key.

```bash
# Recommended default: Ed25519 key pair
python3 generate_ssh_keys.py my_key

# RSA key pair with a specific bit size
python3 generate_ssh_keys.py my_key --type rsa --bits 4096

# Place directly in your SSH directory
python3 generate_ssh_keys.py ~/.ssh/id_ed25519_deploy
```

#### Sample output
```
$ python3 generate_ssh_keys.py my_key
Generating a ED25519 SSH key pair...
Private key saved to: my_key (permissions set to 600)
Public key saved to:  my_key.pub

Done. Add the .pub file's content to a server's ~/.ssh/authorized_keys
to allow login using the private key. Never share the private key file.
```

#### Variations / flags
- Add a `--passphrase` flag that swaps `serialization.NoEncryption()` for `serialization.BestAvailableEncryption(passphrase.encode())`, so the private key itself is encrypted at rest and requires a passphrase to use — recommended for any key that will live on a laptop or shared machine.
- Add a `--comment` flag to append a `user@host`-style comment to the public key file, matching `ssh-keygen`'s convention.
- Add an `--overwrite` flag to intentionally allow replacing an existing key pair (the script currently refuses by default, which is the safer choice).

#### Common pitfalls
- **Windows permission semantics**: `os.chmod(path, stat.S_IRUSR | stat.S_IWUSR)` has limited effect on NTFS — Windows OpenSSH clients instead rely on ACLs, and an overly-permissive private key file may trigger a "UNPROTECTED PRIVATE KEY FILE" warning from `ssh`. On Windows, additionally restrict the file via `icacls` or generate the key inside WSL.
- **Overwriting an existing key pair silently**: the script explicitly refuses to overwrite an existing `output_basename`/`output_basename.pub`, which prevents accidentally destroying a key that's already deployed to servers — don't remove this check without a good reason.
- **Forgetting to add the public key to `authorized_keys`**: generating the pair doesn't grant access to anything by itself; you still need to append the `.pub` file's contents to the target server's `~/.ssh/authorized_keys`.

### 5.11 Read SSH Keys

#### Use case
You need to quickly check what kind of SSH key a file contains, its size or curve, and its fingerprint — for example, to confirm you're about to deploy the right public key, or to identify an unlabeled key file you found on a server — without risking exposure of any private key material.

#### Prerequisites
Requires the third-party `cryptography` package.

```bash
python3 -m pip install cryptography
```

Cross-platform: Linux, macOS, Windows.

#### Cautions
This script never prints private key contents, only metadata (type, size, fingerprint) — that's intentional and shouldn't be changed. It also does not support password-protected (encrypted) private keys; it will detect and report them without attempting to decrypt.

#### Script
```python
#!/usr/bin/env python3
"""
read_ssh_keys.py
----------------------
Reads and displays information about an SSH key file (public or private):
key type, bit size/curve, and fingerprint. Does not print private key material.

Requires: pip install cryptography

Usage:
    python read_ssh_keys.py <key_path>

Example:
    python read_ssh_keys.py ~/.ssh/id_ed25519
    python read_ssh_keys.py ~/.ssh/id_ed25519.pub
"""

import argparse
import base64
import hashlib
import os
import sys

try:
    from cryptography.hazmat.primitives import serialization
    from cryptography.hazmat.primitives.asymmetric import rsa, ed25519
except ImportError:
    print("Error: the 'cryptography' package is required.")
    print("Install it with: pip install -r requirements.txt")
    sys.exit(1)


def fingerprint_from_public_bytes(public_openssh_bytes: bytes) -> str:
    # OpenSSH public key format: "<type> <base64-data> [comment]"
    parts = public_openssh_bytes.decode().strip().split()
    if len(parts) < 2:
        return "unavailable"
    key_data = base64.b64decode(parts[1])
    digest = hashlib.sha256(key_data).digest()
    return "SHA256:" + base64.b64encode(digest).decode().rstrip("=")


def read_key(path: str) -> None:
    if not os.path.isfile(path):
        print(f"Error: '{path}' does not exist.")
        sys.exit(1)

    with open(path, "rb") as f:
        data = f.read()

    is_public = path.endswith(".pub") or data.startswith((b"ssh-", b"ecdsa-"))

    print(f"Reading key: {path}\n")

    if is_public:
        try:
            public_key = serialization.load_ssh_public_key(data)
        except ValueError as e:
            print(f"Error: could not parse public key: {e}")
            sys.exit(1)

        fingerprint = fingerprint_from_public_bytes(data)
        print("Key kind:     PUBLIC")
        print(f"Fingerprint:  {fingerprint}")

    else:
        password = None
        try:
            private_key = serialization.load_ssh_private_key(data, password=password)
        except TypeError:
            print("This private key is password-protected. Reading encrypted keys is not supported by this script.")
            sys.exit(1)
        except ValueError as e:
            print(f"Error: could not parse private key: {e}")
            sys.exit(1)

        print("Key kind:     PRIVATE (contents not displayed)")

        if isinstance(private_key, rsa.RSAPrivateKey):
            print("Key type:     RSA")
            print(f"Key size:     {private_key.key_size} bits")
        elif isinstance(private_key, ed25519.Ed25519PrivateKey):
            print("Key type:     Ed25519")
        else:
            print(f"Key type:     {type(private_key).__name__}")

        public_bytes = private_key.public_key().public_bytes(
            encoding=serialization.Encoding.OpenSSH,
            format=serialization.PublicFormat.OpenSSH,
        )
        fingerprint = fingerprint_from_public_bytes(public_bytes)
        print(f"Fingerprint:  {fingerprint}")

    print("\nNote: private key material is never printed by this script.")


def main():
    parser = argparse.ArgumentParser(description="Read and display info about an SSH key file.")
    parser.add_argument("key_path", help="Path to a public (.pub) or private SSH key file")
    args = parser.parse_args()

    read_key(args.key_path)


if __name__ == "__main__":
    main()
```

#### How it works and how to run
The script guesses whether a file is a public or private key by checking the `.pub` extension or a `ssh-`/`ecdsa-` prefix in the raw bytes. Public keys are parsed with `load_ssh_public_key()` directly; private keys are parsed with `load_ssh_private_key()` and, on success, their *derived* public key is used to compute the same SHA256 fingerprint format `ssh-keygen -l` produces — the standard `SHA256:base64...` form you'll recognize from `ssh`'s "are you sure you want to continue connecting" prompts.

```bash

python3 read_ssh_keys.py ~/.ssh/id_ed25519.pub

# Inspect the matching private key (unencrypted only)
python3 read_ssh_keys.py ~/.ssh/id_ed25519
```

#### Variations / flags
- Add an `--md5` flag to also print the legacy MD5 colon-separated fingerprint format for compatibility with very old tooling that still expects it.
- Add a `--batch` mode that accepts a directory and reports on every key file found inside `~/.ssh/`.
- Compare fingerprints between two files with a `--compare other_key.pub` flag, useful for confirming a public/private pair actually match.

#### Common pitfalls
- **Encrypted private keys**: the script deliberately does not prompt for a passphrase and will report "password-protected... not supported" rather than trying to guess or bypass it — this is intentional, not a bug.
- **Ambiguous file detection**: a private key file with an unusual name (no `.pub` suffix, doesn't start with the expected OpenSSH markers because it's in an older PEM format) may be misclassified. When in doubt, check the file's first line manually (`head -1 keyfile`) — OpenSSH private keys begin with `-----BEGIN OPENSSH PRIVATE KEY-----`.
- **Newer/exotic key types**: keys generated with ECDSA or other curve types will still parse but will only be reported generically via `type(private_key).__name__` rather than a friendly label, since only RSA and Ed25519 have dedicated `isinstance` branches in this script.

### 5.12 Manage API Keys

#### Use case
You work with several third-party API keys (OpenAI, Stripe, AWS, etc.) on a personal workstation or a shared lab machine and want a single, password-protected local store instead of scattering them across shell profile files, plaintext `.env` files, or sticky notes.

#### Prerequisites
Requires the third-party `cryptography` package.

```bash
python3 -m pip install cryptography
```

Cross-platform: Linux, macOS, Windows. Creates two files in the current working directory: `api_keys.vault` (encrypted data) and `api_keys.salt` (the KDF salt, not secret but must be kept alongside the vault).

#### Cautions
This is a single-user, single-machine tool — it is not a substitute for a real team secrets manager (Vault, AWS Secrets Manager, 1Password, Doppler, etc.) once more than one person needs access. The `get` and `list` subcommands print key values directly to the terminal; be mindful of shoulder-surfing and terminal recording when using them.

#### Script
```python
#!/usr/bin/env python3
"""
manage_api_keys.py
------------------------
A simple local API key manager: add, list, get, and remove named API keys,
stored encrypted (password-protected) in a local JSON file.

Requires: pip install cryptography

Usage:
    python manage_api_keys.py add <name> <key_value> --password MyPass123
    python manage_api_keys.py list --password MyPass123
    python manage_api_keys.py get <name> --password MyPass123
    python manage_api_keys.py remove <name> --password MyPass123

Example:
    python manage_api_keys.py add openai sk-abc123... --password MyVaultPass
    python manage_api_keys.py list --password MyVaultPass
    python manage_api_keys.py get openai --password MyVaultPass
    python manage_api_keys.py remove openai --password MyVaultPass

Storage file: api_keys.vault (created in the current directory, encrypted)
"""

import argparse
import base64
import json
import os
import sys

try:
    from cryptography.fernet import Fernet, InvalidToken
    from cryptography.hazmat.primitives import hashes
    from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
except ImportError:
    print("Error: the 'cryptography' package is required.")
    print("Install it with: pip install -r requirements.txt")
    sys.exit(1)

VAULT_FILE = "api_keys.vault"
SALT_FILE = "api_keys.salt"


def get_or_create_salt() -> bytes:
    if os.path.exists(SALT_FILE):
        with open(SALT_FILE, "rb") as f:
            return f.read()
    salt = os.urandom(16)
    with open(SALT_FILE, "wb") as f:
        f.write(salt)
    return salt


def derive_key(password: str, salt: bytes) -> bytes:
    kdf = PBKDF2HMAC(algorithm=hashes.SHA256(), length=32, salt=salt, iterations=390000)
    return base64.urlsafe_b64encode(kdf.derive(password.encode()))


def load_vault(fernet: Fernet) -> dict:
    if not os.path.exists(VAULT_FILE):
        return {}

    with open(VAULT_FILE, "rb") as f:
        encrypted = f.read()

    if not encrypted:
        return {}

    try:
        decrypted = fernet.decrypt(encrypted)
    except InvalidToken:
        print("Error: incorrect password or corrupted vault file.")
        sys.exit(1)

    return json.loads(decrypted.decode("utf-8"))


def save_vault(fernet: Fernet, data: dict) -> None:
    encrypted = fernet.encrypt(json.dumps(data).encode("utf-8"))
    with open(VAULT_FILE, "wb") as f:
        f.write(encrypted)
    os.chmod(VAULT_FILE, 0o600)


def main():
    parser = argparse.ArgumentParser(description="Manage API keys in a local encrypted vault.")
    subparsers = parser.add_subparsers(dest="command", required=True)

    add_parser = subparsers.add_parser("add", help="Add or update an API key")
    add_parser.add_argument("name", help="Name/label for the key (e.g. 'openai')")
    add_parser.add_argument("value", help="The actual API key value")
    add_parser.add_argument("--password", required=True, help="Vault password")

    list_parser = subparsers.add_parser("list", help="List stored key names (not values)")
    list_parser.add_argument("--password", required=True, help="Vault password")

    get_parser = subparsers.add_parser("get", help="Retrieve a stored API key value")
    get_parser.add_argument("name", help="Name/label of the key to retrieve")
    get_parser.add_argument("--password", required=True, help="Vault password")

    remove_parser = subparsers.add_parser("remove", help="Remove a stored API key")
    remove_parser.add_argument("name", help="Name/label of the key to remove")
    remove_parser.add_argument("--password", required=True, help="Vault password")

    args = parser.parse_args()

    salt = get_or_create_salt()
    key = derive_key(args.password, salt)
    fernet = Fernet(key)

    vault = load_vault(fernet)

    if args.command == "add":
        vault[args.name] = args.value
        save_vault(fernet, vault)
        print(f"API key '{args.name}' saved to vault ({VAULT_FILE}).")

    elif args.command == "list":
        if not vault:
            print("Vault is empty.")
        else:
            print(f"Stored API key names ({len(vault)}):")
            for name in vault:
                print(f"  - {name}")

    elif args.command == "get":
        if args.name not in vault:
            print(f"Error: no key named '{args.name}' found.")
            sys.exit(1)
        print(f"{args.name}: {vault[args.name]}")

    elif args.command == "remove":
        if args.name not in vault:
            print(f"Error: no key named '{args.name}' found.")
            sys.exit(1)
        del vault[args.name]
        save_vault(fernet, vault)
        print(f"API key '{args.name}' removed.")


if __name__ == "__main__":
    main()
```

#### How it works and how to run
On first use, a random salt is generated once and saved to `api_keys.salt`; every subsequent command reuses that same salt so the same password always derives the same encryption key. The entire vault (a simple `{name: value}` JSON dictionary) is decrypted into memory, modified in place for `add`/`remove`, and re-encrypted and rewritten to disk as a single Fernet-encrypted blob — there's no partial-file update, so each write replaces the whole vault file atomically-ish (a full rewrite, not a true atomic rename in this version).

```bash
# Add a key
python3 manage_api_keys.py add openai sk-abc123xyz --password MyVaultPass

# List stored key names (values hidden)
python3 manage_api_keys.py list --password MyVaultPass

# Retrieve a specific key's value
python3 manage_api_keys.py get openai --password MyVaultPass

# Remove a key
python3 manage_api_keys.py remove openai --password MyVaultPass
```

#### Sample output
```
$ python3 manage_api_keys.py add openai sk-abc123xyz --password MyVaultPass
API key 'openai' saved to vault (api_keys.vault).

$ python3 manage_api_keys.py list --password MyVaultPass
Stored API key names (1):
  - openai

$ python3 manage_api_keys.py get openai --password MyVaultPass
openai: sk-abc123xyz
```

#### Variations / flags
- Add a `rotate` subcommand that re-encrypts the whole vault under a new password (generate a new salt, decrypt with the old key, re-encrypt with the new one).
- Add an `export --format env` subcommand that prints all keys as `KEY_NAME=value` lines suitable for sourcing into a shell session.
- Write the vault to a temp file and `os.replace()` it into place instead of writing directly, to avoid a corrupted vault file if the process is killed mid-write.

#### Common pitfalls
- **Losing `api_keys.salt`**: the salt file isn't secret, but it *is* required — deleting it while keeping `api_keys.vault` makes the vault permanently undecryptable even with the correct password, since a new random salt would derive a different key.
- **`--password` on the command line**: as with other scripts in this chapter, this leaks into shell history and `ps` output; for daily use consider wrapping this script with a `getpass`-based prompt (see 5.1) instead of passing `--password` directly.
- **Vault file location**: because `VAULT_FILE`/`SALT_FILE` are relative paths, running the script from different working directories creates separate, unrelated vaults — always run it from the same directory, or modify the script to use an absolute path like `~/.config/api_keys/`.

### 5.13 Validate Certificates

#### Use case
You're troubleshooting an HTTPS/TLS issue, auditing a fleet of servers for soon-to-expire certificates, or just verifying that a `.pem`/`.crt` file you were handed is what it claims to be before deploying it.

#### Prerequisites
Requires the third-party `cryptography` package.

```bash
python3 -m pip install cryptography
```

Cross-platform: Linux, macOS, Windows.

#### Cautions
This script checks the certificate's *self-reported* validity window, subject, and issuer — it does **not** verify the certificate against a trusted root CA chain, check revocation (CRL/OCSP), or confirm the hostname it's being used for matches the certificate's SAN/CN. A certificate can pass every check this script performs and still be untrusted, revoked, or wrong for your domain. For full chain-of-trust validation, use `openssl verify` or your TLS library's built-in verification.

#### Script
```python
#!/usr/bin/env python3
"""
validate_certificate.py
-----------------------------
Validates and inspects an X.509 certificate (.pem or .crt file):
checks expiry, subject, issuer, and validity dates.

Requires: pip install cryptography

Usage:
    python validate_certificate.py <certificate_path>

Example:
    python validate_certificate.py server.pem
    python validate_certificate.py /etc/ssl/certs/example.crt
"""

import argparse
import os
import sys
from datetime import datetime, timezone

try:
    from cryptography import x509
    from cryptography.hazmat.backends import default_backend
except ImportError:
    print("Error: the 'cryptography' package is required.")
    print("Install it with: pip install -r requirements.txt")
    sys.exit(1)


def load_certificate(path: str):
    with open(path, "rb") as f:
        data = f.read()

    try:
        return x509.load_pem_x509_certificate(data, default_backend())
    except ValueError:
        pass

    try:
        return x509.load_der_x509_certificate(data, default_backend())
    except ValueError:
        print("Error: could not parse certificate as PEM or DER format.")
        sys.exit(1)


def validate_certificate(path: str) -> None:
    if not os.path.isfile(path):
        print(f"Error: '{path}' does not exist.")
        sys.exit(1)

    cert = load_certificate(path)

    now = datetime.now(timezone.utc)
    not_before = cert.not_valid_before_utc
    not_after = cert.not_valid_after_utc

    print(f"Certificate: {path}\n")
    print(f"Subject:        {cert.subject.rfc4514_string()}")
    print(f"Issuer:         {cert.issuer.rfc4514_string()}")
    print(f"Serial Number:  {cert.serial_number}")
    print(f"Valid From:     {not_before}")
    print(f"Valid Until:    {not_after}")
    print(f"Signature Algo: {cert.signature_algorithm_oid._name}")

    if now < not_before:
        print("\nStatus: NOT YET VALID (starts in the future)")
    elif now > not_after:
        days_expired = (now - not_after).days
        print(f"\nStatus: EXPIRED ({days_expired} day(s) ago)")
    else:
        days_remaining = (not_after - now).days
        print(f"\nStatus: VALID ({days_remaining} day(s) remaining)")
        if days_remaining <= 30:
            print("Warning: this certificate will expire soon.")


def main():
    parser = argparse.ArgumentParser(description="Validate and inspect an X.509 certificate.")
    parser.add_argument("certificate_path", help="Path to the certificate file (.pem or .crt)")
    args = parser.parse_args()

    validate_certificate(args.certificate_path)


if __name__ == "__main__":
    main()
```

#### How it works and how to run
The script first tries to parse the file as PEM (the common base64 text format starting with `-----BEGIN CERTIFICATE-----`), and falls back to DER (raw binary format) if that fails — so it transparently handles both common on-disk encodings. It then reads the certificate's subject, issuer, serial number, validity window, and signature algorithm directly from the parsed X.509 structure, and compares the current UTC time against the `not_before`/`not_after` fields to report VALID / EXPIRED / NOT YET VALID, with an extra warning if fewer than 30 days remain.

```bash
# Validate a PEM certificate
python3 validate_certificate.py server.pem

# Validate a certificate from the system trust store
python3 validate_certificate.py /etc/ssl/certs/example.crt
```

#### Sample output
```
$ python3 validate_certificate.py server.pem

Subject:        CN=example.com
Issuer:         CN=Let's Encrypt Authority X3,O=Let's Encrypt,C=US
Serial Number:  123456789012345678901234567890
Valid From:     2026-05-01 00:00:00+00:00
Valid Until:    2026-07-30 23:59:59+00:00
Signature Algo: sha256WithRSAEncryption

Status: VALID (12 day(s) remaining)
Warning: this certificate will expire soon.
```

```
$ python3 validate_certificate.py old-cert.pem

...

```

#### Variations / flags
- Add a `--warn-days N` flag to make the "expiring soon" threshold configurable instead of the hardcoded 30 days.
- Add a `--check-hostname example.com` flag that inspects the Subject Alternative Name extension and confirms the given hostname is covered.
- Add a `--batch` mode that accepts a glob of certificate files and prints a summary table, useful for auditing an entire `/etc/ssl/certs/` directory or a fleet of servers via a config-management run.

#### Common pitfalls
- **Trusting this as full chain validation**: reiterating the caution above — this script only reads the certificate's own claims. A self-signed or expired-root-signed certificate will still print "VALID" if its own dates are in range; use `openssl verify -CAfile ca-bundle.pem server.pem` for real chain validation.
- **Timezone confusion**: all comparisons are done in UTC (`not_valid_before_utc`/`not_valid_after_utc` and `datetime.now(timezone.utc)`) — if you manually compare printed dates against a local-time clock, remember the printed times are UTC, not your local timezone.
- **DER vs. PEM mixups**: a `.crt` or `.cer` file can be either PEM or DER depending on where it came from; the script handles both automatically, but if you're editing the file with a text editor first, make sure you haven't corrupted a binary DER file by opening/saving it as text.

### 5.14 Verify File Integrity

#### Use case
You downloaded an installer, ISO, or archive and the publisher provided an official checksum — you want to confirm the file wasn't corrupted in transit or tampered with before you run/install it.

#### Prerequisites
Standard library only (`hashlib`, `argparse`, `os`).

```bash
python3 --version   # 3.6+ (uses the walrus operator := , so 3.8+ is required)
# No installation required
```

Cross-platform: Linux, macOS, Windows.

#### Cautions
A matching checksum confirms the file wasn't *accidentally* corrupted, and — if the checksum came from a trusted source over a trusted channel (e.g. HTTPS, a signed release page) — that it wasn't tampered with either. A checksum published on the exact same untrusted download page as the file itself provides much weaker guarantees, since an attacker who could swap the file could just as easily swap the listed checksum. MD5 and SHA1 are included for compatibility but are cryptographically broken for adversarial tampering scenarios — prefer SHA256 or SHA512 when you have a choice.

#### Script
```python
#!/usr/bin/env python3
"""
verify_file_integrity.py
------------------------------
Verifies that a file matches an expected checksum (hash), to confirm
it has not been corrupted or tampered with.

Usage:
    python verify_file_integrity.py <file_path> <expected_hash> [--algo sha256]

Example:
    python verify_file_integrity.py installer.exe a1b2c3...  --algo sha256
"""

import argparse
import hashlib
import os
import sys


def calculate_hash(file_path: str, algorithm: str) -> str:
    hasher = hashlib.new(algorithm)
    with open(file_path, "rb") as f:
        while chunk := f.read(65536):
            hasher.update(chunk)
    return hasher.hexdigest()


def verify_integrity(file_path: str, expected_hash: str, algorithm: str) -> None:
    if not os.path.isfile(file_path):
        print(f"Error: '{file_path}' does not exist.")
        sys.exit(1)

    actual_hash = calculate_hash(file_path, algorithm)
    expected_hash = expected_hash.strip().lower()
    actual_hash_lower = actual_hash.lower()

    print(f"File:          {file_path}")
    print(f"Algorithm:     {algorithm.upper()}")
    print(f"Expected hash: {expected_hash}")
    print(f"Actual hash:   {actual_hash_lower}")

    if actual_hash_lower == expected_hash:
        print("\nResult: MATCH — file integrity verified.")
    else:
        print("\nResult: MISMATCH — file may be corrupted or tampered with!")
        sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Verify file integrity against an expected checksum.")
    parser.add_argument("file_path", help="Path to the file to verify")
    parser.add_argument("expected_hash", help="The expected checksum/hash value")
    parser.add_argument("--algo", default="sha256", choices=["md5", "sha1", "sha256", "sha512"],
                         help="Hash algorithm (default: sha256)")
    args = parser.parse_args()

    verify_integrity(args.file_path, args.expected_hash, args.algo)


if __name__ == "__main__":
    main()
```

#### How it works and how to run
The file is read in 64KB chunks (`f.read(65536)`) rather than all at once, so this works fine even on multi-gigabyte ISOs without loading the whole file into memory. The computed hash and the expected hash are both lowercased before comparison so that case differences in how the checksum was originally published (some tools print uppercase hex) don't cause a false mismatch.

```bash
# Verify against a published SHA256 checksum
python3 verify_file_integrity.py installer.exe a1b2c3d4e5f6...

# Verify with a different algorithm
python3 verify_file_integrity.py installer.iso 9f8e7d6c5b4a... --algo sha512
```

#### Sample output
```
$ python3 verify_file_integrity.py installer.exe a1b2c3d4e5f67890abcdef1234567890abcdef1234567890abcdef1234567890
File:          installer.exe
Algorithm:     SHA256
Expected hash: a1b2c3d4e5f67890abcdef1234567890abcdef1234567890abcdef1234567890
Actual hash:   a1b2c3d4e5f67890abcdef1234567890abcdef1234567890abcdef1234567890

Result: MATCH — file integrity verified.
```

```
$ python3 verify_file_integrity.py installer.exe wrong_hash_value
...
Result: MISMATCH — file may be corrupted or tampered with!
```

#### Variations / flags
- Add a `--url` flag that downloads the file first (via `urllib`/`requests`) and verifies it in one step, useful for a "download and verify" one-liner.
- Add a `--quiet` flag that suppresses the detail lines and only prints `MATCH`/`MISMATCH`, for easy use in scripted pipelines checking the exit code.
- Read the expected hash from a `SHA256SUMS`-style file (see 5.15) and look up the right line by filename automatically, instead of requiring the hash to be pasted on the command line.

#### Common pitfalls
- **Walrus operator requires Python 3.8+**: the `while chunk := f.read(65536):` line will raise a `SyntaxError` on Python 3.7 or older — check your interpreter version if this script fails to even start.
- **Algorithm mismatch with the publisher**: if the publisher's checksum page says "SHA-256" but you run with the default and it still mismatches, double check you copied the *correct* hash from the *correct* algorithm's column — many download pages list MD5, SHA1, and SHA256 side by side.
- **Trailing whitespace/newlines in a copy-pasted hash**: the script strips the expected hash for you, but if you're piping it in from another command, make sure you're not accidentally including quotes or a trailing `%` prompt artifact from your terminal.

### 5.15 Generate Checksums

#### Use case
You're publishing a file for others to download (a release build, a backup archive, an installer) and want to generate and distribute checksums alongside it, so recipients can verify integrity themselves with `verify_file_integrity.py` (5.14) or standard tools like `sha256sum -c`.

#### Prerequisites
Standard library only (`hashlib`, `argparse`, `os`).

```bash
python3 --version   # 3.6+
# No installation required
```

Cross-platform: Linux, macOS, Windows.

#### Cautions
This script computes checksums for you; it does not sign them. A checksum file on its own doesn't prove *who* produced it — for real authenticity guarantees (not just accidental-corruption detection), sign the checksum file itself with GPG or a similar tool and publish the signature alongside it.

#### Script
```python
#!/usr/bin/env python3
"""
generate_checksums.py
---------------------------
Generates checksums (hashes) for one or more files, and optionally
writes them to a checksum file (e.g. SHA256SUMS) for later verification.

Usage:
    python generate_checksums.py <file1> [file2 ...] [--algo sha256] [--output SHA256SUMS.txt]

Example:
    python generate_checksums.py installer.exe
    python generate_checksums.py *.iso --algo sha512 --output checksums.txt
"""

import argparse
import hashlib
import os
import sys


def calculate_hash(file_path: str, algorithm: str) -> str:
    hasher = hashlib.new(algorithm)
    with open(file_path, "rb") as f:
        while chunk := f.read(65536):
            hasher.update(chunk)
    return hasher.hexdigest()


def generate_checksums(file_paths, algorithm: str, output: str = None) -> None:
    results = []

    for file_path in file_paths:
        if not os.path.isfile(file_path):
            print(f"Skipped (not found): {file_path}")
            continue

        digest = calculate_hash(file_path, algorithm)
        results.append((digest, file_path))
        print(f"{digest}  {file_path}")

    if output and results:
        with open(output, "w", encoding="utf-8") as f:
            for digest, path in results:
                f.write(f"{digest}  {path}\n")
        print(f"\nChecksums written to: {output}")
        print(f"Verify later with: sha256sum -c {output}  (or use verify_file_integrity.py per file)")


def main():
    parser = argparse.ArgumentParser(description="Generate checksums for one or more files.")
    parser.add_argument("files", nargs="+", help="File path(s) to checksum")
    parser.add_argument("--algo", default="sha256", choices=["md5", "sha1", "sha256", "sha512"],
                         help="Hash algorithm (default: sha256)")
    parser.add_argument("--output", default=None, help="Optional output file to save checksums")
    args = parser.parse_args()

    generate_checksums(args.files, args.algo, args.output)


if __name__ == "__main__":
    main()
```

#### How it works and how to run
`generate_checksums()` accepts any number of file paths (your shell's glob expansion handles `*.iso`-style wildcards before Python ever sees them), skips anything that isn't a regular file, and prints each `hash  filename` pair as it goes. If `--output` is given, the same lines are written to a file in the exact two-space-separated format that `sha256sum`, `sha1sum`, and `md5sum` use — meaning the output is directly consumable by those standard Unix tools' `-c`/`--check` flag, not just by this chapter's own scripts.

```bash
# Checksum a single file, print only
python3 generate_checksums.py installer.exe

# Checksum multiple files with a glob, write to a checksums file
python3 generate_checksums.py *.iso --algo sha512 --output checksums.txt

# Default SHA256, save to the conventional filename
python3 generate_checksums.py release.tar.gz --output SHA256SUMS
```

#### Sample output
```
$ python3 generate_checksums.py release.tar.gz --output SHA256SUMS
a1b2c3d4e5f67890abcdef1234567890abcdef1234567890abcdef1234567890  release.tar.gz

Checksums written to: SHA256SUMS
Verify later with: sha256sum -c SHA256SUMS  (or use verify_file_integrity.py per file)
```

#### Variations / flags
- Add a `--recursive DIR` mode that walks a directory tree and checksums every file inside it.
- Compute multiple algorithms in one pass (e.g. both SHA256 and SHA512) and write separate output files, avoiding re-reading large files twice.
- Add a `--sign` flag that shells out to `gpg --detach-sign` on the resulting checksum file for an authenticity guarantee beyond plain integrity.

#### Common pitfalls
- **Relative vs. absolute paths in the output file**: the checksum file records whatever path string you passed on the command line — if you later run `sha256sum -c SHA256SUMS` from a different directory, the recorded (relative) paths may not resolve. Keep the checksum file alongside the files it describes, or use consistent relative paths.
- **Skipped files fail silently in `--output` mode**: a missing file is printed as "Skipped" to the console but doesn't appear in the output file at all — double check the console output, don't assume the output file covers every file you listed.
- **Very large files and memory**: like 5.14, this script streams in 64KB chunks, so multi-gigabyte files are fine — but computing SHA512 for many large files back-to-back can still take a while; consider adding a progress indicator for very large batches.

### 5.16 Secure Credential Storage

#### Use case
You need to keep track of a handful of username/password pairs for services you interact with from a script or the command line (test accounts, internal tools, automation logins) without storing them in plaintext anywhere on disk.

#### Prerequisites
Requires the third-party `cryptography` package.

```bash
python3 -m pip install cryptography
```

Cross-platform: Linux, macOS, Windows. Creates `credentials.vault` and `credentials.salt` in the current working directory.

#### Cautions
This is architecturally the same pattern as `manage_api_keys.py` (5.12) but for username/password pairs instead of single key values — the same caveats apply: it's a single-user local tool, not a team secrets manager, and the `get`/`list` output prints credentials in plaintext to the terminal.

#### Script
```python
#!/usr/bin/env python3
"""
secure_credential_storage.py
----------------------------------
Stores and retrieves username/password credentials in a local, encrypted
JSON vault, protected by a master password. Useful as a minimal personal
credential manager or as a building block for automation scripts.

Requires: pip install cryptography

Usage:
    python secure_credential_storage.py add <service> <username> --password <master_password>
    python secure_credential_storage.py get <service> --password <master_password>
    python secure_credential_storage.py list --password <master_password>
    python secure_credential_storage.py remove <service> --password <master_password>

Example:
    python secure_credential_storage.py add gmail myuser@gmail.com --password MyMasterPass
    # (You will be prompted to enter the credential's password securely)
    python secure_credential_storage.py get gmail --password MyMasterPass
    python secure_credential_storage.py list --password MyMasterPass

Storage file: credentials.vault (created in the current directory, encrypted)
"""

import argparse
import base64
import getpass
import json
import os
import sys

try:
    from cryptography.fernet import Fernet, InvalidToken
    from cryptography.hazmat.primitives import hashes
    from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
except ImportError:
    print("Error: the 'cryptography' package is required.")
    print("Install it with: pip install -r requirements.txt")
    sys.exit(1)

VAULT_FILE = "credentials.vault"
SALT_FILE = "credentials.salt"


def get_or_create_salt() -> bytes:
    if os.path.exists(SALT_FILE):
        with open(SALT_FILE, "rb") as f:
            return f.read()
    salt = os.urandom(16)
    with open(SALT_FILE, "wb") as f:
        f.write(salt)
    return salt


def derive_key(password: str, salt: bytes) -> bytes:
    kdf = PBKDF2HMAC(algorithm=hashes.SHA256(), length=32, salt=salt, iterations=390000)
    return base64.urlsafe_b64encode(kdf.derive(password.encode()))


def load_vault(fernet: Fernet) -> dict:
    if not os.path.exists(VAULT_FILE):
        return {}

    with open(VAULT_FILE, "rb") as f:
        encrypted = f.read()

    if not encrypted:
        return {}

    try:
        decrypted = fernet.decrypt(encrypted)
    except InvalidToken:
        print("Error: incorrect master password or corrupted vault file.")
        sys.exit(1)

    return json.loads(decrypted.decode("utf-8"))


def save_vault(fernet: Fernet, data: dict) -> None:
    encrypted = fernet.encrypt(json.dumps(data).encode("utf-8"))
    with open(VAULT_FILE, "wb") as f:
        f.write(encrypted)
    os.chmod(VAULT_FILE, 0o600)


def main():
    parser = argparse.ArgumentParser(description="Store and retrieve credentials in a local encrypted vault.")
    subparsers = parser.add_subparsers(dest="command", required=True)

    add_parser = subparsers.add_parser("add", help="Add or update a credential")
    add_parser.add_argument("service", help="Service name/label (e.g. 'gmail')")
    add_parser.add_argument("username", help="Username for this service")
    add_parser.add_argument("--password", required=True, help="Master vault password")

    get_parser = subparsers.add_parser("get", help="Retrieve a stored credential")
    get_parser.add_argument("service", help="Service name/label to retrieve")
    get_parser.add_argument("--password", required=True, help="Master vault password")

    list_parser = subparsers.add_parser("list", help="List stored service names")
    list_parser.add_argument("--password", required=True, help="Master vault password")

    remove_parser = subparsers.add_parser("remove", help="Remove a stored credential")
    remove_parser.add_argument("service", help="Service name/label to remove")
    remove_parser.add_argument("--password", required=True, help="Master vault password")

    args = parser.parse_args()

    salt = get_or_create_salt()
    master_key = derive_key(args.password, salt)
    fernet = Fernet(master_key)

    vault = load_vault(fernet)

    if args.command == "add":
        try:
            credential_password = getpass.getpass(f"Enter password for '{args.username}' on '{args.service}': ")
        except (EOFError, KeyboardInterrupt):
            print("\nCancelled.")
            sys.exit(1)

        vault[args.service] = {"username": args.username, "password": credential_password}
        save_vault(fernet, vault)
        print(f"\nCredential for '{args.service}' saved to vault ({VAULT_FILE}).")

    elif args.command == "get":
        if args.service not in vault:
            print(f"Error: no credential found for '{args.service}'.")
            sys.exit(1)
        entry = vault[args.service]
        print(f"Service:  {args.service}")
        print(f"Username: {entry['username']}")
        print(f"Password: {entry['password']}")

    elif args.command == "list":
        if not vault:
            print("Vault is empty.")
        else:
            print(f"Stored credentials ({len(vault)}):")
            for service, entry in vault.items():
                print(f"  - {service} (username: {entry['username']})")

    elif args.command == "remove":
        if args.service not in vault:
            print(f"Error: no credential found for '{args.service}'.")
            sys.exit(1)
        del vault[args.service]
        save_vault(fernet, vault)
        print(f"Credential for '{args.service}' removed.")


if __name__ == "__main__":
    main()
```

#### How it works and how to run
The master password (passed via `--password`) derives the vault's encryption key exactly like 5.12, using a persisted salt so the same master password always regenerates the same key across runs. Unlike 5.12, the credential's *own* password is never taken as a command-line argument — `add` explicitly calls `getpass.getpass()` to prompt for it interactively, so the sensitive value at least avoids landing in shell history even though the master password (a lesser secret, since it protects an empty/unlock-only vault by itself) is still passed as a flag.

```bash
# Add a credential (you'll be prompted for the service's own password)
python3 secure_credential_storage.py add gmail myuser@gmail.com --password MyMasterPass

# List services (usernames shown, passwords hidden)
python3 secure_credential_storage.py list --password MyMasterPass

# Retrieve a full credential
python3 secure_credential_storage.py get gmail --password MyMasterPass

# Remove a credential
python3 secure_credential_storage.py remove gmail --password MyMasterPass
```

#### Sample output
```
$ python3 secure_credential_storage.py add gmail myuser@gmail.com --password MyMasterPass
Enter password for 'myuser@gmail.com' on 'gmail': 

Credential for 'gmail' saved to vault (credentials.vault).

$ python3 secure_credential_storage.py list --password MyMasterPass
Stored credentials (1):
  - gmail (username: myuser@gmail.com)

$ python3 secure_credential_storage.py get gmail --password MyMasterPass
Service:  gmail
Username: myuser@gmail.com
Password: ************
```

#### Variations / flags
- Add a `--copy-to-clipboard` option on `get` (via `pyperclip`) so the retrieved password never has to be printed to the visible terminal at all.
- Add per-entry metadata (notes, URL, last-updated timestamp) to the stored dictionary.
- Add a `change-master-password` subcommand that decrypts with the old master password and re-encrypts everything under a new one with a freshly generated salt.

#### Common pitfalls
- **Same salt-loss issue as 5.12**: losing `credentials.salt` while keeping `credentials.vault` makes the vault permanently unreadable, even with the correct master password.
- **Master password on the command line**: it's technically less sensitive than the individual credential passwords (since without it the vault is just encrypted noise), but it still shouldn't be treated casually — avoid typing it into shared or logged terminal sessions.
- **`get` prints the password in plaintext to the terminal**: if you're demoing this script or sharing your screen, be aware `get` will display the stored password directly — there's no masking on output the way there is on `add`'s input prompt.

### 5.17 Implement Multi-Factor Authentication (MFA)

#### Use case
You're adding a second authentication factor to a login flow (on top of the password hashing/verification from 5.4/5.5) — enrolling a user with a QR code they scan into Google Authenticator/Authy/1Password, and later checking the 6-digit code they type against the stored secret.

> **Note:** no source script was supplied for this topic, so the implementation below was written from scratch, following RFC 6238 (TOTP) and RFC 4226 (HOTP), and has been executed and verified end-to-end (secret generation → code generation → successful verification → correct rejection of a wrong code) before being included here.

#### Prerequisites
Core functionality (`setup`, `generate`, `verify`) uses only the Python standard library (`hmac`, `hashlib`, `base64`, `struct`, `secrets`, `time`, `urllib.parse`). Rendering an actual scannable QR code in the terminal is optional and needs one extra package:

```bash
python3 --version   # 3.6+
python3 -m pip install "qrcode[pil]"   # optional — only needed for the --qr flag
```

Cross-platform: Linux, macOS, Windows.

#### Cautions
The generated secret is the single most sensitive piece of MFA state — anyone who obtains it can generate valid codes indefinitely, unlike a password which can at least be observed being typed once. Store it encrypted (e.g. via the same PBKDF2/Fernet pattern as 5.8/5.12), never log it, and never send it anywhere except the one-time enrollment step. This script also does not implement replay protection (rejecting a code that was already used once) — a production system should track the last successfully used counter per user and reject reuse within the same window.

#### Script
```python
#!/usr/bin/env python3
"""
mfa_totp.py
-----------------
Implements Time-based One-Time Password (TOTP) multi-factor authentication,
compatible with apps like Google Authenticator, Authy, and 1Password.
Can generate a new MFA secret + QR-code provisioning URI, and verify a
6-digit code against a secret (RFC 6238 / RFC 4226).

Requires: pip install qrcode[pil]   (optional, only for --qr)

Usage:
    python mfa_totp.py setup <account_name> [--issuer "MyApp"] [--qr]
    python mfa_totp.py generate <base32_secret>
    python mfa_totp.py verify <base32_secret> <code> [--window 1]

Example:
    python mfa_totp.py setup alice@example.com --issuer "Acme Corp" --qr
    python mfa_totp.py generate JBSWY3DPEHPK3PXP
    python mfa_totp.py verify JBSWY3DPEHPK3PXP 123456
"""

import argparse
import base64
import hashlib
import hmac
import secrets
import struct
import sys
import time
import urllib.parse

DEFAULT_DIGITS = 6
DEFAULT_PERIOD = 30  # seconds
DEFAULT_ALGO = "SHA1"  # SHA1 is what nearly all authenticator apps expect


def generate_secret(num_bytes: int = 20) -> str:
    """Generate a random base32-encoded shared secret (default 160 bits, per RFC 4226)."""
    raw = secrets.token_bytes(num_bytes)
    return base64.b32encode(raw).decode("ascii")


def hotp(secret_b32: str, counter: int, digits: int = DEFAULT_DIGITS, algo: str = DEFAULT_ALGO) -> str:
    """RFC 4226 HMAC-based One-Time Password for a given counter value."""
    key = base64.b32decode(secret_b32.upper() + "=" * ((8 - len(secret_b32) % 8) % 8))
    counter_bytes = struct.pack(">Q", counter)

    hash_func = {"SHA1": hashlib.sha1, "SHA256": hashlib.sha256, "SHA512": hashlib.sha512}[algo]
    digest = hmac.new(key, counter_bytes, hash_func).digest()

    offset = digest[-1] & 0x0F
    truncated = digest[offset:offset + 4]
    code_int = struct.unpack(">I", truncated)[0] & 0x7FFFFFFF
    code = code_int % (10 ** digits)

    return str(code).zfill(digits)


def totp(secret_b32: str, for_time: float = None, period: int = DEFAULT_PERIOD,
          digits: int = DEFAULT_DIGITS, algo: str = DEFAULT_ALGO) -> str:
    """RFC 6238 Time-based One-Time Password."""
    if for_time is None:
        for_time = time.time()
    counter = int(for_time // period)
    return hotp(secret_b32, counter, digits, algo)


def verify_totp(secret_b32: str, code: str, window: int = 1, period: int = DEFAULT_PERIOD,
                 digits: int = DEFAULT_DIGITS, algo: str = DEFAULT_ALGO) -> bool:
    """
    Verify a code, allowing +/- `window` time steps of clock drift.
    Uses constant-time comparison for each candidate.
    """
    now = time.time()
    current_counter = int(now // period)

    for offset in range(-window, window + 1):
        candidate = hotp(secret_b32, current_counter + offset, digits, algo)
        if hmac.compare_digest(candidate, code):
            return True
    return False


def provisioning_uri(secret_b32: str, account_name: str, issuer: str = "MyApp",
                      digits: int = DEFAULT_DIGITS, period: int = DEFAULT_PERIOD) -> str:
    """Build an otpauth:// URI that authenticator apps can scan as a QR code."""
    label = urllib.parse.quote(f"{issuer}:{account_name}")
    params = urllib.parse.urlencode({
        "secret": secret_b32,
        "issuer": issuer,
        "algorithm": DEFAULT_ALGO,
        "digits": digits,
        "period": period,
    })
    return f"otpauth://totp/{label}?{params}"


def cmd_setup(args):
    secret = generate_secret()
    uri = provisioning_uri(secret, args.account_name, args.issuer)

    print("New MFA secret generated.\n")
    print(f"Account:        {args.account_name}")
    print(f"Issuer:         {args.issuer}")
    print(f"Secret (base32): {secret}")
    print(f"\nProvisioning URI:\n{uri}")
    print("\nStore the secret securely (e.g. alongside the user's hashed password).")
    print("Do not log or transmit it in plaintext after initial setup.")

    if args.qr:
        try:
            import qrcode
        except ImportError:
            print("\nNote: install 'qrcode[pil]' to render a scannable QR code (pip install qrcode[pil]).")
            return
        qr = qrcode.QRCode(border=2)
        qr.add_data(uri)
        qr.make(fit=True)
        qr.print_ascii(invert=True)


def cmd_generate(args):
    code = totp(args.secret)
    seconds_remaining = DEFAULT_PERIOD - int(time.time()) % DEFAULT_PERIOD
    print(f"Current TOTP code: {code}")
    print(f"Valid for {seconds_remaining} more second(s).")


def cmd_verify(args):
    is_valid = verify_totp(args.secret, args.code, window=args.window)
    if is_valid:
        print("Result: Code is VALID.")
    else:
        print("Result: Code is INVALID.")
        sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="TOTP-based multi-factor authentication (RFC 6238).")
    subparsers = parser.add_subparsers(dest="command", required=True)

    setup_parser = subparsers.add_parser("setup", help="Generate a new MFA secret and enrollment QR/URI")
    setup_parser.add_argument("account_name", help="Account identifier shown in the authenticator app (e.g. an email)")
    setup_parser.add_argument("--issuer", default="MyApp", help="Issuer/app name shown in the authenticator app")
    setup_parser.add_argument("--qr", action="store_true", help="Also render an ASCII QR code (requires 'qrcode' package)")

    generate_parser = subparsers.add_parser("generate", help="Generate the current TOTP code for a secret")
    generate_parser.add_argument("secret", help="Base32-encoded shared secret")

    verify_parser = subparsers.add_parser("verify", help="Verify a code against a secret")
    verify_parser.add_argument("secret", help="Base32-encoded shared secret")
    verify_parser.add_argument("code", help="6-digit code to verify")
    verify_parser.add_argument("--window", type=int, default=1,
                                help="Number of 30s time-steps of clock drift to tolerate on each side (default: 1)")

    args = parser.parse_args()

    if args.command == "setup":
        cmd_setup(args)
    elif args.command == "generate":
        cmd_generate(args)
    elif args.command == "verify":
        cmd_verify(args)


if __name__ == "__main__":
    main()
```

#### How it works and how to run
`generate_secret()` produces a random 160-bit key, base32-encoded because that's the format every authenticator app expects a user to be able to type manually as a fallback to scanning a QR code. `hotp()` implements RFC 4226 exactly: HMAC the counter value with the shared secret, then use "dynamic truncation" (take 4 bytes starting at an offset derived from the last nibble of the digest) to turn the HMAC output into a 6-digit number. `totp()` is just HOTP where the counter is the current Unix time divided by the 30-second period, which is what makes the code change automatically every 30 seconds without either side needing to track a counter. `verify_totp()` checks not just the current time-step but also `window` steps before and after it, so a user's phone clock or server clock being a few seconds to a minute out of sync doesn't cause valid codes to be rejected. `provisioning_uri()` builds the standard `otpauth://totp/...` URI format that every major authenticator app can scan or import.

```bash
# Enroll a new user, printing the secret and a scannable otpauth:// URI
python3 mfa_totp.py setup alice@example.com --issuer "Acme Corp"

# Same, but also render a scannable QR code in the terminal (needs `pip install qrcode[pil]`)
python3 mfa_totp.py setup alice@example.com --issuer "Acme Corp" --qr

# Generate the current 6-digit code for a known secret (e.g. for testing)
python3 mfa_totp.py generate JBSWY3DPEHPK3PXP

# Verify a code a user typed in against their stored secret
python3 mfa_totp.py verify JBSWY3DPEHPK3PXP 123456
```

#### Variations / flags
- Add a `--counter-file` mode implementing full HOTP (event-based, not time-based) for hardware tokens that increment a counter on each button press instead of using the clock.
- Add replay protection by persisting the last-accepted counter per user and rejecting any code at or before that counter, closing the "same code used twice within 30 seconds" gap.
- Add backup/recovery codes: generate 10 single-use random codes at enrollment time (using the same `secrets` module as 5.6/5.7) as a fallback if the user loses their authenticator device.

#### Common pitfalls
- **Server/client clock drift**: TOTP is entirely time-based, so if the server's clock is off by more than the `--window` tolerance, correct codes will be rejected. Keep servers synced via NTP; this is the single most common real-world MFA support issue.
- **Base32 padding**: some authenticator apps display or export secrets without the trailing `=` padding base32 normally requires; `hotp()` re-pads before decoding, but if you're hand-typing a secret from elsewhere, don't add extra padding characters yourself.
- **Reusing a code twice in the same 30-second window**: this script's `verify_totp()` doesn't prevent a captured code from being replayed until it naturally expires — for anything beyond a demo/lab setting, add the counter-tracking replay protection described above.
- **Treating the secret like a password you can "reset via email"**: unlike a password, there's no way to "forget" a TOTP secret and have the user re-derive it — losing it means re-enrollment (scanning a new QR code) is required, so plan your account-recovery flow accordingly.
