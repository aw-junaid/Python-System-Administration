## Chapter 8: Web Operations

Almost every sysadmin, DevOps engineer, or automation script eventually has to talk to the web. Whether it's pulling a report from a REST API, checking that a production site is still alive at 3 a.m., archiving a page before a client redesign, or logging into a portal to submit a form on someone's behalf — these are the bread-and-butter tasks that separate a script that "works on my machine" from one that reliably runs unattended on a server or in a cron job.

This chapter is a practical toolbox of small, self-contained Python scripts for interacting with the web: opening pages, downloading and uploading files, calling REST APIs, authenticating against them, parsing the JSON and XML they return, scraping HTML, watching pages for changes, automating a real browser, and managing cookies and sessions across multiple requests. Each script is deliberately narrow in scope — one job, done well, with clear command-line flags — so you can drop it straight into a larger automation pipeline or use it as a starting point for something more specific to your environment.

#### What You'll Learn

By the end of this chapter you will be able to:

- Send and inspect raw HTTP requests (GET, POST, PUT, DELETE, PATCH) from the command line
- Authenticate against APIs using API keys, HTTP Basic Auth, and OAuth2/JWT bearer tokens
- Download and upload files, including images and PDFs, safely and efficiently
- Parse JSON and XML responses without crashing on malformed or missing data
- Scrape and monitor websites for content changes
- Automate a real browser with Selenium for pages that require JavaScript rendering
- Persist cookies and sessions across multiple requests, the same way a browser stays "logged in"
- Understand where WebSockets fit in when you need a persistent, two-way connection instead of one-off requests

#### Core Libraries Used in This Chapter

###### requests
The workhorse HTTP client for nearly every script in this chapter. Simple, synchronous, and battle-tested. Install with `pip install requests`.

###### beautifulsoup4
Used for HTML parsing and web scraping in section 8.11. Install with `pip install beautifulsoup4`.

###### selenium + webdriver-manager
Used for full browser automation in section 8.13, needed whenever a page relies on JavaScript to render content that a plain HTTP GET won't return. Install with `pip install selenium webdriver-manager`.

###### xml.etree.ElementTree
Part of the Python standard library — no install needed for basic XML parsing (section 8.8).

###### websockets
Used for the persistent, bidirectional connections covered in section 8.19. Install with `pip install websockets`.


#### General Cautions Before You Start

###### Respect robots.txt and Terms of Service
Several scripts in this chapter (especially the web scraper and website monitor) can technically hit any URL you give them. That doesn't mean you should. Always check a site's `robots.txt` and Terms of Service before scraping or repeatedly polling it, and respect any rate limits it publishes.

###### Never hard-code credentials
API keys, passwords, and tokens should come from environment variables, a secrets manager, or command-line flags at runtime — never committed into a script or checked into version control.

###### Timeouts are not optional
Every network call in this chapter uses an explicit `timeout` value. A script without a timeout can hang forever if a remote server stops responding, which is especially dangerous in unattended cron jobs.

###### Handle failure, not just success
Production web scripts spend most of their code handling the unhappy path — connection errors, timeouts, non-2xx status codes, malformed responses. Every script in this chapter wraps its network calls in explicit exception handling for exactly this reason.

### 8.1 Open a Web Page

#### Use Case
A sysadmin writing a deployment script wants to automatically pop open a monitoring dashboard, a freshly generated report, or a login page in the user's default browser right after a task finishes, instead of asking the user to go find the URL themselves.

#### Prerequisites

```bash
# No third-party packages required — webbrowser is part of the Python standard library.
# Just make sure you have Python 3 installed:
python3 --version
```

This script works on Linux, macOS, and Windows, but only on machines with a graphical desktop and a registered default browser. It will fail silently (or print a warning) on a headless server or bare container with no display.

#### Cautions
This only works where a graphical browser is available — don't expect it to succeed over SSH into a headless server. It also trusts whatever URL you pass it, so validate input if the URL comes from an untrusted source.

#### Script

```python
#!/usr/bin/env python3
"""
open_webpage.py
-------------------
Opens a URL in the system's default web browser using the built-in
`webbrowser` module. Useful for automation scripts that need to pop
open a dashboard, report, or login page for the user.

Usage:
    python3 open_webpage.py --url https://example.com
    python3 open_webpage.py --url https://example.com --new-tab
"""

import argparse
import webbrowser


def open_page(url, new_tab=False):
    if not url.startswith(("http://", "https://")):
        url = "https://" + url

    if new_tab:
        opened = webbrowser.open_new_tab(url)
    else:
        opened = webbrowser.open(url)

    if opened:
        print(f"Opened in default browser: {url}")
    else:
        print(f"Could not open a browser for: {url}")
        print("Note: this only works on machines with a graphical browser available "
              "(headless servers/containers will fail here).")


def main():
    parser = argparse.ArgumentParser(description="Open a URL in the default web browser")
    parser.add_argument("--url", required=True, help="URL to open")
    parser.add_argument("--new-tab", action="store_true", help="Open in a new browser tab")
    args = parser.parse_args()

    open_page(args.url, args.new_tab)


if __name__ == "__main__":
    main()
```

#### How It Works
The script relies on Python's built-in `webbrowser` module, which looks up the operating system's registered default browser and asks it to load a URL. If the URL is missing its scheme (`http://` or `https://`), the script prepends `https://` automatically so a lazy input like `example.com` still works. The `--new-tab` flag switches from `webbrowser.open()` (which may reuse an existing window) to `webbrowser.open_new_tab()`.

Run it like this:

```bash
# Open in the default window/tab behavior
python3 open_webpage.py --url https://example.com

# Force a new browser tab
python3 open_webpage.py --url example.com --new-tab
```

#### Sample Output

```bash
$ python3 open_webpage.py --url https://example.com
Opened in default browser: https://example.com

$ python3 open_webpage.py --url internal-dashboard.local --new-tab
Opened in default browser: https://internal-dashboard.local
```

On a headless machine:

```bash
$ python3 open_webpage.py --url https://example.com
Could not open a browser for: https://example.com
Note: this only works on machines with a graphical browser available (headless servers/containers will fail here).
```

#### Variations / Flags
- Add a `--browser firefox` flag using `webbrowser.get("firefox")` to force a specific browser instead of the OS default.
- Wrap the call in a `try/except webbrowser.Error` block to catch environments with no browser controller registered at all.
- Chain it at the end of a longer script (e.g., after `download_file.py` finishes generating a report) to auto-open the result.

#### Common Pitfalls
- **No display available**: running this over SSH on a headless server will fail or silently do nothing, since there's no GUI browser to hand the URL to.
- **Missing scheme**: URLs without `http://`/`https://` can be misinterpreted by some browsers; this script guards against that, but double-check custom URLs (e.g., `file://`) behave as expected.
- **Sandboxed containers**: Docker containers typically have no browser at all — this script has nothing to open unless you've explicitly installed and configured one.

### 8.2 Download Files

#### Use Case
A sysadmin needs to fetch a build artifact, a backup archive, or a large log bundle from a remote URL as part of a deployment or maintenance script, without loading the entire file into memory first.

#### Prerequisites

```bash
pip install requests
```

Cross-platform (Linux/macOS/Windows). Requires outbound network access to the target URL.

#### Cautions
Large files downloaded to disk can fill up storage if you don't check available space first, and streaming to an interrupted connection can leave a truncated file behind — always verify file size/checksum for anything critical.

#### Script

```python
#!/usr/bin/env python3
"""
download_file.py
--------------------
Downloads a file from a URL and saves it to disk, streaming the
content so large files don't get loaded fully into memory. Shows a
simple progress indicator. Requires the `requests` library.

Usage:
    python3 download_file.py --url https://example.com/file.zip --output file.zip
"""

import argparse
import os
import sys
import requests


def download_file(url, output_path, chunk_size=8192):
    try:
        with requests.get(url, stream=True, timeout=30) as response:
            response.raise_for_status()
            total_size = int(response.headers.get("content-length", 0))
            downloaded = 0

            os.makedirs(os.path.dirname(output_path) or ".", exist_ok=True)

            with open(output_path, "wb") as f:
                for chunk in response.iter_content(chunk_size=chunk_size):
                    if chunk:
                        f.write(chunk)
                        downloaded += len(chunk)
                        if total_size:
                            percent = downloaded / total_size * 100
                            sys.stdout.write(f"\rDownloading... {percent:.1f}%")
                            sys.stdout.flush()

            print(f"\nDownloaded {downloaded} bytes to: {output_path}")
            return True

    except requests.RequestException as e:
        print(f"\nDownload failed: {e}")
        return False


def main():
    parser = argparse.ArgumentParser(description="Download a file from a URL")
    parser.add_argument("--url", required=True, help="File URL")
    parser.add_argument("--output", required=True, help="Local path to save the file")
    args = parser.parse_args()

    download_file(args.url, args.output)


if __name__ == "__main__":
    main()
```

#### How It Works
The script uses `requests.get(..., stream=True)` so the response body is pulled down in small chunks (`8192` bytes by default) rather than being buffered entirely in memory — essential for multi-gigabyte files. It reads the `Content-Length` header, if the server sends one, to calculate and print a live percentage progress indicator. `response.raise_for_status()` immediately raises an exception on any 4xx/5xx response so a broken link doesn't silently save an error page as your "file."

```bash
# Basic download
python3 download_file.py --url https://example.com/file.zip --output file.zip

# Download into a subdirectory (created automatically)
python3 download_file.py --url https://example.com/backup.tar.gz --output backups/backup.tar.gz
```

#### Sample Output

```bash
$ python3 download_file.py --url https://example.com/file.zip --output file.zip
Downloading... 100.0%
Downloaded 4831201 bytes to: file.zip
```

If the server doesn't send a `Content-Length`, the percentage indicator is skipped but the byte count still prints at the end. On failure:

```bash
$ python3 download_file.py --url https://example.com/missing.zip --output file.zip
Download failed: 404 Client Error: Not Found for url: https://example.com/missing.zip
```

#### Variations / Flags
- Add a `--resume` flag using the `Range` header to continue a partially downloaded file instead of starting over.
- Add a `--checksum sha256:...` flag to verify integrity after download using `hashlib`.
- Wrap this in a retry loop (e.g., using the `tenacity` library) for flaky networks.

#### Common Pitfalls
- **No disk space check**: the script doesn't verify free disk space before writing, so a huge file can fail mid-download and leave a partial file behind.
- **Redirects to HTML error pages**: some servers return a 200 status with an HTML "file not found" page instead of a real 404 — always sanity-check the downloaded file's size/type for critical pipelines.
- **Permissions**: writing to a directory the script's user doesn't have write access to will raise a `PermissionError` — this is not caught separately from other exceptions in the current script.

### 8.3 Upload Files

#### Use Case
A sysadmin needs to push a log bundle, a generated report, or an image asset to a remote API or file-intake endpoint as part of an automated pipeline, the same way a person would use a browser's "Choose File" upload button.

#### Prerequisites

```bash
pip install requests
```

Cross-platform (Linux/macOS/Windows). The target endpoint must accept `multipart/form-data` uploads, which is the standard for HTTP file uploads and what nearly all upload APIs expect.

#### Cautions
Uploading sensitive files (credentials, PII, internal logs) to the wrong endpoint is a real risk — always double-check the target URL, and be aware the whole file is read into the multipart request, so very large files may need chunked/resumable upload APIs instead.

#### Script

```python
#!/usr/bin/env python3
"""
upload_file.py
------------------
Uploads a local file to a remote server via an HTTP POST multipart
form request (the standard way most APIs and web forms accept file
uploads). Requires the `requests` library.

Usage:
    python3 upload_file.py --url https://httpbin.org/post --file report.pdf
    python3 upload_file.py --url https://api.example.com/upload --file image.png --field avatar
"""

import argparse
import os
import requests


def upload_file(url, filepath, field_name="file"):
    if not os.path.isfile(filepath):
        print(f"File not found: {filepath}")
        return False

    try:
        with open(filepath, "rb") as f:
            files = {field_name: (os.path.basename(filepath), f)}
            response = requests.post(url, files=files, timeout=30)

        print(f"Status code: {response.status_code}")
        print(f"Response body (first 500 chars):\n{response.text[:500]}")

        if response.ok:
            print("Upload succeeded.")
            return True
        else:
            print("Upload failed (non-2xx status code).")
            return False

    except requests.RequestException as e:
        print(f"Upload failed: {e}")
        return False


def main():
    parser = argparse.ArgumentParser(description="Upload a file to a server via HTTP POST")
    parser.add_argument("--url", required=True, help="Upload endpoint URL")
    parser.add_argument("--file", required=True, help="Path to the local file to upload")
    parser.add_argument("--field", default="file", help="Form field name expected by the server")
    args = parser.parse_args()

    upload_file(args.url, args.file, args.field)


if __name__ == "__main__":
    main()
```

#### How It Works
The script opens the target file in binary mode and hands it to `requests.post(..., files=...)`, which builds a proper `multipart/form-data` request — identical in shape to what a browser sends when you submit a `<form enctype="multipart/form-data">`. The `field_name` parameter lets you match whatever form field name the target API expects (commonly `file`, but sometimes `avatar`, `attachment`, etc.). A local existence check (`os.path.isfile`) runs first so the script fails fast with a clear message instead of a confusing traceback.

```bash
# Upload using the default "file" field name
python3 upload_file.py --url https://httpbin.org/post --file report.pdf

# Upload using a custom field name required by the API
python3 upload_file.py --url https://api.example.com/upload --file image.png --field avatar
```

#### Sample Output

```bash
$ python3 upload_file.py --url https://httpbin.org/post --file report.pdf
Status code: 200
Response body (first 500 chars):
{
  "args": {},
  "files": {
    "file": "..."
  },
  ...
}
Upload succeeded.
```

```bash
$ python3 upload_file.py --url https://api.example.com/upload --file missing.pdf
File not found: missing.pdf
```

#### Variations / Flags
- Add `--headers '{"Authorization": "Bearer TOKEN"}'` support for endpoints that require authentication (combine with section 8.6).
- Add extra form fields alongside the file (e.g., `data={"description": "monthly report"}`) for APIs that expect metadata with the upload.
- Add a progress callback for large files using a streaming multipart encoder such as `requests-toolbelt`.

#### Common Pitfalls
- **Wrong field name**: many APIs silently reject or ignore an upload if the multipart field name doesn't match exactly what they expect — check the API docs, not just the flag default.
- **File handle left open on error**: because the file is opened with a `with` block, this is handled correctly here, but custom modifications should preserve that pattern.
- **Large files timing out**: the default 30-second timeout may be too short for large uploads on slow connections; increase it or switch to a resumable/chunked upload strategy for anything beyond a few dozen megabytes.

### 8.4 Send HTTP Requests

#### Use Case
A sysadmin needs a quick command-line tool to poke an endpoint with an arbitrary HTTP method, custom headers, and a request body — useful for debugging APIs, testing webhooks, or scripting one-off requests without reaching for `curl` syntax.

#### Prerequisites

```bash
pip install requests
```

Cross-platform (Linux/macOS/Windows). Standard library `json` module is also used but requires no install.

#### Cautions
This script prints full response bodies (up to 1000 characters) to stdout — be careful running it against endpoints that return sensitive data in logs or CI output that others can see.

#### Script

```python
#!/usr/bin/env python3
"""
http_requests.py
--------------------
A generic HTTP client demonstrating GET, POST, PUT, DELETE requests
with custom headers using the `requests` library. Useful as a quick
command-line HTTP testing tool.

Usage:
    python3 http_requests.py --method GET --url https://example.com
    python3 http_requests.py --method POST --url https://example.com/api --data '{"key": "value"}'
    python3 http_requests.py --method GET --url https://example.com --headers '{"User-Agent": "MyBot/1.0"}'
"""

import argparse
import json
import requests


def send_request(method, url, data=None, headers=None):
    method = method.upper()

    try:
        response = requests.request(
            method=method,
            url=url,
            data=data,
            headers=headers,
            timeout=15,
        )

        print(f"Status code: {response.status_code}")
        print(f"Response headers: {dict(response.headers)}")
        print(f"\nResponse body (first 1000 chars):\n{response.text[:1000]}")
        return response

    except requests.RequestException as e:
        print(f"Request failed: {e}")
        return None


def main():
    parser = argparse.ArgumentParser(description="Send an HTTP request")
    parser.add_argument("--method", default="GET", choices=["GET", "POST", "PUT", "DELETE", "PATCH"])
    parser.add_argument("--url", required=True, help="Target URL")
    parser.add_argument("--data", help="Request body as a JSON string")
    parser.add_argument("--headers", help="Request headers as a JSON string")
    args = parser.parse_args()

    headers = json.loads(args.headers) if args.headers else None
    data = args.data if args.data else None

    send_request(args.method, args.url, data=data, headers=headers)


if __name__ == "__main__":
    main()
```

#### How It Works
The script wraps `requests.request()`, which is the low-level method that GET/POST/PUT/DELETE all call under the hood in the `requests` library — passing `method` as a string lets one function handle every HTTP verb. Headers and body data are supplied as JSON strings on the command line and parsed with `json.loads` before being handed to the request. A 15-second timeout prevents the script from hanging indefinitely on an unresponsive server.

```bash
# Simple GET
python3 http_requests.py --method GET --url https://example.com

# POST with a JSON body
python3 http_requests.py --method POST --url https://example.com/api --data '{"key": "value"}'

# GET with a custom header
python3 http_requests.py --method GET --url https://example.com --headers '{"User-Agent": "MyBot/1.0"}'
```

#### Sample Output

```bash
$ python3 http_requests.py --method GET --url https://httpbin.org/get
Status code: 200
Response headers: {'Content-Type': 'application/json', 'Content-Length': '312', ...}

Response body (first 1000 chars):
{
  "args": {},
  "headers": {...},
  "origin": "203.0.113.5",
  "url": "https://httpbin.org/get"
}
```

#### Variations / Flags
- Add a `--json` flag (distinct from `--data`) that sets `Content-Type: application/json` automatically and passes the payload via `requests`' `json=` parameter instead of raw `data=`.
- Add `--output response.json` to save the full body to disk instead of truncating at 1000 characters.
- Add `--verbose` to also print the request headers that were actually sent, useful for debugging auth issues.

#### Common Pitfalls
- **Malformed `--data`/`--headers` JSON**: `json.loads` will raise a `json.JSONDecodeError` that isn't caught here, producing a raw traceback instead of a friendly message — validate your JSON string first (e.g., with `python3 -m json.tool`).
- **Sending `--data` without `Content-Type`**: raw `data=` doesn't set `Content-Type: application/json` automatically, so some APIs may reject the body as malformed even though it looks like JSON.
- **Truncated output**: the 1000-character cutoff can hide the actual error detail in a longer JSON error response — increase the slice or add a `--output` flag for full inspection.

### 8.5 Perform REST API Calls

#### Use Case
A sysadmin or backend engineer needs to interact with a JSON-based REST service — creating a ticket, fetching a resource, updating a record — from a script, with proper JSON request/response handling instead of raw text manipulation.

#### Prerequisites

```bash
pip install requests
```

Cross-platform (Linux/macOS/Windows). Requires network access to the target API; some real-world APIs will also require authentication (see section 8.6).

#### Cautions
Non-idempotent verbs like POST, PUT, and DELETE can create, modify, or destroy real data — always test against a staging/sandbox endpoint (like the public `jsonplaceholder.typicode.com` used by default here) before pointing this at production.

#### Script

```python
#!/usr/bin/env python3
"""
rest_api_call.py
---------------------
Demonstrates calling a REST API (GET/POST/PUT/DELETE) with a JSON
body and JSON response handling — the typical shape of interacting
with a REST service. Uses https://jsonplaceholder.typicode.com as a
free public test API by default.

Usage:
    python3 rest_api_call.py --method GET --url https://jsonplaceholder.typicode.com/posts/1
    python3 rest_api_call.py --method POST --url https://jsonplaceholder.typicode.com/posts \
        --json '{"title": "foo", "body": "bar", "userId": 1}'
"""

import argparse
import json
import requests


def call_rest_api(method, url, json_body=None):
    try:
        response = requests.request(
            method=method.upper(),
            url=url,
            json=json_body,
            headers={"Content-Type": "application/json"},
            timeout=15,
        )
        response.raise_for_status()

        print(f"Status code: {response.status_code}")

        try:
            data = response.json()
            print("Response JSON:")
            print(json.dumps(data, indent=2)[:1500])
            return data
        except ValueError:
            print("Response was not valid JSON:")
            print(response.text[:500])
            return None

    except requests.RequestException as e:
        print(f"API call failed: {e}")
        return None


def main():
    parser = argparse.ArgumentParser(description="Perform a REST API call")
    parser.add_argument("--method", default="GET", choices=["GET", "POST", "PUT", "DELETE", "PATCH"])
    parser.add_argument(
        "--url",
        default="https://jsonplaceholder.typicode.com/posts/1",
        help="REST API endpoint URL",
    )
    parser.add_argument("--json", help="JSON request body as a string, e.g. '{\"key\": \"value\"}'")
    args = parser.parse_args()

    json_body = json.loads(args.json) if args.json else None
    call_rest_api(args.method, args.url, json_body)


if __name__ == "__main__":
    main()
```

#### How It Works
Unlike the generic HTTP client in section 8.4, this script uses `requests`' `json=` parameter, which automatically serializes a Python dict to a JSON string *and* sets the `Content-Type: application/json` header correctly — the proper way to talk to REST APIs. `response.raise_for_status()` turns any 4xx/5xx response into an exception immediately, and the response body is parsed back into Python with `response.json()`, falling back gracefully to raw text if the server didn't actually return JSON.

```bash
# Fetch a single resource
python3 rest_api_call.py --method GET --url https://jsonplaceholder.typicode.com/posts/1

# Create a new resource
python3 rest_api_call.py --method POST --url https://jsonplaceholder.typicode.com/posts \
    --json '{"title": "foo", "body": "bar", "userId": 1}'
```

#### Sample Output

```bash
$ python3 rest_api_call.py --method GET --url https://jsonplaceholder.typicode.com/posts/1
Status code: 200
Response JSON:
{
  "userId": 1,
  "id": 1,
  "title": "sunt aut facere repellat provident occaecati excepturi optio reprehenderit",
  "body": "quia et suscipit\nsuscipit recusandae consequuntur expedita et cum..."
}
```

```bash
$ python3 rest_api_call.py --method POST --url https://jsonplaceholder.typicode.com/posts \
    --json '{"title": "foo", "body": "bar", "userId": 1}'
Status code: 201
Response JSON:
{
  "title": "foo",
  "body": "bar",
  "userId": 1,
  "id": 101
}
```

#### Variations / Flags
- Add a `--query` flag to pass URL query parameters via `params=` instead of hand-building the URL string.
- Add pagination support (`--page`, `--per-page`) for APIs that paginate list endpoints.
- Add automatic retry with exponential backoff for `429 Too Many Requests` responses, respecting any `Retry-After` header.

#### Common Pitfalls
- **Confusing `--data` and `--json`**: this script only accepts `--json`; passing a raw string without valid JSON syntax will raise a `json.JSONDecodeError` before the request is even sent.
- **Assuming every endpoint returns JSON**: some error pages return HTML even from a JSON API (e.g., a gateway timeout page) — the fallback to raw text handles this, but downstream code consuming the return value should check for `None`.
- **Testing destructive calls on production**: always confirm the target host before running `--method DELETE` against anything other than a test/sandbox API.

### 8.6 Authenticate with APIs

#### Use Case
A sysadmin needs to call a protected API — a cloud provider's control plane, an internal service, a SaaS integration — and must send credentials using whatever authentication scheme that API expects: an API key, a username/password pair, or an OAuth2/JWT bearer token.

#### Prerequisites

```bash
pip install requests
```

Cross-platform (Linux/macOS/Windows). You will need valid credentials for whichever API you're targeting; this script demonstrates the mechanics, not credential issuance.

#### Cautions
Credentials should always be read from environment variables or a secrets manager, never hard-coded into a script or committed to version control. HTTP Basic Auth sends credentials base64-encoded, not encrypted — always use HTTPS, never plain HTTP, when sending any of these auth types.

#### Script

```python
#!/usr/bin/env python3
"""
api_authentication.py
--------------------------
Demonstrates three common API authentication patterns:
  1. API Key (in headers)
  2. Basic Auth (username/password)
  3. Bearer Token (OAuth2 / JWT style)

Credentials are read from environment variables or command-line
flags - never hard-code real credentials into a script you commit
to version control.

Usage:
    # API key auth
    python3 api_authentication.py --mode apikey --url https://api.example.com/data --api-key YOUR_KEY

    # Basic auth
    python3 api_authentication.py --mode basic --url https://api.example.com/data --username user --password pass

    # Bearer token auth
    python3 api_authentication.py --mode bearer --url https://api.example.com/data --token YOUR_TOKEN
"""

import argparse
import requests


def auth_with_api_key(url, api_key, header_name="X-API-Key"):
    headers = {header_name: api_key}
    response = requests.get(url, headers=headers, timeout=15)
    return response


def auth_with_basic(url, username, password):
    response = requests.get(url, auth=(username, password), timeout=15)
    return response


def auth_with_bearer_token(url, token):
    headers = {"Authorization": f"Bearer {token}"}
    response = requests.get(url, headers=headers, timeout=15)
    return response


def main():
    parser = argparse.ArgumentParser(description="Demonstrate common API authentication methods")
    parser.add_argument("--mode", required=True, choices=["apikey", "basic", "bearer"])
    parser.add_argument("--url", required=True, help="API endpoint URL")
    parser.add_argument("--api-key", help="API key (for --mode apikey)")
    parser.add_argument("--username", help="Username (for --mode basic)")
    parser.add_argument("--password", help="Password (for --mode basic)")
    parser.add_argument("--token", help="Bearer token (for --mode bearer)")
    args = parser.parse_args()

    try:
        if args.mode == "apikey":
            if not args.api_key:
                print("--api-key is required for --mode apikey")
                return
            response = auth_with_api_key(args.url, args.api_key)

        elif args.mode == "basic":
            if not (args.username and args.password):
                print("--username and --password are required for --mode basic")
                return
            response = auth_with_basic(args.url, args.username, args.password)

        else:  # bearer
            if not args.token:
                print("--token is required for --mode bearer")
                return
            response = auth_with_bearer_token(args.url, args.token)

        print(f"Status code: {response.status_code}")
        if response.status_code == 200:
            print("Authenticated successfully.")
        elif response.status_code in (401, 403):
            print("Authentication failed - check your credentials.")
        print(f"Response body (first 500 chars):\n{response.text[:500]}")

    except requests.RequestException as e:
        print(f"Request failed: {e}")


if __name__ == "__main__":
    main()
```

#### How It Works
Each authentication mode builds the request differently, matching how real APIs commonly expect credentials:
- **API key** — passed as a custom header (commonly `X-API-Key`, but this varies by provider).
- **Basic Auth** — `requests` accepts an `auth=(username, password)` tuple and handles the base64 encoding of the `Authorization: Basic ...` header internally.
- **Bearer token** — the standard OAuth2/JWT pattern, sent as `Authorization: Bearer <token>`.

The script checks the response status code and gives a plain-English hint for `401`/`403` responses, which almost always mean bad or expired credentials.

```bash

# Basic auth (read password from env var, not hard-coded)
python3 api_authentication.py --mode basic --url https://api.example.com/data --username admin --password "$API_PASSWORD"

# Bearer token auth
python3 api_authentication.py --mode bearer --url https://api.example.com/data --token "$ACCESS_TOKEN"
```


#### Variations / Flags
- Add an `--oauth2-refresh` mode that exchanges a refresh token for a fresh access token before making the request, for APIs with short-lived bearer tokens.
- Read credentials exclusively from environment variables (e.g., `os.environ["API_KEY"]`) rather than accepting them as CLI flags, to avoid credentials showing up in shell history or `ps` output.
- Add a `--header-name` flag to support APIs that use a non-standard header name for API keys (e.g., `Authorization: Apikey ...` instead of `X-API-Key`).

#### Common Pitfalls
- **Credentials in shell history**: passing `--password` or `--api-key` directly on the command line leaves them visible in shell history and `ps aux` output — prefer environment variables for anything real.
- **Wrong header name for API keys**: unlike Basic Auth and Bearer tokens, there's no universal standard for API key headers — always check the specific API's documentation (`X-API-Key`, `Api-Key`, `apikey`, etc. all appear in the wild).
- **Expired tokens**: bearer tokens typically expire; a script that worked yesterday may start failing with `401` today purely because the token needs to be refreshed, not because credentials are wrong.

### 8.7 Parse JSON Responses

#### Use Case
A sysadmin pulls a JSON API response (or a local JSON config/log file) and needs to safely navigate its structure — lists, nested objects, and possibly-missing fields — without the script crashing the moment a field is absent.

#### Prerequisites

```bash
pip install requests
```

The `json` module used for parsing is part of the Python standard library. Cross-platform (Linux/macOS/Windows).

#### Cautions
Never assume a JSON API's structure is guaranteed to stay the same between calls — always use safe accessors like `.get()` rather than direct key indexing (`data["key"]`) in production code, or a missing field will raise a `KeyError` and crash the script.

#### Script

```python
#!/usr/bin/env python3
"""
parse_json.py
------------------
Fetches JSON from a URL (or reads a local JSON file) and demonstrates
parsing/navigating it: accessing keys, iterating lists, and handling
missing fields safely.

Usage:
    python3 parse_json.py --url https://jsonplaceholder.typicode.com/users
    python3 parse_json.py --file data.json
"""

import argparse
import json
import requests


def get_json_data(url=None, filepath=None):
    if url:
        response = requests.get(url, timeout=15)
        response.raise_for_status()
        return response.json()
    elif filepath:
        with open(filepath, "r") as f:
            return json.load(f)
    else:
        raise ValueError("Either url or filepath must be provided")


def explore_json(data):
    print(f"Top-level type: {type(data).__name__}\n")

    if isinstance(data, list):
        print(f"List with {len(data)} item(s). Showing first item:")
        if data:
            print(json.dumps(data[0], indent=2)[:800])
            print(f"\nExample - safely accessing a field from item 0:")
            first = data[0]
            if isinstance(first, dict):
                # .get() avoids KeyError if the field doesn't exist
                print(f"  name/title: {first.get('name', first.get('title', 'N/A'))}")

    elif isinstance(data, dict):
        print("Top-level keys:", list(data.keys()))
        print("\nFull structure (first 800 chars):")
        print(json.dumps(data, indent=2)[:800])

    else:
        print("Value:", data)


def main():
    parser = argparse.ArgumentParser(description="Fetch and parse a JSON response")
    parser.add_argument("--url", help="URL returning JSON")
    parser.add_argument("--file", help="Local JSON file path")
    args = parser.parse_args()

    if not args.url and not args.file:
        print("Provide either --url or --file")
        return

    try:
        data = get_json_data(url=args.url, filepath=args.file)
        explore_json(data)
    except (requests.RequestException, json.JSONDecodeError, FileNotFoundError) as e:
        print(f"Failed to fetch/parse JSON: {e}")


if __name__ == "__main__":
    main()
```

#### How It Works
`get_json_data()` fetches JSON from either a live URL (using `response.json()`, which parses the body automatically) or a local file (using `json.load()`), giving the rest of the script one unified `data` structure to work with either way. `explore_json()` then branches on whether the top level is a list or a dict, since JSON APIs commonly return either shape, and demonstrates the critical habit of using `.get("field", default)` instead of `data["field"]` so a missing key returns a fallback value instead of raising an exception.

```bash
# Parse JSON from a live API
python3 parse_json.py --url https://jsonplaceholder.typicode.com/users

# Parse a local JSON file
python3 parse_json.py --file data.json
```

#### Sample Output

```bash
$ python3 parse_json.py --url https://jsonplaceholder.typicode.com/users
Top-level type: list

List with 10 item(s). Showing first item:
{
  "id": 1,
  "name": "Leanne Graham",
  "username": "Bret",
  "email": "Sincere@april.biz",
  ...
}

Example - safely accessing a field from item 0:
  name/title: Leanne Graham
```

```bash
$ python3 parse_json.py --file nonexistent.json

```

#### Variations / Flags
- Add a `--path users.0.address.city` flag using `jmespath` or `jsonpath-ng` for deep, dotted-path field extraction without writing custom navigation code.
- Add `--pretty` / `--compact` output flags to control `json.dumps` indentation for piping into other tools.
- Add schema validation with the `jsonschema` library to catch structural drift from an API before it breaks downstream processing.

#### Common Pitfalls
- **Direct key indexing**: `data["field"]` crashes with a `KeyError` the moment an API omits an optional field — always default to `.get()` in scripts that run unattended.
- **Assuming top-level shape**: some APIs wrap list results in an object (e.g., `{"results": [...], "page": 1}`) rather than returning a bare list — check the actual response shape before writing parsing logic.
- **Truncated preview output**: this script's `[:800]` character preview is for readability only — don't mistake it for the full response when debugging a real issue.

### 8.8 Parse XML Responses

#### Use Case
A sysadmin needs to consume a legacy SOAP API, an RSS/Atom feed, or an XML-based config export, and extract specific elements and attributes without pulling in a heavyweight dependency.

#### Prerequisites

```bash
pip install requests
```

`xml.etree.ElementTree` is part of the Python standard library — no extra install needed for basic XML parsing. Cross-platform (Linux/macOS/Windows).

#### Cautions
Python's built-in XML parser does not protect against maliciously crafted XML (e.g., "billion laughs" entity expansion attacks) — for XML from untrusted sources, use the `defusedxml` package instead of raw `xml.etree.ElementTree`.

#### Script

```python
#!/usr/bin/env python3
"""
parse_xml.py
-----------------
Fetches XML from a URL (or reads a local XML file) and demonstrates
parsing it using Python's built-in `xml.etree.ElementTree` - no
extra dependency needed for basic XML parsing.

Usage:
    python3 parse_xml.py --url https://www.w3schools.com/xml/note.xml
    python3 parse_xml.py --file data.xml
"""

import argparse
import xml.etree.ElementTree as ET
import requests


def get_xml_root(url=None, filepath=None):
    if url:
        response = requests.get(url, timeout=15)
        response.raise_for_status()
        return ET.fromstring(response.content)
    elif filepath:
        tree = ET.parse(filepath)
        return tree.getroot()
    else:
        raise ValueError("Either url or filepath must be provided")


def explore_xml(root, max_children=10):
    print(f"Root tag: <{root.tag}>")
    print(f"Root attributes: {root.attrib}\n")

    print("Children of root:")
    for i, child in enumerate(root):
        if i >= max_children:
            print(f"  ... ({len(root) - max_children} more)")
            break
        text = (child.text or "").strip()
        print(f"  <{child.tag}> attrib={child.attrib} text='{text[:60]}'")

    print("\nExample - find all elements matching a tag name:")
    if len(root) > 0:
        first_tag = root[0].tag
        matches = root.findall(f".//{first_tag}")
        print(f"  Found {len(matches)} element(s) with tag <{first_tag}>")


def main():
    parser = argparse.ArgumentParser(description="Fetch and parse an XML response")
    parser.add_argument("--url", help="URL returning XML")
    parser.add_argument("--file", help="Local XML file path")
    args = parser.parse_args()

    if not args.url and not args.file:
        print("Provide either --url or --file")
        return

    try:
        root = get_xml_root(url=args.url, filepath=args.file)
        explore_xml(root)
    except (requests.RequestException, ET.ParseError, FileNotFoundError) as e:
        print(f"Failed to fetch/parse XML: {e}")


if __name__ == "__main__":
    main()
```

#### How It Works
`get_xml_root()` fetches raw bytes from a URL and hands them to `ET.fromstring()`, or parses a local file directly with `ET.parse()`, returning the root `Element` of the XML document tree either way. `explore_xml()` then demonstrates the core navigation patterns: reading a tag's name and attributes, iterating its direct children, and using `.findall(".//tagname")` — an XPath-lite expression meaning "find this tag anywhere in the subtree" — to locate all matching elements regardless of nesting depth.

```bash
# Parse XML from a live URL
python3 parse_xml.py --url https://www.w3schools.com/xml/note.xml

# Parse a local XML file
python3 parse_xml.py --file data.xml
```

#### Sample Output

```bash
$ python3 parse_xml.py --url https://www.w3schools.com/xml/note.xml
Root tag: <note>
Root attributes: {}

Children of root:
  <to> attrib={} text='Tove'
  <from> attrib={} text='Jani'
  <heading> attrib={} text='Reminder'
  <body> attrib={} text="Don't forget me this weekend!"

Example - find all elements matching a tag name:
  Found 1 element(s) with tag <to>
```

```bash
$ python3 parse_xml.py --file broken.xml
Failed to fetch/parse XML: syntax error: line 3, column 0
```

#### Variations / Flags
- Add a `--xpath ".//item[@id='42']"` flag to run arbitrary XPath queries directly against the parsed tree.
- Swap in `defusedxml.ElementTree` instead of the standard library module when parsing XML from any source you don't fully trust.
- Add namespace handling (`{namespace}tag` syntax or an explicit `namespaces={}` dict) for XML documents that use XML namespaces, which trips up naive `.find()`/`.findall()` calls.

#### Common Pitfalls
- **XML namespaces**: many real-world XML documents (RSS feeds, SOAP responses) use namespaces, which change how tags must be referenced in `.find()`/`.findall()` — a tag that looks like `<title>` in the raw text may actually need to be queried as `{http://...}title`.
- **Untrusted XML sources**: the standard library's `ElementTree` is explicitly documented as not secure against maliciously crafted XML — use `defusedxml` for anything not from a fully trusted source.
- **Encoding mismatches**: passing `response.text` (a decoded string) instead of `response.content` (raw bytes) to `ET.fromstring()` can cause encoding-declaration conflicts on documents with a non-UTF-8 `<?xml encoding="...">` header.

### 8.9 Download Web Pages

#### Use Case
A sysadmin needs to archive a page's raw HTML for offline analysis, compliance record-keeping, or as an input to a separate parsing/scraping step run later — without immediately extracting structured data from it (see section 8.11 for that).

#### Prerequisites

```bash
pip install requests
```

Cross-platform (Linux/macOS/Windows). Requires network access to the target page.

#### Cautions
Saving raw HTML captures only the server-rendered markup — pages that build their content with JavaScript after load will save as a nearly empty shell (use section 8.13's browser automation for those instead).

#### Script

```python
#!/usr/bin/env python3
"""
download_webpage.py
------------------------
Downloads the raw HTML of a web page and saves it to a local file -
useful for archiving, offline analysis, or feeding into a separate
parsing step. Requires the `requests` library.

Usage:
    python3 download_webpage.py --url https://example.com --output page.html
"""

import argparse
import requests


def download_webpage(url, output_path):
    try:
        headers = {"User-Agent": "Mozilla/5.0 (compatible; WebOpsBot/1.0)"}
        response = requests.get(url, headers=headers, timeout=15)
        response.raise_for_status()

        with open(output_path, "w", encoding="utf-8") as f:
            f.write(response.text)

        print(f"Saved {len(response.text)} characters of HTML to: {output_path}")
        print(f"Status code: {response.status_code}")
        return True

    except requests.RequestException as e:
        print(f"Failed to download page: {e}")
        return False


def main():
    parser = argparse.ArgumentParser(description="Download a web page's HTML to a local file")
    parser.add_argument("--url", required=True, help="Web page URL")
    parser.add_argument("--output", default="page.html", help="Output HTML file path")
    args = parser.parse_args()

    download_webpage(args.url, args.output)


if __name__ == "__main__":
    main()
```

#### How It Works
The script sends a GET request with a realistic browser-style `User-Agent` header, since some servers block or serve degraded content to requests that identify as a generic script/bot. `response.text` (already decoded to a Python string using the encoding detected from response headers) is written straight to a local `.html` file with explicit UTF-8 encoding, avoiding the mangled-character issues that can occur with platform-default encodings on Windows.

```bash
# Save with the default filename
python3 download_webpage.py --url https://example.com

# Save with a custom output path
python3 download_webpage.py --url https://example.com --output archive/example-2026-07-17.html
```

#### Sample Output

```bash

Status code: 200
```


#### Variations / Flags
- Add a `--render-js` flag that switches to Selenium (section 8.13) for pages requiring JavaScript execution before the meaningful HTML exists.
- Add `--timestamp-filename` to auto-append the current date/time to the output path for repeated archival snapshots of the same page.
- Add gzip/brotli decompression handling verification — `requests` handles this automatically, but confirm with `response.headers.get("Content-Encoding")` if debugging odd output.

#### Common Pitfalls
- **JavaScript-rendered pages**: single-page applications (React, Vue, Angular) often deliver an almost-empty `<div id="root"></div>` in the raw HTML — the actual content only appears after JavaScript runs in a real browser, which this script does not execute.
- **Missing User-Agent leads to blocks**: some sites return a 403 or a CAPTCHA challenge page to requests without a browser-like `User-Agent` — this script sets one by default, but some sites require more (cookies, JS challenges) to pass through.
- **Overwriting without warning**: the script overwrites `--output` silently if the file already exists — add an existence check first if you need to preserve prior archives.

### 8.10 Check Website Availability

#### Use Case
A sysadmin needs a simple uptime check for a website or API endpoint — the kind of thing you'd run from cron every few minutes to alert if a production service goes down, or to sanity-check a host before running a deeper healthcheck.

#### Prerequisites

```bash
pip install requests
```

Cross-platform (Linux/macOS/Windows). No special permissions needed beyond outbound network access.

#### Cautions
A successful HTTP response doesn't guarantee the application itself is healthy — a server can return `200 OK` on a generic landing page while the actual service behind it is broken; pair this with an application-level healthcheck endpoint where possible.

#### Script

```python
#!/usr/bin/env python3
"""
check_website.py
---------------------
Checks whether a website is up, and reports its HTTP status code and
response time. Useful as a simple uptime check, e.g. run from cron.

Usage:
    python3 check_website.py --url https://example.com
    python3 check_website.py --url https://example.com --timeout 5
"""

import argparse
import sys
import time
import requests


def check_website(url, timeout=10):
    if not url.startswith(("http://", "https://")):
        url = "https://" + url

    try:
        start = time.time()
        response = requests.get(url, timeout=timeout)
        elapsed = time.time() - start

        print(f"URL: {url}")
        print(f"Status code: {response.status_code}")
        print(f"Response time: {elapsed:.3f} seconds")

        if response.ok:
            print("[UP] Website is reachable and returned a success status code.")
            return True
        else:
            print(f"[WARNING] Website responded but with a non-success status code: {response.status_code}")
            return False

    except requests.ConnectionError:
        print(f"[DOWN] Could not connect to {url}")
        return False
    except requests.Timeout:
        print(f"[DOWN] Request to {url} timed out after {timeout} seconds")
        return False
    except requests.RequestException as e:
        print(f"[DOWN] Request failed: {e}")
        return False


def main():
    parser = argparse.ArgumentParser(description="Check whether a website is up")
    parser.add_argument("--url", required=True, help="Website URL to check")
    parser.add_argument("--timeout", type=int, default=10, help="Timeout in seconds")
    args = parser.parse_args()

    is_up = check_website(args.url, args.timeout)
    sys.exit(0 if is_up else 1)


if __name__ == "__main__":
    main()
```

#### How It Works
The script times a single GET request using `time.time()` before and after the call, then reports elapsed time alongside the HTTP status code. `response.ok` is a `requests` shortcut that's `True` for any status code under 400. Crucially, it distinguishes between three different failure modes — connection errors (DNS failure, refused connection), timeouts, and other request exceptions — so alerting logic downstream can tell "the server is unreachable" apart from "the server responded but slowly" or "the server responded with an error."

The script exits with status code `0` on success and `1` on failure, following standard shell/cron conventions so it can be used directly in conditional scripts.

```bash
# Basic check
python3 check_website.py --url https://example.com

# Shorter timeout for a stricter check
python3 check_website.py --url https://example.com --timeout 5

# Use in a shell conditional
python3 check_website.py --url https://example.com && echo "site is fine" || echo "ALERT: site is down"
```

#### Sample Output

```bash
$ python3 check_website.py --url https://example.com
URL: https://example.com
Status code: 200
Response time: 0.184 seconds
[UP] Website is reachable and returned a success status code.
```

```bash
$ python3 check_website.py --url https://this-does-not-exist-xyz.com --timeout 5
URL: https://this-does-not-exist-xyz.com
[DOWN] Could not connect to https://this-does-not-exist-xyz.com
```

#### Variations / Flags
- Schedule with cron for periodic checks, e.g. every 5 minutes:

```bash
*/5 * * * * /usr/bin/python3 /opt/scripts/check_website.py --url https://example.com >> /var/log/site_check.log 2>&1
```

- Add a `--notify slack-webhook-url` flag to POST an alert to Slack/PagerDuty/etc. when a check fails, instead of only printing to stdout.
- Add a `--expect-status 200` flag for endpoints where a non-200 "success" code (e.g., a `204 No Content` healthcheck) is actually the expected happy path.

#### Common Pitfalls
- **Redirect chains counted as "up"**: `requests` follows redirects by default, so a domain that redirects to a parked/error page may still report `[UP]` with a 200 from the final destination — check `response.history` if you need to detect redirects.
- **Cron environment differences**: scripts that work fine interactively can fail under cron due to a different `PATH` or missing environment variables — always use full paths and test with `env -i` to simulate cron's minimal environment.
- **Single check false positives/negatives**: one failed check could be a transient network blip rather than real downtime — production monitoring should require multiple consecutive failures before alerting.

### 8.11 Web Scraping

#### Use Case
A sysadmin or data engineer needs to extract structured information — headings, links, prices, table data — from a web page's HTML, rather than just archiving the raw markup or calling a proper API (which may not exist for the target site).

#### Prerequisites

```bash
pip install requests beautifulsoup4
```

Cross-platform (Linux/macOS/Windows).

#### Cautions
Always check a site's `robots.txt` and Terms of Service before scraping, respect rate limits and crawl-delay directives, and only scrape data you are authorized to collect — scraping can violate a site's terms of service or, in some jurisdictions, the law, particularly for content behind a login or paywall.

#### Script

```python
#!/usr/bin/env python3
"""
web_scraper.py
------------------
Scrapes a web page and extracts data using BeautifulSoup: page title,
all links, and all headings. Requires `requests` and `beautifulsoup4`.

IMPORTANT: Always check a site's robots.txt and Terms of Service
before scraping, respect rate limits, and only scrape data you're
authorized to collect.

Usage:
    python3 web_scraper.py --url https://example.com
    python3 web_scraper.py --url https://example.com --selector "h2.title"
"""

import argparse
import requests
from bs4 import BeautifulSoup


def scrape_page(url, css_selector=None):
    headers = {"User-Agent": "Mozilla/5.0 (compatible; WebOpsBot/1.0)"}

    try:
        response = requests.get(url, headers=headers, timeout=15)
        response.raise_for_status()
    except requests.RequestException as e:
        print(f"Failed to fetch page: {e}")
        return

    soup = BeautifulSoup(response.text, "html.parser")

    title = soup.title.string.strip() if soup.title else "(no title found)"
    print(f"Page title: {title}\n")

    if css_selector:
        elements = soup.select(css_selector)
        print(f"Found {len(elements)} element(s) matching selector '{css_selector}':")
        for el in elements[:20]:
            print(f"  - {el.get_text(strip=True)[:100]}")
        return

    # Default: show headings and links
    headings = soup.find_all(["h1", "h2", "h3"])
    print(f"Headings found: {len(headings)}")
    for h in headings[:10]:
        print(f"  <{h.name}> {h.get_text(strip=True)[:80]}")

    links = soup.find_all("a", href=True)
    print(f"\nLinks found: {len(links)}")
    for link in links[:10]:
        print(f"  {link.get_text(strip=True)[:40] or '(no text)'} -> {link['href']}")


def main():
    parser = argparse.ArgumentParser(description="Scrape a web page using BeautifulSoup")
    parser.add_argument("--url", required=True, help="Page URL to scrape")
    parser.add_argument("--selector", help="Optional CSS selector to extract specific elements")
    args = parser.parse_args()

    scrape_page(args.url, args.selector)


if __name__ == "__main__":
    main()
```

#### How It Works
After fetching the raw HTML with `requests`, the script hands it to `BeautifulSoup`, which builds a navigable parse tree out of the markup. `soup.title.string` pulls the page's `<title>` text, `soup.find_all([...])` locates every element matching a list of tag names (here, all heading levels), and `soup.select(css_selector)` supports full CSS selector syntax (e.g., `h2.title`, `div#main > p`) when the caller wants a more specific target than the default "show me headings and links" behavior.

```bash
# Default scrape: title, headings, and links
python3 web_scraper.py --url https://example.com

# Target specific elements with a CSS selector
python3 web_scraper.py --url https://example.com --selector "h2.title"
```

#### Sample Output

```bash
$ python3 web_scraper.py --url https://example.com
Page title: Example Domain

Headings found: 1
  <h1> Example Domain

Links found: 1
```

#### Variations / Flags
- Add a `--delay 2` flag with `time.sleep()` between requests when scraping multiple pages in a loop, to respect the target site's rate limits.
- Add `--output data.csv` to write extracted rows to CSV using the standard library `csv` module.
- Swap `html.parser` for `lxml` (`pip install lxml`) for significantly faster parsing on large pages.

#### Common Pitfalls
- **Ignoring robots.txt**: this script does not check `robots.txt` automatically — that's a manual responsibility before running it against any site you don't own.
- **JavaScript-rendered content**: like `download_webpage.py`, this scrapes only server-rendered HTML — pages that build content client-side will return mostly empty results (use section 8.13's Selenium-based automation instead).
- **Fragile selectors**: CSS selectors and class names on third-party sites change without notice; scrapers built against a specific site's markup should be expected to break and need maintenance over time.
- **Rate limiting / IP bans**: scraping too aggressively (many requests in rapid succession) can get your IP temporarily or permanently blocked — always add delays and respect any published rate limits.

### 8.12 Monitor Website Changes

#### Use Case
A sysadmin wants to be alerted when a specific web page changes — a vendor's pricing page, a job listing board, a compliance/status announcement page — without manually checking it every day.

#### Prerequisites

```bash
pip install requests
```

`hashlib`, `os`, and `time` are all part of the Python standard library. Cross-platform (Linux/macOS/Windows). Needs a writable local directory to store hash state between checks.

#### Cautions
Hashing the entire page means any change — including a rotating ad, a timestamp, or an A/B test variant — will trigger a "changed" alert even if the content you actually care about didn't change; for noisy pages, target a specific section instead of the whole page (see Variations below).

#### Script

```python
#!/usr/bin/env python3
"""
monitor_website.py
-----------------------
Periodically checks a web page and alerts (prints) when its content
changes, by comparing a hash of the page content between checks.
Useful for watching a pricing page, job listing, or announcement
page for updates.

Usage:
    python3 monitor_website.py --url https://example.com --interval 300
    (checks every 300 seconds / 5 minutes; Ctrl+C to stop)

    # Single check against a previously saved hash file (good for cron):
    python3 monitor_website.py --url https://example.com --once
"""

import argparse
import hashlib
import os
import time
import requests

HASH_STORE_DIR = "monitor_state"


def get_page_hash(url):
    headers = {"User-Agent": "Mozilla/5.0 (compatible; WebOpsBot/1.0)"}
    response = requests.get(url, headers=headers, timeout=15)
    response.raise_for_status()
    return hashlib.sha256(response.text.encode("utf-8")).hexdigest()


def get_state_file(url):
    os.makedirs(HASH_STORE_DIR, exist_ok=True)
    safe_name = hashlib.md5(url.encode("utf-8")).hexdigest()
    return os.path.join(HASH_STORE_DIR, f"{safe_name}.hash")


def check_once(url):
    state_file = get_state_file(url)
    current_hash = get_page_hash(url)

    if os.path.isfile(state_file):
        with open(state_file, "r") as f:
            previous_hash = f.read().strip()

        if current_hash != previous_hash:
            print(f"[CHANGED] Page content has changed: {url}")
            changed = True
        else:
            print(f"[NO CHANGE] Page content is the same: {url}")
            changed = False
    else:
        print(f"[BASELINE] First check - saving initial hash for: {url}")
        changed = False

    with open(state_file, "w") as f:
        f.write(current_hash)

    return changed


def monitor_loop(url, interval):
    print(f"Monitoring '{url}' every {interval} second(s). Press Ctrl+C to stop.\n")
    try:
        while True:
            check_once(url)
            time.sleep(interval)
    except KeyboardInterrupt:
        print("\nStopped monitoring.")


def main():
    parser = argparse.ArgumentParser(description="Monitor a website for content changes")
    parser.add_argument("--url", required=True, help="URL to monitor")
    parser.add_argument("--interval", type=int, default=300, help="Check interval in seconds")
    parser.add_argument("--once", action="store_true", help="Check once and exit (good for cron)")
    args = parser.parse_args()

    try:
        if args.once:
            check_once(args.url)
        else:
            monitor_loop(args.url, args.interval)
    except requests.RequestException as e:
        print(f"Failed to check page: {e}")


if __name__ == "__main__":
    main()
```

#### How It Works
Rather than storing entire page snapshots, the script computes a SHA-256 hash of the page's HTML text — a fixed-length fingerprint that changes completely if even one character of the source changes. It saves that hash to a small state file in `monitor_state/`, named using an MD5 hash of the URL itself (so multiple monitored URLs don't collide). On each subsequent check, it recomputes the hash and compares it to the saved value: a mismatch means the page changed since the last check. The `--once` mode is designed for cron, where each invocation is a fresh process with no persistent memory except the state file on disk; the `--interval` loop mode is for interactive, long-running use.

```bash
# Continuous monitoring, checking every 5 minutes
python3 monitor_website.py --url https://example.com --interval 300

# Single check, suitable for cron
python3 monitor_website.py --url https://example.com --once
```

Example crontab entry to check every 10 minutes:

```bash
*/10 * * * * /usr/bin/python3 /opt/scripts/monitor_website.py --url https://example.com --once >> /var/log/page_monitor.log 2>&1
```

#### Sample Output

```bash
$ python3 monitor_website.py --url https://example.com --once
[BASELINE] First check - saving initial hash for: https://example.com

$ python3 monitor_website.py --url https://example.com --once
[NO CHANGE] Page content is the same: https://example.com

$ python3 monitor_website.py --url https://example.com --once
[CHANGED] Page content has changed: https://example.com
```

#### Variations / Flags
- Add a `--selector` flag (combined with BeautifulSoup from section 8.11) to hash only a specific section of the page (e.g., the price element) instead of the entire HTML, dramatically reducing false positives from unrelated page changes.
- Add a `--notify` flag to send an email, Slack message, or webhook call when a change is detected, instead of only printing to stdout.
- Add a `--diff` flag that stores the full previous content (not just its hash) so you can show exactly what changed, using Python's `difflib`.

#### Common Pitfalls
- **Noisy false positives**: dynamic elements (ads, "last updated" timestamps, CSRF tokens embedded in the HTML) will cause the whole-page hash to change on every single check, even with zero meaningful content change — scope down to a specific selector for anything but the simplest static pages.
- **State file location**: the `monitor_state/` directory is created relative to the current working directory, so running the script from different directories (e.g., manually vs. via cron) can create separate, disconnected state files for the same URL.
- **No historical record**: this script only ever keeps the most recent hash — it can tell you *that* something changed but not *when* it first changed if you don't check frequently enough, or *what* changed without adding a diff feature.

### 8.13 Automate Browser Tasks

#### Use Case
A sysadmin needs to interact with a page that only renders its real content via JavaScript (a single-page app, a dashboard behind a login form with a JS-heavy flow, or a site that actively blocks plain HTTP clients) — something the `requests`-based scripts in this chapter fundamentally cannot handle because they never execute JavaScript.

#### Prerequisites

```bash
pip install selenium webdriver-manager
```

Requires a Chrome or Chromium browser installed on the machine. `webdriver-manager` automatically downloads a matching `chromedriver` binary on first run, so you don't need to manage driver versions by hand. Cross-platform (Linux/macOS/Windows), but headless servers/containers need this to run in `--headless` mode (the default here) since there's no display to render a visible window.

#### Cautions
Real browser automation is far heavier and slower than a plain HTTP request — expect seconds, not milliseconds, per page load — and it consumes significantly more memory and CPU. It can also trigger bot-detection systems on some sites even more readily than a plain scraper, since some sites specifically fingerprint Selenium's automation signals.

#### Script

```python
#!/usr/bin/env python3
"""
browser_automation.py
--------------------------
Demonstrates basic browser automation using Selenium WebDriver:
opening a page, waiting for it to load, taking a screenshot, and
reading the page title. Requires `selenium` and a Chrome/Chromium
browser plus a matching chromedriver (handled automatically via
`webdriver-manager`).

Usage:
    python3 browser_automation.py --url https://example.com --screenshot page.png
    python3 browser_automation.py --url https://example.com --headless

Notes:
    - Runs headless by default in this script's --headless flag off
      means it will try to open a visible browser window, which
      requires a display (won't work on a bare server/container
      without a virtual display like Xvfb).
    - On first run, webdriver-manager downloads the correct
      chromedriver version automatically.
"""

import argparse
from selenium import webdriver
from selenium.webdriver.chrome.options import Options
from selenium.webdriver.chrome.service import Service
from webdriver_manager.chrome import ChromeDriverManager


def run_browser_task(url, screenshot_path=None, headless=True):
    options = Options()
    if headless:
        options.add_argument("--headless=new")
    options.add_argument("--no-sandbox")
    options.add_argument("--disable-dev-shm-usage")
    options.add_argument("--window-size=1280,800")

    service = Service(ChromeDriverManager().install())
    driver = webdriver.Chrome(service=service, options=options)

    try:
        print(f"Opening: {url}")
        driver.get(url)

        print(f"Page title: {driver.title}")
        print(f"Current URL: {driver.current_url}")

        if screenshot_path:
            driver.save_screenshot(screenshot_path)
            print(f"Screenshot saved to: {screenshot_path}")

        # Example: extract all visible link text on the page
        links = driver.find_elements("tag name", "a")
        print(f"\nFound {len(links)} link(s) on the page. First 10:")
        for link in links[:10]:
            text = link.text.strip() or "(no visible text)"
            href = link.get_attribute("href")
            print(f"  {text} -> {href}")

    finally:
        driver.quit()


def main():
    parser = argparse.ArgumentParser(description="Automate a basic browser task with Selenium")
    parser.add_argument("--url", required=True, help="URL to open")
    parser.add_argument("--screenshot", help="Path to save a screenshot (e.g. page.png)")
    parser.add_argument("--headless", action="store_true", default=True, help="Run without a visible window (default: on)")
    parser.add_argument("--visible", dest="headless", action="store_false", help="Run with a visible browser window")
    args = parser.parse_args()

    run_browser_task(args.url, args.screenshot, args.headless)


if __name__ == "__main__":
    main()
```

#### How It Works
`ChromeDriverManager().install()` checks the locally installed Chrome/Chromium version and downloads a matching `chromedriver` binary automatically, avoiding the classic "driver version doesn't match browser version" failure. The script then launches a real Chrome instance (headless by default, meaning no visible window is drawn — required on servers) and drives it exactly as a human would: navigating to a URL with `driver.get()`, reading the rendered page's title and current URL (which reflects any JavaScript redirects), optionally saving a screenshot, and finding elements — here, all `<a>` tags — using Selenium's DOM query methods. Because Selenium waits for the page to load and executes any JavaScript on it, this correctly captures content that a plain `requests.get()` would miss entirely. The `finally: driver.quit()` block ensures the browser process is always cleaned up, even if something above it raises an exception.

```bash
# Headless run with a screenshot
python3 browser_automation.py --url https://example.com --screenshot page.png

# Explicit headless flag (same as default)
python3 browser_automation.py --url https://example.com --headless

# Visible browser window (requires a display / desktop environment)
python3 browser_automation.py --url https://example.com --visible
```

#### Sample Output

```bash
$ python3 browser_automation.py --url https://example.com --screenshot page.png
Opening: https://example.com
Page title: Example Domain
Current URL: https://example.com/
Screenshot saved to: page.png

Found 1 link(s) on the page. First 10:
  More information... -> https://www.iana.org/domains/example
```

#### Variations / Flags
- Add explicit waits (`WebDriverWait` + `expected_conditions`) instead of relying on Selenium's implicit page-load wait, for pages that load content asynchronously after the initial page load event.
- Add form-filling and click actions (`driver.find_element(...).send_keys(...)`, `.click()`) to automate logins or multi-step workflows.
- Swap Chrome for Firefox using `selenium.webdriver.Firefox` and `webdriver_manager.firefox.GeckoDriverManager` if Chrome isn't available in your environment.

#### Common Pitfalls
- **No display on headless servers**: forgetting `--headless` (or running `--visible`) on a bare Linux server without a virtual display like Xvfb will cause Selenium to fail to launch the browser entirely.
- **Driver/browser version mismatches**: without `webdriver-manager`, a manually downloaded `chromedriver` can silently fail if it doesn't match the installed Chrome version — this script avoids that problem by resolving the driver automatically.
- **Resource usage in loops**: launching a fresh browser instance per URL in a tight loop is slow and memory-hungry; for scraping many pages, reuse a single `driver` instance across requests instead of recreating it each time.
- **Bot detection**: some sites detect and block Selenium's default automation fingerprint even in `--headless=new` mode — this is a legitimate defense some sites use and should be respected, not circumvented.

### 8.15 Handle Sessions

#### Use Case
A sysadmin needs to make several related requests to the same service — log in, then fetch data, then submit an action — where cookies, headers, and connection state need to persist automatically across all of them, rather than being managed by hand on every call.

#### Prerequisites

```bash
pip install requests
```

Cross-platform (Linux/macOS/Windows).

#### Cautions
A `requests.Session()` will hold onto cookies (including auth session cookies) for its entire lifetime — make sure to call `.close()` when done, and avoid reusing a single session object across unrelated user contexts in a multi-user application, or you risk leaking one user's session state into another's requests.

#### Script

```python
#!/usr/bin/env python3
"""
handle_sessions.py
-----------------------
Demonstrates using requests.Session() to persist cookies, headers,
and connection pooling across multiple requests - the standard
pattern for scripts that need to log in once and then make several
authenticated requests (e.g. a login page followed by API calls).

Usage:
    python3 handle_sessions.py --base-url https://httpbin.org
"""

import argparse
import requests


def demo_session(base_url):
    session = requests.Session()

    # Common headers applied to every request made through this session
    session.headers.update({"User-Agent": "WebOpsBot/1.0"})

    print("=== Request 1: sets a cookie ===")
    r1 = session.get(f"{base_url}/cookies/set?session_demo=hello")
    print(f"Status: {r1.status_code}")
    print(f"Session cookies after request 1: {dict(session.cookies)}")

    print("\n=== Request 2: same session, cookie is sent automatically ===")
    r2 = session.get(f"{base_url}/cookies")
    print(f"Status: {r2.status_code}")
    print(f"Response body (first 300 chars): {r2.text[:300]}")

    print("\n=== Request 3: check what headers the session sent ===")
    r3 = session.get(f"{base_url}/headers")
    print(f"Status: {r3.status_code}")
    print(f"Response body (first 300 chars): {r3.text[:300]}")

    session.close()
    print("\nSession closed.")


def main():
    parser = argparse.ArgumentParser(description="Demonstrate persistent sessions with requests.Session()")
    parser.add_argument(
        "--base-url",
        default="https://httpbin.org",
        help="Base URL of a test API that supports /cookies and /headers endpoints",
    )
    args = parser.parse_args()

    demo_session(args.base_url)


if __name__ == "__main__":
    main()
```

#### How It Works
`requests.Session()` creates a persistent object that automatically stores any cookies set by the server across every subsequent call made through it, and reuses an underlying TCP connection pool for better performance on multiple requests to the same host. `session.headers.update({...})` sets headers once that then apply to every future request via that session — no need to re-pass `headers=` on every call. The three example requests demonstrate the payoff: request 1 causes the server to set a cookie, and request 2 — with zero extra code — automatically sends that cookie back, exactly like a browser tab would across page loads. This is the standard pattern for "log in once, then make several authenticated calls."

```bash
# Run against the default httpbin.org test API
python3 handle_sessions.py

# Run against a different base URL that supports the same test endpoints
python3 handle_sessions.py --base-url https://httpbin.org
```

#### Sample Output

```bash
$ python3 handle_sessions.py
=== Request 1: sets a cookie ===
Status: 200
Session cookies after request 1: {'session_demo': 'hello'}

=== Request 2: same session, cookie is sent automatically ===
Status: 200
Response body (first 300 chars): {
  "cookies": {
    "session_demo": "hello"
  }
}

=== Request 3: check what headers the session sent ===
Status: 200
Response body (first 300 chars): {
  "headers": {
    "User-Agent": "WebOpsBot/1.0",
    ...
  }
}

Session closed.
```

#### Variations / Flags
- Wrap the session in a `with requests.Session() as session:` block instead of manually calling `.close()`, so cleanup happens automatically even on exceptions.
- Add a real login flow: `session.post(login_url, data={"username": ..., "password": ...})` followed by subsequent authenticated `session.get()`/`session.post()` calls — the session automatically carries the resulting auth cookie forward.
- Combine with retry logic via `requests.adapters.HTTPAdapter` and `urllib3.util.Retry` mounted onto the session for automatic retries on transient failures across all requests made through it.

#### Common Pitfalls
- **Forgetting to close the session**: long-running scripts that create many `Session()` objects without closing them can leak file descriptors/connections over time — always close or use a context manager.
- **Sharing one session across users**: in a multi-user server-side script, reusing a single global `Session()` object for different users' requests can cause one user's cookies/auth to leak into another user's requests — create a fresh session per user context.
- **Assuming headers set on the session override per-request headers**: per-call `headers=` passed to an individual `session.get(..., headers={...})` call merge with, rather than fully replace, the session-level headers — check for unexpected duplicate or conflicting values.

### 8.16 Download Images

#### Use Case
A sysadmin or content pipeline needs to bulk-download product photos, avatars, or other images referenced by URL — for example, mirroring assets before a CDN migration, or archiving images linked from a scraped page.

#### Prerequisites

```bash
pip install requests
```

Cross-platform (Linux/macOS/Windows).

#### Cautions
Only download images you have the right to use/store — copyright applies to images the same as any other content. Also validate `Content-Type` (as this script does) so an HTML error page returned with a 200 status doesn't get silently saved with a `.jpg` extension.

#### Script

```python
#!/usr/bin/env python3
"""
download_images.py
-----------------------
Downloads one or more images from URLs and saves them to a local
folder, with content-type validation so you don't accidentally save
an error page as a ".jpg". Requires the `requests` library.

Usage:
    python3 download_images.py --url https://example.com/photo.jpg --output-dir images
    python3 download_images.py --urls-file image_urls.txt --output-dir images
"""

import argparse
import os
from urllib.parse import urlparse
import requests


def download_image(url, output_dir):
    try:
        headers = {"User-Agent": "Mozilla/5.0 (compatible; WebOpsBot/1.0)"}
        response = requests.get(url, headers=headers, timeout=15, stream=True)
        response.raise_for_status()

        content_type = response.headers.get("Content-Type", "")
        if not content_type.startswith("image/"):
            print(f"[SKIPPED] Not an image (Content-Type: {content_type}): {url}")
            return False

        os.makedirs(output_dir, exist_ok=True)

        filename = os.path.basename(urlparse(url).path) or "downloaded_image"
        if "." not in filename:
            ext = content_type.split("/")[-1].split(";")[0]
            filename = f"{filename}.{ext}"

        output_path = os.path.join(output_dir, filename)

        with open(output_path, "wb") as f:
            for chunk in response.iter_content(8192):
                f.write(chunk)

        print(f"[OK] Saved: {output_path}")
        return True

    except requests.RequestException as e:
        print(f"[FAILED] {url} - {e}")
        return False


def main():
    parser = argparse.ArgumentParser(description="Download images from one or more URLs")
    parser.add_argument("--url", help="A single image URL")
    parser.add_argument("--urls-file", help="Text file with one image URL per line")
    parser.add_argument("--output-dir", default="images", help="Directory to save images to")
    args = parser.parse_args()

    if not args.url and not args.urls_file:
        print("Provide either --url or --urls-file")
        return

    urls = []
    if args.url:
        urls.append(args.url)
    if args.urls_file:
        with open(args.urls_file, "r") as f:
            urls.extend(line.strip() for line in f if line.strip())

    success_count = 0
    for url in urls:
        if download_image(url, args.output_dir):
            success_count += 1

    print(f"\nDone. {success_count}/{len(urls)} image(s) downloaded successfully.")


if __name__ == "__main__":
    main()
```

#### How It Works
Before saving anything, the script checks that the response's `Content-Type` header actually starts with `image/` — this guards against the common failure mode of a broken link silently returning an HTML "404" page that would otherwise get saved with an image extension and quietly corrupt a downstream pipeline. The filename is derived from the URL path via `urlparse()`, and if the URL doesn't include a file extension (common with dynamically generated image URLs), the script infers one from the `Content-Type` header itself (e.g., `image/jpeg` becomes `.jpeg`). Batch downloading is supported via `--urls-file`, a plain text file with one URL per line.

```bash
# Download a single image
python3 download_images.py --url https://example.com/photo.jpg --output-dir images

# Batch download from a list of URLs
python3 download_images.py --urls-file image_urls.txt --output-dir images
```

#### Sample Output

```bash
$ python3 download_images.py --url https://example.com/photo.jpg --output-dir images
[OK] Saved: images/photo.jpg

Done. 1/1 image(s) downloaded successfully.
```

```bash
$ python3 download_images.py --urls-file image_urls.txt --output-dir images
[OK] Saved: images/banner.png
[SKIPPED] Not an image (Content-Type: text/html): https://example.com/broken-link
[OK] Saved: images/logo.svg

Done. 2/3 image(s) downloaded successfully.
```

#### Variations / Flags
- Add a `--concurrency 5` flag using `concurrent.futures.ThreadPoolExecutor` to download multiple images in parallel instead of sequentially, which matters a lot for large batches.
- Add a `--min-size 1024` flag to skip suspiciously tiny "images" that are often tracking pixels or placeholder graphics rather than real content.
- Add deduplication by hashing each downloaded image's bytes and skipping duplicates already present in the output directory.

#### Common Pitfalls
- **Filename collisions**: two different URLs that happen to share the same basename (e.g., `.../a/image.jpg` and `.../b/image.jpg`) will overwrite each other in a flat output directory — consider prefixing filenames with a hash or preserving directory structure.
- **Sequential downloads are slow**: downloading hundreds of images one at a time over a loop can take a long time; this script does not parallelize by default.
- **Content-Type spoofing**: a malicious or misconfigured server could still lie about `Content-Type` — for security-sensitive pipelines, also validate the actual file signature/magic bytes after download, not just the header.

### 8.17 Download PDFs

#### Use Case
A sysadmin needs to bulk-download reports, invoices, or documentation published as PDFs at known URLs — for example, archiving monthly vendor statements or pulling a batch of published compliance documents.

#### Prerequisites

```bash
pip install requests
```

Cross-platform (Linux/macOS/Windows).

#### Cautions
As with any downloaded content, verify you're authorized to access and store the PDFs in question, especially for anything behind authentication — this script does not handle login flows (combine with sections 8.6 or 8.15 if the PDFs require authentication).

#### Script

```python
#!/usr/bin/env python3
"""
download_pdfs.py
---------------------
Downloads one or more PDF files from URLs and saves them locally,
verifying the content type is actually a PDF before saving. Requires
the `requests` library.

Usage:
    python3 download_pdfs.py --url https://example.com/document.pdf --output-dir pdfs
    python3 download_pdfs.py --urls-file pdf_urls.txt --output-dir pdfs
"""

import argparse
import os
from urllib.parse import urlparse
import requests


def download_pdf(url, output_dir):
    try:
        headers = {"User-Agent": "Mozilla/5.0 (compatible; WebOpsBot/1.0)"}
        response = requests.get(url, headers=headers, timeout=20, stream=True)
        response.raise_for_status()

        content_type = response.headers.get("Content-Type", "")
        if "pdf" not in content_type.lower() and not url.lower().endswith(".pdf"):
            print(f"[SKIPPED] Not a PDF (Content-Type: {content_type}): {url}")
            return False

        os.makedirs(output_dir, exist_ok=True)

        filename = os.path.basename(urlparse(url).path) or "downloaded_document.pdf"
        if not filename.lower().endswith(".pdf"):
            filename += ".pdf"

        output_path = os.path.join(output_dir, filename)

        with open(output_path, "wb") as f:
            for chunk in response.iter_content(8192):
                f.write(chunk)

        size_kb = os.path.getsize(output_path) / 1024
        print(f"[OK] Saved: {output_path} ({size_kb:.1f} KB)")
        return True

    except requests.RequestException as e:
        print(f"[FAILED] {url} - {e}")
        return False


def main():
    parser = argparse.ArgumentParser(description="Download PDF files from one or more URLs")
    parser.add_argument("--url", help="A single PDF URL")
    parser.add_argument("--urls-file", help="Text file with one PDF URL per line")
    parser.add_argument("--output-dir", default="pdfs", help="Directory to save PDFs to")
    args = parser.parse_args()

    if not args.url and not args.urls_file:
        print("Provide either --url or --urls-file")
        return

    urls = []
    if args.url:
        urls.append(args.url)
    if args.urls_file:
        with open(args.urls_file, "r") as f:
            urls.extend(line.strip() for line in f if line.strip())

    success_count = 0
    for url in urls:
        if download_pdf(url, args.output_dir):
            success_count += 1

    print(f"\nDone. {success_count}/{len(urls)} PDF(s) downloaded successfully.")


if __name__ == "__main__":
    main()
```

#### How It Works
The validation check here is intentionally more lenient than the image downloader's: it accepts a file either if the server's `Content-Type` header contains `"pdf"` (case-insensitively, since real-world servers send both `application/pdf` and non-standard variants) *or* if the URL itself ends in `.pdf` — a pragmatic fallback for misconfigured servers that serve PDFs with an incorrect or missing content type. After saving, the script reports the final file size in KB by checking the actual bytes written to disk with `os.path.getsize()`, giving a quick sanity check that a real document was downloaded rather than a truncated or empty file.

```bash
# Download a single PDF
python3 download_pdfs.py --url https://example.com/document.pdf --output-dir pdfs

# Batch download from a list of URLs
python3 download_pdfs.py --urls-file pdf_urls.txt --output-dir pdfs
```

#### Variations / Flags
- Add a `--verify-pdf` flag that opens each downloaded file with a PDF library (e.g., `pypdf`) to confirm it actually parses as a valid PDF, not just that the extension/content-type look right.
- Add `--rename "{index}_{filename}"` templating for batches where source filenames collide or aren't descriptive enough.
- Add resumable downloads for very large PDF files using the `Range` header, similar to the variation suggested for section 8.2.

#### Common Pitfalls
- **Password-protected or authenticated PDFs**: this script has no login handling — PDFs behind an authentication wall will return an HTML login page (correctly skipped by the content-type check) rather than the actual document; combine with session/auth handling from earlier sections.
- **Extension-based fallback can be fooled**: accepting any URL ending in `.pdf` regardless of content type means a cleverly named non-PDF URL could still pass the check — for strict pipelines, rely on content-type or a post-download validity check instead.
- **Large PDF timeouts**: the 20-second timeout may not be enough for very large scanned-document PDFs on slow connections; increase it for bulk archival jobs.

### 8.18 Submit HTML Forms

#### Use Case
A sysadmin needs to automate submitting a web form — a support ticket, a status update, a configuration change on an internal admin panel — the same way a browser does when a user clicks "Submit," without opening an actual browser.

#### Prerequisites

```bash
pip install requests
```

Cross-platform (Linux/macOS/Windows). You'll typically need to inspect the target form's HTML (via browser dev tools) first to identify the correct field names and submission URL.

#### Cautions
Automating form submissions against a site you don't control or own can violate its Terms of Service, and repeated automated submissions can trigger anti-bot defenses (CAPTCHAs, rate limiting, IP bans) — this pattern is best suited to internal tools and APIs you're authorized to automate.

#### Script

```python
#!/usr/bin/env python3
"""
submit_form.py
-------------------
Submits an HTML form via HTTP POST (the same request a browser would
send when you click "Submit"), using the `requests` library. Field
data is supplied as JSON so you can submit any form fields your
target expects.

Usage:
    python3 submit_form.py --url https://httpbin.org/post --data '{"username": "ahmad", "password": "secret"}'
"""

import argparse
import json
import requests


def submit_form(url, form_data):
    try:
        response = requests.post(url, data=form_data, timeout=15)

        print(f"Status code: {response.status_code}")
        print(f"Response body (first 800 chars):\n{response.text[:800]}")

        if response.ok:
            print("\nForm submitted successfully.")
        else:
            print("\nForm submission returned a non-success status code.")

        return response

    except requests.RequestException as e:
        print(f"Form submission failed: {e}")
        return None


def main():
    parser = argparse.ArgumentParser(description="Submit an HTML form via HTTP POST")
    parser.add_argument("--url", required=True, help="Form action URL")
    parser.add_argument(
        "--data",
        required=True,
        help='Form fields as a JSON string, e.g. \'{"username": "ahmad", "password": "secret"}\'',
    )
    args = parser.parse_args()

    form_data = json.loads(args.data)
    submit_form(args.url, form_data)


if __name__ == "__main__":
    main()
```

#### How It Works
Passing a Python dict to `requests.post(url, data=form_data)` (as opposed to `json=form_data`) encodes it as `application/x-www-form-urlencoded` — the exact same content type and encoding an HTML `<form>` sends by default when a user clicks Submit. The target `url` should be the form's `action` attribute, which is often a different path than the page the form visually appears on — always check the rendered HTML's `<form action="...">` value rather than assuming it matches the page URL.

```bash
python3 submit_form.py --url https://httpbin.org/post --data '{"username": "ahmad", "password": "secret"}'
```

#### Sample Output

```bash
$ python3 submit_form.py --url https://httpbin.org/post --data '{"username": "ahmad", "password": "secret"}'
Status code: 200
Response body (first 800 chars):
{
  "form": {
    "password": "secret",
    "username": "ahmad"
  },
  "url": "https://httpbin.org/post"
}

Form submitted successfully.
```

```bash
$ python3 submit_form.py --url https://httpbin.org/status/403 --data '{"foo": "bar"}'
Status code: 403
Response body (first 800 chars):

Form submission returned a non-success status code.
```

#### Variations / Flags
- Add CSRF token handling: many real forms require first GETting the form page to extract a hidden CSRF token, then including it in the POST body — combine with `requests.Session()` (section 8.15) so cookies from the GET carry through to the POST.
- Add multipart support (switch `data=` to `files=`) for forms that include a file upload field alongside regular text fields (combine with section 8.3).
- Add `--follow-redirects false` (via `allow_redirects=False`) to inspect the raw redirect response instead of following it automatically, useful for debugging a form that isn't behaving as expected.

#### Common Pitfalls
- **Missing CSRF tokens**: many modern web forms require a CSRF token obtained from the form page itself; submitting without one will typically be rejected with a 403 even if all the visible field values are correct.
- **Wrong action URL**: submitting to the page URL instead of the form's actual `action` URL is one of the most common reasons a scripted form submission silently fails or gets redirected somewhere unexpected.
- **Using `json=` instead of `data=`**: sending form fields as `json=form_data` instead of `data=form_data` produces a `Content-Type: application/json` request, which most classic HTML form handlers won't parse correctly — they expect `application/x-www-form-urlencoded`.

### 8.19 Interact with WebSockets

#### Use Case
A sysadmin needs to consume a persistent, two-way connection — a live log stream, a real-time metrics feed, a chat/notification service — where a single request/response HTTP call (as used everywhere else in this chapter) isn't the right model, because the server needs to keep pushing messages to the client (and vice versa) over one long-lived connection.

#### Prerequisites

```bash
pip install websockets
```

Cross-platform (Linux/macOS/Windows). Requires Python 3.8+ and outbound network access to the target `ws://` or `wss://` endpoint. `wss://` is the encrypted equivalent of `https://` for WebSockets and should always be preferred over plain `ws://` outside of local testing.

#### Cautions
Unlike the request/response scripts elsewhere in this chapter, a WebSocket connection stays open indefinitely — a script that doesn't handle disconnects, timeouts, and reconnection logic can hang forever or silently stop receiving messages after a network blip. Always set a reasonable timeout and close the connection explicitly when done.

#### Script

```python
#!/usr/bin/env python3
"""
websocket_client.py
------------------------
Demonstrates connecting to a WebSocket endpoint, sending a message,
and listening for messages pushed from the server. Requires the
`websockets` library.

Usage:
    python3 websocket_client.py --url wss://echo.websocket.org --message "hello"
    python3 websocket_client.py --url wss://example.com/stream --listen-only --duration 30
"""

import argparse
import asyncio
import websockets


async def send_and_receive(url, message, timeout=10):
    try:
        async with websockets.connect(url, open_timeout=timeout) as ws:
            print(f"Connected to: {url}")

            await ws.send(message)
            print(f"Sent: {message}")

            reply = await asyncio.wait_for(ws.recv(), timeout=timeout)
            print(f"Received: {reply}")

    except (websockets.exceptions.WebSocketException, asyncio.TimeoutError, OSError) as e:
        print(f"WebSocket error: {e}")


async def listen_only(url, duration, timeout=10):
    try:
        async with websockets.connect(url, open_timeout=timeout) as ws:
            print(f"Connected to: {url}")
            print(f"Listening for {duration} second(s)... (Ctrl+C to stop early)\n")

            try:
                async with asyncio.timeout(duration):
                    async for message in ws:
                        print(f"Received: {message}")
            except asyncio.TimeoutError:
                print("\nListen duration elapsed.")

    except (websockets.exceptions.WebSocketException, OSError) as e:
        print(f"WebSocket error: {e}")


def main():
    parser = argparse.ArgumentParser(description="Connect to a WebSocket endpoint")
    parser.add_argument("--url", required=True, help="WebSocket URL (ws:// or wss://)")
    parser.add_argument("--message", help="A single message to send and then wait for one reply")
    parser.add_argument("--listen-only", action="store_true", help="Only listen for incoming messages, don't send anything")
    parser.add_argument("--duration", type=int, default=30, help="How long to listen in --listen-only mode (seconds)")
    parser.add_argument("--timeout", type=int, default=10, help="Connection/response timeout in seconds")
    args = parser.parse_args()

    if args.listen_only:
        asyncio.run(listen_only(args.url, args.duration, args.timeout))
    else:
        if not args.message:
            print("--message is required unless --listen-only is set")
            return
        asyncio.run(send_and_receive(args.url, args.message, args.timeout))


if __name__ == "__main__":
    main()
```

#### How It Works
WebSockets require an asynchronous connection model since the client and server can both send messages at any time, so this script is built on Python's `asyncio` rather than the synchronous style used by `requests` elsewhere in this chapter. `websockets.connect(url)` used as an `async with` context manager opens the connection and guarantees it's closed cleanly afterward, even on error. Two modes are provided: `send_and_receive()` sends one message and waits for exactly one reply (useful for simple request/response-style WebSocket APIs or an echo test server), while `listen_only()` subscribes to a stream and prints every message the server pushes for a bounded duration, using `asyncio.timeout()` so the script doesn't run forever by accident. `open_timeout` and `asyncio.wait_for()` both guard against a connection or response that never arrives.

```bash
# Send one message and print the server's reply
python3 websocket_client.py --url wss://echo.websocket.org --message "hello"

# Listen to a live stream for 30 seconds without sending anything
python3 websocket_client.py --url wss://example.com/stream --listen-only --duration 30
```

#### Sample Output

```bash
$ python3 websocket_client.py --url wss://echo.websocket.org --message "hello"
Connected to: wss://echo.websocket.org
Sent: hello
Received: hello
```

```bash
$ python3 websocket_client.py --url wss://example.com/stream --listen-only --duration 10

Listening for 10 second(s)... (Ctrl+C to stop early)

Received: {"event": "metric", "cpu": 42.1}

Listen duration elapsed.
```

#### Variations / Flags
- Add automatic reconnection with exponential backoff — wrap the `async with websockets.connect(...)` block in a retry loop so a dropped connection (a common real-world occurrence) doesn't end the script.
- Add authentication support for endpoints that require a token, typically passed either as a query parameter (`wss://example.com/stream?token=...`) or an `Authorization` header via `websockets.connect(url, additional_headers={...})`.
- Add a `--json` flag to parse each incoming message as JSON and pretty-print it, for APIs that push structured event data rather than plain text.

#### Common Pitfalls
- **Treating WebSockets like HTTP**: unlike every other script in this chapter, there's no single "response" to a WebSocket send — the server can push zero, one, or many messages at unpredictable times, so code that assumes a strict request/reply pattern will misbehave against real streaming APIs.
- **Forgetting `wss://` in production**: plain `ws://` is unencrypted, exposing all message content and any auth tokens to anyone on the network path — always use `wss://` outside of local development.
- **No reconnection handling**: this script exits on any connection error rather than retrying — a long-running monitor built on this pattern needs explicit reconnect logic to survive normal network hiccups.
- **Blocking the event loop**: any long-running synchronous code (e.g., heavy computation or blocking file I/O) added inside the `async for message in ws:` loop will stall the ability to receive further messages promptly — offload such work with `asyncio.to_thread()` if needed.
