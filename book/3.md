## Chapter 3: File & Directory Operations

Every system administrator eventually lives on the command line, and the filesystem is where nearly all of that time is spent — copying, comparing, renaming, archiving, cleaning, and occasionally destroying files and directories. This chapter is a working toolbox of small, single-purpose Python scripts that automate the file and directory chores that would otherwise eat up minutes of manual work, dozens of times a day.

None of these scripts try to be clever frameworks. Each one does exactly one job, takes its input from the command line, and prints exactly what it did so it can be trusted, scripted, scheduled, or dropped straight into a larger pipeline. That's deliberate — a sysadmin script you can't read in thirty seconds is a script you shouldn't run at 2 a.m.


**A Note on Scope and Safety:** File and directory operations are inherently destructive in ways that, say, a text-formatting script is not. Deleting the wrong directory or overwriting the wrong file cannot be undone by re-running a script. Every script in this chapter that touches existing data in a destructive way — deleting, overwriting, or bulk-renaming — either prompts for confirmation by default or clearly documents the flag that skips the prompt (`--force`), so automation is possible without making destructive behavior the accidental default.

A second theme worth flagging up front: several of Python's convenience functions in `shutil` and `filecmp` make assumptions that are easy to miss. `shutil.copytree()` refuses to run if the destination already exists. `filecmp.dircmp()` defaults to a *shallow* comparison — checking file size and modification time rather than actual content — which can report two files as identical when their contents have quietly diverged. These aren't bugs, but they're the kind of behavior that only becomes obvious the first time it bites you, so each relevant script below calls it out explicitly.

**Standard Library vs. Third-Party Dependencies:** Most of the scripts in this chapter rely only on the Python standard library (`os`, `shutil`, `argparse`, `zipfile`, `tarfile`, `tempfile`, `filecmp`, `difflib`, `hashlib`) and will run on any machine with Python 3.8+ installed — Linux, macOS, or Windows. The two exceptions are the encryption scripts (3.25, 3.26), which depend on the third-party `cryptography` package. Each script's own page notes its exact requirements and installation command so you never have to guess before running one.

### 3.1 List Directory Contents

#### Use Case

A sysadmin needs a fast readout of what's inside a directory — before an archive job, a cleanup pass, or just to sanity-check a deployment — without relying on a GUI file browser or memorizing `ls` flag combinations.

#### Prerequisites

Standard library only (`os`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

This lists only the immediate contents of the given directory — it does not recurse into subdirectories (use `traverse_directories.py`, 3.10, for that). Directory sizes shown in `--long` mode reflect the size of the directory *entry* itself (typically 4096 bytes on many filesystems), not the total size of its contents.

#### Script

```python
#!/usr/bin/env python3
"""
list_directory.py
------------------
Lists the contents of a directory (files and subdirectories).

Usage:
    python list_directory.py <directory_path> [--long]

Example:
    python list_directory.py .
    python list_directory.py /home/user/Documents --long
"""

import argparse
import os
import sys
from datetime import datetime


def list_directory(path: str, long_format: bool = False) -> None:
    if not os.path.exists(path):
        print(f"Error: Path '{path}' does not exist.")
        sys.exit(1)

    if not os.path.isdir(path):
        print(f"Error: '{path}' is not a directory.")
        sys.exit(1)

    entries = sorted(os.listdir(path))

    if not entries:
        print(f"Directory '{path}' is empty.")
        return

    print(f"Contents of '{path}':\n")

    for entry in entries:
        full_path = os.path.join(path, entry)
        entry_type = "DIR " if os.path.isdir(full_path) else "FILE"

        if long_format:
            size = os.path.getsize(full_path)
            mtime = datetime.fromtimestamp(os.path.getmtime(full_path))
            print(f"[{entry_type}] {entry:<40} {size:>10} bytes  {mtime.strftime('%Y-%m-%d %H:%M:%S')}")
        else:
            print(f"[{entry_type}] {entry}")

    print(f"\nTotal items: {len(entries)}")


def main():
    parser = argparse.ArgumentParser(description="List contents of a directory.")
    parser.add_argument("path", help="Path to the directory to list")
    parser.add_argument("--long", action="store_true", help="Show detailed info (size, modified time)")
    args = parser.parse_args()

    list_directory(args.path, args.long)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Simple listing
python3 list_directory.py .

# Detailed listing with size and modified time
python3 list_directory.py /home/user/Documents --long
```

The script validates the path exists and is a directory, then sorts and iterates `os.listdir()` results, classifying each entry as `DIR` or `FILE`. In `--long` mode it also prints size (via `os.path.getsize()`) and last-modified time (via `os.path.getmtime()`), formatted for readability.

#### Sample Output (Verified)

```bash
$ python3 list_directory.py listtest
Contents of 'listtest':

[FILE] a.txt
[FILE] b.txt
[DIR ] subdir

Total items: 3

$ python3 list_directory.py listtest --long
Contents of 'listtest':

[FILE] a.txt                                             2 bytes  2026-07-17 18:41:07
[FILE] b.txt                                             3 bytes  2026-07-17 18:41:07
[DIR ] subdir                                         4096 bytes  2026-07-17 18:41:07

Total items: 3
```

#### Variations / Flags

- Add `--recursive` to fold in `traverse_directories.py`-style walking as an option instead of a separate script.
- Add `--sort-by size` or `--sort-by modified` instead of the fixed alphabetical sort.
- Add `--filter "*.txt"` using `fnmatch` to only show matching entries.

#### Common Pitfalls

- **Directory "size" is misleading** — the bytes shown for a `DIR` entry in `--long` mode is the directory inode's own size, not the total size of everything inside it; don't use this output to estimate disk usage.
- **Not recursive by default** — a directory with deeply nested content will just show its immediate children; combine with 3.10 for a full tree view.
- **Hidden files are included** — unlike some `ls` defaults, `os.listdir()` returns dotfiles too, which is usually desirable for a sysadmin tool but worth knowing if you expected them hidden.

### 3.2 Create Files

#### Use Case

A sysadmin is setting up a new project skeleton or needs to create several empty marker files (e.g., `.gitkeep`, lock files, placeholder configs) in one shot without opening an editor for each one.

#### Prerequisites

Standard library only (`os`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

The script will not overwrite a file that already exists — it skips it instead. This protects against accidental data loss, but it also means re-running the script with different `--content` won't update files created on a previous run; you'll need to delete them first if you want fresh content.

#### Script

```python
#!/usr/bin/env python3
"""
create_files.py
----------------
Creates one or more new files, optionally with content.

Usage:
    python create_files.py <file1> [file2 ...] [--content "text"]

Example:
    python create_files.py notes.txt
    python create_files.py a.txt b.txt c.txt --content "Hello World"
"""

import argparse
import os
import sys


def create_files(file_paths, content: str = "") -> None:
    for file_path in file_paths:
        try:
            directory = os.path.dirname(file_path)
            if directory and not os.path.exists(directory):
                os.makedirs(directory)

            if os.path.exists(file_path):
                print(f"Skipped (already exists): {file_path}")
                continue

            with open(file_path, "w", encoding="utf-8") as f:
                f.write(content)

            print(f"Created: {file_path}")
        except OSError as e:
            print(f"Error creating '{file_path}': {e}")
            sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Create one or more files.")
    parser.add_argument("files", nargs="+", help="File path(s) to create")
    parser.add_argument("--content", default="", help="Optional text content to write into each file")
    args = parser.parse_args()

    create_files(args.files, args.content)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Empty file
python3 create_files.py notes.txt

# Multiple files with the same content
python3 create_files.py a.txt b.txt c.txt --content "Hello World"
```

For each path given, the script first ensures any parent directory exists (creating it if needed), then checks whether the file already exists. If not, it opens the file in write mode and writes the (optionally empty) `--content` string to it.

#### Sample Output (Verified)

```bash
$ python3 create_files.py notesA.txt notesB.txt --content "Hello World"
Created: notesA.txt
Created: notesB.txt

$ cat notesA.txt
Hello World
$ python3 create_files.py notesA.txt
Skipped (already exists): notesA.txt
```

#### Variations / Flags

- Add `--overwrite` to intentionally replace existing files instead of skipping them.
- Add `--from-file template.txt` to read content from an existing file rather than passing it inline as a string.
- Add `--binary` mode for writing raw bytes (e.g., a minimal valid PNG or zero-byte placeholder) instead of text.

#### Common Pitfalls

- **Every listed file gets identical content** — there's no per-file content option, so creating several files with different starting content requires separate script invocations.
- **Parent directory auto-creation** can be surprising: `create_files.py deep/nested/path/file.txt` silently creates `deep/nested/path/` too, which is convenient but means a typo'd path creates a new directory tree rather than failing loudly.
- **No content = truly empty file** (0 bytes), not a file containing a newline — distinguish this from tools like `echo >` which typically add a trailing newline.

### 3.3 Delete Files

#### Use Case

A sysadmin needs to remove a handful of obsolete files — old exports, stale lock files, superseded backups — as part of routine cleanup, with the option to skip confirmation prompts when running non-interactively (e.g., from a cron job).

#### Prerequisites

Standard library only (`os`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

Deletion via `os.remove()` is immediate and permanent — there is no recycle bin / trash step. The confirmation prompt is per-file by default (not one prompt for the whole batch), which is safer but means a large batch requires either `--force` or a lot of `y` presses.

#### Script

```python
#!/usr/bin/env python3
"""
delete_files.py
-----------------
Deletes one or more files. Asks for confirmation unless --force is used.

Usage:
    python delete_files.py <file1> [file2 ...] [--force]

Example:
    python delete_files.py old.txt
    python delete_files.py a.txt b.txt --force
"""

import argparse
import os
import sys


def delete_files(file_paths, force: bool = False) -> None:
    for file_path in file_paths:
        if not os.path.exists(file_path):
            print(f"Skipped (not found): {file_path}")
            continue

        if not os.path.isfile(file_path):
            print(f"Skipped (not a file): {file_path}")
            continue

        if not force:
            answer = input(f"Delete '{file_path}'? [y/N]: ").strip().lower()
            if answer != "y":
                print(f"Skipped: {file_path}")
                continue

        try:
            os.remove(file_path)
            print(f"Deleted: {file_path}")
        except OSError as e:
            print(f"Error deleting '{file_path}': {e}")
            sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Delete one or more files.")
    parser.add_argument("files", nargs="+", help="File path(s) to delete")
    parser.add_argument("--force", action="store_true", help="Delete without confirmation prompt")
    args = parser.parse_args()

    delete_files(args.files, args.force)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Interactive, one confirmation per file
python3 delete_files.py old.txt

# Batch, no prompts — for scripts and automation
python3 delete_files.py a.txt b.txt --force
```

For each path given, the script checks it exists and is a regular file (not a directory), asks for confirmation unless `--force` is set, then calls `os.remove()`. Missing files or non-file paths (like directories) are skipped with a message rather than causing the whole run to fail.

#### Sample Output (Verified)

```bash
$ python3 delete_files.py old_temp.txt --force
Deleted: old_temp.txt

$ ls old_temp.txt
ls: cannot access 'old_temp.txt': No such file or directory
```

#### Variations / Flags

- Add `--yes-to-all` as an alias to prompt once for the whole batch rather than per-file, for a middle ground between full `--force` and per-file prompts.
- Add `--pattern "*.log"` glob support to delete matching files in a directory without listing each one explicitly.
- Route deleted files to a `.trash/` folder (move instead of remove) as a lower-risk default, reserving true permanent deletion for an explicit `--permanent` flag.

#### Common Pitfalls

- **Directories passed as arguments are silently skipped** ("not a file") rather than raising an error — this is intentional (use `delete_directories.py`, 3.8, for those) but easy to overlook in a mixed batch.
- **`os.remove()` on a symlink deletes the symlink itself, not its target** — this is standard Unix behavior and usually what you want, but worth confirming if working with linked configuration files.
- **Per-file prompts on a large batch without `--force`** can make the script feel "stuck" if run non-interactively (e.g., in a cron job with no attached terminal) — `input()` will raise `EOFError` in that context, so always use `--force` for scheduled/automated runs.
- **No dry-run mode built in** — test the exact file list against a throwaway copy first if you're not fully confident in a glob expansion or variable substitution feeding the argument list.

### 3.4 Rename Files

#### Use Case

A sysadmin needs to rename one specific file or directory — for example, promoting `report_draft.txt` to `final_report.txt` — as a discrete, deliberate action distinct from the bulk/sequential renaming handled by `batch_rename_files.py` (3.32).

#### Prerequisites

Standard library only (`os`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

The script refuses to overwrite an existing target — this is a safety feature, not a limitation, and distinguishes it from a raw `mv` command, which would silently overwrite. If you need overwrite behavior, delete the target first with `delete_files.py` (3.3) or add an explicit `--overwrite` flag.

#### Script

```python
#!/usr/bin/env python3
"""
rename_files.py
-----------------
Renames a single file or folder.

Usage:
    python rename_files.py <old_name> <new_name>

Example:
    python rename_files.py report.txt final_report.txt
"""

import argparse
import os
import sys


def rename_file(old_path: str, new_path: str) -> None:
    if not os.path.exists(old_path):
        print(f"Error: '{old_path}' does not exist.")
        sys.exit(1)

    if os.path.exists(new_path):
        print(f"Error: target '{new_path}' already exists.")
        sys.exit(1)

    try:
        os.rename(old_path, new_path)
        print(f"Renamed: '{old_path}' -> '{new_path}'")
    except OSError as e:
        print(f"Error renaming file: {e}")
        sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Rename a file or directory.")
    parser.add_argument("old_name", help="Current name/path")
    parser.add_argument("new_name", help="New name/path")
    args = parser.parse_args()

    rename_file(args.old_name, args.new_name)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 rename_files.py report.txt final_report.txt
```

The script checks the source exists and the target doesn't, then calls `os.rename()`, which works for both files and directories on the same filesystem. Under the hood this is a metadata-only operation on most filesystems — the data itself isn't copied, so it's essentially instantaneous even for large files.

#### Sample Output (Verified)

```bash
$ python3 rename_files.py listtest/a.txt listtest/renamed.txt
Renamed: 'listtest/a.txt' -> 'listtest/renamed.txt'

$ ls listtest
b.txt  renamed.txt  subdir
```

#### Variations / Flags

- Add `--overwrite` to explicitly allow replacing an existing target, for cases where that's genuinely intended.
- Support renaming multiple pairs in one invocation by accepting a mapping file (`old,new` per line).
- Add case-only rename handling — on case-insensitive filesystems (default macOS, Windows), renaming `Report.txt` to `report.txt` requires an intermediate temp name since `os.rename()` may treat them as the same path.

#### Common Pitfalls

- **Cross-filesystem/cross-drive renames fail** — `os.rename()` only works within the same filesystem/drive; renaming across different mounted volumes raises `OSError` and requires a copy+delete instead (see `move_files.py`, 3.5, which uses `shutil.move()` and handles this transparently).
- **Case-insensitive filesystems**: on macOS and Windows, renaming `file.txt` to `File.txt` may not behave as expected since the OS may treat them as the same file — the "target already exists" check can trigger unexpectedly here.
- **This script renames exactly one item per call** — for bulk renaming, use `batch_rename_files.py` (3.32) instead.

### 3.5 Move Files

#### Use Case

A sysadmin needs to relocate a batch of files — say, moving completed reports out of a working directory into an archive folder — where the files should no longer exist at their original location afterward, unlike a copy operation.

#### Prerequisites

Standard library only (`shutil`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

`shutil.move()` overwrites a same-named file already present at the destination with no confirmation prompt. Unlike `rename_files.py` (3.4), this script also accepts folders as sources and works across different filesystems/drives (falling back to copy+delete automatically when a fast rename isn't possible).

#### Script

```python
#!/usr/bin/env python3
"""
move_files.py
---------------
Moves one or more files/folders into a destination directory.

Usage:
    python move_files.py <destination_dir> <source1> [source2 ...]

Example:
    python move_files.py /home/user/Archive report.txt notes.txt
"""

import argparse
import os
import shutil
import sys


def move_files(sources, destination: str) -> None:
    if not os.path.exists(destination):
        os.makedirs(destination)
        print(f"Created destination directory: {destination}")

    if not os.path.isdir(destination):
        print(f"Error: destination '{destination}' is not a directory.")
        sys.exit(1)

    for src in sources:
        if not os.path.exists(src):
            print(f"Skipped (not found): {src}")
            continue

        try:
            shutil.move(src, destination)
            print(f"Moved: '{src}' -> '{destination}'")
        except (OSError, shutil.Error) as e:
            print(f"Error moving '{src}': {e}")


def main():
    parser = argparse.ArgumentParser(description="Move files/folders into a destination directory.")
    parser.add_argument("destination", help="Destination directory")
    parser.add_argument("sources", nargs="+", help="File(s)/folder(s) to move")
    args = parser.parse_args()

    move_files(args.sources, args.destination)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 move_files.py /home/user/Archive report.txt notes.txt
```

The script ensures the destination exists (creating it if necessary), then hands each source — file or directory — to `shutil.move()`. On the same filesystem this is a fast rename; across filesystems, `shutil.move()` transparently falls back to copying the data then deleting the original.

#### Sample Output (Verified)

```bash
$ python3 move_files.py Archive report.txt notes.txt
Created destination directory: Archive
Moved: 'report.txt' -> 'Archive'
Moved: 'notes.txt' -> 'Archive'

$ ls Archive
notes.txt  report.txt
$ ls report.txt
ls: cannot access 'report.txt': No such file or directory
```

Confirmed: source files no longer exist at their original location after the move.

#### Variations / Flags

- Add `--overwrite-policy skip|rename|overwrite` to make same-named-destination behavior explicit rather than defaulting to silent overwrite.
- Add `--dry-run` to preview the moves before committing, especially useful when moving many files at once.
- Combine with `find_files_by_extension.py` (3.11) to move only files matching a specific type into an archive folder.

#### Common Pitfalls

- **Silent overwrite at destination** — same risk as `copy_files.py` (3.6); check for name collisions before running against a populated destination.
- **Moving a folder into a destination that already contains a same-named folder** raises `shutil.Error` rather than merging — the error is caught and printed per-item, but the move for that item won't happen.
- **Partial batch failures don't stop the whole run** — one failed move (permission error, name collision) is reported and the script continues to the next source, so always check the full output rather than assuming success from the absence of a fatal error.

### 3.6 Copy Files

#### Use Case

A sysadmin needs to gather a handful of scattered report and log files into a single backup folder before they get rotated out or deleted elsewhere, without manually creating the destination directory first.

#### Prerequisites

Standard library only (`shutil`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

`shutil.copy2()` silently overwrites a file of the same name already in the destination — there's no confirmation prompt or `--force` flag needed because overwriting is simply the default behavior. Be careful running this against a destination that already holds files with the same names as your sources.

#### Script

```python
#!/usr/bin/env python3
"""
copy_files.py
---------------
Copies one or more files into a destination directory.

Usage:
    python copy_files.py <destination_dir> <source1> [source2 ...]

Example:
    python copy_files.py /home/user/Backup report.txt notes.txt
"""

import argparse
import os
import shutil
import sys


def copy_files(sources, destination: str) -> None:
    if not os.path.exists(destination):
        os.makedirs(destination)
        print(f"Created destination directory: {destination}")

    if not os.path.isdir(destination):
        print(f"Error: destination '{destination}' is not a directory.")
        sys.exit(1)

    for src in sources:
        if not os.path.isfile(src):
            print(f"Skipped (not a file or not found): {src}")
            continue

        try:
            shutil.copy2(src, destination)
            print(f"Copied: '{src}' -> '{destination}'")
        except (OSError, shutil.Error) as e:
            print(f"Error copying '{src}': {e}")


def main():
    parser = argparse.ArgumentParser(description="Copy files into a destination directory.")
    parser.add_argument("destination", help="Destination directory")
    parser.add_argument("sources", nargs="+", help="File(s) to copy")
    args = parser.parse_args()

    copy_files(args.sources, args.destination)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 copy_files.py /home/user/Backup report.txt notes.txt
```

The script first ensures the destination directory exists, creating it with `os.makedirs()` if not. It then loops over each source path, skipping anything that isn't a regular file, and copies the rest with `shutil.copy2()`, which preserves metadata (timestamps, permission bits) in addition to content — unlike plain `shutil.copy()`.

#### Sample Output (Verified)

```bash
$ python3 copy_files.py Backup report.txt notes.txt
Created destination directory: Backup
Copied: 'report.txt' -> 'Backup'
Copied: 'notes.txt' -> 'Backup'

$ ls Backup
notes.txt
report.txt
```

#### Variations / Flags

- Add `--rename-on-conflict` to append a numeric suffix (`report(1).txt`) instead of silently overwriting same-named files.
- Add glob support (`copy_files.py Backup *.log`) by expanding patterns with the `glob` module before passing them in, since not all shells expand globs the same way.
- Add `--flatten` vs `--preserve-structure` options if sources come from different subdirectories and you want to control whether directory structure is preserved under the destination.

#### Common Pitfalls

- **Silent overwrite** is the single biggest surprise here — always double check destination contents first if source and destination might share filenames.
- **`shutil.copy2` doesn't copy ownership (UID/GID)**, only timestamps and permission bits — copied files will be owned by whoever ran the script.
- **Copying a directory path by mistake**: passing a directory as one of the `sources` is silently skipped ("not a file or not found") rather than erroring loudly — easy to miss in a long batch.
- **Destination as a file, not a directory**: if `destination` happens to already exist as a regular file (not a folder), the script correctly detects this and exits with an error rather than corrupting anything.

## 3.7 Copy Directories

#### Use Case

A sysadmin is about to modify configuration files or run a risky script inside a project directory and wants a complete, exact copy of the current state first, so anything can be rolled back by simply restoring the copy.

#### Prerequisites

Standard library only (`shutil`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

`shutil.copytree()` **refuses to run if the destination already exists** — this script exits with an error rather than silently merging or overwriting, which is a deliberate safety behavior worth understanding before scripting around it. Copying very large trees can also be slow and disk-hungry since every byte is duplicated.

#### Script

```python
#!/usr/bin/env python3
"""
copy_directories.py
----------------------
Recursively copies an entire directory tree to a new location.

Usage:
    python copy_directories.py <source_dir> <destination_dir>

Example:
    python copy_directories.py /home/user/Project /home/user/Project_backup
"""

import argparse
import os
import shutil
import sys


def copy_directory(source: str, destination: str) -> None:
    if not os.path.isdir(source):
        print(f"Error: source '{source}' is not a valid directory.")
        sys.exit(1)

    if os.path.exists(destination):
        print(f"Error: destination '{destination}' already exists.")
        sys.exit(1)

    try:
        shutil.copytree(source, destination)
        print(f"Copied directory: '{source}' -> '{destination}'")
    except (OSError, shutil.Error) as e:
        print(f"Error copying directory: {e}")
        sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Recursively copy a directory tree.")
    parser.add_argument("source", help="Source directory")
    parser.add_argument("destination", help="Destination directory (must not already exist)")
    args = parser.parse_args()

    copy_directory(args.source, args.destination)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 copy_directories.py /home/user/Project /home/user/Project_backup
```

The script validates that the source exists and the destination does not, then delegates entirely to `shutil.copytree()`, which recursively copies files, subdirectories, and permission bits, creating the destination directory structure from scratch.

#### Sample Output (Verified)

```bash
$ find Project
Project/sub
Project/sub/b.txt
Project/a.txt

$ python3 copy_directories.py Project Project_backup
Copied directory: 'Project' -> 'Project_backup'

$ find Project_backup
Project_backup
Project_backup/sub
Project_backup/sub/b.txt
Project_backup/a.txt
```

#### Variations / Flags

- Add `--overwrite` that removes an existing destination with `shutil.rmtree()` first (with a confirmation prompt) if you specifically want merge/replace behavior instead of the default refuse-if-exists safety.
- Pass `dirs_exist_ok=True` to `shutil.copytree()` (Python 3.8+) behind a `--merge` flag to copy into an existing destination without erroring.
- Add `ignore=shutil.ignore_patterns('*.log', '__pycache__')` to skip junk files during the copy, similar in spirit to `clean_temporary_files.py` (3.30).

#### Common Pitfalls

- **Destination-exists error is by design, not a bug** — new users often expect this script to behave like `cp -r`, which merges into an existing folder. It won't; you must give it a destination path that doesn't exist yet.
- **Symlinks are followed by default** (`symlinks=False` in `copytree`), meaning a symlink in the source tree gets replaced with a copy of its *target's* content in the destination — not preserved as a symlink. Pass `symlinks=True` if you need the reverse.
- **Permissions and ownership**: `copytree` preserves file permission bits but not ownership (UID/GID) across users — running as a different user than the original owner can produce a copy owned by whoever ran the script.
- **Partial failures**: if the copy fails partway through (e.g., disk full, permission denied on one file), `shutil.Error` is raised with details on all failures, but a partially-copied destination directory may be left behind — check before assuming a failed run left nothing.

### 3.8 Delete Directories

#### Use Case

A sysadmin needs to remove an entire old project folder, a failed build output directory, or a decommissioned application's data directory — all contents included — without manually deleting files one by one.

#### Prerequisites

Standard library only (`shutil`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

This is one of the most destructive scripts in the chapter. `shutil.rmtree()` recursively deletes every file and subdirectory inside the target with no per-item confirmation and, critically, **no recycle bin / trash — deletion is immediate and permanent**. Always double-check the path before confirming, and strongly prefer running without `--force` the first time you point this at a new directory.

#### Script

```python
#!/usr/bin/env python3
"""
delete_directories.py
------------------------
Deletes a directory and all its contents. Asks for confirmation unless --force is used.

Usage:
    python delete_directories.py <directory_path> [--force]

Example:
    python delete_directories.py old_project
    python delete_directories.py old_project --force
"""

import argparse
import os
import shutil
import sys


def delete_directory(path: str, force: bool = False) -> None:
    if not os.path.exists(path):
        print(f"Error: '{path}' does not exist.")
        sys.exit(1)

    if not os.path.isdir(path):
        print(f"Error: '{path}' is not a directory.")
        sys.exit(1)

    if not force:
        answer = input(f"Delete directory '{path}' and ALL its contents? [y/N]: ").strip().lower()
        if answer != "y":
            print("Aborted.")
            return

    try:
        shutil.rmtree(path)
        print(f"Deleted directory: {path}")
    except OSError as e:
        print(f"Error deleting directory: {e}")
        sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Delete a directory and all its contents.")
    parser.add_argument("path", help="Directory path to delete")
    parser.add_argument("--force", action="store_true", help="Delete without confirmation prompt")
    args = parser.parse_args()

    delete_directory(args.path, args.force)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Interactive — asks for confirmation
python3 delete_directories.py old_project

# Non-interactive, for scripts/cron jobs where the safety check has already happened elsewhere
python3 delete_directories.py old_project --force
```

The script validates the path exists and is actually a directory (not a file), prompts for a single `y/N` confirmation unless `--force` is set, and then calls `shutil.rmtree()`, which recursively removes the directory and everything inside it in one operation.

#### Sample Output (Verified)

```bash
$ python3 delete_directories.py old_project --force
Deleted directory: old_project

$ ls old_project
ls: cannot access 'old_project': No such file or directory
```

#### Variations / Flags

- Add `--dry-run` to list every file and subdirectory that would be deleted (via `os.walk()`) before any confirmation prompt, giving a full preview for large or unfamiliar trees.
- Add a "move to trash" mode using the `send2trash` package instead of `shutil.rmtree()`, giving a recoverable safety net for interactive use.
- Log every deleted path to an audit file before deletion, useful for compliance or post-incident review in shared environments.

#### Common Pitfalls

- **No undo.** This is the single most important thing to internalize about this script — there is no OS-level recycle bin involved, `shutil.rmtree()` deletes immediately and permanently.
- **`--force` bypasses the only safety net** — reserve it for well-tested automation, not ad hoc interactive use, and never as a copy-paste default in a new script you haven't reviewed.
- **Symlinked directories**: `shutil.rmtree()` will refuse to operate directly on a path that is itself a symlink to a directory (raises `OSError` by design, as a safety feature) — but it *will* follow symlinks it encounters while descending into a real directory's contents, which can lead to deleting files outside the intended tree if you're not careful about symlinks inside the target.
- **Permission errors mid-delete** can leave a partially-deleted directory behind — check the error message and the directory's remaining contents rather than assuming a failed run left everything untouched.

### 3.9 Create Nested Directories

#### Use Case

A sysadmin is setting up a dated folder structure for logs or reports (e.g., `projects/2026/january/reports`) and needs every intermediate level created automatically, without manually creating and stepping into each parent folder.

#### Cautions

#### Script

```python
#!/usr/bin/env python3
"""
create_nested_directories.py

Creates a nested directory structure (like mkdir -p).

def main():
```

```bash
python3 create_nested_directories.py projects/2026/january/reports
```

`os.makedirs(path, exist_ok=True)` creates every missing directory in the path in one call — equivalent to running `mkdir -p` in a shell — and does not raise an error if some or all of the path already exists.

#### Sample Output (Verified)


#### Variations / Flags

- Add `--mode 0o750` to control permission bits on the newly created directories (`os.makedirs(path, mode=..., exist_ok=True)`), noting that `mode` only affects the *final* directory on some platforms due to umask interaction — worth testing on your target OS.

- Accept multiple paths in one invocation (`nargs="+"`) to batch-create several directory trees at once.

#### Common Pitfalls

- **No feedback on what was actually new** — because `exist_ok=True` suppresses the "already exists" error, you can't tell from the output whether the whole path was freshly created or already there.

### 3.10 Traverse Directories Recursively

#### Use Case

A sysadmin needs a complete picture of a directory's structure — every subfolder and file, nested arbitrarily deep — for documentation, auditing, or just to understand an unfamiliar project layout before making changes.

#### Prerequisites

Standard library only (`os`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

On very large trees (tens of thousands of files), the printed output can be enormous and slow to scroll through — consider redirecting to a file (`> tree.txt`) or combining with `--extension`/filtering variations for large codebases or data directories.

#### Script

```python
#!/usr/bin/env python3
"""
traverse_directories.py
---------------------------
Recursively walks through a directory tree and prints every file and folder.

Usage:
    python traverse_directories.py <directory_path>

Example:
    python traverse_directories.py /home/user/Documents
"""

import argparse
import os
import sys


def traverse_directory(path: str) -> None:
    if not os.path.isdir(path):
        print(f"Error: '{path}' is not a valid directory.")
        sys.exit(1)

    file_count = 0
    dir_count = 0

    for root, dirs, files in os.walk(path):
        level = root.replace(path, "").count(os.sep)
        indent = "  " * level
        print(f"{indent}[DIR] {os.path.basename(root) or root}")

        sub_indent = "  " * (level + 1)
        for f in files:
            print(f"{sub_indent}{f}")
            file_count += 1
        dir_count += len(dirs)

    print(f"\nTotal directories: {dir_count + 1}")
    print(f"Total files: {file_count}")


def main():
    parser = argparse.ArgumentParser(description="Recursively traverse a directory tree.")
    parser.add_argument("path", help="Directory to traverse")
    args = parser.parse_args()

    traverse_directory(args.path)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 traverse_directories.py /home/user/Documents
```

`os.walk()` yields every directory in the tree top-down along with its immediate subdirectories and files. The script computes an indentation level from how many path separators appear between the root path and the current directory, printing a visually nested tree, and tallies running totals for files and directories along the way.

#### Sample Output (Verified)

```bash
$ python3 traverse_directories.py docs2
[DIR] docs2
  x.txt
  [DIR] sub1
    y.txt
    [DIR] sub2
      z.txt

Total directories: 3
Total files: 3
```

#### Variations / Flags

- Add `--max-depth 2` to stop descending beyond a given nesting level, useful for a quick overview of a huge tree without full output.
- Add `--exclude __pycache__,node_modules` to skip noisy directories, mirroring the exclusion logic in `clean_temporary_files.py` (3.30).
- Add file size annotations per entry (reusing `os.path.getsize()`), turning this into a lightweight `du`-style tool.

#### Common Pitfalls

- **The indentation logic depends on `os.sep`**, which differs between Windows (`\`) and Unix (`/`) — the script's `root.replace(path, "").count(os.sep)` approach works correctly on both since it uses the platform's actual separator, but custom modifications that hardcode `/` will break on Windows.
- **Symlinked directories are not followed by default** (`os.walk()`'s `followlinks=False` default) — this avoids infinite loops from circular symlinks but means linked content won't appear in the tree unless `followlinks=True` is explicitly set.
- **No filtering built in** — every file and folder is printed regardless of type; for large trees this can produce output that's more overwhelming than useful without one of the variations above.

### 3.11 Find Files by Extension

#### Use Case

A sysadmin needs to find every PDF in a Documents folder before an archive run, or every `.log` file across a deep application directory tree to feed into a cleanup or analysis pipeline — without manually browsing each subfolder.

#### Prerequisites

Standard library only (`os`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

Matching is a simple case-insensitive suffix check (`str.endswith()`), not a true extension parser — a file named `report.xlsx` would also match a search for `.x` if you passed that, so keep the extension argument precise.

#### Script

```python
#!/usr/bin/env python3
"""
find_files_by_extension.py
-------------------------------
Searches a directory tree for files matching a given extension.

Usage:
    python find_files_by_extension.py <directory_path> <extension>

Example:
    python find_files_by_extension.py /home/user/Documents .pdf
"""

import argparse
import os
import sys


def find_files_by_extension(path: str, extension: str) -> None:
    if not os.path.isdir(path):
        print(f"Error: '{path}' is not a valid directory.")
        sys.exit(1)

    if not extension.startswith("."):
        extension = "." + extension

    matches = []

    for root, _, files in os.walk(path):
        for f in files:
            if f.lower().endswith(extension.lower()):
                matches.append(os.path.join(root, f))

    if matches:
        print(f"Found {len(matches)} file(s) with extension '{extension}':\n")
        for m in matches:
            print(m)
    else:
        print(f"No files with extension '{extension}' found in '{path}'.")


def main():
    parser = argparse.ArgumentParser(description="Find files by extension in a directory tree.")
    parser.add_argument("path", help="Directory to search")
    parser.add_argument("extension", help="File extension to search for (e.g. .txt or txt)")
    args = parser.parse_args()

    find_files_by_extension(args.path, args.extension)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 find_files_by_extension.py /home/user/Documents .pdf
```

The script normalizes the extension argument (adding a leading `.` if missing), then walks the entire directory tree with `os.walk()`, collecting every file whose lowercased name ends with the lowercased extension. Results are printed as full relative paths.

#### Sample Output (Verified)

```bash
$ python3 find_files_by_extension.py docstest .pdf
Found 2 file(s) with extension '.pdf':

docstest/a.pdf
docstest/b.pdf
```

`docstest/c.txt` was correctly excluded.

#### Variations / Flags

- Add support for multiple extensions in one call (`--extensions .pdf,.docx`) for a broader "documents" style search.
- Pipe the resulting paths directly into `compress_files.py` (3.21) or `move_files.py` (3.5) to archive or relocate matches in one pipeline.
- Add `--min-size` / `--max-size` filters to narrow results by file size alongside extension.

#### Common Pitfalls

- **Suffix matching, not true extension parsing** — a search for `.ar` would incorrectly match `report.tar` (since `"report.tar".endswith(".ar")` is False actually — but a search for `.tar` would match `backup.tar` correctly; the risk is with short/ambiguous extension strings that could accidentally match unrelated suffixes, so always include the leading dot and be as specific as possible).
- **Case sensitivity is intentionally relaxed** — `.PDF` and `.pdf` both match, which is usually desired but can surprise users expecting exact-case matching on case-sensitive filesystems.
- **No result limit** — searching a massive tree for a very common extension (e.g., `.txt`) can produce thousands of lines of output; redirect to a file for large searches.

### 3.12 Find Duplicate Files

#### Use Case

A sysadmin suspects a Downloads folder or shared drive has accumulated duplicate copies of the same files under different names, and wants a reliable, content-based (not just name-based) way to find and review them before deleting anything.

#### Prerequisites

Standard library only (`hashlib`, `collections`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

This script only *identifies* duplicates — it doesn't delete or offer to delete any of them, which is the correct default for something this potentially destructive. On very large directory trees, hashing every file can take a long time and a lot of I/O; consider filtering by size first (files of different sizes can never be duplicates) before hashing, for large-scale use.

#### Script

```python
#!/usr/bin/env python3
"""
find_duplicate_files.py
-----------------------------
Finds duplicate files in a directory tree by comparing content hashes (SHA-256).

Usage:
    python find_duplicate_files.py <directory_path>

Example:
    python find_duplicate_files.py /home/user/Downloads
"""

import argparse
import hashlib
import os
import sys
from collections import defaultdict


def hash_file(path: str, block_size: int = 65536) -> str:
    hasher = hashlib.sha256()
    with open(path, "rb") as f:
        while chunk := f.read(block_size):
            hasher.update(chunk)
    return hasher.hexdigest()


def find_duplicates(path: str) -> None:
    if not os.path.isdir(path):
        print(f"Error: '{path}' is not a valid directory.")
        sys.exit(1)

    hashes = defaultdict(list)

    for root, _, files in os.walk(path):
        for f in files:
            full_path = os.path.join(root, f)
            try:
                file_hash = hash_file(full_path)
                hashes[file_hash].append(full_path)
            except (OSError, PermissionError) as e:
                print(f"Skipping '{full_path}': {e}")

    duplicates_found = False
    for file_hash, paths in hashes.items():
        if len(paths) > 1:
            duplicates_found = True
            print(f"\nDuplicate group (hash {file_hash[:12]}...):")
            for p in paths:
                print(f"  {p}")

    if not duplicates_found:
        print("No duplicate files found.")


def main():
    parser = argparse.ArgumentParser(description="Find duplicate files by content hash.")
    parser.add_argument("path", help="Directory to search for duplicates")
    args = parser.parse_args()

    find_duplicates(args.path)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 find_duplicate_files.py /home/user/Downloads
```

The script walks the entire tree, computing a SHA-256 hash for every file's full content (reading in 64 KB chunks to stay memory-efficient, the same pattern used in `calculate_file_hashes.py`, 3.14). Files are grouped by hash in a dictionary; any hash with more than one associated path represents a set of byte-for-byte identical files, printed together as a "duplicate group."

#### Sample Output (Verified)

```bash
$ python3 find_duplicate_files.py duptest
Duplicate group (hash f953bbd204bb...):
  duptest/file2.txt
  duptest/file1.txt
```

`duptest/file3.txt`, which had different content, was correctly excluded from the duplicate group.

#### Variations / Flags

- Add a size pre-filter (group by `os.path.getsize()` first, only hash files that share a size with at least one other file) to dramatically speed up scans of large directories, since hashing is the expensive step.
- Add `--delete-duplicates --keep-first` to automatically remove all but one file per duplicate group — pair with a confirmation prompt or `--dry-run` given how destructive this would be.
- Add `--min-size 1024` to skip hashing trivially small files (e.g., empty files, which are all "duplicates" of each other by definition and rarely useful to report).

#### Common Pitfalls

- **Every empty file in the tree will be reported as one giant duplicate group** — an empty file's SHA-256 hash is always the same constant value, so a tree with many zero-byte placeholder files (like `.gitkeep`) will show them all grouped together, which is technically correct but rarely the duplicates you're looking for.
- **Full-content hashing is thorough but slow** — for very large files or trees, this is I/O and CPU intensive; the size-prefilter variation above is worth implementing before running this against anything beyond a few thousand files.
- **This script never deletes anything on its own** — treat its output as a report to review manually or feed into a separate, more careful deletion step.

### 3.13 Search File Contents

#### Use Case

A sysadmin needs to find every place a particular string appears across a codebase or config directory — for example, locating all `TODO` comments before a release, or finding every file that references a deprecated hostname.

#### Prerequisites

Standard library only (`os`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

Every file that matches the (optional) extension filter is opened and read as text — pointed at a directory containing large binary files, this can be slow and will produce garbled or meaningless "matches" from binary content interpreted as text (mitigated somewhat by `errors="ignore"`, but not eliminated). Always narrow with `--extension` when searching a mixed-content directory.

#### Script

```python
#!/usr/bin/env python3
"""
search_file_contents.py
-----------------------------
Searches for a text string inside all files in a directory tree (like grep -r).

Usage:
    python search_file_contents.py <directory_path> <search_text> [--extension .txt]

Example:
    python search_file_contents.py /home/user/Project "TODO"
    python search_file_contents.py /home/user/Project "import os" --extension .py
"""

import argparse
import os
import sys


def search_file_contents(path: str, search_text: str, extension: str = None) -> None:
    if not os.path.isdir(path):
        print(f"Error: '{path}' is not a valid directory.")
        sys.exit(1)

    matches_found = 0

    for root, _, files in os.walk(path):
        for f in files:
            if extension and not f.lower().endswith(extension.lower()):
                continue

            full_path = os.path.join(root, f)
            try:
                with open(full_path, "r", encoding="utf-8", errors="ignore") as file_obj:
                    for line_num, line in enumerate(file_obj, start=1):
                        if search_text in line:
                            print(f"{full_path}:{line_num}: {line.strip()}")
                            matches_found += 1
            except (OSError, PermissionError):
                continue

    if matches_found == 0:
        print(f"No matches found for '{search_text}' in '{path}'.")
    else:
        print(f"\nTotal matches: {matches_found}")


def main():
    parser = argparse.ArgumentParser(description="Search for text inside all files in a directory tree.")
    parser.add_argument("path", help="Directory to search")
    parser.add_argument("search_text", help="Text string to search for")
    parser.add_argument("--extension", default=None, help="Only search files with this extension (e.g. .py)")
    args = parser.parse_args()

    search_file_contents(args.path, args.search_text, args.extension)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Search every file
python3 search_file_contents.py /home/user/Project "TODO"

# Narrow to Python files only
python3 search_file_contents.py /home/user/Project "import os" --extension .py
```

The script walks the directory tree, optionally filtering by file extension, and opens each candidate file as UTF-8 text (silently ignoring undecodable bytes). It reads line by line, checking for a plain substring match, and prints `path:line_number: line_content` for each hit — a format familiar to anyone who's used `grep -n`.

#### Sample Output (Verified)

```bash
$ python3 search_file_contents.py searchtest "TODO" --extension .py
searchtest/a.py:2: TODO fix this

Total matches: 1
```

`searchtest/b.py`, which had no match, correctly contributed nothing to the output.

#### Variations / Flags

- Add `--regex` to support pattern matching via the `re` module instead of plain substring search.
- Add `--case-insensitive` to lowercase both the search text and each line before comparing.
- Add `--context 2` to print surrounding lines around each match, similar to `grep -C`.

#### Common Pitfalls

- **Plain substring search, not regex** — special characters in the search text are matched literally, which is usually what you want but means you can't search for patterns (e.g., "any TODO followed by a name") without the `--regex` variation.
- **Binary files can produce noisy false "matches"** — even with `errors="ignore"`, decoded binary content can occasionally contain your search string by coincidence; always scope with `--extension` on directories with mixed content types.
- **Performance on huge trees**: every matching-extension file is opened and read fully, line by line, with no early exit — for a very large directory this can be slow; consider first narrowing scope with `find_files_by_extension.py` (3.11) if you need to search a huge tree repeatedly.

### 3.14 Calculate File Hashes

#### Use Case

A sysadmin downloads a large ISO or backup archive and needs to confirm it wasn't corrupted in transit by comparing its checksum against a published value. Hashes are also used to fingerprint files for deduplication, to prove a file hasn't changed since it was signed off, or to verify integrity after copying data between servers.

#### Prerequisites

Standard library only (`hashlib`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

MD5 and SHA1 are fast but are not cryptographically secure against a determined attacker — use them only for accidental-corruption checks, not for verifying a file hasn't been maliciously tampered with. For anything security-sensitive, use SHA256 (the script's default).

#### Script

```python
#!/usr/bin/env python3
"""
calculate_file_hashes.py
------------------------------
Calculates the hash (MD5, SHA1, or SHA256) of one or more files.

Usage:
    python calculate_file_hashes.py <file1> [file2 ...] [--algo sha256]

Example:
    python calculate_file_hashes.py document.pdf
    python calculate_file_hashes.py a.txt b.txt --algo md5
"""

import argparse
import hashlib
import os
import sys


def calculate_hash(file_path: str, algorithm: str) -> str:
    hasher = hashlib.new(algorithm)
    with open(file_path, "rb") as f:
        while chunk := f.read(65536):
            hasher.update(chunk)
    return hasher.hexdigest()


def calculate_file_hashes(file_paths, algorithm: str) -> None:
    for file_path in file_paths:
        if not os.path.isfile(file_path):
            print(f"Skipped (not found): {file_path}")
            continue

        try:
            digest = calculate_hash(file_path, algorithm)
            print(f"{algorithm.upper()}  {digest}  {file_path}")
        except (OSError, ValueError) as e:
            print(f"Error hashing '{file_path}': {e}")


def main():
    parser = argparse.ArgumentParser(description="Calculate file hashes (MD5/SHA1/SHA256).")
    parser.add_argument("files", nargs="+", help="File path(s) to hash")
    parser.add_argument(
        "--algo", default="sha256", choices=["md5", "sha1", "sha256"], help="Hash algorithm (default: sha256)"
    )
    args = parser.parse_args()

    calculate_file_hashes(args.files, args.algo)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Default: SHA256
python3 calculate_file_hashes.py document.pdf

# Multiple files, MD5
python3 calculate_file_hashes.py a.txt b.txt --algo md5
```

The script reads each file in 64 KB chunks (so it never loads a huge file entirely into memory), feeds each chunk into the chosen hash algorithm, and prints the final hex digest alongside the algorithm name and file path.

#### Sample Output (Verified)


#### Variations / Flags

- Add `--check <file>` mode that reads a `hash  filename` manifest (like `sha256sum -c`) and reports PASS/FAIL for each entry.
- Add `sha512` or `blake2b` to the `choices` list for stronger or faster alternatives.
- For very large directories, combine with `os.walk()` and write results straight to a `.sha256` manifest file instead of stdout.

#### Common Pitfalls

- **`hashlib.new(algorithm)` will raise `ValueError` for unsupported algorithm names** — the `--algo` choices list already guards against typos, but a custom fork that removes that constraint should keep validating input.
- **Comparing hashes across platforms is case-sensitive as text but not as data** — always lowercase both sides before comparing strings.
- **A matching hash confirms integrity, not authenticity** — the file wasn't corrupted, but a matching hash alone doesn't prove who produced it unless the hash came from a trusted, separately-verified source.

### 3.15 Compare Two Files

#### Use Case

A sysadmin has two versions of a config file — say, one from before a change and one from after — and needs to confirm whether anything actually changed, and if so, see the precise lines that differ before deciding whether to roll back.

#### Prerequisites

Standard library only (`filecmp`, `difflib`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

The line-by-line diff only works for text files; binary files (images, executables, compiled archives) will report as differing but without a usable diff. The script also decodes text as UTF-8 with `errors="ignore"`, which silently drops or mangles bytes that aren't valid UTF-8 — fine for typical config/log files, but not a safe way to compare files in other encodings.

#### Script

```python
#!/usr/bin/env python3
"""
compare_two_files.py
--------------------------
Compares two files and reports whether they are identical, plus a line-by-line diff for text files.

Usage:
    python compare_two_files.py <file1> <file2>

Example:
    python compare_two_files.py old_version.txt new_version.txt
"""

import argparse
import difflib
import filecmp
import os
import sys


def compare_files(file1: str, file2: str) -> None:
    for f in (file1, file2):
        if not os.path.isfile(f):
            print(f"Error: '{f}' does not exist or is not a file.")
            sys.exit(1)

    identical = filecmp.cmp(file1, file2, shallow=False)

    if identical:
        print(f"Files are IDENTICAL: '{file1}' == '{file2}'")
        return

    print(f"Files DIFFER: '{file1}' != '{file2}'\n")

    try:
        with open(file1, "r", encoding="utf-8", errors="ignore") as f1, \
             open(file2, "r", encoding="utf-8", errors="ignore") as f2:
            lines1 = f1.readlines()
            lines2 = f2.readlines()

        diff = difflib.unified_diff(lines1, lines2, fromfile=file1, tofile=file2, lineterm="")
        diff_output = list(diff)

        if diff_output:
            print("Line-by-line diff:")
            for line in diff_output:
                print(line)
        else:
            print("Binary or non-text files: content differs but no text diff available.")
    except UnicodeDecodeError:
        print("Could not generate text diff (binary file detected).")


def main():
    parser = argparse.ArgumentParser(description="Compare two files and show differences.")
    parser.add_argument("file1", help="First file")
    parser.add_argument("file2", help="Second file")
    args = parser.parse_args()

    compare_files(args.file1, args.file2)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 compare_two_files.py old_version.txt new_version.txt
```

`filecmp.cmp(..., shallow=False)` forces a full byte-for-byte comparison (unlike the default directory comparison in 3.16, this script deliberately opts out of the shallow shortcut). If the files differ, it reads both as text and feeds them into `difflib.unified_diff()` to produce a standard `---`/`+++`/`@@` style diff.

#### Sample Output (Verified)

```bash
$ printf "line1\nline2\nline3\n" > old.txt
$ printf "line1\nline2-changed\nline3\nline4\n" > new.txt
$ python3 compare_two_files.py old.txt new.txt
Files DIFFER: 'old.txt' != 'new.txt'

Line-by-line diff:
--- old.txt
+++ new.txt
@@ -1,3 +1,4 @@
 line1

-line2

+line2-changed

 line3

+line4

```

Note the extra blank line after every diff row — since `readlines()` keeps each line's trailing `\n` and the script's own `print()` adds another, every row is followed by a blank line. Purely cosmetic, but worth knowing before piping this output into another tool that expects strict unified-diff format.

#### Variations / Flags

- Strip the double-newline artifact by passing `lines1 = [l.rstrip("\n") for l in f1]` (and the same for `lines2`) before diffing, or by writing diff lines with `print(line, end="")`/`sys.stdout.write`.
- Add `--context 5` to control how many surrounding lines `unified_diff` shows (its `n=` parameter).
- Add a `--side-by-side` mode using `difflib.HtmlDiff` for a browser-viewable comparison.

#### Common Pitfalls

- **Binary files report as differing with no diff body** — the script correctly detects this (falls through to "Binary or non-text files..." or catches `UnicodeDecodeError`), but don't expect a byte-level diff for binaries; use a dedicated binary diff tool if you need that.
- **`errors="ignore"` hides real content** — if a file has stray non-UTF-8 bytes, those bytes are silently dropped from the diff view, which can make a file look identical when it technically isn't.
- **Trailing newline differences count** — a file with vs. without a final newline will be reported as DIFFER, which surprises people comparing hand-edited config files.

### 3.16 Compare Directories

#### Use Case

A sysadmin has two copies of a project — say, a production directory and a staging directory — and needs to know which files exist only in one, which are missing from the other, and which have diverged, before deciding what to sync or overwrite.

#### Prerequisites

Standard library only (`filecmp`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

`filecmp.dircmp` performs a **shallow** comparison by default — it compares file size and modification time, not actual byte content. Two files with identical size and mtime but different content can be silently reported as identical (or simply not flagged as different). For anything where content accuracy matters, cross-check suspect files with `compare_two_files.py` (3.15), which does a full byte-for-byte comparison.

#### Script

```python
#!/usr/bin/env python3
"""
compare_directories.py
----------------------------
Compares two directories and reports files that are only in one, or that differ.

Usage:
    python compare_directories.py <directory1> <directory2>

Example:
    python compare_directories.py /home/user/ProjectA /home/user/ProjectB
"""

import argparse
import filecmp
import os
import sys


def compare_directories(dir1: str, dir2: str) -> None:
    if not os.path.isdir(dir1) or not os.path.isdir(dir2):
        print("Error: both arguments must be valid directories.")
        sys.exit(1)

    comparison = filecmp.dircmp(dir1, dir2)
    _print_report(comparison, dir1, dir2)


def _print_report(comparison: filecmp.dircmp, dir1: str, dir2: str, indent: str = "") -> None:
    if comparison.left_only:
        print(f"{indent}Only in {dir1}:")
        for item in sorted(comparison.left_only):
            print(f"{indent}  {item}")

    if comparison.right_only:
        print(f"{indent}Only in {dir2}:")
        for item in sorted(comparison.right_only):
            print(f"{indent}  {item}")

    if comparison.diff_files:
        print(f"{indent}Files that differ:")
        for item in sorted(comparison.diff_files):
            print(f"{indent}  {item}")

    if comparison.same_files:
        print(f"{indent}Identical files: {len(comparison.same_files)}")

    for sub_dir, sub_comparison in comparison.subdirs.items():
        print(f"{indent}Subdirectory: {sub_dir}")
        _print_report(sub_comparison, os.path.join(dir1, sub_dir), os.path.join(dir2, sub_dir), indent + "  ")


def main():
    parser = argparse.ArgumentParser(description="Compare two directories recursively.")
    parser.add_argument("directory1", help="First directory")
    parser.add_argument("directory2", help="Second directory")
    args = parser.parse_args()

    compare_directories(args.directory1, args.directory2)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 compare_directories.py /home/user/ProjectA /home/user/ProjectB
```

`filecmp.dircmp()` builds a comparison object for the two top-level directories, categorizing entries into "only in dir1," "only in dir2," "files that differ," and "identical files." The script then recurses into `comparison.subdirs` to print the same report for every matching subdirectory, indenting each level for readability.

#### Sample Output (Verified)

```bash
$ python3 compare_directories.py dirA dirB
Only in dirA:
  onlyA.txt
Identical files: 2
```

**Verified caveat in action:** in this test run, `dirA/diff.txt` and `dirB/diff.txt` had different *content* ("v1" vs "v2") but were created in the same second and happened to be the same byte size — so `filecmp`'s shallow comparison counted them as identical rather than listing them under "Files that differ." This is exactly the shallow-comparison limitation described above, not a bug in the script.

#### Variations / Flags

- Force a full content comparison instead of the default shallow one by calling `filecmp.cmpfiles(dir1, dir2, common_files, shallow=False)` in place of relying on `dircmp.diff_files`.
- Add `--summary-only` to suppress subdirectory recursion and just print top-level counts.
- Pipe `comparison.left_only` / `right_only` into `copy_files.py` (3.6) to auto-sync one directory toward matching the other.

#### Common Pitfalls

- **Shallow comparison false negatives** (see Cautions) — the single most common surprise with this script. If exact content accuracy matters, don't trust `diff_files` alone.
- **`common_funny` and `funny_files`** attributes on the `dircmp` object (entries that couldn't be compared, e.g., due to permission errors) are not surfaced by this script — silent comparison failures won't show up in the printed report.
- **Symlinks** are compared as their target's stat info by default, which can be surprising if you expected symlink-vs-symlink identity checks.
- **Large trees**: `dircmp` builds the comparison recursively in memory; on extremely large directory trees (hundreds of thousands of files) this can be slow — consider `rsync --dry-run` for very large-scale comparisons instead.


### 3.17 Get File Metadata

#### Use Case

A sysadmin needs to quickly check when a file was last modified, what permissions it has, or whether a suspicious path is actually a symlink — the kind of forensic detail that's tedious to gather with several separate commands.

#### Prerequisites

Standard library only (`os`, `stat`) — no installation required. Works on Linux, macOS, and Windows, though see the caution below about `st_ctime`'s different meaning across platforms.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

On Windows, `st_ctime` genuinely reflects file *creation* time; on Linux, `st_ctime` reflects the last time the file's *metadata* changed (permissions, ownership, rename — not necessarily creation), which is a well-known source of confusion. Don't treat "Created" in this script's output as a reliable creation timestamp on Linux.

#### Script

```python
#!/usr/bin/env python3
"""
get_file_metadata.py
--------------------------
Displays metadata about a file: size, creation/modification time, permissions, etc.

Usage:
    python get_file_metadata.py <file_path>

Example:
    python get_file_metadata.py report.pdf
"""

import argparse
import os
import stat
import sys
from datetime import datetime


def get_file_metadata(path: str) -> None:
    if not os.path.exists(path):
        print(f"Error: '{path}' does not exist.")
        sys.exit(1)

    info = os.stat(path)

    print(f"Metadata for: {path}\n")
    print(f"Size:              {info.st_size} bytes")
    print(f"Created:           {datetime.fromtimestamp(info.st_ctime)}")
    print(f"Last Modified:     {datetime.fromtimestamp(info.st_mtime)}")
    print(f"Last Accessed:     {datetime.fromtimestamp(info.st_atime)}")
    print(f"Permissions:       {stat.filemode(info.st_mode)}")
    print(f"Is Directory:      {os.path.isdir(path)}")
    print(f"Is File:           {os.path.isfile(path)}")
    print(f"Is Symbolic Link:  {os.path.islink(path)}")
    print(f"Absolute Path:     {os.path.abspath(path)}")


def main():
    parser = argparse.ArgumentParser(description="Get metadata for a file or directory.")
    parser.add_argument("path", help="Path to the file or directory")
    args = parser.parse_args()

    get_file_metadata(args.path)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 get_file_metadata.py report.pdf
```

The script calls `os.stat()` once to gather all metadata in a single system call, then formats the relevant fields: size in bytes, three timestamps (`st_ctime`, `st_mtime`, `st_atime`) converted from Unix epoch to human-readable form, permission bits rendered in `ls -l` style via `stat.filemode()`, and boolean checks for directory/file/symlink status.

#### Sample Output (Verified)

```bash
$ python3 get_file_metadata.py meta.txt
Metadata for: meta.txt

Size:              20 bytes
Created:           2026-07-17 18:41:23.230006
Last Modified:     2026-07-17 18:41:23.230006
Last Accessed:     2026-07-17 18:41:23.230006
Permissions:       -rw-r--r--
Is Directory:      False
Is File:           True
Is Symbolic Link:  False
Absolute Path:     /home/claude/work/sandbox/meta.txt
```

#### Variations / Flags

- Add `--json` output mode for feeding metadata into other tooling programmatically instead of parsing the human-readable text.
- Add owner/group lookup via `pwd.getpwuid(info.st_uid).pw_name` and `grp.getgrgid(info.st_gid).gr_name` on Unix systems (not available on Windows without extra handling).
- Accept multiple paths in one call for a quick side-by-side metadata comparison.

#### Common Pitfalls

- **`st_ctime` means different things on different platforms** — see Cautions. Don't build automation that assumes it's always "creation time" without accounting for this Linux/Windows difference.
- **Symlinks are followed by `os.stat()`** — metadata shown (size, timestamps) reflects the *target* of a symlink, not the symlink file itself; use `os.lstat()` instead if you specifically need the symlink's own metadata.
- **Timestamps shown include microsecond precision** but are still only as accurate as the underlying filesystem's timestamp resolution — some filesystems only track modification time to the nearest second, which can make near-simultaneous operations look identical (the same caveat that affected `compare_directories.py`, 3.16, in earlier testing).

### 3.18 Monitor File Changes

#### Use Case

A sysadmin needs to know the instant a specific config file gets modified by another process or user — for example, watching for unauthorized edits to `/etc/passwd`-style files, or waiting for a long-running job to finish writing its output file.

#### Prerequisites

Standard library only (`os`, `time`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

This is a polling-based watcher, not an event-driven one — it wakes up every `--interval` seconds and compares size/mtime rather than being notified instantly by the OS. Two rapid changes to the file that happen within the same interval and coincidentally leave the same final size and mtime could be missed. For production-grade instant notification, an OS-level file-watching API (like `inotify` on Linux via the `watchdog` package) is more appropriate; this script is best suited for casual monitoring, not security-critical change detection.

#### Script

```python
#!/usr/bin/env python3
"""
monitor_file_changes.py
-----------------------------
Monitors a single file for changes (modification time / size) and prints a message when it changes.
Press Ctrl+C to stop.

Usage:
    python monitor_file_changes.py <file_path> [--interval 2]

Example:
    python monitor_file_changes.py config.json
    python monitor_file_changes.py config.json --interval 5
"""

import argparse
import os
import sys
import time
from datetime import datetime


def monitor_file(path: str, interval: int) -> None:
    if not os.path.isfile(path):
        print(f"Error: '{path}' does not exist or is not a file.")
        sys.exit(1)

    print(f"Monitoring '{path}' for changes every {interval} second(s). Press Ctrl+C to stop.\n")

    last_mtime = os.path.getmtime(path)
    last_size = os.path.getsize(path)

    try:
        while True:
            time.sleep(interval)

            if not os.path.exists(path):
                print(f"[{datetime.now()}] File was deleted!")
                break

            current_mtime = os.path.getmtime(path)
            current_size = os.path.getsize(path)

            if current_mtime != last_mtime or current_size != last_size:
                print(f"[{datetime.now()}] Change detected: size={current_size} bytes, modified={datetime.fromtimestamp(current_mtime)}")
                last_mtime = current_mtime
                last_size = current_size

    except KeyboardInterrupt:
        print("\nStopped monitoring.")


def main():
    parser = argparse.ArgumentParser(description="Monitor a file for changes.")
    parser.add_argument("path", help="File to monitor")
    parser.add_argument("--interval", type=int, default=2, help="Check interval in seconds (default: 2)")
    args = parser.parse_args()

    monitor_file(args.path, args.interval)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Default 2-second polling interval
python3 monitor_file_changes.py config.json

# Slower polling to reduce overhead
python3 monitor_file_changes.py config.json --interval 5
```

The script records the file's initial modification time and size, then loops forever, sleeping for `--interval` seconds between checks. Each wake-up it re-reads the file's current mtime and size; if either differs from the last recorded values, it prints a timestamped change notice and updates its baseline. If the file disappears entirely, it reports the deletion and exits. `Ctrl+C` stops the loop cleanly.

#### Sample Output (Verified)

```bash
$ python3 monitor_file_changes.py watched.txt --interval 1
Monitoring 'watched.txt' for changes every 1 second(s). Press Ctrl+C to stop.

[2026-07-17 18:41:27.902504] Change detected: size=16 bytes, modified=2026-07-17 18:41:27.858006
```
(Change was triggered mid-run by appending a line to `watched.txt` with `echo "updated" >> watched.txt` from another terminal.)

#### Variations / Flags

- Add `--command "notify-send 'file changed'"` to run an arbitrary shell command on each detected change, turning this into a lightweight trigger for other automation.
- Add content-hash comparison (reusing `calculate_file_hashes.py`'s approach, 3.14) instead of just size/mtime, to catch same-size-same-second edits that a coarse mtime/size check could miss.
- Swap the polling loop for the `watchdog` package's `Observer`/`FileSystemEventHandler` for true OS-level event notification instead of periodic polling.

#### Common Pitfalls

- **Polling interval is a tradeoff** — a short interval catches changes faster but uses more CPU/disk I/O; a long interval is cheaper but can miss rapid successive changes that net out to no visible size/mtime difference.
- **This blocks the terminal indefinitely** until `Ctrl+C` or file deletion — it's meant to run in the foreground of a dedicated terminal/session, or be backgrounded explicitly (`&` on Unix, or run under `nohup`/`systemd`/a scheduler for unattended use).
- **File-rename-in-place patterns can confuse it**: some programs "save" by writing to a temp file and renaming it over the original — depending on timing, this can look like a delete-then-recreate rather than a modification, and the script's "File was deleted!" branch would fire even though a new file with the same name reappears moments later.

### 3.19 Create Temporary Files

#### Use Case

A sysadmin's script needs a throwaway file to hold intermediate output — say, a staging file for a data transformation step — without hardcoding a filename that might collide with another running instance of the same script.

#### Prerequisites

Standard library only (`tempfile`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

As with the temporary-directory script, without `--keep` the file is deleted the instant the script's `with` block finishes — meaning by default this script is a self-contained demo rather than something whose output another shell command can reliably pick up afterward.

#### Script

```python
#!/usr/bin/env python3
"""
create_temporary_files.py
-------------------------------
Creates a temporary file using Python's tempfile module.
By default the file is deleted automatically after the script finishes,
unless --keep is specified.

Usage:
    python create_temporary_files.py [--content "text"] [--keep] [--suffix .txt]

Example:
    python create_temporary_files.py --content "scratch data" --keep
"""

import argparse
import tempfile


def create_temp_file(content: str, keep: bool, suffix: str) -> None:
    if keep:
        fd, path = tempfile.mkstemp(suffix=suffix)
        with open(fd, "w", encoding="utf-8") as f:
            f.write(content)
        print(f"Temporary file created and KEPT at: {path}")
    else:
        with tempfile.NamedTemporaryFile(mode="w", suffix=suffix, delete=True, encoding="utf-8") as f:
            f.write(content)
            f.flush()
            print(f"Temporary file created at: {f.name}")
            print("This file will be deleted automatically when the script exits.")


def main():
    parser = argparse.ArgumentParser(description="Create a temporary file.")
    parser.add_argument("--content", default="", help="Content to write into the temp file")
    parser.add_argument("--keep", action="store_true", help="Keep the file after the script exits")
    parser.add_argument("--suffix", default=".tmp", help="File suffix/extension (default: .tmp)")
    args = parser.parse_args()

    create_temp_file(args.content, args.keep, args.suffix)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Kept, with a specific extension
python3 create_temporary_files.py --content "scratch data" --keep --suffix .txt

# Auto-deleted immediately after the script prints its path
python3 create_temporary_files.py --content "scratch data"
```

With `--keep`, the script uses the lower-level `tempfile.mkstemp()`, which returns an open file descriptor and a guaranteed-unique path that the caller owns and must clean up manually. Without it, `tempfile.NamedTemporaryFile(delete=True)` is used as a context manager — the file exists on disk only while the `with` block is open, and is removed automatically the moment it closes.

#### Sample Output (Verified)

```bash
$ python3 create_temporary_files.py --content "scratch data" --keep --suffix .txt
Temporary file created and KEPT at: /tmp/tmpnetgtx71.txt

$ python3 create_temporary_files.py --content "scratch data"
Temporary file created at: /tmp/tmp1nq8l9wn.tmp
This file will be deleted automatically when the script exits.
```

#### Variations / Flags

- Add `--prefix scratch_` for a more identifiable filename pattern in a busy `/tmp` directory.
- On Windows specifically, note that `NamedTemporaryFile` with `delete=True` cannot be reopened by name while still held open in some Python versions/platforms — if a downstream process needs to read the file while this script's process is still running, use `--keep` (or `delete=False`) instead.
- Add a `--stdin` mode that reads content to write from standard input rather than a `--content` string argument, useful for piping larger payloads in.

#### Common Pitfalls

- **`--keep` files are never auto-removed** — just like the temp-directory script, these will accumulate in `/tmp` (or `%TEMP%`) unless something else cleans them up.
- **Without `--keep`, the file is gone by the time the script exits** — there is no way for a calling shell script to `cat` this file afterward; the printed path is informational only in that mode.
- **Windows file-locking quirk**: an open `NamedTemporaryFile` on Windows may not be openable by a second process while the first still holds it — a subtle cross-platform difference from Linux/macOS behavior worth testing if this pattern is reused in production automation.

### 3.20 Create Temporary Directories


#### Use Case

A sysadmin is writing an automation script that needs somewhere to unpack an archive, stage files, or run intermediate build steps — a location guaranteed not to collide with anything else on the system and that ideally cleans itself up when the job finishes.

#### Prerequisites

Standard library only (`tempfile`, `shutil`) — no installation required. Works on Linux, macOS, and Windows (the OS-appropriate temp location, e.g. `/tmp` or `%TEMP%`, is selected automatically by `tempfile`).

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

Without `--keep`, the directory (and everything in it) is deleted the moment the `with` block exits — meaning by the time the script's process ends, the directory is already gone. This script is really meant as a demonstration/building block for other scripts to import and use `tempfile.TemporaryDirectory()` inside a longer-running process, rather than something you'd invoke standalone expecting the directory to persist for later steps in a shell pipeline (unless you pass `--keep`).

#### Script

```python
#!/usr/bin/env python3
"""
create_temporary_directories.py
-------------------------------------
Creates a temporary directory using Python's tempfile module.
By default it is deleted automatically after the script finishes,
unless --keep is specified.

Usage:
    python create_temporary_directories.py [--keep]

Example:
    python create_temporary_directories.py --keep
"""

import argparse
import shutil
import tempfile


def create_temp_dir(keep: bool) -> None:
    if keep:
        path = tempfile.mkdtemp()
        print(f"Temporary directory created and KEPT at: {path}")
    else:
        with tempfile.TemporaryDirectory() as path:
            print(f"Temporary directory created at: {path}")
            print("This directory will be deleted automatically when the script exits.")


def main():
    parser = argparse.ArgumentParser(description="Create a temporary directory.")
    parser.add_argument("--keep", action="store_true", help="Keep the directory after the script exits")
    args = parser.parse_args()

    create_temp_dir(args.keep)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash

# Persists after the script exits — caller is responsible for cleanup

```

Without `--keep`, the script uses `tempfile.TemporaryDirectory()` as a context manager, which creates a uniquely-named directory under the OS temp path and guarantees its deletion (via `shutil.rmtree` internally) when the `with` block exits — even if an exception occurs inside it. With `--keep`, it instead calls `tempfile.mkdtemp()`, which creates the directory but leaves cleanup entirely to the caller.

#### Variations / Flags

- Add `--prefix myapp_` and `--dir /custom/path` (both supported natively by `tempfile.mkdtemp()`/`TemporaryDirectory()`) to control the naming and location instead of relying entirely on OS defaults.
- Print the path to stdout *only* (no extra log lines) behind a `--quiet` flag, so the output can be captured directly by another script: `TMPDIR=$(python3 create_temporary_directories.py --keep --quiet)`.
- Register the kept directory's path in a manifest file so a separate cleanup script can find and remove orphaned temp directories later.

#### Common Pitfalls

- **Without `--keep`, nothing outside the script can use the directory** — since it's deleted the instant the `with` block ends (effectively immediately, since the script does nothing else inside it), this mode is only useful as example code to copy into a larger script, not as a standalone CLI tool.
- **`--keep` directories are never cleaned up automatically** — over time, forgetting this flag was used can quietly fill up `/tmp` with orphaned directories; track or periodically sweep them with `clean_temporary_files.py` (3.30) style logic.
- **Permissions on the temp directory** default to being readable/writable only by the owning user on most Unix systems — don't assume other users or services can access it without adjusting permissions explicitly.

### 3.21 Compress Files

#### Use Case

A sysadmin needs to send a handful of log files to a colleague, or package a project folder into a single ZIP before uploading it to shared storage — without manually right-clicking through a GUI archive tool.

#### Prerequisites

Standard library only (`zipfile`) — no installation required. Works on Linux, macOS, and Windows, and produces archives that any OS can open natively.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

The script overwrites the output ZIP file if it already exists (opens in `"w"` mode), with no confirmation prompt — double-check the output filename before running it against something you already have. Very large directories can take a while and use significant CPU during `ZIP_DEFLATED` compression.

#### Script

```python
#!/usr/bin/env python3
"""
compress_files.py
-----------------------
Compresses one or more files/folders into a ZIP archive.

Usage:
    python compress_files.py <output_zip> <source1> [source2 ...]

Example:
    python compress_files.py backup.zip file1.txt file2.txt
    python compress_files.py project.zip my_project_folder
"""

import argparse
import os
import sys
import zipfile


def compress_files(output_zip: str, sources) -> None:
    if not output_zip.lower().endswith(".zip"):
        output_zip += ".zip"

    try:
        with zipfile.ZipFile(output_zip, "w", zipfile.ZIP_DEFLATED) as zf:
            for src in sources:
                if not os.path.exists(src):
                    print(f"Skipped (not found): {src}")
                    continue

                if os.path.isfile(src):
                    zf.write(src, arcname=os.path.basename(src))
                    print(f"Added file: {src}")
                elif os.path.isdir(src):
                    for root, _, files in os.walk(src):
                        for f in files:
                            full_path = os.path.join(root, f)
                            arcname = os.path.relpath(full_path, os.path.dirname(src))
                            zf.write(full_path, arcname=arcname)
                    print(f"Added directory: {src}")

        print(f"\nArchive created: {output_zip}")
    except OSError as e:
        print(f"Error creating archive: {e}")
        sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Compress files/folders into a ZIP archive.")
    parser.add_argument("output_zip", help="Output ZIP file path")
    parser.add_argument("sources", nargs="+", help="File(s)/folder(s) to compress")
    args = parser.parse_args()

    compress_files(args.output_zip, args.sources)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Zip individual files
python3 compress_files.py backup.zip file1.txt file2.txt

# Zip an entire folder (preserves its internal structure)
python3 compress_files.py project.zip my_project_folder
```

The script opens a new ZIP archive in write mode using DEFLATE compression. Plain files are added directly under their basename. Directories are walked recursively with `os.walk()`, and each file's archive path (`arcname`) is computed relative to the parent of the source folder — so the folder itself becomes the top-level entry inside the ZIP, matching how most GUI archive tools behave.

#### Sample Output (Verified)

```bash
$ python3 compress_files.py backup.zip zipsrc/a.txt zipsrc/b.txt
Added file: zipsrc/a.txt
Added file: zipsrc/b.txt

Archive created: backup.zip

$ python3 compress_files.py project zipsrc
Added directory: zipsrc

Archive created: project.zip

$ unzip -l project.zip
Archive:  project.zip
  Length      Date    Time    Name
---------  ---------- -----   ----
        2  2026-07-17 18:30   zipsrc/b.txt
        2  2026-07-17 18:30   zipsrc/a.txt
---------                     -------
        4                     2 files
```

Note the `.zip` extension was auto-appended to `project` in the second call.

#### Variations / Flags

- Add `--level 9` to control `zipfile.ZipFile(..., compresslevel=9)` for maximum compression at the cost of speed.
- Add `--password` support via `pyzipper` (the standard `zipfile` module can't write password-protected ZIPs) if encrypted archives are needed instead of the separate encrypt_files.py workflow.
- Add `--exclude "*.log"` with `fnmatch` to skip unwanted files while walking a directory.

#### Common Pitfalls

- **Silent overwrite** — running the script twice with the same output name replaces the old archive with no warning.
- **Path separators inside the ZIP**: on Windows, `os.path.relpath` may produce backslash-separated arcnames; some cross-platform unzip tools expect forward slashes. If the archive needs to be opened on Linux/macOS, normalize arcnames with `arcname.replace(os.sep, "/")`.
- **Symlinks are followed and their target content is stored**, not the symlink itself — archives can balloon in size if a symlink points to something large or circular.
- **Missing sources are skipped, not fatal** — a typo'd path won't stop the whole archive from being created, so check the printed "Skipped" lines carefully.

### 3.22 Extract ZIP Archives

#### Use Case

A sysadmin receives a ZIP archive — a software distribution, a data export, a backup — and needs to unpack it into a specific directory as part of a deployment or restore script, rather than double-clicking through a GUI archive manager.

#### Prerequisites

Standard library only (`zipfile`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

`zipfile.extractall()` on an untrusted or malicious ZIP can, in principle, contain crafted paths designed to write outside the intended destination (a "zip slip" path-traversal issue) on older Python versions or naive extraction code. Modern Python's `zipfile` mitigates absolute paths and most `..` traversal automatically, but it's still good practice to only extract ZIP files from sources you trust, especially in automated/unattended pipelines.

#### Script

```python
#!/usr/bin/env python3
"""
extract_zip_archives.py
-----------------------------
Extracts the contents of a ZIP archive to a destination directory.

Usage:
    python extract_zip_archives.py <zip_file> [destination_dir]

Example:
    python extract_zip_archives.py backup.zip
    python extract_zip_archives.py backup.zip ./extracted
"""

import argparse
import os
import sys
import zipfile


def extract_zip(zip_path: str, destination: str) -> None:
    if not os.path.isfile(zip_path):
        print(f"Error: '{zip_path}' does not exist.")
        sys.exit(1)

    if not zipfile.is_zipfile(zip_path):
        print(f"Error: '{zip_path}' is not a valid ZIP file.")
        sys.exit(1)

    os.makedirs(destination, exist_ok=True)

    try:
        with zipfile.ZipFile(zip_path, "r") as zf:
            zf.extractall(destination)
            file_count = len(zf.namelist())

        print(f"Extracted {file_count} item(s) from '{zip_path}' to '{destination}'")
    except (OSError, zipfile.BadZipFile) as e:
        print(f"Error extracting archive: {e}")
        sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Extract a ZIP archive.")
    parser.add_argument("zip_file", help="ZIP file to extract")
    parser.add_argument("destination", nargs="?", default=".", help="Destination directory (default: current directory)")
    args = parser.parse_args()

    extract_zip(args.zip_file, args.destination)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Extract into the current directory
python3 extract_zip_archives.py backup.zip

# Extract into a specific destination (created if it doesn't exist)
python3 extract_zip_archives.py backup.zip ./extracted
```

The script first confirms the file exists and is genuinely a ZIP archive (via `zipfile.is_zipfile()`, which checks the file's actual structure rather than trusting the extension), creates the destination directory if needed, then calls `extractall()` to unpack everything in one step and reports how many items were extracted.

#### Sample Output (Verified)

```bash
$ python3 extract_zip_archives.py project.zip ./extracted
Extracted 2 item(s) from 'project.zip' to './extracted'

$ find extracted
extracted
extracted/zipsrc
extracted/zipsrc/b.txt
extracted/zipsrc/a.txt
```

(This round-trips correctly against `compress_files.py`, 3.21 — the archive extracted here was created by that script earlier in testing.)

#### Variations / Flags

- Add `--list-only` to print archive contents via `zf.namelist()` without actually extracting, useful for previewing before committing to an extraction.
- Add `--password` support for password-protected ZIPs via `zf.extractall(destination, pwd=password.encode())`.
- Add per-member path validation (reject any member whose resolved path falls outside the destination directory) for extra safety when handling ZIPs from less-trusted sources.

#### Common Pitfalls

- **Extension doesn't guarantee format** — the script correctly checks `is_zipfile()` rather than trusting the `.zip` extension, so a renamed non-ZIP file is caught with a clear error rather than a confusing crash.
- **Existing files at the destination are silently overwritten** by `extractall()` if the archive contains files with matching names — there's no conflict check before extraction.
- **Untrusted archives**: while modern Python's `zipfile` guards against most path-traversal tricks, don't treat this as a hardened sandboxed extractor for arbitrary internet-sourced ZIPs — apply the same caution you would with any archive extraction tool.

### 3.23 Create TAR Archives

#### Use Case

A sysadmin needs to package a set of configuration files or an application directory for transfer to a Linux server where `.tar.gz` is the expected format, or needs a backup archive that preserves Unix file permissions and ownership metadata more faithfully than a ZIP would.

#### Prerequisites

Standard library only (`tarfile`) — no installation required. Works on Linux, macOS, and Windows (though TAR is most idiomatic on Unix-like systems).

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

Like `compress_files.py`, this overwrites the output archive if it already exists, with no confirmation prompt. Gzip compression (`--gzip`) uses noticeably more CPU than plain TAR for large inputs, trading time for smaller output size.

#### Script

```python
#!/usr/bin/env python3
"""
create_tar_archives.py
----------------------------
Creates a TAR archive (optionally gzip-compressed) from one or more files/folders.

Usage:
    python create_tar_archives.py <output_tar> <source1> [source2 ...] [--gzip]

Example:
    python create_tar_archives.py backup.tar file1.txt folder1
    python create_tar_archives.py backup.tar.gz file1.txt folder1 --gzip
"""

import argparse
import os
import sys
import tarfile


def create_tar(output_tar: str, sources, use_gzip: bool) -> None:
    mode = "w:gz" if use_gzip else "w"

    if use_gzip and not (output_tar.endswith(".tar.gz") or output_tar.endswith(".tgz")):
        output_tar += ".tar.gz"
    elif not use_gzip and not output_tar.endswith(".tar"):
        output_tar += ".tar"

    try:
        with tarfile.open(output_tar, mode) as tar:
            for src in sources:
                if not os.path.exists(src):
                    print(f"Skipped (not found): {src}")
                    continue
                tar.add(src, arcname=os.path.basename(src))
                print(f"Added: {src}")

        print(f"\nArchive created: {output_tar}")
    except (OSError, tarfile.TarError) as e:
        print(f"Error creating TAR archive: {e}")
        sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Create a TAR archive.")
    parser.add_argument("output_tar", help="Output TAR file path")
    parser.add_argument("sources", nargs="+", help="File(s)/folder(s) to archive")
    parser.add_argument("--gzip", action="store_true", help="Compress the archive with gzip (.tar.gz)")
    args = parser.parse_args()

    create_tar(args.output_tar, args.sources, args.gzip)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Plain, uncompressed TAR
python3 create_tar_archives.py backup.tar file1.txt folder1

# Gzip-compressed TAR
python3 create_tar_archives.py backup.tar.gz file1.txt folder1 --gzip
```

`tarfile.open()` is used in either `"w"` (plain) or `"w:gz"` (gzip-compressed) mode depending on the `--gzip` flag. Each source is added recursively via `tar.add()`, using just the basename as the archive path so the resulting archive doesn't embed the full source directory structure from your machine.

#### Sample Output (Verified)

```bash
$ python3 create_tar_archives.py backup.tar zipsrc/a.txt zipsrc/b.txt
Added: zipsrc/a.txt
Added: zipsrc/b.txt

Archive created: backup.tar

$ python3 create_tar_archives.py backup zipsrc --gzip
Added: zipsrc

Archive created: backup.tar.gz

$ tar -tvf backup.tar.gz
drwxr-xr-x root/root         0 2026-07-17 18:30 zipsrc/
-rw-r--r-- root/root         2 2026-07-17 18:30 zipsrc/a.txt
-rw-r--r-- root/root         2 2026-07-17 18:30 zipsrc/b.txt
```

Note the extension auto-correction: passing `backup` with `--gzip` produced `backup.tar.gz` automatically.

#### Variations / Flags

- Add `--bzip2` / `--xz` options by switching the mode string to `"w:bz2"` or `"w:xz"` for alternate compression algorithms with different speed/size tradeoffs.
- Add `--exclude` pattern support using `tarfile`'s `filter=` callback (Python 3.12+) or a custom filter function for older versions, to skip files matching a pattern while adding a directory.
- Combine with `calculate_file_hashes.py` (3.14) to hash the resulting archive immediately after creation for an integrity-checked backup workflow.

#### Common Pitfalls

- **Extension auto-correction only checks suffixes, not existing files** — running the command twice with the same base name will silently overwrite the previous archive.
- **`tar.add()` follows symlinks by default** and stores the target's content — pass `recursive=True` (already the tarfile default) with care if your sources contain symlinked loops, which can cause the archive creation to hang or fail.
- **User/group ownership stored in the archive reflects the machine that created it** (shown as `root/root` in the sample output when run as root) — extracting on another machine as a different user may require `--same-owner`/`--no-same-owner` handling depending on the extraction tool.
- **Cross-platform extraction**: `.tar` and `.tar.gz` files extract natively on Linux/macOS; on Windows they typically require a third-party tool (7-Zip, WinRAR) unless using WSL.

### 3.24 Extract TAR Archives

#### Use Case

A sysadmin receives a `.tar` or `.tar.gz` archive — commonly used for Linux software distributions and backups — and needs to unpack it into a specific directory as part of a deployment or restore script.

#### Prerequisites

Standard library only (`tarfile`) — no installation required. Works on Linux, macOS, and Windows, though TAR extraction is most commonly needed on Unix-like systems.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

Like ZIP extraction, `tarfile.extractall()` on an untrusted archive can in principle contain crafted member paths intended to write outside the destination directory ("tar slip"/path traversal). Only extract TAR archives from sources you trust, especially in unattended automation. TAR archives can also preserve Unix ownership/permission metadata that may not translate meaningfully when extracted on Windows.

#### Script

```python
#!/usr/bin/env python3
"""
extract_tar_archives.py
-----------------------------
Extracts the contents of a TAR archive (plain or gzip-compressed) to a destination directory.

Usage:
    python extract_tar_archives.py <tar_file> [destination_dir]

Example:
    python extract_tar_archives.py backup.tar
    python extract_tar_archives.py backup.tar.gz ./extracted
"""

import argparse
import os
import sys
import tarfile


def extract_tar(tar_path: str, destination: str) -> None:
    if not os.path.isfile(tar_path):
        print(f"Error: '{tar_path}' does not exist.")
        sys.exit(1)

    if not tarfile.is_tarfile(tar_path):
        print(f"Error: '{tar_path}' is not a valid TAR file.")
        sys.exit(1)

    os.makedirs(destination, exist_ok=True)

    try:
        with tarfile.open(tar_path, "r:*") as tar:
            members = tar.getmembers()
            tar.extractall(destination)
            file_count = len(members)

        print(f"Extracted {file_count} item(s) from '{tar_path}' to '{destination}'")
    except (OSError, tarfile.TarError) as e:
        print(f"Error extracting archive: {e}")
        sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Extract a TAR archive (plain or gzip-compressed).")
    parser.add_argument("tar_file", help="TAR file to extract (.tar, .tar.gz, .tgz)")
    parser.add_argument("destination", nargs="?", default=".", help="Destination directory (default: current directory)")
    args = parser.parse_args()

    extract_tar(args.tar_file, args.destination)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Extract a plain TAR into the current directory
python3 extract_tar_archives.py backup.tar

# Extract a gzip-compressed TAR into a specific destination
python3 extract_tar_archives.py backup.tar.gz ./extracted
```

The script confirms the file exists and is genuinely a TAR archive using `tarfile.is_tarfile()`, then opens it in `"r:*"` mode — a wildcard mode that auto-detects whether the archive is plain, gzip-, bzip2-, or xz-compressed, so both `.tar` and `.tar.gz` inputs are handled by the same code path without needing a separate flag. It then extracts every member into the destination directory (created if needed) and reports the count.

#### Sample Output (Verified)

```bash

extracted_tar/zipsrc/b.txt
```

#### Variations / Flags

- Add `--list-only` to print archive member names via `tar.getnames()` without extracting, for previewing contents first.
- Add member-path validation (reject any member whose resolved path would land outside the destination directory) for safer handling of archives from less-trusted sources — Python 3.12+ added a built-in `filter="data"` argument to `extractall()` for exactly this purpose, worth using directly on newer Python versions.
- Add `--strip-components 1` style logic (not built into `tarfile` natively) to drop a leading path segment from every member, useful when an archive wraps its contents in a single top-level folder you don't want to keep.

#### Common Pitfalls

- **`"r:*"` mode requires Python to correctly auto-detect compression** — this works reliably for standard `.tar`, `.tar.gz`/`.tgz`, `.tar.bz2`, and `.tar.xz` files, but a corrupted or unusually-crafted archive can cause detection to fail with a `tarfile.ReadError` rather than a clean fallback.
- **Ownership and permission metadata**: TAR archives can embed Unix UID/GID and permission bits; extracting as an unprivileged user may silently fail to restore original ownership (this is normal and expected — full restoration typically requires root/administrator privileges).
- **Untrusted archives**: as with ZIP extraction, don't treat this as a hardened sandbox for arbitrary internet-sourced TAR files without adding the path-validation safeguard mentioned above.

### 3.25 Encrypt Files

#### Use Case

A sysadmin needs to send a file containing credentials, private keys, or other sensitive data over an untrusted channel, or store it somewhere that isn't fully trusted, and wants it unreadable to anyone without the password.

#### Prerequisites

Requires the third-party `cryptography` package.

```bash
pip install cryptography --break-system-packages
```

Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.7+
```

#### Cautions

The derived encryption key depends entirely on the password's strength — a short or common password provides very little real protection even though the underlying AES/Fernet cryptography is sound. There is no password recovery: losing the password means losing the data permanently. Also note the *encrypted* `.enc` file does not securely erase the original plaintext file — `encrypt_files.py` leaves the original in place, so make sure to explicitly delete the plaintext with `delete_files.py` (3.3) afterward if that's the intent.

#### Script

```python
#!/usr/bin/env python3
"""
encrypt_files.py
----------------------
Encrypts a file using a password, with the Fernet symmetric encryption
scheme (AES-based, provided by the 'cryptography' library).

Produces a .enc file. Use decrypt_files.py with the SAME password to reverse it.

Requires: pip install cryptography

Usage:
    python encrypt_files.py <file_path> <password>

Example:
    python encrypt_files.py secret.txt MyStrongPassword123
"""

import argparse
import base64
import os
import sys

try:
    from cryptography.fernet import Fernet
    from cryptography.hazmat.primitives import hashes
    from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
except ImportError:
    print("Error: the 'cryptography' package is required.")
    print("Install it with: pip install -r requirements.txt")
    sys.exit(1)


SALT = b"file_encryption_salt_v1"  # Fixed salt for reproducible key derivation from a password


def derive_key(password: str) -> bytes:
    kdf = PBKDF2HMAC(
        algorithm=hashes.SHA256(),
        length=32,
        salt=SALT,
        iterations=390000,
    )
    return base64.urlsafe_b64encode(kdf.derive(password.encode()))


def encrypt_file(file_path: str, password: str) -> None:
    if not os.path.isfile(file_path):
        print(f"Error: '{file_path}' does not exist.")
        sys.exit(1)

    key = derive_key(password)
    fernet = Fernet(key)

    with open(file_path, "rb") as f:
        data = f.read()

    encrypted_data = fernet.encrypt(data)
    output_path = file_path + ".enc"

    with open(output_path, "wb") as f:
        f.write(encrypted_data)

    print(f"Encrypted: '{file_path}' -> '{output_path}'")
    print("Keep your password safe — it is required to decrypt this file.")


def main():
    parser = argparse.ArgumentParser(description="Encrypt a file using a password.")
    parser.add_argument("file_path", help="File to encrypt")
    parser.add_argument("password", help="Password used to encrypt the file")
    args = parser.parse_args()

    encrypt_file(args.file_path, args.password)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 encrypt_files.py secret.txt MyStrongPassword123
```

The password is stretched into a 32-byte key using PBKDF2-HMAC-SHA256 with a fixed salt and 390,000 iterations — deliberately slow, which is the point of a KDF: it makes brute-forcing the password computationally expensive. That key is base64-encoded and handed to `Fernet`, which encrypts the file's raw bytes using AES in CBC mode with a random IV and an HMAC for authenticity, then writes the result to `<original>.enc`.

#### Sample Output (Verified)

```bash
$ echo "top secret data" > secret.txt
$ python3 encrypt_files.py secret.txt "MyStrongPassword123"
Encrypted: 'secret.txt' -> 'secret.txt.enc'
Keep your password safe — it is required to decrypt this file.

$ ls secret*
secret.txt  secret.txt.enc
```

(Round-trip verified against `decrypt_files.py`, 3.26 — decrypting `secret.txt.enc` with the same password reproduces the original `top secret data` content exactly.)

#### Variations / Flags

- Add `--delete-original` to automatically remove the plaintext file after successful encryption (using the safer confirmation-based logic from `delete_files.py`, 3.3).
- Add `--stdin-password` with `getpass.getpass()` instead of a plaintext CLI argument, avoiding password exposure in shell history.
- Generate a random salt per file (stored alongside the `.enc` output, e.g. as the first 16 bytes) instead of a single hardcoded `SALT` shared across every encrypted file — a stronger design for anything beyond casual/internal use, since a fixed salt means identical passwords always derive identical keys.

#### Common Pitfalls

- **Password visible in shell history and `ps` output** — same issue as the decrypt script; avoid this pattern for anything beyond casual/internal use.
- **Fixed `SALT` across all files** means two files encrypted with the same password will use the same derived key — not a practical vulnerability for Fernet's authenticated encryption per se, but a weaker design than a per-file random salt, and it's worth understanding this tradeoff before using the script for anything highly sensitive.
- **Plaintext original is not deleted automatically** — encrypting a file does not remove the unencrypted source; you must delete it yourself if that's required.
- **No progress indicator for large files** — the entire file is read into memory at once (`f.read()`), which can be slow or memory-heavy for very large files (multi-GB); for those, a streaming encryption approach would be more appropriate.

### 3.26 Decrypt Files

#### Use Case

A sysadmin previously encrypted a sensitive file (credentials, a backup archive, a private key) before storing or transmitting it, and now needs to restore the original file using the password it was encrypted with.

#### Prerequisites

Requires the third-party `cryptography` package.

```bash
pip install cryptography --break-system-packages
```

Works on Linux, macOS, and Windows (anywhere the `cryptography` package installs, which is essentially everywhere with a supported Python version).

```bash
python3 --version   # requires Python 3.7+ (cryptography's minimum supported version varies by release)
```

#### Cautions

There is no password recovery mechanism — if the password is lost, the encrypted file is permanently unrecoverable; this is inherent to the encryption scheme, not a limitation of the script. The script must be paired with the exact same `SALT` constant used at encryption time (see `encrypt_files.py`, 3.25) — changing that constant in one script without updating the other will make previously encrypted files undecryptable.

#### Script

```python
#!/usr/bin/env python3
"""
decrypt_files.py
----------------------
Decrypts a file that was encrypted with encrypt_files.py, using the same password.

Requires: pip install cryptography

Usage:
    python decrypt_files.py <encrypted_file> <password>

Example:
    python decrypt_files.py secret.txt.enc MyStrongPassword123
"""

import argparse
import base64
import os
import sys

try:
    from cryptography.fernet import Fernet, InvalidToken
    from cryptography.hazmat.primitives import hashes
    from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
except ImportError:
    print("Error: the 'cryptography' package is required.")
    print("Install it with: pip install -r requirements.txt")
    sys.exit(1)


SALT = b"file_encryption_salt_v1"  # Must match the salt used in encrypt_files.py


def derive_key(password: str) -> bytes:
    kdf = PBKDF2HMAC(
        algorithm=hashes.SHA256(),
        length=32,
        salt=SALT,
        iterations=390000,
    )
    return base64.urlsafe_b64encode(kdf.derive(password.encode()))


def decrypt_file(file_path: str, password: str) -> None:
    if not os.path.isfile(file_path):
        print(f"Error: '{file_path}' does not exist.")
        sys.exit(1)

    key = derive_key(password)
    fernet = Fernet(key)

    with open(file_path, "rb") as f:
        encrypted_data = f.read()

    try:
        decrypted_data = fernet.decrypt(encrypted_data)
    except InvalidToken:
        print("Error: decryption failed. Wrong password or corrupted file.")
        sys.exit(1)

    if file_path.endswith(".enc"):
        output_path = file_path[:-4]
    else:
        output_path = file_path + ".decrypted"

    with open(output_path, "wb") as f:
        f.write(decrypted_data)

    print(f"Decrypted: '{file_path}' -> '{output_path}'")


def main():
    parser = argparse.ArgumentParser(description="Decrypt a file encrypted with encrypt_files.py.")
    parser.add_argument("file_path", help="Encrypted file (.enc)")
    parser.add_argument("password", help="Password used to originally encrypt the file")
    args = parser.parse_args()

    decrypt_file(args.file_path, args.password)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 decrypt_files.py secret.txt.enc MyStrongPassword123
```

The script derives the same AES-based Fernet key from the password using PBKDF2-HMAC-SHA256 with 390,000 iterations and the fixed `SALT` — identical to the derivation used by `encrypt_files.py`. It then attempts to decrypt the file's bytes; if the password is wrong or the file was corrupted/tampered with, Fernet's built-in authentication check raises `InvalidToken` rather than producing garbage output. On success, the `.enc` suffix is stripped to reconstruct the original filename (or `.decrypted` is appended if the input didn't end in `.enc`).

#### Sample Output (Verified)

```bash
$ python3 encrypt_files.py secret.txt "MyStrongPassword123"
Encrypted: 'secret.txt' -> 'secret.txt.enc'
Keep your password safe — it is required to decrypt this file.

$ python3 decrypt_files.py secret.txt.enc "MyStrongPassword123"
Decrypted: 'secret.txt.enc' -> 'secret.txt'

$ cat secret.txt
top secret data
```

Wrong-password case, verified:

```bash
$ python3 decrypt_files.py secret.txt.enc "WrongPassword"
Error: decryption failed. Wrong password or corrupted file.
```

#### Variations / Flags

- Add `--output custom_name.txt` to control the output path explicitly instead of relying on stripping `.enc`.
- Add `--stdin-password` to prompt interactively (via `getpass`) instead of passing the password as a plaintext CLI argument, which avoids it showing up in shell history or `ps` output.
- Batch mode: accept multiple `.enc` files and a single password to decrypt them all in one run.

#### Common Pitfalls

- **Password on the command line is visible in shell history and process listings** — the biggest practical security gap in this script's interface. Prefer an interactive password prompt (`getpass.getpass()`) for anything sensitive.
- **`InvalidToken` covers both "wrong password" and "corrupted/tampered file"** — the script can't tell you which one happened, only that decryption failed.
- **Overwriting the original file**: if `secret.txt` already exists again when you decrypt `secret.txt.enc`, the script silently overwrites it since it opens the output path in `"wb"` mode with no existence check.
- **The `SALT` constant must never change once files have been encrypted with it** — it's hardcoded identically in both `encrypt_files.py` and `decrypt_files.py`; if you fork one script and change its salt without updating the other, previously encrypted files become permanently undecryptable.

### 3.27 Split Large Files

#### Use Case

A sysadmin needs to move a multi-gigabyte file — a database dump, a video, a disk image — through a channel with a size limit (email attachment caps, certain upload portals, older FAT32-formatted USB drives), and needs it split into predictable, reassemblable chunks.

#### Prerequisites

Standard library only (`os`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

The original file is left untouched — splitting produces new `.partN` files alongside it, so make sure there's enough free disk space for both the original and all its parts simultaneously (roughly double the original file's size at peak). This script must be paired with `merge_split_files.py` (3.28) to reconstruct the original; the `.partN` files are not independently useful.

#### Script

```python
#!/usr/bin/env python3
"""
split_large_files.py
--------------------------
Splits a large file into smaller chunks of a specified size (in MB).
Produces files named <original_name>.part0, .part1, .part2, etc.

Usage:
    python split_large_files.py <file_path> <chunk_size_mb>

Example:
    python split_large_files.py bigvideo.mp4 100
"""

import argparse
import os
import sys


def split_file(file_path: str, chunk_size_mb: float) -> None:
    if not os.path.isfile(file_path):
        print(f"Error: '{file_path}' does not exist.")
        sys.exit(1)

    chunk_size_bytes = int(chunk_size_mb * 1024 * 1024)
    if chunk_size_bytes <= 0:
        print("Error: chunk size must be greater than 0.")
        sys.exit(1)

    part_num = 0
    total_size = os.path.getsize(file_path)

    with open(file_path, "rb") as f:
        while True:
            chunk = f.read(chunk_size_bytes)
            if not chunk:
                break

            part_path = f"{file_path}.part{part_num}"
            with open(part_path, "wb") as part_file:
                part_file.write(chunk)

            print(f"Created: {part_path} ({len(chunk)} bytes)")
            part_num += 1

    print(f"\nSplit '{file_path}' ({total_size} bytes) into {part_num} part(s).")
    print("Use merge_split_files.py to reassemble them.")


def main():
    parser = argparse.ArgumentParser(description="Split a large file into smaller chunks.")
    parser.add_argument("file_path", help="File to split")
    parser.add_argument("chunk_size_mb", type=float, help="Size of each chunk in megabytes (e.g. 100)")
    args = parser.parse_args()

    split_file(args.file_path, args.chunk_size_mb)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 split_large_files.py bigvideo.mp4 100
```

The script reads the source file in fixed-size blocks (`chunk_size_mb` converted to bytes), writing each block out to a sequentially numbered `.partN` file until the source is exhausted. Because it streams rather than loading the whole file into memory, it scales to files much larger than available RAM.

#### Sample Output (Verified)

```bash
$ python3 split_large_files.py splittest/big.bin 1
Created: splittest/big.bin.part0 (1048576 bytes)
Created: splittest/big.bin.part1 (1048576 bytes)
Created: splittest/big.bin.part2 (500000 bytes)

Split 'splittest/big.bin' (2597152 bytes) into 3 part(s).
Use merge_split_files.py to reassemble them.
```

A 2,597,152-byte test file was split at a 1 MB (1,048,576-byte) chunk size into two full 1 MB parts and one final partial part — exactly as expected.

#### Variations / Flags

- Add `--delete-original` to remove the source file after a successful split (paired with a confirmation prompt, given the destructive nature).
- Add a manifest file (`bigvideo.mp4.manifest`) recording the original filename, total size, part count, and a SHA-256 hash of the original (via `calculate_file_hashes.py`, 3.14) so `merge_split_files.py` can verify a successful reassembly automatically.
- Support splitting by part *count* instead of chunk *size* (`--parts 5` instead of a fixed MB size) for cases where you need exactly N pieces.

#### Common Pitfalls

- **Disk space during the split**: the original file and all its parts exist simultaneously, so make sure you have roughly 2x the original file's size in free space before starting.
- **Part file ordering depends on correct numeric parsing at merge time** — the numbering itself (`part0`, `part1`, ... `part10`, `part11`) sorts incorrectly as plain text (`part10` before `part2`) if a naive alphabetical sort is used downstream; `merge_split_files.py` (3.28) handles this correctly by extracting and sorting on the numeric suffix, but any custom tooling built around these files needs to do the same.
- **No integrity verification built in** — the script doesn't hash the original or the parts, so silent corruption during transfer (e.g., over a flaky connection) wouldn't be caught automatically; verify manually with `calculate_file_hashes.py` (3.14) before and after transfer if integrity matters.

### 3.28 Merge Split Files

#### Use Case

A sysadmin has received a set of `.part0`, `.part1`, `.part2`... files — produced by `split_large_files.py` (3.27) — after transferring a large file through a size-limited channel, and needs to reconstruct the original file on the receiving end.

#### Prerequisites

Standard library only (`glob`, `re`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

The script trusts that all part files matching the pattern belong to the same original file and are complete — it doesn't verify a checksum against the original, so a missing or corrupted part will produce a corrupted merged output with no automatic warning. Always verify the result (e.g., with `calculate_file_hashes.py`, 3.14, against a known-good hash) for anything important.

#### Script

```python
#!/usr/bin/env python3
"""
merge_split_files.py
--------------------------
Merges files created by split_large_files.py (.part0, .part1, ...) back into the original file.

Usage:
    python merge_split_files.py <original_file_base_name> [--output output_file]

Example:
    python merge_split_files.py bigvideo.mp4
    # Looks for bigvideo.mp4.part0, bigvideo.mp4.part1, etc. in the current directory
    # and reassembles them into bigvideo.mp4 (or the --output path if given).
"""

import argparse
import glob
import os
import re
import sys


def merge_files(base_name: str, output_path: str) -> None:
    pattern = f"{base_name}.part*"
    parts = glob.glob(pattern)

    if not parts:
        print(f"Error: no part files found matching pattern '{pattern}'.")
        sys.exit(1)

    def part_number(path: str) -> int:
        match = re.search(r"\.part(\d+)$", path)
        return int(match.group(1)) if match else -1

    parts.sort(key=part_number)

    print(f"Found {len(parts)} part file(s). Merging in order:")
    for p in parts:
        print(f"  {p}")

    with open(output_path, "wb") as outfile:
        for part in parts:
            with open(part, "rb") as infile:
                outfile.write(infile.read())

    print(f"\nMerged into: {output_path}")


def main():
    parser = argparse.ArgumentParser(description="Merge split file parts back into the original file.")
    parser.add_argument("base_name", help="Original file base name (without .partN suffix)")
    parser.add_argument("--output", default=None, help="Output file path (default: same as base_name)")
    args = parser.parse_args()

    output_path = args.output if args.output else args.base_name
    merge_files(args.base_name, output_path)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Merge into a file with the same name as the original
python3 merge_split_files.py bigvideo.mp4

# Merge into a different output path
python3 merge_split_files.py bigvideo.mp4 --output restored_video.mp4
```

The script globs for every file matching `<base_name>.part*`, then uses a regular expression to extract the numeric suffix from each match and sorts by that number — critically, as a true integer sort rather than alphabetical, so `part2` correctly comes before `part10`. It then reads each part in order and writes its bytes sequentially to the output file.

#### Sample Output (Verified)

```bash
$ python3 merge_split_files.py big.bin --output merged.bin
Found 3 part file(s). Merging in order:
  big.bin.part0
  big.bin.part1
  big.bin.part2

Merged into: merged.bin

$ cmp big.bin merged.bin && echo "MATCH - merge verified correct"
MATCH - merge verified correct
```

The merged file was confirmed byte-for-byte identical to the original via `cmp`, verifying a correct round trip with `split_large_files.py` (3.27).

#### Variations / Flags

- Add `--verify-hash <sha256>` to compute the merged output's hash immediately and compare it against a known-good value, catching missing or corrupted parts automatically instead of relying on a separate manual check.
- Add a check that part numbers are contiguous (no gaps like `part0`, `part1`, `part3` with `part2` missing) before merging, failing fast with a clear error instead of silently producing a corrupted file.
- Add `--delete-parts-after` to clean up the `.partN` files once a successful (and ideally hash-verified) merge completes.

#### Common Pitfalls

- **No completeness check** — if a part file is missing (lost in transfer, accidentally deleted), the script merges whatever parts it *does* find without warning, producing a truncated/corrupted output. Always verify the result against a checksum.
- **Glob pattern matches loosely** — `<base_name>.part*` could accidentally pick up unrelated files if your working directory has a naming collision (e.g., a stray `bigvideo.mp4.part_backup`); keep split parts in a dedicated directory when possible.
- **Output path collision**: if the default output path (same as `base_name`) already exists — for instance, if the original file is still present alongside its parts — the script will silently overwrite it, since it opens the output in `"wb"` mode without an existence check.

### 3.29 Watch Directories for Changes

#### Use Case

A sysadmin needs to know when new files land in a drop folder (an upload/inbox directory that a pipeline should react to), or wants to keep an eye on a shared directory to catch unexpected changes as they happen, without manually re-listing the folder over and over.

#### Prerequisites

Standard library only (`os`, `time`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

Like `monitor_file_changes.py` (3.18), this is a polling-based watcher, not an OS-level event notifier — it re-scans the entire directory tree every `--interval` seconds, which means very short-lived files (created and deleted between two polls) can be missed entirely, and large directory trees can make each poll itself slow. For directories with tens of thousands of files or a need for instant notification, an event-driven library like `watchdog` (which wraps `inotify`/`FSEvents`/`ReadDirectoryChangesW`) is a better fit.

#### Script

```python
#!/usr/bin/env python3
"""
watch_directories.py
--------------------------
Watches a directory for file additions, deletions, and modifications.
Press Ctrl+C to stop.

Usage:
    python watch_directories.py <directory_path> [--interval 2]

Example:
    python watch_directories.py /home/user/Downloads
    python watch_directories.py /home/user/Downloads --interval 5
"""

import argparse
import os
import sys
import time
from datetime import datetime


def snapshot(path: str) -> dict:
    state = {}
    for root, _, files in os.walk(path):
        for f in files:
            full_path = os.path.join(root, f)
            try:
                state[full_path] = os.path.getmtime(full_path)
            except OSError:
                continue
    return state


def watch_directory(path: str, interval: int) -> None:
    if not os.path.isdir(path):
        print(f"Error: '{path}' is not a valid directory.")
        sys.exit(1)

    print(f"Watching '{path}' for changes every {interval} second(s). Press Ctrl+C to stop.\n")

    previous_state = snapshot(path)

    try:
        while True:
            time.sleep(interval)
            current_state = snapshot(path)

            added = current_state.keys() - previous_state.keys()
            removed = previous_state.keys() - current_state.keys()
            modified = {
                f for f in current_state.keys() & previous_state.keys()
                if current_state[f] != previous_state[f]
            }

            timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")

            for f in added:
                print(f"[{timestamp}] ADDED:    {f}")
            for f in removed:
                print(f"[{timestamp}] REMOVED:  {f}")
            for f in modified:
                print(f"[{timestamp}] MODIFIED: {f}")

            previous_state = current_state

    except KeyboardInterrupt:
        print("\nStopped watching.")


def main():
    parser = argparse.ArgumentParser(description="Watch a directory for file changes.")
    parser.add_argument("path", help="Directory to watch")
    parser.add_argument("--interval", type=int, default=2, help="Check interval in seconds (default: 2)")
    args = parser.parse_args()

    watch_directory(args.path, args.interval)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 watch_directories.py /home/user/Downloads --interval 5
```

The script takes an initial "snapshot" — a dictionary mapping every file's full path to its modification time, built by recursively walking the tree — then loops forever, sleeping between polls. Each cycle it takes a new snapshot and uses set operations to find paths present now but not before (`added`), present before but not now (`removed`), and present in both but with a changed mtime (`modified`), printing timestamped notices for each category.

#### Sample Output (Verified)

```bash
$ python3 watch_directories.py watchdir --interval 1
Watching 'watchdir' for changes every 1 second(s). Press Ctrl+C to stop.

[2026-07-17 18:41:35] ADDED:    watchdir/new.txt
[2026-07-17 18:41:37] REMOVED:  watchdir/new.txt
```
(A file was created with `echo "new" > watchdir/new.txt` and then removed with `rm watchdir/new.txt` from another terminal during the run, both correctly detected on the next poll cycle.)

#### Variations / Flags

- Add `--command "./on_change.sh {event} {path}"` to trigger arbitrary automation on each detected change, turning this into a lightweight file-drop trigger for a pipeline.
- Add `--extension .csv` filtering so the watcher (and its snapshot cost) only tracks files of interest in a directory with a lot of irrelevant noise.
- Swap the polling loop for the `watchdog` package for real OS-level event notification — recommended for production use, especially on directories with many files where repeated full-tree snapshots become expensive.

#### Common Pitfalls

- **Full-tree re-scan every interval** — for directories with a very large number of files, each poll itself takes real time and I/O, which both slows detection and adds load; narrow the watched path or increase the interval for large trees.
- **Rapid create-then-delete within one interval is invisible** — since only two snapshots are ever compared (previous vs. current), a file that appears and disappears entirely between polls leaves no trace in the output.
- **Blocks the terminal indefinitely**, same as `monitor_file_changes.py` (3.18) — run it in a dedicated session or background it appropriately for unattended use.
- **Memory grows with directory size** — the snapshot dictionary holds one entry per file in the entire tree; for directories with millions of files this could become a meaningful memory footprint, worth considering before pointing this at, say, an entire filesystem root.

### 3.30 Clean Temporary Files

#### Use Case

A sysadmin is about to archive a project directory for long-term storage, or is trying to free up disk space on a build server, and wants to strip out `.tmp`, `.bak`, `.log`, cache files, and `__pycache__` folders without hand-picking each one.

#### Prerequisites

Standard library only (`os`, `shutil`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

This is a destructive, recursive operation. It walks the *entire* directory tree given to it and deletes every matching file/folder it finds, with no per-item confirmation (only one bulk confirmation up front, unless `--force` is used). Point it at the wrong root directory and it will happily clean out something you didn't mean to touch. Always review the printed "will be deleted" list before confirming.

#### Script

```python
#!/usr/bin/env python3
"""
clean_temporary_files.py
------------------------------
Scans a directory tree and deletes common temporary/junk files
(e.g. .tmp, .bak, .log, __pycache__ folders). Asks for confirmation unless --force is used.

Usage:
    python clean_temporary_files.py <directory_path> [--force]

Example:
    python clean_temporary_files.py /home/user/Project
    python clean_temporary_files.py /home/user/Project --force
"""

import argparse
import os
import shutil
import sys

TEMP_EXTENSIONS = (".tmp", ".bak", ".log", ".cache", ".swp")
TEMP_DIR_NAMES = ("__pycache__", ".pytest_cache", "node_modules_tmp")


def find_temp_items(path: str):
    temp_files = []
    temp_dirs = []

    for root, dirs, files in os.walk(path):
        for d in list(dirs):
            if d in TEMP_DIR_NAMES:
                temp_dirs.append(os.path.join(root, d))
                dirs.remove(d)  # don't descend into it

        for f in files:
            if f.lower().endswith(TEMP_EXTENSIONS):
                temp_files.append(os.path.join(root, f))

    return temp_files, temp_dirs


def clean_temp_files(path: str, force: bool) -> None:
    if not os.path.isdir(path):
        print(f"Error: '{path}' is not a valid directory.")
        sys.exit(1)

    temp_files, temp_dirs = find_temp_items(path)

    if not temp_files and not temp_dirs:
        print("No temporary files or folders found.")
        return

    print("The following will be deleted:\n")
    for f in temp_files:
        print(f"  FILE: {f}")
    for d in temp_dirs:
        print(f"  DIR:  {d}")

    if not force:
        answer = input(f"\nDelete these {len(temp_files) + len(temp_dirs)} item(s)? [y/N]: ").strip().lower()
        if answer != "y":
            print("Aborted.")
            return

    for f in temp_files:
        try:
            os.remove(f)
            print(f"Deleted file: {f}")
        except OSError as e:
            print(f"Error deleting '{f}': {e}")

    for d in temp_dirs:
        try:
            shutil.rmtree(d)
            print(f"Deleted directory: {d}")
        except OSError as e:
            print(f"Error deleting '{d}': {e}")

    print("\nCleanup complete.")


def main():
    parser = argparse.ArgumentParser(description="Clean temporary/junk files from a directory tree.")
    parser.add_argument("path", help="Directory to clean")
    parser.add_argument("--force", action="store_true", help="Delete without confirmation prompt")
    args = parser.parse_args()

    clean_temp_files(args.path, args.force)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Interactive: lists everything that would be deleted, asks y/N once
python3 clean_temporary_files.py /home/user/Project

# Non-interactive, for cron jobs or CI pipelines
python3 clean_temporary_files.py /home/user/Project --force
```

`os.walk()` traverses the tree top-down. Whenever it finds a directory name matching `TEMP_DIR_NAMES` it records the full path and removes it from the `dirs` list in place — which stops `os.walk()` from descending into it, avoiding wasted work on a folder that's about to be deleted anyway. Files matching `TEMP_EXTENSIONS` are recorded individually. The script prints the full plan, asks for one confirmation (unless `--force`), then deletes files with `os.remove()` and directories with `shutil.rmtree()`.

#### Sample Output (Verified)

```bash
$ find proj
proj/app.py
proj/debug.log
proj/cache.tmp
proj/backup.bak
proj/__pycache__/x.pyc

$ python3 clean_temporary_files.py proj --force
The following will be deleted:

  FILE: proj/backup.bak
  FILE: proj/debug.log
  FILE: proj/cache.tmp
  DIR:  proj/__pycache__
Deleted file: proj/backup.bak
Deleted file: proj/debug.log
Deleted file: proj/cache.tmp
Deleted directory: proj/__pycache__

Cleanup complete.
```

`proj/app.py` was correctly left alone.

#### Variations / Flags

- Add `--dry-run` to print the plan and exit without prompting or deleting — useful for a first pass on an unfamiliar directory.
- Add `--extensions .tmp,.bak` to override `TEMP_EXTENSIONS` from the command line instead of editing the script.
- Add `--older-than 30` (days) using `os.path.getmtime()` so only stale junk gets removed, not files created seconds ago.

#### Common Pitfalls

- **The extension and directory-name lists are hardcoded** — if your junk files use a different convention (e.g., `.old`, `dist/`), edit `TEMP_EXTENSIONS` / `TEMP_DIR_NAMES` or add a `--extensions` flag before relying on this in a new environment.
- **One confirmation covers everything** — there's no per-file undo once you type `y`. Skim the printed list carefully first.
- **Symlinks inside the tree**: `os.walk()` does not follow symlinked directories by default, so junk inside a symlinked folder won't be touched — this is usually what you want, but worth knowing.
- **Permissions**: files owned by another user or in a read-only mount will raise `OSError` per-item, which is caught and printed, but the overall run will still report "Cleanup complete" even if some deletions failed — check the per-item output, not just the final line.

### 3.31 Automatically Organize Files

#### Use Case

A sysadmin (or anyone) has a Downloads folder that's accumulated hundreds of mixed files — images, documents, videos, archives, code — and wants them automatically sorted into tidy category subfolders in one pass, instead of dragging files around manually.

#### Prerequisites

Standard library only (`os`, `shutil`) — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

This moves files (not copies) — the originals no longer exist at their prior location once organized. It only processes the top level of the given directory (files directly inside it, not files already nested in subfolders), so running it repeatedly on an already-organized folder is safe and simply won't find anything new to move at the top level.

#### Script

```python
#!/usr/bin/env python3
"""
organize_files.py
------------------------
Automatically organizes files in a directory into subfolders based on file extension/type
(e.g. Images, Documents, Videos, Music, Archives, Others).

Usage:
    python organize_files.py <directory_path>

Example:
    python organize_files.py /home/user/Downloads
"""

import argparse
import os
import shutil
import sys

CATEGORY_MAP = {
    "Images": [".jpg", ".jpeg", ".png", ".gif", ".bmp", ".svg", ".webp", ".tiff"],
    "Documents": [".pdf", ".doc", ".docx", ".txt", ".odt", ".rtf", ".xls", ".xlsx", ".ppt", ".pptx", ".csv"],
    "Videos": [".mp4", ".mkv", ".avi", ".mov", ".wmv", ".flv", ".webm"],
    "Music": [".mp3", ".wav", ".flac", ".aac", ".ogg", ".m4a"],
    "Archives": [".zip", ".rar", ".tar", ".gz", ".7z", ".bz2"],
    "Code": [".py", ".js", ".html", ".css", ".java", ".c", ".cpp", ".sh", ".json", ".xml"],
}


def get_category(extension: str) -> str:
    for category, extensions in CATEGORY_MAP.items():
        if extension.lower() in extensions:
            return category
    return "Others"


def organize_files(path: str) -> None:
    if not os.path.isdir(path):
        print(f"Error: '{path}' is not a valid directory.")
        sys.exit(1)

    moved_count = 0

    for entry in os.listdir(path):
        full_path = os.path.join(path, entry)

        if not os.path.isfile(full_path):
            continue

        extension = os.path.splitext(entry)[1]
        category = get_category(extension)
        category_dir = os.path.join(path, category)

        os.makedirs(category_dir, exist_ok=True)

        destination = os.path.join(category_dir, entry)
        try:
            shutil.move(full_path, destination)
            print(f"Moved: {entry} -> {category}/")
            moved_count += 1
        except (OSError, shutil.Error) as e:
            print(f"Error moving '{entry}': {e}")

    print(f"\nOrganized {moved_count} file(s) in '{path}'.")


def main():
    parser = argparse.ArgumentParser(description="Automatically organize files into category subfolders.")
    parser.add_argument("path", help="Directory to organize")
    args = parser.parse_args()

    organize_files(args.path)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
python3 organize_files.py /home/user/Downloads
```

The script lists the top-level contents of the given directory and, for each regular file, looks up its extension in a hardcoded `CATEGORY_MAP` (Images, Documents, Videos, Music, Archives, Code) to decide which subfolder it belongs in — anything unmatched falls into an `Others` category. It creates the target subfolder if needed and moves the file into it with `shutil.move()`.

#### Sample Output (Verified)

```bash
$ python3 organize_files.py organizetest
Moved: song.mp3 -> Music/
Moved: archive.zip -> Archives/
Moved: photo.jpg -> Images/
Moved: unknown.xyz -> Others/
Moved: script.py -> Code/
Moved: doc.pdf -> Documents/

Organized 6 file(s) in 'organizetest'.

$ find organizetest
organizetest/Documents/doc.pdf
organizetest/Music/song.mp3
organizetest/Code/script.py
organizetest/Archives/archive.zip
organizetest/Images/photo.jpg
organizetest/Others/unknown.xyz
```

Every file landed in the correct category subfolder, including the unrecognized `.xyz` extension correctly falling back to `Others`.

#### Variations / Flags

- Add `--dry-run` to preview the planned moves (`file -> Category/`) before committing, especially valuable the first time you point this at an unfamiliar, cluttered folder.
- Add `--recursive` to also organize files already nested in subdirectories, not just the top level — with care, since this changes the script's blast radius considerably.
- Make `CATEGORY_MAP` configurable via an external JSON/YAML file instead of hardcoding it, so categories and extensions can be customized without editing the script itself.

#### Common Pitfalls

- **The category list is hardcoded and opinionated** — a file type not covered by `CATEGORY_MAP` (e.g., `.psd`, `.epub`, `.iso`) lands in a generic `Others` folder rather than a more specific one; extend the map for extensions you use often.
- **Name collisions at the destination**: if `Images/photo.jpg` already exists from a previous run and a new `photo.jpg` needs to move there, `shutil.move()` will overwrite it silently — there's no rename-on-conflict handling.
- **Not recursive by default** — files already inside subfolders (including ones this script created on a prior run) are left alone; this is generally the safer, expected behavior but worth knowing if you expected a deep reorganization.
- **This is a move, not a copy** — always sure of the outcome before running on anything irreplaceable; test on a copy of the folder first if unsure how the categorization will land.

### 3.32 Batch Rename Files

#### Use Case

A sysadmin dumps 400 photos off a phone or camera SD card and needs them renamed from `IMG_2931.jpg`, `IMG_2932.jpg`... into something sortable and meaningful like `vacation_001.jpg`, `vacation_002.jpg`. The same pattern applies to renaming log exports, scanned invoices, or backup snapshots before archiving them.

#### Prerequisites

Standard library only — no installation required. Works on Linux, macOS, and Windows.

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

Renames are immediate and not reversible by the script — there's no undo. Files are processed in sorted filename order, so if the sort order doesn't match the order you actually want (e.g., `img2.jpg` sorting before `img10.jpg`), rename a copy first or fix the source names before running this at scale.

#### Script

```python
#!/usr/bin/env python3
"""
batch_rename_files.py
---------------------------
Batch renames all files in a directory using a prefix and sequential numbering.

Usage:
    python batch_rename_files.py <directory_path> <prefix> [--start 1] [--extension .jpg]

Example:
    python batch_rename_files.py /home/user/Photos vacation_
    # vacation_001.jpg, vacation_002.jpg, ...

    python batch_rename_files.py /home/user/Photos photo_ --start 100 --extension .png
"""

import argparse
import os
import sys


def batch_rename(path: str, prefix: str, start: int, extension_filter: str = None) -> None:
    if not os.path.isdir(path):
        print(f"Error: '{path}' is not a valid directory.")
        sys.exit(1)

    files = sorted(os.listdir(path))

    if extension_filter:
        if not extension_filter.startswith("."):
            extension_filter = "." + extension_filter
        files = [f for f in files if f.lower().endswith(extension_filter.lower())]

    files = [f for f in files if os.path.isfile(os.path.join(path, f))]

    if not files:
        print("No matching files found to rename.")
        return

    counter = start
    renamed = 0

    for f in files:
        ext = os.path.splitext(f)[1]
        new_name = f"{prefix}{counter:03d}{ext}"
        old_path = os.path.join(path, f)
        new_path = os.path.join(path, new_name)

        if os.path.exists(new_path):
            print(f"Skipped (target exists): {new_name}")
            continue

        try:
            os.rename(old_path, new_path)
            print(f"Renamed: {f} -> {new_name}")
            renamed += 1
            counter += 1
        except OSError as e:
            print(f"Error renaming '{f}': {e}")

    print(f"\nRenamed {renamed} file(s).")


def main():
    parser = argparse.ArgumentParser(description="Batch rename files with a prefix and sequential numbers.")
    parser.add_argument("path", help="Directory containing files to rename")
    parser.add_argument("prefix", help="Prefix for new filenames")
    parser.add_argument("--start", type=int, default=1, help="Starting number (default: 1)")
    parser.add_argument("--extension", default=None, help="Only rename files with this extension (e.g. .jpg)")
    args = parser.parse_args()

    batch_rename(args.path, args.prefix, args.start, args.extension)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Rename every file in the directory, sequential numbering starting at 1
python3 batch_rename_files.py /home/user/Photos vacation_

# Only rename .jpg files, starting the counter at 100
python3 batch_rename_files.py /home/user/Photos photo_ --start 100 --extension .png
```

The script lists the directory, optionally filters by extension, sorts the remaining filenames alphabetically, then walks through them assigning `prefix + zero-padded counter + original extension`. If a target name already exists it skips that file rather than overwriting it.

#### Sample Output (Verified)

```bash
$ ls photos
a.jpg  b.jpg  c.png  notes.txt
$ python3 batch_rename_files.py photos vacation_ --extension .jpg
Renamed: a.jpg -> vacation_001.jpg
Renamed: b.jpg -> vacation_002.jpg

Renamed 2 file(s).
```

`c.png` and `notes.txt` were left untouched because of the `--extension .jpg` filter.

#### Variations / Flags

- Add a `--dry-run` flag that prints the planned `old -> new` mapping without calling `os.rename()`, useful for previewing large batches.
- Add `--recursive` with `os.walk()` to rename files inside subdirectories too.
- Pad width is hardcoded to 3 digits (`:03d`); expose it as `--pad 4` for folders with more than 999 files.

#### Common Pitfalls

- **Extension filter is case-insensitive but must match exactly** — `.jpeg` files won't match a `.jpg` filter.
- **Alphabetical sort ≠ chronological order.** If you want photos renamed in the order they were taken, sort by modification time instead of filename before feeding the list in.
- **Re-running the script on already-renamed files** will skip everything because the target names already exist — that's a safety feature, not a bug, but it can be confusing if you expected the counter to keep climbing.
- **No file locking** — don't run this against a directory that another process is actively writing to.

### 3.33 Change File Timestamps

#### Use Case

A sysadmin needs to force a build system to treat a file as freshly modified (triggering a rebuild), or needs to backdate a restored file's timestamp to match its original creation date after recovering it from a backup where the timestamp metadata was lost.

#### Prerequisites

Standard library only (`os`, `time`, `datetime`) — no installation required. Works on Linux, macOS, and Windows (Windows note: file creation time is a separate, OS-specific attribute that `os.utime()` does not touch — see pitfalls).

```bash
python3 --version   # requires Python 3.6+
```

#### Cautions

This overwrites both the access time and modification time of the target file, which can affect backup tools, cache-invalidation logic, or `find -newer` style scripts that rely on real modification history. Don't run it against files that other tooling depends on for accurate change tracking.

#### Script

```python
#!/usr/bin/env python3
"""
change_file_timestamps.py
-------------------------------
Changes the access and modification timestamps of a file (like the 'touch' command).
If no date is given, sets the timestamps to the current time.

Usage:
    python change_file_timestamps.py <file_path> [--date "YYYY-MM-DD HH:MM:SS"]

Example:
    python change_file_timestamps.py report.txt
    python change_file_timestamps.py report.txt --date "2025-01-15 10:30:00"
"""

import argparse
import os
import sys
import time
from datetime import datetime


def change_timestamp(file_path: str, date_str: str = None) -> None:
    if not os.path.exists(file_path):
        # Create the file if it doesn't exist (like 'touch')
        open(file_path, "a").close()
        print(f"File did not exist, created: {file_path}")

    if date_str:
        try:
            dt = datetime.strptime(date_str, "%Y-%m-%d %H:%M:%S")
            timestamp = time.mktime(dt.timetuple())
        except ValueError:
            print("Error: date must be in format 'YYYY-MM-DD HH:MM:SS'")
            sys.exit(1)
    else:
        timestamp = time.time()

    try:
        os.utime(file_path, (timestamp, timestamp))
        new_time = datetime.fromtimestamp(timestamp)
        print(f"Updated timestamps for '{file_path}' to {new_time}")
    except OSError as e:
        print(f"Error updating timestamps: {e}")
        sys.exit(1)


def main():
    parser = argparse.ArgumentParser(description="Change file access/modification timestamps.")
    parser.add_argument("file_path", help="File to update")
    parser.add_argument("--date", default=None, help="Target date/time in 'YYYY-MM-DD HH:MM:SS' format (default: now)")
    args = parser.parse_args()

    change_timestamp(args.file_path, args.date)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

```bash
# Touch to current time (creates the file if it doesn't exist)
python3 change_file_timestamps.py report.txt

# Backdate to a specific date/time
python3 change_file_timestamps.py report.txt --date "2025-01-15 10:30:00"
```

If the target file doesn't exist, the script creates it first (matching `touch` behavior), then calls `os.utime()` to set both the access time and modification time to either the current time or the parsed `--date` value.

#### Sample Output (Verified)

```bash
$ python3 change_file_timestamps.py report.txt --date "2025-01-15 10:30:00"
File did not exist, created: report.txt
Updated timestamps for 'report.txt' to 2025-01-15 10:30:00

$ stat report.txt | grep Modify
Modify: 2025-01-15 10:30:00.000000000 +0000
```

#### Variations / Flags

- Add `--access-only` or `--modify-only` flags to set just one of the two timestamps instead of both together.
- Accept a reference file (`--reference other_file.txt`) to copy its timestamps, mirroring `touch -r`.
- Support glob patterns (`*.log`) to timestamp many files in one call.

#### Common Pitfalls

- **Date format is strict** — only `YYYY-MM-DD HH:MM:SS` is accepted; anything else exits with an error rather than guessing the intended format.
- **`time.mktime()` interprets the date in the local system timezone**, so the same command run on machines in different timezones will produce different absolute timestamps.
- **File *creation* time is not the same as modification time on most filesystems.** On Linux/ext4 there often isn't a user-settable creation time at all; on Windows and macOS, changing it requires separate OS-specific APIs that this script does not touch — `os.utime()` only affects access and modification time.
- **Permissions** — you need write access to the file (and to the parent directory if the file doesn't exist yet) or the script exits with an `OSError`.
