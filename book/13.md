## Chapter 13: Backup & Synchronization

Every system administrator eventually learns the same lesson, usually the hard way: a server is only as trustworthy as its last good backup. Disks fail, deployments go wrong, a `rm -rf` gets typed into the wrong terminal tab, and ransomware doesn't ask permission. Backup and synchronization tooling is the boring, unglamorous discipline that turns those disasters from career events into minor inconveniences.

This chapter walks through ten practical scripts that cover the full lifecycle of keeping data safe: creating backups, keeping them small and current without re-copying everything each time, mirroring directories across machines, proving that a backup actually restores what it claims to, and shipping copies off-site so a single burning building can't take out both the original and the backup. Every script in this chapter is written in Python 3, uses only the standard library unless a section explicitly says otherwise, and is designed to run unmodified on Linux and macOS, with Windows notes where behavior differs (path separators, permission bits, and the availability of tools like `rsync`).

### 13.1 Create Backups

#### Use Case

A sysadmin needs a quick, reliable, self-contained snapshot of a directory before a risky change — a deployment, a config edit, an OS upgrade — so that if something breaks, the previous state is one command away from being restored. This is also the natural first backup in any retention scheme (see 13.2/13.3), since incremental and differential backups both need a full backup as their baseline.

#### Prerequisites

No third-party packages required — uses only the Python standard library (`tarfile`, `os`, `argparse`).

```bash
# Nothing to install; verify Python 3 is available
python3 --version
```

OS scope: Linux, macOS, Windows (all paths use `os.path`/`pathlib` so this is fully cross-platform).

#### Cautions

- A full backup captures a point-in-time copy; anything that changes on disk *during* the backup (an actively-written log file, a database file) may be captured mid-write. For live databases, back up via the database's own dump tool instead (see the database backup scripts in this book) and only use this script for static files/config.
- This script does not exclude anything by default — be careful not to accidentally archive `node_modules`, `.git`, or other huge/irrelevant directories; use `--exclude` to skip them.

#### Script

```python
#!/usr/bin/env python3
"""
backup_create.py
==================
Create a full, timestamped, compressed backup of a directory tree.

Usage
-----
    python backup_create.py --source /etc/myapp --dest /var/backups

If --source is omitted, a small sample directory is created and backed
up so you can see the full flow immediately.
"""

import argparse
import fnmatch
import os
import sys
import tarfile
from datetime import datetime


def create_sample_source(path: str) -> None:
    os.makedirs(path, exist_ok=True)
    with open(os.path.join(path, "config.yaml"), "w", encoding="utf-8") as f:
        f.write("app_name: demo\nversion: 1.0\n")
    with open(os.path.join(path, "notes.txt"), "w", encoding="utf-8") as f:
        f.write("Sample file for backup_create.py demo run.\n")
    print(f"[info] No --source given, created a sample directory at: {path}")


def should_exclude(rel_path: str, patterns) -> bool:
    return any(fnmatch.fnmatch(rel_path, pat) for pat in patterns)


def create_backup(source: str, dest_dir: str, name: str, exclude_patterns) -> str:
    if not os.path.isdir(source):
        print(f"[error] Source directory not found: {source}")
        sys.exit(1)

    os.makedirs(dest_dir, exist_ok=True)
    timestamp = datetime.now().strftime("%Y%m%d-%H%M%S")
    archive_name = f"{name}-{timestamp}.tar.gz"
    archive_path = os.path.join(dest_dir, archive_name)

    file_count = 0
    total_bytes = 0

    def filter_fn(tarinfo):
        nonlocal file_count, total_bytes
        rel = tarinfo.name
        if should_exclude(rel, exclude_patterns):
            print(f"[skip] {rel}")
            return None
        if tarinfo.isfile():
            file_count += 1
            total_bytes += tarinfo.size
        return tarinfo

    with tarfile.open(archive_path, "w:gz") as tar:
        tar.add(source, arcname=os.path.basename(source.rstrip("/")), filter=filter_fn)

    size = os.path.getsize(archive_path)
    print(f"[success] Backup written to: {archive_path}")
    print(f"          Files archived: {file_count}, uncompressed: {total_bytes} bytes, "
          f"compressed: {size} bytes")
    return archive_path


def main():
    parser = argparse.ArgumentParser(description="Create a full timestamped tar.gz backup of a directory.")
    parser.add_argument("--source", default=None, help="Directory to back up")
    parser.add_argument("--dest", default="./backups", help="Directory to write the archive into")
    parser.add_argument("--name", default="backup", help="Base name for the archive file")
    parser.add_argument("--exclude", nargs="*", default=[".git", "__pycache__", "*.pyc", "node_modules"],
                         help="Glob patterns (relative paths) to exclude from the backup")
    args = parser.parse_args()

    source = args.source
    if source is None:
        source = "sample_source"
        create_sample_source(source)

    create_backup(source, args.dest, args.name, args.exclude)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

The script walks the source directory using `tarfile`'s built-in recursive `add()`, streaming files directly into a gzip-compressed archive rather than buffering the whole tree in memory first — so it scales to large directories. A `filter` callback both applies `--exclude` glob patterns and tallies file count/size as it goes, so the summary at the end is free (no second pass over the archive).

```bash
# Basic run: back up /etc/myapp into ./backups
python backup_create.py --source /etc/myapp --dest ./backups

# Custom archive name prefix
python backup_create.py --source /etc/myapp --dest ./backups --name myapp-config

# Exclude extra patterns on top of the defaults
python backup_create.py --source ./project --dest ./backups --exclude "*.log" "tmp/*" ".git"

# No arguments: runs against an auto-created sample directory
python backup_create.py
```

#### Sample Output

```
$ python backup_create.py
[info] No --source given, created a sample directory at: sample_source
[success] Backup written to: ./backups/backup-20260718-143012.tar.gz
          Files archived: 2, uncompressed: 78 bytes, compressed: 178 bytes
```

#### Variations / Flags

- Add a `--dry-run` flag that walks the source and prints what *would* be archived (file list + total size) without writing an archive — useful for sanity-checking `--exclude` patterns before a large run.
- Add `--format zip` to switch from `tar.gz` to `.zip` for environments where recipients don't have `tar` (e.g., Windows users without WSL) — swap `tarfile` for `zipfile.ZipFile`.
- Schedule nightly with cron: `0 2 * * * /usr/bin/python3 /opt/scripts/backup_create.py --source /etc/myapp --dest /var/backups >> /var/log/backup.log 2>&1`.

#### Common Pitfalls

- **Permissions:** running as a non-root user against directories owned by other users (e.g., `/etc`) will silently skip unreadable files rather than erroring loudly — run with `sudo` when backing up system directories, and check the printed file count against expectations.
- **Disk space:** the destination needs enough free space for the compressed archive; a failed write mid-archive leaves a truncated, unusable `.tar.gz` — always verify with 13.6 before relying on it.
- **Symlinks:** by default `tarfile.add` follows symlinks into their targets; a symlink pointing outside the source tree (or a circular symlink) can cause huge or infinite archives — pass `arcname` carefully and consider `dereference=False` if the source contains symlinks you want preserved as-is rather than followed.

### 13.2 Incremental Backups

#### Use Case

A sysadmin running nightly backups of a large, slowly-changing dataset (a home directory, a media library, a log archive) doesn't want to re-copy gigabytes of unchanged files every night. Incremental backups copy only files that changed since the *most recent backup of any kind* (full or incremental), keeping each night's backup small and fast — at the cost of needing the entire chain to restore.

#### Prerequisites

No third-party packages required — uses only the standard library (`json`, `hashlib`, `shutil`).

```bash
python3 --version
```

OS scope: Linux, macOS, Windows.

#### Cautions

- Restoring from an incremental chain requires the original full backup **plus every incremental since**, applied in order. Losing or corrupting one incremental in the middle of the chain breaks everything after it — this is the main trade-off versus differential backups (13.3).
- The manifest file (`manifest.json`) is the source of truth for "what's already backed up." Don't hand-edit it, and don't delete it between runs unless you intend to force a full re-backup.

#### Script

```python
#!/usr/bin/env python3
"""
backup_incremental.py
========================
Back up only files that are new or changed since the last run (full or
incremental), tracked via a JSON manifest of path -> (mtime, size).

Usage
-----
    # First run acts as the baseline (backs up everything)
    python backup_incremental.py --source ./data --dest ./backups

    # Later runs only copy what changed since the manifest was last updated
    python backup_incremental.py --source ./data --dest ./backups
"""

import argparse
import json
import os
import shutil
import sys
from datetime import datetime

MANIFEST_NAME = "manifest.json"


def create_sample_source(path: str) -> None:
    os.makedirs(path, exist_ok=True)
    with open(os.path.join(path, "report.txt"), "w", encoding="utf-8") as f:
        f.write("Quarterly report draft.\n")
    print(f"[info] No --source given, created a sample directory at: {path}")


def load_manifest(dest_dir: str) -> dict:
    path = os.path.join(dest_dir, MANIFEST_NAME)
    if os.path.exists(path):
        with open(path, "r", encoding="utf-8") as f:
            return json.load(f)
    return {}


def save_manifest(dest_dir: str, manifest: dict) -> None:
    path = os.path.join(dest_dir, MANIFEST_NAME)
    with open(path, "w", encoding="utf-8") as f:
        json.dump(manifest, f, indent=2)


def scan_source(source: str) -> dict:
    state = {}
    for root, _dirs, files in os.walk(source):
        for name in files:
            full = os.path.join(root, name)
            rel = os.path.relpath(full, source)
            st = os.stat(full)
            state[rel] = {"mtime": st.st_mtime, "size": st.st_size}
    return state


def run_incremental(source: str, dest_dir: str) -> None:
    if not os.path.isdir(source):
        print(f"[error] Source directory not found: {source}")
        sys.exit(1)

    os.makedirs(dest_dir, exist_ok=True)
    manifest = load_manifest(dest_dir)
    current_state = scan_source(source)

    changed = {
        rel: meta for rel, meta in current_state.items()
        if rel not in manifest or manifest[rel] != meta
    }

    if not changed:
        print("[info] No new or changed files since last backup — nothing to do.")
        return

    timestamp = datetime.now().strftime("%Y%m%d-%H%M%S")
    inc_dir = os.path.join(dest_dir, f"incremental-{timestamp}")
    os.makedirs(inc_dir, exist_ok=True)

    for rel in changed:
        src_path = os.path.join(source, rel)
        dst_path = os.path.join(inc_dir, rel)
        os.makedirs(os.path.dirname(dst_path), exist_ok=True)
        shutil.copy2(src_path, dst_path)
        print(f"[copy] {rel}")

    manifest.update(changed)
    save_manifest(dest_dir, manifest)

    print(f"[success] Incremental backup written to: {inc_dir}")
    print(f"          {len(changed)} file(s) copied out of {len(current_state)} total tracked")


def main():
    parser = argparse.ArgumentParser(description="Back up only files changed since the last run.")
    parser.add_argument("--source", default=None)
    parser.add_argument("--dest", default="./backups")
    args = parser.parse_args()

    source = args.source
    if source is None:
        source = "sample_source"
        create_sample_source(source)

    run_incremental(source, args.dest)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

On each run, the script walks `--source` and builds a dict of every file's modification time and size. It compares this against `manifest.json` in `--dest` (the state as of the last run). Anything new or with a different mtime/size is copied into a fresh, timestamped `incremental-<timestamp>/` folder, and the manifest is updated to reflect the new state — so the *next* run only picks up changes made after this one.

```bash
# Baseline run (everything is "new", so this backs up the whole source)
python backup_incremental.py --source ./data --dest ./backups

# Change a file, then run again — only that file is copied
echo "updated" >> ./data/report.txt
python backup_incremental.py --source ./data --dest ./backups

# Run with nothing changed
python backup_incremental.py --source ./data --dest ./backups
```

#### Sample Output

```
$ python backup_incremental.py --source ./data --dest ./backups
[copy] report.txt
[success] Incremental backup written to: ./backups/incremental-20260718-143521
          1 file(s) copied out of 1 total tracked

$ python backup_incremental.py --source ./data --dest ./backups
[info] No new or changed files since last backup — nothing to do.
```

#### Variations / Flags

- Add `--hash` to compare SHA-256 content hashes instead of mtime/size — slower, but catches changes where mtime was reset without content actually changing (e.g., after a `git checkout`).
- Add `--dry-run` to print which files *would* be copied without actually copying or updating the manifest.
- Schedule nightly with cron, right after 13.1's full backup runs weekly: `0 2 * * 1-6 python3 backup_incremental.py --source /data --dest /backups` (full backup on day 0/Sunday via 13.1, incrementals the other six days).

#### Common Pitfalls

- **Clock skew / mtime unreliability:** if files are copied from another system or an editor rewrites a file without changing content, mtime-based comparison can mis-detect "changed" files (false positives are harmless, just wasteful) or, worse, miss real changes if a backup tool preserves the original mtime (false negatives) — use `--hash` mode when correctness matters more than speed.
- **Chain fragility:** deleting an old `incremental-*` folder in the middle of the sequence silently breaks any restore that depends on it; keep the full backup and every incremental together until you're certain you no longer need that recovery point.
- **Manifest and dest on the same disk as source:** if source and dest share a disk that fails, the whole chain is lost — combine with 13.10 for off-site copies of both the manifest and the incrementals.

### 13.3 Differential Backups

#### Use Case

A sysadmin wants faster, simpler restores than an incremental chain offers (see 13.2) without paying the full cost of nightly full backups (13.1). Differential backups solve this by always comparing against the *last full backup only* — each differential grows a little larger than the last, but a restore only ever needs two pieces: the full backup, and the most recent differential.

#### Prerequisites

No third-party packages required — standard library only (`json`, `shutil`, `os`).

```bash
python3 --version
```

OS scope: Linux, macOS, Windows.

#### Cautions

- Unlike 13.2, a differential run does **not** update its baseline — it always measures against the last *full* backup's manifest. If you never take a new full backup, differentials will keep growing indefinitely; take a fresh full backup periodically (e.g., weekly) and reset the baseline.
- This script assumes a full backup (with a `full-manifest.json` baseline) has already been created via `--action full` before any `--action diff` run.

#### Script

```python
#!/usr/bin/env python3
"""
backup_differential.py
=========================
Take a full backup that establishes a baseline manifest, then take
differential backups that always compare against that same baseline
(not against the previous differential).

Usage
-----
    # Establish the baseline (copies everything, records full-manifest.json)
    python backup_differential.py --action full --source ./data --dest ./backups

    # Each differential run copies everything changed since the FULL backup
    python backup_differential.py --action diff --source ./data --dest ./backups
"""

import argparse
import json
import os
import shutil
import sys
from datetime import datetime

BASELINE_MANIFEST = "full-manifest.json"


def create_sample_source(path: str) -> None:
    os.makedirs(path, exist_ok=True)
    with open(os.path.join(path, "ledger.csv"), "w", encoding="utf-8") as f:
        f.write("date,amount\n2026-07-01,100\n")
    print(f"[info] No --source given, created a sample directory at: {path}")


def scan_source(source: str) -> dict:
    state = {}
    for root, _dirs, files in os.walk(source):
        for name in files:
            full = os.path.join(root, name)
            rel = os.path.relpath(full, source)
            st = os.stat(full)
            state[rel] = {"mtime": st.st_mtime, "size": st.st_size}
    return state


def run_full(source: str, dest_dir: str) -> None:
    if not os.path.isdir(source):
        print(f"[error] Source directory not found: {source}")
        sys.exit(1)

    os.makedirs(dest_dir, exist_ok=True)
    timestamp = datetime.now().strftime("%Y%m%d-%H%M%S")
    full_dir = os.path.join(dest_dir, f"full-{timestamp}")
    state = scan_source(source)

    for rel in state:
        src_path = os.path.join(source, rel)
        dst_path = os.path.join(full_dir, rel)
        os.makedirs(os.path.dirname(dst_path), exist_ok=True)
        shutil.copy2(src_path, dst_path)

    with open(os.path.join(dest_dir, BASELINE_MANIFEST), "w", encoding="utf-8") as f:
        json.dump(state, f, indent=2)

    print(f"[success] Full backup written to: {full_dir}")
    print(f"          Baseline manifest recorded ({len(state)} file(s)) — future --action diff "
          f"runs compare against this until the next full backup.")


def run_diff(source: str, dest_dir: str) -> None:
    baseline_path = os.path.join(dest_dir, BASELINE_MANIFEST)
    if not os.path.exists(baseline_path):
        print(f"[error] No baseline found at {baseline_path}. Run --action full first.")
        sys.exit(1)

    with open(baseline_path, "r", encoding="utf-8") as f:
        baseline = json.load(f)

    current_state = scan_source(source)
    changed = {
        rel: meta for rel, meta in current_state.items()
        if rel not in baseline or baseline[rel] != meta
    }

    if not changed:
        print("[info] No changes since the last full backup — nothing to do.")
        return

    timestamp = datetime.now().strftime("%Y%m%d-%H%M%S")
    diff_dir = os.path.join(dest_dir, f"differential-{timestamp}")

    for rel in changed:
        src_path = os.path.join(source, rel)
        dst_path = os.path.join(diff_dir, rel)
        os.makedirs(os.path.dirname(dst_path), exist_ok=True)
        shutil.copy2(src_path, dst_path)
        print(f"[copy] {rel}")

    print(f"[success] Differential backup written to: {diff_dir}")
    print(f"          {len(changed)} file(s) changed since the last full backup "
          f"(baseline: {len(baseline)} file(s))")


def main():
    parser = argparse.ArgumentParser(description="Full + differential backups against a fixed baseline.")
    parser.add_argument("--action", choices=["full", "diff"], default="full")
    parser.add_argument("--source", default=None)
    parser.add_argument("--dest", default="./backups")
    args = parser.parse_args()

    source = args.source
    if source is None:
        source = "sample_source"
        create_sample_source(source)

    if args.action == "full":
        run_full(source, args.dest)
    else:
        run_diff(source, args.dest)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

`--action full` walks the source, copies every file into a timestamped `full-*` folder, and writes `full-manifest.json` — the fixed reference point. `--action diff` walks the source again and compares against that same `full-manifest.json` (never against a previous differential), copying anything that's new or changed into a fresh `differential-*` folder. Restoring means: extract the latest full backup, then overlay the most recent differential on top.

```bash
# Week 1: establish the baseline
python backup_differential.py --action full --source /data --dest /backups

# Every day after: differential against that same baseline
python backup_differential.py --action diff --source /data --dest /backups

# Following week: take a new full backup to reset growth
python backup_differential.py --action full --source /data --dest /backups
```

#### Sample Output

```
$ python backup_differential.py --action full --source ./data --dest ./backups
[success] Full backup written to: ./backups/full-20260713-020000
          Baseline manifest recorded (1 file(s)) — future --action diff runs
          compare against this until the next full backup.

$ python backup_differential.py --action diff --source ./data --dest ./backups
[copy] ledger.csv
[success] Differential backup written to: ./backups/differential-20260714-020000
          1 file(s) changed since the last full backup (baseline: 1 file(s))
```

#### Variations / Flags

- Add `--hash` for content-hash comparison instead of mtime/size, same rationale as 13.2.
- Add a `--report-only` flag to print what a diff *would* include without copying — useful for estimating tonight's backup size before running it.
- Schedule with cron: full backup weekly (`0 2 * * 0`), differential daily the other six days (`0 2 * * 1-6`) — both invoking this same script with different `--action` values.

#### Common Pitfalls

- **Forgetting to take a new full backup:** if `--action full` is never re-run, every differential keeps comparing against an ever-more-stale baseline and grows toward the size of a full backup anyway, defeating the purpose — schedule full backups on a fixed cadence (e.g., weekly).
- **Restore requires exactly two pieces, in order:** applying a differential without first restoring its corresponding full backup will leave the restore incomplete (only the *changed* files will be present) — always restore full, then differential, never differential alone.
- **Multiple full backups in the destination:** since `full-manifest.json` is a single file, taking a second full backup overwrites the baseline that old differentials were computed against — don't mix differentials from before and after a new full backup when restoring.

### 13.4 Synchronize Directories

#### Use Case

A sysadmin is staging content — deploying a website's static assets, merging a contractor's deliverables into a shared drive, pushing config updates to a shared folder — and wants new and changed files copied across, but doesn't want anything already present at the destination removed, even if it's no longer in the source. This is the safe, additive counterpart to mirroring (13.5).

#### Prerequisites

No third-party packages required — standard library only (`os`, `shutil`).

```bash
python3 --version
```

OS scope: Linux, macOS, Windows.

#### Cautions

- This script never deletes anything at the destination — if you need an exact one-way replica (including deletions), use 13.5 Mirror Folders instead.
- Files are compared by size + modification time, not content hash by default; two files with identical content but different mtimes will be re-copied unnecessarily (harmless, just extra I/O — use `--hash` for a stricter comparison).

#### Script

```python
#!/usr/bin/env python3
"""
sync_directories.py
======================
One-way, additive synchronization: copy new and modified files from
--source to --dest. Files that exist only in --dest are left alone.

Usage
-----
    python sync_directories.py --source ./project --dest ./staging
"""

import argparse
import hashlib
import os
import shutil
import sys


def create_sample_dirs(source: str, dest: str) -> None:
    os.makedirs(source, exist_ok=True)
    os.makedirs(dest, exist_ok=True)
    with open(os.path.join(source, "index.html"), "w", encoding="utf-8") as f:
        f.write("<html><body>Hello</body></html>\n")
    with open(os.path.join(dest, "old_notes.txt"), "w", encoding="utf-8") as f:
        f.write("This file only exists in dest and will NOT be deleted.\n")
    print(f"[info] No --source/--dest given, created sample dirs: {source}, {dest}")


def file_hash(path: str) -> str:
    h = hashlib.sha256()
    with open(path, "rb") as f:
        for chunk in iter(lambda: f.read(65536), b""):
            h.update(chunk)
    return h.hexdigest()


def needs_copy(src_path: str, dst_path: str, use_hash: bool) -> bool:
    if not os.path.exists(dst_path):
        return True
    src_stat, dst_stat = os.stat(src_path), os.stat(dst_path)
    if use_hash:
        return file_hash(src_path) != file_hash(dst_path)
    return src_stat.st_size != dst_stat.st_size or src_stat.st_mtime > dst_stat.st_mtime


def sync(source: str, dest: str, use_hash: bool, dry_run: bool) -> None:
    if not os.path.isdir(source):
        print(f"[error] Source directory not found: {source}")
        sys.exit(1)

    os.makedirs(dest, exist_ok=True)
    copied, skipped = 0, 0

    for root, _dirs, files in os.walk(source):
        rel_root = os.path.relpath(root, source)
        dst_root = os.path.join(dest, rel_root) if rel_root != "." else dest
        os.makedirs(dst_root, exist_ok=True)

        for name in files:
            src_path = os.path.join(root, name)
            dst_path = os.path.join(dst_root, name)

            if needs_copy(src_path, dst_path, use_hash):
                action = "[would copy]" if dry_run else "[copy]"
                print(f"{action} {os.path.relpath(src_path, source)}")
                if not dry_run:
                    shutil.copy2(src_path, dst_path)
                copied += 1
            else:
                skipped += 1

    verb = "would copy" if dry_run else "copied"
    print(f"[success] Sync complete: {copied} file(s) {verb}, {skipped} file(s) already up to date")


def main():
    parser = argparse.ArgumentParser(description="One-way additive directory synchronization.")
    parser.add_argument("--source", default=None)
    parser.add_argument("--dest", default=None)
    parser.add_argument("--hash", action="store_true", help="Compare file contents via SHA-256 instead of size/mtime")
    parser.add_argument("--dry-run", action="store_true", help="Show what would be copied without copying")
    args = parser.parse_args()

    source, dest = args.source, args.dest
    if source is None or dest is None:
        source, dest = "sample_source", "sample_dest"
        create_sample_dirs(source, dest)

    sync(source, dest, args.hash, args.dry_run)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

The script walks `--source` recursively, mirroring the directory structure (not the files) into `--dest` as needed with `os.makedirs`. For each file it decides whether to copy using `needs_copy()`: by default that's a cheap size/mtime comparison; with `--hash` it reads and hashes both files for a byte-accurate comparison. Files unique to `--dest` are never touched or reported as different — they're simply out of scope for this script.

```bash

python sync_directories.py --source ./project --dest ./staging

# Preview only, no files touched
python sync_directories.py --source ./project --dest ./staging --dry-run

# Strict content comparison
python sync_directories.py --source ./project --dest ./staging --hash

# No arguments: runs against auto-created sample dirs
python sync_directories.py
```

#### Variations / Flags

- Add `--exclude` glob patterns (see 13.1) to skip build artifacts, `.git`, or temp files during sync.
- Add `--delete` to opt into destination-side deletions on a per-run basis — at that point it becomes equivalent to 13.5 Mirror Folders, so consider using that script directly instead.
- Wrap with `watchdog` (pip install watchdog) for continuous sync on file-change events instead of periodic cron runs, for near-real-time staging environments.

#### Common Pitfalls

- **Case-insensitive filesystems:** on macOS (default) and Windows, `File.txt` and `file.txt` are the same path — a source and dest that differ only in case can produce surprising overwrite behavior; Linux ext4 is case-sensitive and won't have this issue.
- **Permissions on the destination:** `shutil.copy2` preserves metadata (including mode bits) from the source but will fail if the destination directory itself isn't writable by the running user — check destination ownership before scheduling as a cron job under a specific service account.
- **Symlinks in the source:** by default `os.walk` does not follow symlinked directories (avoiding infinite loops), so symlinked subdirectories are silently skipped unless you pass `followlinks=True` to `os.walk` — do this deliberately, since it can introduce cycles if the source tree is untrusted.


### 13.5 Mirror Folders

#### Use Case

A sysadmin needs the destination to be an *exact* replica of the source — a deploy target that should contain nothing but the current release, a backup drive that should reflect exactly what's on the primary disk right now, a config directory pushed identically to every server in a fleet. Unlike 13.4's additive sync, mirroring also removes anything from the destination that isn't in the source.

#### Prerequisites

No third-party packages required — standard library only (`os`, `shutil`).

```bash
python3 --version
```

OS scope: Linux, macOS, Windows.

#### Cautions

- **This script deletes files and directories.** A wrong `--source`/`--dest` pair (or swapping the two arguments) can silently wipe out real data with no way back short of restoring from a separate backup. Always run with `--dry-run` first and read the output before running for real.
- Never point `--dest` at a directory that also contains files you care about but that aren't meant to be mirrored from `--source` — this script treats "not in source" as "should not exist," full stop.

#### Script

```python
#!/usr/bin/env python3
"""
mirror_folders.py
====================
Exact one-way mirror: --dest ends up containing exactly what --source
contains — new/changed files are copied, and anything in --dest that
is not in --source is deleted.

Usage
-----
    python mirror_folders.py --source ./release --dest /var/www/html --dry-run
    python mirror_folders.py --source ./release --dest /var/www/html
"""

import argparse
import os
import shutil
import sys


def create_sample_dirs(source: str, dest: str) -> None:
    os.makedirs(source, exist_ok=True)
    os.makedirs(dest, exist_ok=True)
    with open(os.path.join(source, "app.js"), "w", encoding="utf-8") as f:
        f.write("console.log('hello');\n")
    with open(os.path.join(dest, "old_app.js"), "w", encoding="utf-8") as f:
        f.write("// stale file, not in source — will be removed by mirror\n")
    print(f"[info] No --source/--dest given, created sample dirs: {source}, {dest}")


def needs_copy(src_path: str, dst_path: str) -> bool:
    if not os.path.exists(dst_path):
        return True
    s, d = os.stat(src_path), os.stat(dst_path)
    return s.st_size != d.st_size or s.st_mtime > d.st_mtime


def mirror(source: str, dest: str, dry_run: bool) -> None:
    if not os.path.isdir(source):
        print(f"[error] Source directory not found: {source}")
        sys.exit(1)

    os.makedirs(dest, exist_ok=True)

    source_rel_files = set()
    copied = 0

    # Pass 1: copy new/changed files, tracking every relative path seen in source
    for root, _dirs, files in os.walk(source):
        rel_root = os.path.relpath(root, source)
        dst_root = os.path.join(dest, rel_root) if rel_root != "." else dest
        if not dry_run:
            os.makedirs(dst_root, exist_ok=True)

        for name in files:
            src_path = os.path.join(root, name)
            dst_path = os.path.join(dst_root, name)
            rel_path = os.path.normpath(os.path.join(rel_root, name)) if rel_root != "." else name
            source_rel_files.add(rel_path)

            if needs_copy(src_path, dst_path):
                action = "[would copy]" if dry_run else "[copy]"
                print(f"{action} {rel_path}")
                if not dry_run:
                    shutil.copy2(src_path, dst_path)
                copied += 1

    # Pass 2: remove anything in dest that isn't in source
    removed = 0
    for root, dirs, files in os.walk(dest, topdown=False):
        rel_root = os.path.relpath(root, dest)
        for name in files:
            rel_path = os.path.normpath(os.path.join(rel_root, name)) if rel_root != "." else name
            if rel_path not in source_rel_files:
                action = "[would delete]" if dry_run else "[delete]"
                print(f"{action} {rel_path}")
                if not dry_run:
                    os.remove(os.path.join(root, name))
                removed += 1
        # remove now-empty directories (except dest itself)
        if not dry_run and root != dest and not os.listdir(root):
            os.rmdir(root)

    verb_c, verb_d = ("would copy", "would delete") if dry_run else ("copied", "deleted")
    print(f"[success] Mirror complete: {copied} file(s) {verb_c}, {removed} file(s) {verb_d}")


def main():
    parser = argparse.ArgumentParser(description="Exact one-way directory mirror (copies + deletions).")
    parser.add_argument("--source", default=None)
    parser.add_argument("--dest", default=None)
    parser.add_argument("--dry-run", action="store_true", help="Show what would change without changing anything")
    args = parser.parse_args()

    source, dest = args.source, args.dest
    if source is None or dest is None:
        source, dest = "sample_source", "sample_dest"
        create_sample_dirs(source, dest)

    mirror(source, dest, args.dry_run)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

The script runs in two passes. Pass one walks `--source` and copies anything new or changed into `--dest`, while recording the full set of relative paths that exist in the source. Pass two walks `--dest` bottom-up (`topdown=False`, so directories are visited after their contents) and deletes any file whose relative path wasn't in that set, then removes any directory left empty as a result. `--dry-run` runs the exact same logic but skips every actual filesystem mutation, so the printed plan is a reliable preview.

```bash
# ALWAYS preview a mirror before running it for real
python mirror_folders.py --source ./release --dest /var/www/html --dry-run

# Apply for real once the dry-run output looks correct
python mirror_folders.py --source ./release --dest /var/www/html

# No arguments: runs against auto-created sample dirs
python mirror_folders.py
```

#### Sample Output

```
$ python mirror_folders.py --dry-run
[info] No --source/--dest given, created sample dirs: sample_source, sample_dest
[would copy] app.js
[would delete] old_app.js
[success] Mirror complete: 1 file(s) would copy, 1 file(s) would delete

$ python mirror_folders.py --source sample_source --dest sample_dest
[copy] app.js
[delete] old_app.js
[success] Mirror complete: 1 file(s) copied, 1 file(s) deleted
```

#### Variations / Flags

- Add `--exclude` patterns (as in 13.1/13.4) for paths in `--dest` that should survive mirroring regardless of source content — e.g., a `logs/` directory a web server writes to that shouldn't be wiped every deploy.
- Add `--backup-deleted ./trash` to move files instead of permanently deleting them, as a safety net during rollout of this script.
- For production use at scale, prefer the real `rsync -av --delete` (see 13.10) — it's battle-tested, handles permissions/symlinks/hard-links correctly, and is dramatically faster on large trees; this script is best for understanding the mechanics or for environments where `rsync` isn't available.

#### Common Pitfalls

- **Wrong argument order:** because `--source` and `--dest` both delete-and-replace behavior lives at the destination, accidentally swapping them mirrors the (usually smaller/older) destination back onto the source, destroying the real data — the `--dry-run` output is the safety check for exactly this mistake.
- **Empty `--dest` on first run:** the very first mirror of a populated existing `--dest` will delete *everything* not already in `--source` — expected behavior, but surprising if you assumed mirroring only added files (that's 13.4, not this script).
- **Race conditions:** if something else is actively writing to `--dest` while this script runs (e.g., a live web server), the delete pass can remove a file that was just recreated by that process — mirror into a staging path and atomically swap/symlink it into place for zero-downtime deploys instead of mirroring directly into a live serving directory.

### 13.6 Verify Backup Integrity

#### Use Case

A sysadmin has been taking nightly backups for months and needs to know, before a real emergency forces the question, whether those backups actually restore correctly — no missing files, no silent corruption from a flaky disk or an interrupted copy. This script checksums a source tree and a backup tree and reports exactly what differs, so integrity checks can run automatically after every backup job rather than being discovered as a surprise during a real restore.

#### Prerequisites

No third-party packages required — standard library only (`hashlib`, `os`).

```bash
python3 --version
```

OS scope: Linux, macOS, Windows.

#### Cautions

- Hashing every file in a large backup is I/O- and CPU-intensive; running this against multi-terabyte backups nightly can itself become a load concern — consider sampling a subset of files on most nights and doing a full verification weekly.
- This script verifies file-level integrity (contents match), not backup *completeness* against some external record of what should exist — if a file was silently missing from the backup at creation time in both places, this check won't flag it.

#### Script

```python
#!/usr/bin/env python3
"""
verify_backup.py
===================
Compare a source directory against a backup directory using SHA-256
checksums and report any file that's missing, extra, or has drifted.

Usage
-----
    python verify_backup.py --source ./data --backup ./backups/full-20260713-020000
"""

import argparse
import hashlib
import os
import sys


def create_sample_dirs(source: str, backup: str) -> None:
    os.makedirs(source, exist_ok=True)
    os.makedirs(backup, exist_ok=True)
    with open(os.path.join(source, "a.txt"), "w", encoding="utf-8") as f:
        f.write("content A\n")
    with open(os.path.join(source, "b.txt"), "w", encoding="utf-8") as f:
        f.write("content B\n")
    with open(os.path.join(backup, "a.txt"), "w", encoding="utf-8") as f:
        f.write("content A\n")  # matches
    with open(os.path.join(backup, "b.txt"), "w", encoding="utf-8") as f:
        f.write("CORRUPTED\n")  # deliberately different, to demonstrate a mismatch
    print(f"[info] No --source/--backup given, created sample dirs (with one deliberate mismatch): "
          f"{source}, {backup}")


def file_hash(path: str) -> str:
    h = hashlib.sha256()
    with open(path, "rb") as f:
        for chunk in iter(lambda: f.read(65536), b""):
            h.update(chunk)
    return h.hexdigest()


def build_hash_map(root: str) -> dict:
    result = {}
    for dirpath, _dirs, files in os.walk(root):
        for name in files:
            full = os.path.join(dirpath, name)
            rel = os.path.relpath(full, root)
            result[rel] = file_hash(full)
    return result


def verify(source: str, backup: str) -> bool:
    if not os.path.isdir(source):
        print(f"[error] Source directory not found: {source}")
        sys.exit(1)
    if not os.path.isdir(backup):
        print(f"[error] Backup directory not found: {backup}")
        sys.exit(1)

    print("[info] Hashing source tree...")
    source_hashes = build_hash_map(source)
    print("[info] Hashing backup tree...")
    backup_hashes = build_hash_map(backup)

    missing = sorted(set(source_hashes) - set(backup_hashes))
    extra = sorted(set(backup_hashes) - set(source_hashes))
    mismatched = sorted(
        rel for rel in (set(source_hashes) & set(backup_hashes))
        if source_hashes[rel] != backup_hashes[rel]
    )

    for rel in missing:
        print(f"[MISSING]    {rel}  (in source, not in backup)")
    for rel in extra:
        print(f"[EXTRA]      {rel}  (in backup, not in source)")
    for rel in mismatched:
        print(f"[MISMATCH]   {rel}  (checksum differs)")

    ok = not (missing or mismatched)  # extras are reported but don't fail integrity on their own
    total = len(source_hashes)
    verified = total - len(missing) - len(mismatched)

    if ok and not extra:
        print(f"[success] Verified {verified}/{total} file(s) — backup matches source exactly.")
    elif ok:
        print(f"[success] Verified {verified}/{total} file(s) — all source files present and correct "
              f"({len(extra)} extra file(s) in backup, not necessarily a problem).")
    else:
        print(f"[FAILURE] {len(missing)} missing, {len(mismatched)} corrupted, out of {total} source file(s). "
              f"This backup should NOT be trusted for restore until resolved.")

    return ok


def main():
    parser = argparse.ArgumentParser(description="Verify a backup's integrity against its source via SHA-256.")
    parser.add_argument("--source", default=None)
    parser.add_argument("--backup", default=None)
    args = parser.parse_args()

    source, backup = args.source, args.backup
    if source is None or backup is None:
        source, backup = "sample_source", "sample_backup"
        create_sample_dirs(source, backup)

    ok = verify(source, backup)
    sys.exit(0 if ok else 1)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

The script builds two dictionaries — relative path to SHA-256 hex digest — one for `--source` and one for `--backup`, by streaming each file through `hashlib.sha256()` in 64KB chunks (so memory use stays flat regardless of file size). It then diffs the two path sets three ways: files only in source (**MISSING** from the backup), files only in the backup (**EXTRA**, often just old files from a previous run and not automatically a failure), and files in both whose hashes differ (**MISMATCH**, real corruption). The process exit code is `0` only when there are no missing or mismatched files, so this script can gate a deployment pipeline or alert on failure.

```bash
# Verify a specific backup against its source
python verify_backup.py --source /data --backup /backups/full-20260713-020000

# Use the exit code in a script or cron job
python verify_backup.py --source /data --backup /backups/latest && echo "backup OK"

# No arguments: runs against sample dirs with a deliberate mismatch, to show FAILURE output
python verify_backup.py
```

#### Sample Output

```
$ python verify_backup.py
[info] No --source/--backup given, created sample dirs (with one deliberate mismatch): sample_source, sample_backup
[info] Hashing source tree...
[info] Hashing backup tree...
[MISMATCH]   b.txt  (checksum differs)
[FAILURE] 0 missing, 1 corrupted, out of 2 source file(s). This backup should NOT be trusted for restore until resolved.
```

#### Variations / Flags

- Add `--report json` to emit machine-readable output for feeding into a monitoring system (Nagios, Prometheus pushgateway, a Slack webhook) instead of human-readable text.
- Add `--sample 10%` to hash only a random percentage of files for fast nightly spot-checks, reserving a full verification for a weekly job.
- Chain after 13.1/13.2/13.3: `python backup_create.py ... && python verify_backup.py --source ./data --backup ./backups/<latest>` so every backup is verified immediately, catching write errors while the source data is still fresh.

#### Common Pitfalls

- **Comparing against a compressed archive:** this script expects `--backup` to be an extracted directory, not a `.tar.gz`/`.zip` file — extract first (or adapt the script to read from within the archive) before verifying a compressed backup produced by 13.7.
- **Performance on large trees:** full SHA-256 verification of a multi-terabyte backup can take hours; don't schedule it inside the same maintenance window as the backup job itself without measuring how long it actually takes first.
- **False sense of security from "EXTRA only":** a backup with only EXTRA files (no MISSING/MISMATCH) still passes, but "extra" files are often stale data from a previous, unrelated backup that happens to share the directory — periodically clean out `--backup` directories that mix multiple unrelated runs.

### 13.7 Compress Backups

#### Use Case

A sysadmin has a directory of backup files (or an uncompressed backup folder from 13.1/13.2/13.3) that's consuming too much disk space or taking too long to transfer off-site (13.10), and needs it compressed into a single portable archive with a predictable size/speed trade-off.

#### Prerequisites

No third-party packages required — standard library only (`tarfile`, `zipfile`).

```bash
python3 --version
```

OS scope: Linux, macOS, Windows (both `.tar.gz` and `.zip` output work everywhere in Python; `.tar.gz` is more common on Linux/macOS, `.zip` is more universally readable on Windows without extra tools).

#### Cautions

- Compression is CPU-bound; on a busy server, a high compression level applied to a very large directory can noticeably affect other workloads — consider `--level` tuning or running during low-traffic windows (`nice`/`ionice` on Linux).
- Compressing already-compressed data (JPEG images, video, other `.gz`/`.zip` files) yields little to no size reduction but still costs CPU time — exclude such files if the backup contains a lot of them.

#### Script

```python
#!/usr/bin/env python3
"""
compress_backup.py
=====================
Compress a directory into a single .tar.gz or .zip archive, with a
configurable compression level.

Usage
-----
    python compress_backup.py --source ./backups/full-20260713-020000 \
        --output ./backups/full-20260713-020000.tar.gz --format tar.gz --level 9
"""

import argparse
import os
import sys
import tarfile
import zipfile


def create_sample_source(path: str) -> None:
    os.makedirs(path, exist_ok=True)
    with open(os.path.join(path, "data.txt"), "w", encoding="utf-8") as f:
        f.write("Sample backup content. " * 200 + "\n")
    print(f"[info] No --source given, created a sample directory at: {path}")


def compress_tar_gz(source: str, output: str, level: int) -> None:
    with tarfile.open(output, f"w:gz", compresslevel=level) as tar:
        tar.add(source, arcname=os.path.basename(source.rstrip("/")))


def compress_zip(source: str, output: str, level: int) -> None:
    with zipfile.ZipFile(output, "w", compression=zipfile.ZIP_DEFLATED, compresslevel=level) as zf:
        base = os.path.basename(source.rstrip("/"))
        for root, _dirs, files in os.walk(source):
            for name in files:
                full = os.path.join(root, name)
                arcname = os.path.join(base, os.path.relpath(full, source))
                zf.write(full, arcname)


def compress(source: str, output: str, fmt: str, level: int) -> None:
    if not os.path.isdir(source):
        print(f"[error] Source directory not found: {source}")
        sys.exit(1)

    uncompressed_size = sum(
        os.path.getsize(os.path.join(dp, f))
        for dp, _dn, filenames in os.walk(source)
        for f in filenames
    )

    print(f"[info] Compressing {source} ({uncompressed_size} bytes uncompressed) -> {output} "
          f"(format={fmt}, level={level})")

    if fmt == "tar.gz":
        compress_tar_gz(source, output, level)
    else:
        compress_zip(source, output, level)

    compressed_size = os.path.getsize(output)
    ratio = (1 - compressed_size / uncompressed_size) * 100 if uncompressed_size else 0
    print(f"[success] Archive written to: {output}")
    print(f"          {uncompressed_size} bytes -> {compressed_size} bytes ({ratio:.1f}% smaller)")


def main():
    parser = argparse.ArgumentParser(description="Compress a directory into a single tar.gz or zip archive.")
    parser.add_argument("--source", default=None)
    parser.add_argument("--output", default=None)
    parser.add_argument("--format", choices=["tar.gz", "zip"], default="tar.gz")
    parser.add_argument("--level", type=int, default=6, help="Compression level: 1 (fastest) to 9 (smallest)")
    args = parser.parse_args()

    source = args.source
    if source is None:
        source = "sample_source"
        create_sample_source(source)

    output = args.output or f"{source.rstrip('/')}.{args.format}"
    compress(source, output, args.format, args.level)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

For `.tar.gz`, the script streams the source directory straight into `tarfile`'s gzip writer at the requested `compresslevel` (1–9, matching gzip's own scale) without materializing an intermediate uncompressed tarball. For `.zip`, it walks the tree manually and writes each file individually with `ZIP_DEFLATED` compression, since `zipfile` doesn't have a single-call recursive `add()` like `tarfile` does. Either way, it reports the size before and after so you can judge whether a given `--level` is worth the extra CPU time for this particular dataset.

```bash
# Compress a completed full backup folder (default tar.gz, level 6)
python compress_backup.py --source ./backups/full-20260713-020000

# Maximum compression, for archival/off-site storage where speed doesn't matter
python compress_backup.py --source ./backups/full-20260713-020000 --level 9

# Fastest compression, for a quick nightly job where speed matters more than size
python compress_backup.py --source ./backups/full-20260713-020000 --level 1

# Zip format instead, for Windows-friendly distribution
python compress_backup.py --source ./backups/full-20260713-020000 --format zip
```

#### Sample Output

```
$ python compress_backup.py --level 9
[info] No --source given, created a sample directory at: sample_source
[info] Compressing sample_source (4004 bytes uncompressed) -> sample_source.tar.gz (format=tar.gz, level=9)
[success] Archive written to: sample_source.tar.gz
          4004 bytes -> 47 bytes (98.8% smaller)
```

#### Variations / Flags

- Add `--exclude` glob patterns (as in 13.1) to skip files not worth compressing (already-compressed media, `.tmp` files).
- Add a `--threads N` mode using `tarfile` per-file parallel compression via a process pool for very large backups, since gzip compression in the standard library is single-threaded by default.
- Pipe output through `zstd` or `xz` via `subprocess` instead of gzip for meaningfully better compression ratios on text-heavy backups, at the cost of requiring those tools to be installed.

#### Common Pitfalls

- **Level 9 isn't always worth it:** for many real-world datasets, the size difference between level 6 and level 9 is small (a few percent) while the CPU time difference can be large — benchmark on your actual data rather than assuming max compression is always the right default.
- **Zip 32-bit size limits:** the classic ZIP format has a 4GB per-file / 4GB total limit unless ZIP64 extensions are used; Python's `zipfile` enables ZIP64 automatically for large files, but archives created by *other* tools without ZIP64 support may not open correctly if they exceed this — not usually an issue with this script, but worth knowing if archives are consumed by legacy tools.
- **Compressing a backup that's still being written:** compressing a `full-*` or `incremental-*` directory while 13.1/13.2 is still copying into it produces a corrupt, incomplete archive — always run compression as a separate step *after* the backup job reports success, not concurrently with it.

### 13.8 Encrypt Backups

#### Use Case

A sysadmin is about to ship a compressed backup (13.7) to off-site or cloud storage (13.10) and needs it unreadable to anyone who gains access to that storage without the password — a stolen backup drive, a misconfigured cloud bucket, or a compromised third-party storage provider should not mean a data breach.

#### Prerequisites

```bash
pip install cryptography
```

OS scope: Linux, macOS, Windows.

#### Cautions

- **There is no password recovery.** If the password used to encrypt is lost, the backup is permanently unreadable — store the password in a password manager or secrets vault, separately from the encrypted backup itself, before relying on this.
- This script encrypts a single file at a time (typically your compressed archive from 13.7) — compress first, then encrypt, not the other way around, since encrypted data doesn't compress further.
- Never commit passwords to version control or pass them as plain CLI arguments in shared/logged environments — this script supports an environment variable and an interactive prompt specifically to avoid that.

#### Script

```python
#!/usr/bin/env python3
"""
encrypt_backup.py
====================
Encrypt (and decrypt) a backup file using a password-derived key, via
Fernet symmetric encryption (AES-128-CBC + HMAC) from the `cryptography`
library.

Usage
-----
    # Encrypt (prompts for a password if not given via --password or DB_BACKUP_PASSWORD)
    python encrypt_backup.py --action encrypt --input backup.tar.gz --output backup.tar.gz.enc

    # Decrypt
    python encrypt_backup.py --action decrypt --input backup.tar.gz.enc --output backup.tar.gz

Requirements
------------
    pip install cryptography
"""

import argparse
import base64
import getpass
import os
import sys

try:
    from cryptography.fernet import Fernet, InvalidToken
    from cryptography.hazmat.primitives import hashes
    from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
except ImportError:
    print("[error] 'cryptography' is not installed. Run: pip install cryptography")
    sys.exit(1)

SALT_SIZE = 16
PBKDF2_ITERATIONS = 390_000


def create_sample_input(path: str) -> None:
    with open(path, "wb") as f:
        f.write(b"Sample backup archive contents, standing in for a real .tar.gz file.\n")
    print(f"[info] No --input given, created a sample file at: {path}")


def derive_key(password: str, salt: bytes) -> bytes:
    kdf = PBKDF2HMAC(algorithm=hashes.SHA256(), length=32, salt=salt, iterations=PBKDF2_ITERATIONS)
    return base64.urlsafe_b64encode(kdf.derive(password.encode("utf-8")))


def get_password(cli_password: str) -> str:
    if cli_password:
        return cli_password
    env_password = os.environ.get("DB_BACKUP_PASSWORD")
    if env_password:
        return env_password
    return getpass.getpass("Backup encryption password: ")


def encrypt_file(input_path: str, output_path: str, password: str) -> None:
    if not os.path.exists(input_path):
        print(f"[error] Input file not found: {input_path}")
        sys.exit(1)

    salt = os.urandom(SALT_SIZE)
    key = derive_key(password, salt)
    fernet = Fernet(key)

    with open(input_path, "rb") as f:
        plaintext = f.read()
    ciphertext = fernet.encrypt(plaintext)

    with open(output_path, "wb") as f:
        f.write(salt)  # store the salt alongside the ciphertext; it's not secret
        f.write(ciphertext)

    print(f"[success] Encrypted '{input_path}' -> '{output_path}' "
          f"({os.path.getsize(input_path)} -> {os.path.getsize(output_path)} bytes)")


def decrypt_file(input_path: str, output_path: str, password: str) -> None:
    if not os.path.exists(input_path):
        print(f"[error] Input file not found: {input_path}")
        sys.exit(1)

    with open(input_path, "rb") as f:
        salt = f.read(SALT_SIZE)
        ciphertext = f.read()

    key = derive_key(password, salt)
    fernet = Fernet(key)

    try:
        plaintext = fernet.decrypt(ciphertext)
    except InvalidToken:
        print("[error] Decryption failed — wrong password, or the file is corrupted/tampered with.")
        sys.exit(1)

    with open(output_path, "wb") as f:
        f.write(plaintext)

    print(f"[success] Decrypted '{input_path}' -> '{output_path}' ({len(plaintext)} bytes)")


def main():
    parser = argparse.ArgumentParser(description="Encrypt or decrypt a backup file with a password.")
    parser.add_argument("--action", choices=["encrypt", "decrypt"], default="encrypt")
    parser.add_argument("--input", default=None)
    parser.add_argument("--output", default=None)
    parser.add_argument("--password", default=None,
                         help="Password (prefer DB_BACKUP_PASSWORD env var or the interactive prompt instead)")
    args = parser.parse_args()

    input_path = args.input
    if input_path is None:
        input_path = "sample_backup.tar.gz"
        create_sample_input(input_path)

    password = get_password(args.password)

    if args.action == "encrypt":
        output_path = args.output or f"{input_path}.enc"
        encrypt_file(input_path, output_path, password)
    else:
        output_path = args.output or input_path.removesuffix(".enc")
        decrypt_file(input_path, output_path, password)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

The script never uses the password directly as an encryption key. Instead it derives a proper 256-bit key from the password using PBKDF2-HMAC-SHA256 with a random 16-byte salt and a high iteration count (390,000, in line with current OWASP guidance), which makes brute-forcing the password computationally expensive even if the encrypted file is stolen. That salt is stored, unencrypted, as a prefix of the output file — salts aren't secret, only the password is — so decryption can regenerate the exact same key. The actual encryption uses `Fernet`, which combines AES-128 in CBC mode with an HMAC for authenticity, so tampering with the encrypted file is detected (via `InvalidToken`) rather than silently producing corrupted plaintext.

python encrypt_backup.py --action encrypt --input backup.tar.gz --output backup.tar.gz.enc

# Decrypt (needs the same password)
python encrypt_backup.py --action decrypt --input backup.tar.gz.enc --output backup.tar.gz

# No --input: encrypts an auto-created sample file, prompts for a password interactively

#### Variations / Flags

- Swap password-based encryption for asymmetric encryption (e.g., `age` or GPG public-key encryption via `subprocess`) so backup jobs can encrypt with a public key without needing the (secret) private key present on the backed-up server at all — better for automated, unattended backups.
- Add `--chunk-size` streaming encryption for files too large to comfortably fit in memory at once (Fernet as used here reads the whole file into memory; very large backups should use a streaming AEAD construction instead).
- Combine with 13.9: rotate old encrypted backups just like plaintext ones — encryption doesn't change retention policy.

#### Common Pitfalls

- **Losing the password:** repeated for emphasis because it's the single most common real-world backup-encryption failure — there is no backdoor, by design; treat the password with the same care as the data it protects.
- **Reusing a password across many backups without a secrets manager:** if the same password protects months of backups and it leaks, every one of those backups is compromised at once — consider rotating the encryption password periodically alongside key rotation policy for other secrets.
- **Iteration count vs. hardware:** 390,000 PBKDF2 iterations takes a noticeable fraction of a second on typical hardware — if run against very many small files, this per-file KDF cost adds up; encrypt one large compressed archive (13.7) rather than encrypting thousands of small files individually.

### 13.9 Automatic Backup Rotation

#### Use Case

A sysadmin has a nightly backup cron job (13.1/13.2/13.3, optionally compressed via 13.7) that's been running for a year, and the backup disk is now full of backups nobody needs anymore — last Tuesday's backup has little value once this Tuesday's exists. This script applies a retention policy automatically, deleting backups that fall outside the configured window so disk usage stays bounded without manual cleanup.

#### Prerequisites

No third-party packages required — standard library only (`os`, `re`, `datetime`).

```bash
python3 --version
```

OS scope: Linux, macOS, Windows.

#### Cautions

- **This script permanently deletes backup files/folders that fall outside the retention policy.** Always run with `--dry-run` after changing `--keep-daily`/`--keep-weekly`/`--keep-monthly` values, and before pointing it at a real backup directory for the first time.
- This script identifies backups by parsing a timestamp out of each filename (matching the naming convention used by 13.1/13.2/13.3/13.7 — e.g., `full-20260713-020000` or `backup-20260713-020000.tar.gz`). Backups that don't match the expected pattern are left untouched and reported, not silently deleted.

#### Script

```python
#!/usr/bin/env python3
"""
backup_rotation.py
=====================
Apply a grandfather-father-son style retention policy to a directory
of dated backups: keep the N most recent daily backups, plus one
backup per week for --keep-weekly weeks, plus one per month for
--keep-monthly months. Everything else is deleted.

Usage
-----
    python backup_rotation.py --backup-dir ./backups \
        --keep-daily 7 --keep-weekly 4 --keep-monthly 6 --dry-run
"""

import argparse
import os
import re
import shutil
import sys
from datetime import datetime

# Matches names like "full-20260713-020000", "backup-20260713-020000.tar.gz",
# "incremental-20260713-020000", "differential-20260713-020000"
TIMESTAMP_RE = re.compile(r"(\d{8})-(\d{6})")


def create_sample_backups(backup_dir: str) -> None:
    os.makedirs(backup_dir, exist_ok=True)
    from datetime import timedelta
    now = datetime.now()
    # Simulate 40 days of nightly backups
    for i in range(40):
        ts = (now - timedelta(days=i)).strftime("%Y%m%d-%H%M%S")
        name = f"backup-{ts}.tar.gz"
        with open(os.path.join(backup_dir, name), "w", encoding="utf-8") as f:
            f.write("sample backup placeholder\n")
    print(f"[info] No --backup-dir contents found, created 40 days of sample backups in: {backup_dir}")


def parse_timestamp(name: str):
    m = TIMESTAMP_RE.search(name)
    if not m:
        return None
    try:
        return datetime.strptime(m.group(1) + m.group(2), "%Y%m%d%H%M%S")
    except ValueError:
        return None


def classify_retention(backups, keep_daily, keep_weekly, keep_monthly):
    """Return the set of filenames to KEEP."""
    backups_sorted = sorted(backups.items(), key=lambda kv: kv[1], reverse=True)  # newest first
    keep = set()

    # Keep the N most recent, unconditionally (daily tier)
    for name, _ts in backups_sorted[:keep_daily]:
        keep.add(name)

    # Weekly tier: for each of the last keep_weekly weeks, keep the newest backup in that week
    remaining = [(n, t) for n, t in backups_sorted if n not in keep]
    seen_weeks = {}
    for name, ts in remaining:
        week_key = ts.strftime("%Y-W%U")
        if week_key not in seen_weeks and len(seen_weeks) < keep_weekly:
            seen_weeks[week_key] = name
            keep.add(name)

    # Monthly tier: for each of the last keep_monthly months, keep the newest backup in that month
    remaining = [(n, t) for n, t in backups_sorted if n not in keep]
    seen_months = {}
    for name, ts in remaining:
        month_key = ts.strftime("%Y-%m")
        if month_key not in seen_months and len(seen_months) < keep_monthly:
            seen_months[month_key] = name
            keep.add(name)

    return keep


def rotate(backup_dir: str, keep_daily: int, keep_weekly: int, keep_monthly: int, dry_run: bool) -> None:
    if not os.path.isdir(backup_dir):
        print(f"[error] Backup directory not found: {backup_dir}")
        sys.exit(1)

    entries = os.listdir(backup_dir)
    backups, unrecognized = {}, []
    for name in entries:
        ts = parse_timestamp(name)
        if ts:
            backups[name] = ts
        else:
            unrecognized.append(name)

    if not backups:
        print("[info] No timestamped backups found — nothing to rotate.")
        return

    keep = classify_retention(backups, keep_daily, keep_weekly, keep_monthly)
    to_delete = sorted(set(backups) - keep)

    for name in unrecognized:
        print(f"[skip] {name}  (doesn't match the expected timestamp pattern — left untouched)")

    for name in sorted(keep):
        print(f"[keep] {name}")

    for name in to_delete:
        action = "[would delete]" if dry_run else "[delete]"
        print(f"{action} {name}")
        if not dry_run:
            full_path = os.path.join(backup_dir, name)
            if os.path.isdir(full_path):
                shutil.rmtree(full_path)
            else:
                os.remove(full_path)

    verb = "would delete" if dry_run else "deleted"
    print(f"[success] Retention applied: {len(keep)} kept, {len(to_delete)} {verb}, "
          f"{len(unrecognized)} unrecognized (left alone)")


def main():
    parser = argparse.ArgumentParser(description="Apply daily/weekly/monthly retention to a backup directory.")
    parser.add_argument("--backup-dir", default=None)
    parser.add_argument("--keep-daily", type=int, default=7)
    parser.add_argument("--keep-weekly", type=int, default=4)
    parser.add_argument("--keep-monthly", type=int, default=6)
    parser.add_argument("--dry-run", action="store_true")
    args = parser.parse_args()

    backup_dir = args.backup_dir
    if backup_dir is None:
        backup_dir = "sample_backups"
        create_sample_backups(backup_dir)

    rotate(backup_dir, args.keep_daily, args.keep_weekly, args.keep_monthly, args.dry_run)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

The script scans `--backup-dir`, extracting a `YYYYMMDD-HHMMSS` timestamp from each entry's filename via regex. It then applies a three-tier grandfather-father-son policy: the `--keep-daily` most recent backups are always kept regardless of date; beyond that, it keeps the single newest backup from each of the last `--keep-weekly` distinct calendar weeks; beyond that, the newest backup from each of the last `--keep-monthly` distinct calendar months. Everything not selected by any tier is deleted. Entries that don't match the expected naming pattern are reported but never touched, so unrelated files sharing the directory are safe.

```bash
# Preview the retention decision without deleting anything
python backup_rotation.py --backup-dir /backups --keep-daily 7 --keep-weekly 4 --keep-monthly 6 --dry-run

# Apply for real once the dry-run output looks right
python backup_rotation.py --backup-dir /backups --keep-daily 7 --keep-weekly 4 --keep-monthly 6

# Shorter retention for a low-value/high-frequency backup set
python backup_rotation.py --backup-dir /backups/hourly --keep-daily 2 --keep-weekly 0 --keep-monthly 0

# No arguments: generates 40 days of sample backups and rotates them with the defaults
python backup_rotation.py --dry-run
```

#### Sample Output

```
$ python backup_rotation.py --dry-run
[info] No --backup-dir contents found, created 40 days of sample backups in: sample_backups
[keep] backup-20260718-143012.tar.gz
[keep] backup-20260717-143012.tar.gz
...
[would delete] backup-20260610-143012.tar.gz
[would delete] backup-20260609-143012.tar.gz
[success] Retention applied: 17 kept, 23 would delete, 0 unrecognized (left alone)
```

#### Variations / Flags

- Add `--keep-yearly N` for a fourth tier, common in compliance-driven environments that require multi-year retention of at least one backup per year.
- Add `--min-free-gb` to trigger rotation only when free disk space drops below a threshold, rather than (or in addition to) a fixed schedule.
- Schedule immediately after the nightly backup+verify job in cron, so rotation only runs against a set that's already been confirmed good: `... && python verify_backup.py ... && python backup_rotation.py --backup-dir /backups --keep-daily 7 --keep-weekly 4 --keep-monthly 6`.

#### Common Pitfalls

- **Running before a backup completes:** rotating while tonight's backup job is still writing can misclassify or even delete an in-progress backup if its partial filename happens to match the timestamp pattern — always chain rotation *after* a successful, verified backup, never in parallel with one.
- **Overlapping tiers double-counting:** because the script keeps backups tier by tier and skips anything already kept, `--keep-daily` backups don't also consume a slot in the weekly/monthly tiers — but if `--keep-daily` is set very high, it can span multiple weeks/months on its own, making the weekly/monthly settings effectively no-ops; sanity check with `--dry-run` after any change to these values.
- **Naming convention drift:** if a backup is manually renamed or copied in from a different tool with a different filename format, it silently falls into the "unrecognized, left alone" bucket rather than being rotated — periodically audit the directory listing for files this script is quietly ignoring.

### 13.10 Remote Backup via rclone/rsync

#### Use Case

A sysadmin has local backups (13.1–13.9) sitting on the same machine they're protecting against — which means a single disk failure, fire, or ransomware attack that takes out the primary server takes out the backups too. This script ships a copy off-site: over SSH to another server via `rsync`, or to cloud object storage (S3, Backblaze B2, Google Drive, and dozens of others) via `rclone`.

#### Prerequisites

This script is a thin, safety-checked wrapper around external CLI tools — it does not reimplement transfer logic itself.

```bash
# rsync — usually preinstalled on Linux/macOS; on Debian/Ubuntu if missing:
sudo apt install rsync

# rclone — download from https://rclone.org/downloads/ or via package manager:
sudo apt install rclone        # Debian/Ubuntu
brew install rclone            # macOS

# rclone also requires a one-time configuration of the remote target:
rclone config
```

OS scope: Linux and macOS have both tools natively available; on Windows, use WSL, Git Bash with rsync, or rclone's native Windows binary (`rsync` is not native to Windows).

#### Cautions

- **This script executes external `rsync`/`rclone` commands with the flags you pass it** — a mistyped path or an `--rsync-flag ' --delete'` can delete data on the *remote* side just as easily as 13.5 can locally. Always test with `--dry-run` first (both tools support it natively, and this script forwards it).
- SSH-based `rsync` requires key-based authentication already set up to the remote host for unattended/cron use — this script does not manage SSH keys or prompt for passwords.
- Bandwidth and remote storage costs are real; large backups shipped nightly to cloud storage can become expensive quickly — check `rclone`'s `--bwlimit` flag and your storage provider's pricing before scheduling frequent large transfers.

#### Script

```python
#!/usr/bin/env python3
"""
remote_backup.py
===================
Ship a local backup directory off-site using rsync (over SSH to a
remote host) or rclone (to cloud storage), with a required dry-run
preview path.

Usage
-----
    # rsync to a remote server over SSH
    python remote_backup.py --tool rsync --source ./backups/ --dest user@remotehost:/backups/ --dry-run
    python remote_backup.py --tool rsync --source ./backups/ --dest user@remotehost:/backups/

    # rclone to a configured cloud remote (run `rclone config` first)
    python remote_backup.py --tool rclone --source ./backups/ --dest myremote:backups/ --dry-run
    python remote_backup.py --tool rclone --source ./backups/ --dest myremote:backups/
"""

import argparse
import shutil
import subprocess
import sys


def create_sample_source(path: str) -> None:
    import os
    os.makedirs(path, exist_ok=True)
    with open(os.path.join(path, "placeholder.txt"), "w", encoding="utf-8") as f:
        f.write("Sample local backup content for a remote_backup.py dry-run demo.\n")
    print(f"[info] No --source given, created a sample directory at: {path}")


def check_tool(tool: str) -> None:
    if shutil.which(tool) is None:
        print(f"[error] '{tool}' command not found on PATH. See the Prerequisites section for install instructions.")
        sys.exit(1)


def build_rsync_command(source: str, dest: str, dry_run: bool, extra_flags):
    cmd = ["rsync", "-avz", "--progress"]
    if dry_run:
        cmd.append("--dry-run")
    cmd.extend(extra_flags)
    cmd.extend([source, dest])
    return cmd


def build_rclone_command(source: str, dest: str, dry_run: bool, extra_flags):
    cmd = ["rclone", "sync", source, dest, "--progress"]
    if dry_run:
        cmd.append("--dry-run")
    cmd.extend(extra_flags)
    return cmd


def run_transfer(tool: str, source: str, dest: str, dry_run: bool, extra_flags) -> None:
    check_tool(tool)

    if tool == "rsync":
        cmd = build_rsync_command(source, dest, dry_run, extra_flags)
    else:
        cmd = build_rclone_command(source, dest, dry_run, extra_flags)

    print(f"[info] Running: {' '.join(cmd)}")
    result = subprocess.run(cmd, text=True)

    if result.returncode != 0:
        print(f"[error] {tool} exited with code {result.returncode} — transfer may be incomplete.")
        sys.exit(result.returncode)

    mode = "Dry-run preview" if dry_run else "Transfer"
    print(f"[success] {mode} completed via {tool}: {source} -> {dest}")


def main():
    parser = argparse.ArgumentParser(description="Ship a local backup off-site via rsync (SSH) or rclone (cloud).")
    parser.add_argument("--tool", choices=["rsync", "rclone"], default="rsync")
    parser.add_argument("--source", default=None, help="Local source path (trailing slash matters for rsync)")
    parser.add_argument("--dest", default=None, help="Remote destination, e.g. user@host:/path/ or remote:bucket/path/")
    parser.add_argument("--dry-run", action="store_true", help="Preview the transfer without copying any data")
    parser.add_argument("--extra-flags", nargs="*", default=[], help="Additional flags passed through to rsync/rclone")
    args = parser.parse_args()

    source, dest = args.source, args.dest
    if source is None:
        source = "sample_source/"
        create_sample_source("sample_source")
    if dest is None:
        print("[error] --dest is required (e.g. user@remotehost:/backups/ or myremote:backups/)")
        sys.exit(1)

    run_transfer(args.tool, source, dest, args.dry_run, args.extra_flags)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

The script is deliberately a thin wrapper: it checks the chosen tool is installed via `shutil.which`, constructs a well-formed command line, and hands off to `subprocess.run` so `rsync`/`rclone`'s own (mature, well-tested) transfer, retry, and progress-reporting logic does the real work rather than being reimplemented here. `rsync` is used with `-avz` (archive mode, verbose, compressed in transit) over SSH; `rclone sync` makes the destination match the source, similar in spirit to 13.5's local mirroring but for cloud remotes. `--dry-run` is forwarded natively to both tools rather than being simulated by this script, so the preview accurately reflects exactly what each tool would do.

```bash
# Preview an rsync transfer to a remote server over SSH (requires SSH key auth already set up)
python remote_backup.py --tool rsync --source ./backups/ --dest user@backup-host:/mnt/backups/ --dry-run

# Run it for real
python remote_backup.py --tool rsync --source ./backups/ --dest user@backup-host:/mnt/backups/

# Preview an rclone sync to a configured cloud remote (e.g. S3, B2, Drive)
python remote_backup.py --tool rclone --source ./backups/ --dest myremote:company-backups/ --dry-run

# Pass extra flags through, e.g. bandwidth limit and deleting remote files no longer present locally
python remote_backup.py --tool rclone --source ./backups/ --dest myremote:company-backups/ \
    --extra-flags --bwlimit 10M --delete-during
```

#### Variations / Flags

- Add `--tool both` to run `rsync` to a secondary on-prem server *and* `rclone` to cloud storage in the same job, for the classic "3-2-1" backup rule (3 copies, 2 different media, 1 off-site).
- Add `--bwlimit` as a first-class flag (rather than only via `--extra-flags`) if bandwidth throttling is used on every run, to avoid saturating the site's uplink during business hours.
- Schedule after 13.9's rotation step, so only the backups you're actually keeping get shipped off-site: `... && python backup_rotation.py ... && python remote_backup.py --tool rclone --source ./backups/ --dest myremote:backups/`.

#### Common Pitfalls

- **Trailing slashes matter for rsync:** `rsync -avz ./backups user@host:/dest/` (no trailing slash on source) copies the `backups` folder itself into `/dest/`, while `./backups/` (trailing slash) copies its *contents* into `/dest/` — get this backwards and you either nest an extra directory level or scatter files unexpectedly.
- **SSH host key / authentication failures in cron:** a cron job runs without an interactive terminal, so any SSH host-key prompt or password prompt will hang or fail silently — pre-accept the host key (`ssh-keyscan`) and use passwordless key-based auth before scheduling.
- **rclone remotes are named, not typed inline:** `myremote:backups/` refers to a remote named `myremote` set up via `rclone config`, not a literal hostname — a fresh machine running this script needs `rclone config` run (or the config file copied over) before any `--tool rclone` transfer will work.
