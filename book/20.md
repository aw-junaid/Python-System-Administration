## Chapter 20: Report Generation

Every system a sysadmin touches eventually needs to explain itself to someone who isn't sitting at a terminal — a manager who wants a weekly uptime summary, a finance team that wants server costs in a spreadsheet, a monitoring pipeline that wants JSON it can ingest, or a teammate who just wants to glance at a chart instead of scrolling through raw log lines. Report generation is the bridge between the data you can query and the format the person (or system) on the other end can actually use.

This chapter collects ten self-contained Python scripts that each solve one report-generation problem well, rather than one monolithic tool that tries to do everything badly. Every script follows the same shape: it accepts a CSV (or, in a couple of cases, a JSON) file as input, falls back to built-in sample data if you don't give it one, and writes a single, predictable output artifact. That consistency means once you understand one script in this chapter, you can read any of the others in under a minute.

The chapter is organized by output format, moving roughly from "human reads this in a browser or editor" toward "a machine reads this" and finally toward "this updates live." Static document formats (HTML, Markdown, PDF) come first, followed by data-interchange formats (CSV, Excel, JSON, XML) that other tools consume, and the chapter closes with visual and interactive outputs (charts, a web dashboard, and a terminal graph) for when a table of numbers isn't the fastest way to spot a problem.

Each section below is a short pointer to its own dedicated file, where you'll find the use case, prerequisites, the full script, run instructions with flags, sample output, variations, and the pitfalls that actually bite people in production. Treat this chapter page as the index; treat each numbered file as the manual page for that specific tool.

A few conventions apply across every script in this chapter, so they aren't repeated ten times over:

- **No input required to try it.** Every script ships with built-in sample data. Run it with zero arguments first to confirm your environment is set up correctly, before pointing it at real data.
- **`--input` / `-i` selects your data.** Where a script accepts real data, it's a CSV file (two scripts accept JSON instead — noted in their own pages).
- **Output paths are flags, not hardcoded.** Every script lets you choose the output filename or directory with `--output`/`-o` or `--outdir`/`-d`.
- **Dependencies are isolated per script.** Some scripts use only the Python standard library; others need a `pip install`. Each script's page tells you exactly what to install and nothing more — don't install the whole chapter's dependency list just to run one script.
- **These are templates, not black boxes.** They're short enough to read end to end and safe to modify — swap the sample data function, add a column, change the color palette — without breaking the rest of the script.

### 20.1 Generate HTML Reports

#### Use Case

You need a report someone can just double-click and view in a browser — no spreadsheet software, no PDF reader, nothing installed on their end. This script turns any CSV into a self-contained, responsive HTML page with a styled table, so a quick server-status or inventory snapshot looks presentable enough to screenshot into a Slack message or attach to an email.

#### Prerequisites

Standard library only — no installation required.

```bash
# Verify you have Python 3 available; no pip install needed
python3 --version
```

- **OS scope:** Linux, macOS, Windows — anywhere Python 3 runs.
- **Libraries used:** `argparse`, `csv`, `datetime`, `html`, `os`, `sys` (all stdlib).

#### Cautions

The script escapes cell values with `html.escape()` before writing them into the page, which prevents a malicious CSV cell from injecting HTML/JS into the report — don't remove that call if you modify the script. Status-color classes are matched by exact lowercase string (`ok`, `warning`, `critical`, etc.), so unrecognized status text will render with no color coding, not an error.

#### Script

```python
#!/usr/bin/env python3
"""
generate_html_report.py

Generates a responsive, styled HTML report (table-based) from CSV data
or from built-in sample data if no input file is given.

Usage:
    python generate_html_report.py
    python generate_html_report.py --input data.csv --output report.html --title "My Report"

Expected output:
    An .html file that opens in any browser, showing a styled, responsive
    table with alternating row colors, a header banner, and a footer with
    the generation timestamp. Resize the browser window to see the table
    scroll horizontally on small screens.
"""

import argparse
import csv
import datetime
import html
import os
import sys


def load_sample_data():
    headers = ["Server", "CPU Usage (%)", "Memory Usage (%)", "Disk Usage (%)", "Status"]
    rows = [
        ["web-01", "42", "63", "71", "OK"],
        ["web-02", "88", "77", "54", "WARNING"],
        ["db-01", "35", "81", "90", "CRITICAL"],
        ["cache-01", "12", "22", "18", "OK"],
        ["worker-03", "67", "58", "40", "OK"],
    ]
    return headers, rows


def load_csv_data(path):
    with open(path, newline="", encoding="utf-8") as f:
        reader = csv.reader(f)
        rows = list(reader)
    if not rows:
        raise ValueError("CSV file is empty.")
    return rows[0], rows[1:]


def status_class(value):
    v = value.strip().lower()
    if v in ("critical", "error", "fail", "failed"):
        return "status-critical"
    if v in ("warning", "warn"):
        return "status-warning"
    if v in ("ok", "good", "pass", "passed", "success"):
        return "status-ok"
    return ""


def build_html(headers, rows, title):
    generated_at = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")

    header_html = "".join(f"<th>{html.escape(h)}</th>" for h in headers)

    body_rows = []
    for row in rows:
        cells = []
        for i, cell in enumerate(row):
            cls = status_class(cell) if headers[i].lower() == "status" else ""
            cells.append(f'<td class="{cls}">{html.escape(str(cell))}</td>')
        body_rows.append(f"<tr>{''.join(cells)}</tr>")
    body_html = "\n            ".join(body_rows)

    return f"""<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>{html.escape(title)}</title>
<style>
    :root {{
        --primary: #2563eb;
        --bg: #f4f6fb;
        --card-bg: #ffffff;
        --border: #e2e8f0;
        --text: #1e293b;
        --muted: #64748b;
    }}
    * {{ box-sizing: border-box; }}
    body {{
        margin: 0;
        font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Arial, sans-serif;
        background: var(--bg);
        color: var(--text);
        padding: 24px;
    }}
    .container {{
        max-width: 1000px;
        margin: 0 auto;
        background: var(--card-bg);
        border-radius: 12px;
        box-shadow: 0 4px 16px rgba(0,0,0,0.06);
        overflow: hidden;
    }}
    header {{
        background: linear-gradient(135deg, var(--primary), #1d4ed8);
        color: #fff;
        padding: 24px 28px;
    }}
    header h1 {{ margin: 0 0 4px 0; font-size: 22px; }}
    header p {{ margin: 0; opacity: 0.85; font-size: 13px; }}
    .table-wrap {{ overflow-x: auto; padding: 0 0 8px 0; }}
    table {{
        width: 100%;
        border-collapse: collapse;
        min-width: 500px;
        font-size: 14px;
    }}
    thead th {{
        text-align: left;
        background: #f1f5f9;
        color: var(--muted);
        text-transform: uppercase;
        font-size: 11px;
        letter-spacing: 0.05em;
        padding: 12px 16px;
        border-bottom: 2px solid var(--border);
        position: sticky;
        top: 0;
    }}
    tbody td {{
        padding: 12px 16px;
        border-bottom: 1px solid var(--border);
    }}
    tbody tr:nth-child(even) {{ background: #fafbfe; }}
    tbody tr:hover {{ background: #eef2ff; }}
    .status-ok {{ color: #15803d; font-weight: 600; }}
    .status-warning {{ color: #b45309; font-weight: 600; }}
    .status-critical {{ color: #b91c1c; font-weight: 600; }}
    footer {{
        padding: 14px 28px;
        font-size: 12px;
        color: var(--muted);
        border-top: 1px solid var(--border);
    }}
    @media (max-width: 600px) {{
        body {{ padding: 8px; }}
        header {{ padding: 18px; }}
    }}
</style>
</head>
<body>
    <div class="container">
        <header>
            <h1>{html.escape(title)}</h1>
            <p>Generated on {generated_at}</p>
        </header>
        <div class="table-wrap">
            <table>
                <thead><tr>{header_html}</tr></thead>
                <tbody>
            {body_html}
                </tbody>
            </table>
        </div>
        <footer>Report generated automatically by generate_html_report.py &mdash; {len(rows)} row(s).</footer>
    </div>
</body>
</html>
"""


def main():
    parser = argparse.ArgumentParser(description="Generate a responsive HTML report from CSV data.")
    parser.add_argument("--input", "-i", help="Path to input CSV file. If omitted, sample data is used.")
    parser.add_argument("--output", "-o", default="report.html", help="Path to output HTML file (default: report.html)")
    parser.add_argument("--title", "-t", default="System Status Report", help="Report title")
    args = parser.parse_args()

    if args.input:
        if not os.path.isfile(args.input):
            print(f"Error: input file not found: {args.input}", file=sys.stderr)
            sys.exit(1)
        headers, rows = load_csv_data(args.input)
    else:
        headers, rows = load_sample_data()
        print("No --input provided, using built-in sample data.")

    html_content = build_html(headers, rows, args.title)

    with open(args.output, "w", encoding="utf-8") as f:
        f.write(html_content)

    print(f"HTML report written to: {os.path.abspath(args.output)}")
    print("Open it in any web browser to view.")


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

The script reads either your CSV or its built-in sample data into a `(headers, rows)` pair, then hands that off to `build_html()`, which builds one big f-string containing the full page (CSS included, so the output file is self-contained with no external assets). Any column literally named `Status` gets its cell value checked against `status_class()` so it renders in green/amber/red.

```bash
# Zero-config run — uses built-in sample server-status data
python generate_html_report.py

# Point it at your own CSV and set a custom title
python generate_html_report.py --input servers.csv --output status.html --title "Weekly Server Status"

# Short flags work the same way
python generate_html_report.py -i servers.csv -o status.html -t "Weekly Server Status"
```

#### Sample Output

```bash
$ python generate_html_report.py
No --input provided, using built-in sample data.
HTML report written to: /home/user/reports/report.html
Open it in any web browser to view.
```

Opening `report.html` shows a blue header banner titled "System Status Report," a five-row table with the `web-02` and `db-01` rows highlighted amber/red under the Status column, and a footer noting "5 row(s)" with the generation timestamp.

#### Variations / Flags

- Change `--title` per run to reuse the script across different systems (`"Database Cluster Health"`, `"CDN Node Report"`) without touching the code.
- Add a cron entry to regenerate the report on a schedule and serve it from a simple web server: `0 * * * * python3 /opt/scripts/generate_html_report.py -i /var/log/status.csv -o /var/www/html/status.html`.
- Extend `status_class()` with more keywords (e.g., `"degraded"`, `"maintenance"`) if your status vocabulary differs from OK/WARNING/CRITICAL.

#### Common Pitfalls

- **Wrong working directory:** relative `--output` paths are resolved from wherever you run the command, not the script's location — use an absolute path in cron jobs.
- **CSV encoding mismatches:** the script opens files as UTF-8; a CSV exported from older Excel versions as Latin-1/Windows-1252 will raise a `UnicodeDecodeError` — re-save it as UTF-8 first.
- **Empty CSV:** an input file with headers but zero data rows produces a valid but empty-bodied table — this is a silent "success," so check row counts if you're piping this into automation.
- **Browser caching:** if you regenerate the report to the same filename and the browser tab is already open, you'll need a hard refresh to see the update — the file itself has no cache-busting.


### 20.2 Generate Markdown Reports

#### Use Case

You maintain a documentation repo, wiki, or GitHub-tracked project and want a status report that lives in version control alongside the code — reviewable in a diff, renderable natively on GitHub/GitLab, and editable by hand afterward if needed. This script converts a CSV into a clean `.md` file with a title, timestamp, summary line, and a pipe-table.

#### Prerequisites

Standard library only — no installation required.

```bash
# Verify you have Python 3 available; no pip install needed
python3 --version
```

- **OS scope:** Linux, macOS, Windows — anywhere Python 3 runs.
- **Libraries used:** `argparse`, `csv`, `datetime`, `os`, `sys` (all stdlib).

#### Cautions

Markdown pipe-tables break if a cell contains an unescaped `|` character or an embedded newline — the script's `escape_md()` handles both, so don't bypass it if you customize cell formatting. Very wide tables (many columns) render poorly on GitHub's mobile view; consider transposing or trimming columns for reports meant to be read on a phone.

#### Script

```python
#!/usr/bin/env python3
"""
generate_markdown_report.py

Generates a clean Markdown (.md) report suitable for documentation
repositories, wikis, or GitHub README-style output. Reads from CSV
or uses built-in sample data.

Usage:
    python generate_markdown_report.py
    python generate_markdown_report.py --input data.csv --output report.md --title "My Report"

Expected output:
    A .md file containing a title, a generation timestamp, a short
    summary section, and a clean Markdown pipe-table. Renders correctly
    on GitHub, GitLab, and any standard Markdown viewer.
"""

import argparse
import csv
import datetime
import os
import sys


def load_sample_data():
    headers = ["Task", "Owner", "Priority", "Status"]
    rows = [
        ["Set up CI pipeline", "Ayesha", "High", "Done"],
        ["Write unit tests", "Bilal", "Medium", "In Progress"],
        ["Deploy to staging", "Ayesha", "High", "Pending"],
        ["Update documentation", "Sara", "Low", "Pending"],
    ]
    return headers, rows


def load_csv_data(path):
    with open(path, newline="", encoding="utf-8") as f:
        rows = list(csv.reader(f))
    if not rows:
        raise ValueError("CSV file is empty.")
    return rows[0], rows[1:]


def escape_md(value):
    return str(value).replace("|", "\\|").replace("\n", " ")


def build_markdown(headers, rows, title):
    generated_at = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")

    lines = []
    lines.append(f"# {title}")
    lines.append("")
    lines.append(f"_Generated on {generated_at}_")
    lines.append("")
    lines.append(f"**Total records:** {len(rows)}")
    lines.append("")
    lines.append("## Report Data")
    lines.append("")

    lines.append("| " + " | ".join(escape_md(h) for h in headers) + " |")
    lines.append("| " + " | ".join(["---"] * len(headers)) + " |")
    for row in rows:
        lines.append("| " + " | ".join(escape_md(c) for c in row) + " |")

    lines.append("")
    lines.append("---")
    lines.append("*This report was generated automatically by `generate_markdown_report.py`.*")
    lines.append("")

    return "\n".join(lines)


def main():
    parser = argparse.ArgumentParser(description="Generate a Markdown report from CSV data.")
    parser.add_argument("--input", "-i", help="Path to input CSV file. If omitted, sample data is used.")
    parser.add_argument("--output", "-o", default="report.md", help="Path to output Markdown file (default: report.md)")
    parser.add_argument("--title", "-t", default="Project Status Report", help="Report title")
    args = parser.parse_args()

    if args.input:
        if not os.path.isfile(args.input):
            print(f"Error: input file not found: {args.input}", file=sys.stderr)
            sys.exit(1)
        headers, rows = load_csv_data(args.input)
    else:
        headers, rows = load_sample_data()
        print("No --input provided, using built-in sample data.")

    md_content = build_markdown(headers, rows, args.title)

    with open(args.output, "w", encoding="utf-8") as f:
        f.write(md_content)

    print(f"Markdown report written to: {os.path.abspath(args.output)}")
    print("Open it in any Markdown viewer, or view it directly on GitHub/GitLab.")


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

`build_markdown()` assembles the document as a list of lines (title, timestamp, record count, then a pipe-table header/separator/rows) and joins them with newlines. Every cell passes through `escape_md()` first so embedded `|` characters can't break the table structure.

```bash
# Zero-config run — uses built-in sample task-tracker data
python generate_markdown_report.py

# Point it at your own CSV and set a custom title
python generate_markdown_report.py --input tasks.csv --output STATUS.md --title "Sprint 14 Status"

# Short flags
python generate_markdown_report.py -i tasks.csv -o STATUS.md -t "Sprint 14 Status"
```

#### Sample Output

```bash
$ python generate_markdown_report.py
No --input provided, using built-in sample data.
Markdown report written to: /home/user/reports/report.md
Open it in any Markdown viewer, or view it directly on GitHub/GitLab.
```

`report.md` contents:

```markdown
# Project Status Report

_Generated on 2026-07-18 09:12:03_

**Total records:** 4

## Report Data

| Task | Owner | Priority | Status |
| --- | --- | --- | --- |
| Set up CI pipeline | Ayesha | High | Done |
| Write unit tests | Bilal | Medium | In Progress |
| Deploy to staging | Ayesha | High | Pending |
| Update documentation | Sara | Low | Pending |

---
*This report was generated automatically by `generate_markdown_report.py`.*
```

#### Variations / Flags

- Commit the output to a `docs/status/` folder on a schedule to build a history of reports you can diff over time in git.
- Pipe the output into a git commit hook: `python generate_markdown_report.py -i tasks.csv -o STATUS.md && git add STATUS.md && git commit -m "Update status report"`.
- Combine with `--title` templating (e.g., include the date in the title) if you want one file per day instead of overwriting.

#### Common Pitfalls

- **Overwriting history:** the default behavior overwrites `report.md` every run — use a dated filename (`report-2026-07-18.md`) if you need to keep past versions instead of relying on git history alone.
- **Column count mismatch:** if a data row has fewer or more fields than the header row (common with malformed CSV exports), the pipe-table row will be misaligned rather than erroring — validate your CSV's column count first.
- **Rendering differences:** GitHub, GitLab, and IDE Markdown previewers all render pipe-tables slightly differently (column width, alignment); don't assume pixel-perfect consistency across viewers.

### 20.3 Generate PDF Reports

#### Use Case

You need a report that looks identical no matter who opens it or what software they use — an invoice summary to email a client, an audit record to archive, or a document meant to be printed. PDF is the format people trust as a "final" artifact, and this script produces one with a proper running header, footer, and page numbers so multi-page reports stay navigable.

#### Prerequisites

Requires the `reportlab` package.

```bash
pip install reportlab
```

- **OS scope:** Linux, macOS, Windows — `reportlab` ships pure-Python wheels for all three.
- **Libraries used:** `argparse`, `csv`, `datetime`, `os`, `sys` (stdlib) + `reportlab` (pip).

#### Cautions

`reportlab`'s `Table` will overflow the page width if you feed it a CSV with many columns or very long cell text — long values wrap, but a table with 15+ columns will still look cramped on A4. This script hardcodes `A4` page size; U.S.-based reports may want `letter` from `reportlab.lib.pagesizes` instead.

#### Script

```python
#!/usr/bin/env python3
"""
generate_pdf_report.py

Generates a page-oriented PDF report with a running header, footer,
page numbers, and a data table. Requires the 'reportlab' package.

Usage:
    python generate_pdf_report.py
    python generate_pdf_report.py --input data.csv --output report.pdf --title "My Report"

Expected output:
    A .pdf file, one or more pages, each with a title header at the
    top, a footer showing "Page X of Y" and the generation date, and
    a formatted table of the report data in the body.
"""

import argparse
import csv
import datetime
import os
import sys

try:
    from reportlab.lib import colors
    from reportlab.lib.pagesizes import A4
    from reportlab.lib.units import cm
    from reportlab.platypus import SimpleDocTemplate, Table, TableStyle, Spacer
    from reportlab.lib.styles import getSampleStyleSheet
    from reportlab.platypus import Paragraph
except ImportError:
    print("Missing dependency 'reportlab'. Install it with: pip install reportlab", file=sys.stderr)
    sys.exit(1)


def load_sample_data():
    headers = ["Invoice #", "Client", "Amount (USD)", "Due Date", "Status"]
    rows = [
        ["INV-1001", "Acme Corp", "1,250.00", "2026-07-20", "Unpaid"],
        ["INV-1002", "Globex Inc", "3,400.50", "2026-07-15", "Paid"],
        ["INV-1003", "Initech", "890.00", "2026-08-01", "Unpaid"],
        ["INV-1004", "Umbrella LLC", "5,600.00", "2026-07-10", "Overdue"],
    ]
    return headers, rows


def load_csv_data(path):
    with open(path, newline="", encoding="utf-8") as f:
        rows = list(csv.reader(f))
    if not rows:
        raise ValueError("CSV file is empty.")
    return rows[0], rows[1:]


class PDFReport:
    """Handles header/footer drawing on every page."""

    def __init__(self, title):
        self.title = title
        self.generated_at = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")

    def header_footer(self, canvas, doc):
        canvas.saveState()
        width, height = A4

        # Header
        canvas.setFont("Helvetica-Bold", 11)
        canvas.setFillColor(colors.HexColor("#1d4ed8"))
        canvas.drawString(2 * cm, height - 1.4 * cm, self.title)
        canvas.setFont("Helvetica", 8)
        canvas.setFillColor(colors.grey)
        canvas.drawRightString(width - 2 * cm, height - 1.4 * cm, self.generated_at)
        canvas.setStrokeColor(colors.HexColor("#e2e8f0"))
        canvas.line(2 * cm, height - 1.6 * cm, width - 2 * cm, height - 1.6 * cm)

        # Footer
        canvas.setFont("Helvetica", 8)
        canvas.setFillColor(colors.grey)
        page_num_text = f"Page {doc.page}"
        canvas.drawCentredString(width / 2, 1.2 * cm, page_num_text)
        canvas.drawString(2 * cm, 1.2 * cm, "Generated by generate_pdf_report.py")
        canvas.restoreState()


def build_pdf(headers, rows, title, output_path):
    doc = SimpleDocTemplate(
        output_path,
        pagesize=A4,
        topMargin=2.2 * cm,
        bottomMargin=2 * cm,
        leftMargin=2 * cm,
        rightMargin=2 * cm,
    )

    styles = getSampleStyleSheet()
    elements = []

    elements.append(Paragraph(f"<b>{title}</b>", styles["Title"]))
    elements.append(Spacer(1, 6))
    elements.append(Paragraph(f"Total records: {len(rows)}", styles["Normal"]))
    elements.append(Spacer(1, 14))

    table_data = [headers] + rows
    table = Table(table_data, repeatRows=1)
    table.setStyle(TableStyle([
        ("BACKGROUND", (0, 0), (-1, 0), colors.HexColor("#2563eb")),
        ("TEXTCOLOR", (0, 0), (-1, 0), colors.white),
        ("FONTNAME", (0, 0), (-1, 0), "Helvetica-Bold"),
        ("FONTSIZE", (0, 0), (-1, -1), 9),
        ("ROWBACKGROUNDS", (0, 1), (-1, -1), [colors.white, colors.HexColor("#f1f5f9")]),
        ("GRID", (0, 0), (-1, -1), 0.5, colors.HexColor("#cbd5e1")),
        ("VALIGN", (0, 0), (-1, -1), "MIDDLE"),
        ("TOPPADDING", (0, 0), (-1, -1), 6),
        ("BOTTOMPADDING", (0, 0), (-1, -1), 6),
        ("LEFTPADDING", (0, 0), (-1, -1), 8),
    ]))
    elements.append(table)

    report = PDFReport(title)
    doc.build(elements, onFirstPage=report.header_footer, onLaterPages=report.header_footer)


def main():
    parser = argparse.ArgumentParser(description="Generate a page-oriented PDF report from CSV data.")
    parser.add_argument("--input", "-i", help="Path to input CSV file. If omitted, sample data is used.")
    parser.add_argument("--output", "-o", default="report.pdf", help="Path to output PDF file (default: report.pdf)")
    parser.add_argument("--title", "-t", default="Invoice Summary Report", help="Report title")
    args = parser.parse_args()

    if args.input:
        if not os.path.isfile(args.input):
            print(f"Error: input file not found: {args.input}", file=sys.stderr)
            sys.exit(1)
        headers, rows = load_csv_data(args.input)
    else:
        headers, rows = load_sample_data()
        print("No --input provided, using built-in sample data.")

    build_pdf(headers, rows, args.title, args.output)

    print(f"PDF report written to: {os.path.abspath(args.output)}")
    print("Open it with any PDF viewer.")


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

`build_pdf()` uses `reportlab`'s `platypus` layer to flow a title, a record-count line, and a `Table` onto a `SimpleDocTemplate`. The `PDFReport.header_footer` callback is passed to `doc.build()` for both `onFirstPage` and `onLaterPages`, so every page — however many the table spans — gets the same title/timestamp header and "Page N" footer drawn directly on the canvas.

```bash
# Zero-config run — uses built-in sample invoice data
python generate_pdf_report.py

# Point it at your own CSV and set a custom title
python generate_pdf_report.py --input invoices.csv --output invoices_july.pdf --title "July Invoice Summary"

# Short flags
python generate_pdf_report.py -i invoices.csv -o invoices_july.pdf -t "July Invoice Summary"
```

### 20.4 Generate CSV Reports

#### Use Case

You have structured data sitting in JSON (an API response, a log-processing output) and need it as a flat, spreadsheet-ready table someone can open in Excel, Google Sheets, or LibreOffice Calc without any conversion step. CSV is also the common denominator input format the other scripts in this chapter expect, so this script doubles as "the thing that produces the input for everything else."

#### Prerequisites

Standard library only — no installation required.

```bash
# Verify you have Python 3 available; no pip install needed
python3 --version
```

- **OS scope:** Linux, macOS, Windows — anywhere Python 3 runs.
- **Libraries used:** `argparse`, `csv`, `json`, `os`, `sys` (all stdlib).

#### Cautions

The script only accepts JSON shaped as a flat `list[dict]` — a list of objects with no nested objects/arrays inside. Nested structures will be coerced with `str()` on whatever `.get()` returns, which for a nested dict or list produces the Python `repr()` string inside the cell rather than a usable value — flatten nested JSON before feeding it in.

#### Script

```python
#!/usr/bin/env python3
"""
generate_csv_report.py

Generates a simple tabular CSV data extract, ready to be opened in
Excel, Google Sheets, LibreOffice Calc, or any spreadsheet app. Uses
built-in sample data unless a JSON input file is supplied.

Usage:
    python generate_csv_report.py
    python generate_csv_report.py --input data.json --output report.csv

Expected output:
    A .csv file with a header row followed by data rows, comma
    separated, UTF-8 encoded, opens cleanly as a table in any
    spreadsheet application.
"""

import argparse
import csv
import json
import os
import sys


def load_sample_data():
    headers = ["Employee ID", "Name", "Department", "Salary", "Join Date"]
    rows = [
        ["E001", "Ahmed Khan", "Engineering", "95000", "2022-03-15"],
        ["E002", "Fatima Noor", "Marketing", "72000", "2021-11-01"],
        ["E003", "Usman Ali", "Engineering", "101000", "2020-06-23"],
        ["E004", "Hina Malik", "Finance", "88000", "2023-01-09"],
        ["E005", "Bilal Ahmed", "Sales", "67000", "2019-09-30"],
    ]
    return headers, rows


def load_json_data(path):
    with open(path, "r", encoding="utf-8") as f:
        data = json.load(f)

    if isinstance(data, list) and data and isinstance(data[0], dict):
        headers = list(data[0].keys())
        rows = [[str(item.get(h, "")) for h in headers] for item in data]
        return headers, rows

    raise ValueError("JSON input must be a list of flat objects (list[dict]).")


def write_csv(headers, rows, output_path):
    with open(output_path, "w", newline="", encoding="utf-8") as f:
        writer = csv.writer(f)
        writer.writerow(headers)
        writer.writerows(rows)


def main():
    parser = argparse.ArgumentParser(description="Generate a CSV report from JSON data.")
    parser.add_argument("--input", "-i", help="Path to input JSON file (list of flat objects). If omitted, sample data is used.")
    parser.add_argument("--output", "-o", default="report.csv", help="Path to output CSV file (default: report.csv)")
    args = parser.parse_args()

    if args.input:
        if not os.path.isfile(args.input):
            print(f"Error: input file not found: {args.input}", file=sys.stderr)
            sys.exit(1)
        headers, rows = load_json_data(args.input)
    else:
        headers, rows = load_sample_data()
        print("No --input provided, using built-in sample data.")

    write_csv(headers, rows, args.output)

    print(f"CSV report written to: {os.path.abspath(args.output)}")
    print(f"Rows written: {len(rows)}. Open it in Excel, Sheets, or any spreadsheet app.")


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

Unlike most of this chapter's scripts, this one's real input format is JSON, not CSV — it's the "JSON in, CSV out" converter. `load_json_data()` takes the keys of the first object as the header row (assuming every object in the list shares the same keys) and writes one CSV row per object using `csv.writer`, which handles comma/quote escaping automatically.

#### Variations / Flags

- Chain this script ahead of any other script in the chapter: `python generate_csv_report.py -i api_dump.json -o data.csv && python generate_html_report.py -i data.csv`.
- Add a `--delimiter` flag (defaulting to `,`) if downstream tooling expects TSV or semicolon-separated output instead.

#### Common Pitfalls

- **Header mismatch across records:** because headers come only from `data[0].keys()`, any object later in the array with extra keys will silently lose those fields — objects missing a key just get an empty cell (`""`), which can mask a data quality problem.
- **Non-list top-level JSON:** if your JSON file's top level is a single object (`{...}`) rather than a list (`[{...}, {...}]`), the script raises `ValueError` — wrap single-object exports in a list first.
- **Numbers become strings:** every value passes through `str()`, so numeric types lose their original JSON type once in CSV — this is fine for display but matters if you round-trip the CSV back into a type-sensitive tool.
- **Excel and UTF-8 BOM:** some older Excel versions misinterpret UTF-8 CSVs without a byte-order mark, showing garbled special characters — if this bites you, re-save with `encoding="utf-8-sig"` instead of `"utf-8"`.

### 20.5 Generate Excel Reports

#### Use Case

Finance, sales, and ops teams generally don't want a CSV — they want an `.xlsx` file with a styled header, a totals row that's an actual formula (so they can verify or extend it), and a chart they can drop straight into a slide deck. This script builds exactly that from a CSV input, giving you a "real" Excel deliverable without opening Excel at all.

#### Prerequisites

Requires the `openpyxl` package.

```bash
pip install openpyxl
```

- **OS scope:** Linux, macOS, Windows — `openpyxl` is pure Python and works identically everywhere.
- **Libraries used:** `argparse`, `csv`, `os`, `sys` (stdlib) + `openpyxl` (pip).

#### Cautions

`find_numeric_column()` scans columns right-to-left and picks the *last* numeric column for both the `SUM()` formula and the chart — if your CSV has multiple numeric columns and you want a specific one totaled, reorder your columns or adjust the function. The formula is written as a live Excel formula string (`=SUM(...)`), which some non-Excel spreadsheet tools handle slightly differently — verify the total renders correctly in whatever app your recipient will open it with.

#### Script

```python
#!/usr/bin/env python3
"""
generate_excel_report.py

Generates a formatted Excel workbook (.xlsx) with styled headers,
a formula column (computed with real Excel formulas, not just
static values), and an embedded bar chart. Requires 'openpyxl'.

Usage:
    python generate_excel_report.py
    python generate_excel_report.py --input data.csv --output report.xlsx --title "Sales Report"

Expected output:
    An .xlsx file with one sheet named "Report" containing a styled
    table, a "Total" formula row using SUM(), and a bar chart plotted
    from the numeric column. Opens in Excel, LibreOffice Calc, or
    Google Sheets (upload/import).
"""

import argparse
import csv
import os
import sys

try:
    from openpyxl import Workbook
    from openpyxl.styles import Font, PatternFill, Alignment, Border, Side
    from openpyxl.chart import BarChart, Reference
    from openpyxl.utils import get_column_letter
except ImportError:
    print("Missing dependency 'openpyxl'. Install it with: pip install openpyxl", file=sys.stderr)
    sys.exit(1)


def load_sample_data():
    headers = ["Product", "Region", "Units Sold", "Revenue"]
    rows = [
        ["Widget A", "North", 120, 3600],
        ["Widget B", "South", 95, 2850],
        ["Widget C", "East", 150, 6000],
        ["Widget D", "West", 80, 2400],
        ["Widget E", "North", 60, 1800],
    ]
    return headers, rows


def load_csv_data(path):
    with open(path, newline="", encoding="utf-8") as f:
        reader = csv.reader(f)
        rows = list(reader)
    if not rows:
        raise ValueError("CSV file is empty.")
    headers = rows[0]
    data_rows = []
    for r in rows[1:]:
        converted = []
        for cell in r:
            try:
                converted.append(float(cell) if "." in cell else int(cell))
            except ValueError:
                converted.append(cell)
        data_rows.append(converted)
    return headers, data_rows


def find_numeric_column(headers, rows):
    """Return index of the last numeric column, used for the chart + SUM formula."""
    for idx in range(len(headers) - 1, -1, -1):
        if rows and isinstance(rows[0][idx], (int, float)):
            return idx
    return None


def build_workbook(headers, rows, title, output_path):
    wb = Workbook()
    ws = wb.active
    ws.title = "Report"

    header_fill = PatternFill(start_color="2563EB", end_color="2563EB", fill_type="solid")
    header_font = Font(color="FFFFFF", bold=True)
    thin_border = Border(*(Side(style="thin", color="CBD5E1"),) * 4)

    # Title row
    ws.merge_cells(start_row=1, start_column=1, end_row=1, end_column=len(headers))
    title_cell = ws.cell(row=1, column=1, value=title)
    title_cell.font = Font(size=14, bold=True, color="1D4ED8")
    ws.row_dimensions[1].height = 24

    # Header row
    header_row_idx = 3
    for col_idx, header in enumerate(headers, start=1):
        cell = ws.cell(row=header_row_idx, column=col_idx, value=header)
        cell.fill = header_fill
        cell.font = header_font
        cell.alignment = Alignment(horizontal="center")
        cell.border = thin_border

    # Data rows
    for r_offset, row in enumerate(rows, start=1):
        for c_offset, value in enumerate(row, start=1):
            cell = ws.cell(row=header_row_idx + r_offset, column=c_offset, value=value)
            cell.border = thin_border

    numeric_col = find_numeric_column(headers, rows)

    # Totals row using a real Excel formula
    total_row_idx = header_row_idx + len(rows) + 1
    ws.cell(row=total_row_idx, column=1, value="TOTAL").font = Font(bold=True)
    if numeric_col is not None:
        col_letter = get_column_letter(numeric_col + 1)
        first_data_row = header_row_idx + 1
        last_data_row = header_row_idx + len(rows)
        formula = f"=SUM({col_letter}{first_data_row}:{col_letter}{last_data_row})"
        total_cell = ws.cell(row=total_row_idx, column=numeric_col + 1, value=formula)
        total_cell.font = Font(bold=True)

    # Column widths
    for col_idx, header in enumerate(headers, start=1):
        ws.column_dimensions[get_column_letter(col_idx)].width = max(14, len(str(header)) + 4)

    # Chart
    if numeric_col is not None and len(rows) > 1:
        chart = BarChart()
        chart.title = f"{headers[numeric_col]} by {headers[0]}"
        chart.y_axis.title = headers[numeric_col]
        chart.x_axis.title = headers[0]

        data_ref = Reference(
            ws, min_col=numeric_col + 1, max_col=numeric_col + 1,
            min_row=header_row_idx, max_row=header_row_idx + len(rows)
        )
        cats_ref = Reference(
            ws, min_col=1, max_col=1,
            min_row=header_row_idx + 1, max_row=header_row_idx + len(rows)
        )
        chart.add_data(data_ref, titles_from_data=True)
        chart.set_categories(cats_ref)
        chart.width = 16
        chart.height = 9
        ws.add_chart(chart, get_column_letter(len(headers) + 2) + "3")

    wb.save(output_path)


def main():
    parser = argparse.ArgumentParser(description="Generate a formatted Excel report with a chart from CSV data.")
    parser.add_argument("--input", "-i", help="Path to input CSV file. If omitted, sample data is used.")
    parser.add_argument("--output", "-o", default="report.xlsx", help="Path to output XLSX file (default: report.xlsx)")
    parser.add_argument("--title", "-t", default="Sales Performance Report", help="Report title")
    args = parser.parse_args()

    if args.input:
        if not os.path.isfile(args.input):
            print(f"Error: input file not found: {args.input}", file=sys.stderr)
            sys.exit(1)
        headers, rows = load_csv_data(args.input)
    else:
        headers, rows = load_sample_data()
        print("No --input provided, using built-in sample data.")

    build_workbook(headers, rows, args.title, args.output)

    print(f"Excel report written to: {os.path.abspath(args.output)}")
    print("Open it in Excel or LibreOffice Calc. Contains a live SUM() formula and a bar chart.")


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

`load_csv_data()` type-converts each cell to `int`/`float` where possible so numeric columns actually behave as numbers in Excel (not text). `find_numeric_column()` locates the rightmost numeric column, which `build_workbook()` then uses both for the `SUM()` totals formula and as the data series for an embedded `BarChart`, placed a couple of columns to the right of the table.

```bash
# Short flags

```

#### Sample Output

```bash

Open it in Excel or LibreOffice Calc. Contains a live SUM() formula and a bar chart.
```

#### Variations / Flags

- Add a `--sheet-name` flag if you need the tab named something other than "Report" for downstream automation that reads a specific sheet name.
- Swap `BarChart()` for `LineChart()` or `PieChart()` (also from `openpyxl.chart`) if a trend or proportion view fits your data better than a bar comparison.

#### Common Pitfalls

- **Non-numeric-looking numbers:** a column like ZIP codes or IDs that happens to be all digits will be converted to `int` by `load_csv_data()` and lose leading zeros — audit `find_numeric_column()`'s target column and, if needed, force ID-like columns to stay as text.
- **No numeric column found:** if none of your columns are numeric, `numeric_col` is `None`, and the script silently skips both the totals formula and the chart — this is not an error, just a quieter-than-expected output.
- **Google Sheets formula quirks:** `SUM()` imports fine into Google Sheets, but chart formatting from `openpyxl` doesn't always survive the Excel-to-Sheets import faithfully — re-check chart appearance after uploading if that's your target.
- **File locked by another process:** saving will fail with a `PermissionError` if the target `.xlsx` is currently open in Excel — close the file before rerunning the script against the same path.

### 20.6 Generate JSON Reports

#### Use Case

You're not generating this report for a human at all — you're generating it for another program: a monitoring dashboard's ingest endpoint, a CI pipeline artifact, or an internal API that expects structured, machine-parseable data. This script wraps your CSV data in a predictable `report.metadata` + `report.data` JSON envelope so any downstream consumer knows exactly where to look for the title, generation time, record count, and the records themselves.

#### Prerequisites

Standard library only — no installation required.

```bash
# Verify you have Python 3 available; no pip install needed
python3 --version
```

- **OS scope:** Linux, macOS, Windows — anywhere Python 3 runs.
- **Libraries used:** `argparse`, `csv`, `datetime`, `json`, `os`, `sys` (all stdlib).

#### Cautions

`coerce()` decides a value is a `float` purely by checking for a literal `.` character in the string — this means a non-numeric string that happens to contain a period (like a version string `"1.2"` in a non-numeric column, or an IP-like value) will be silently coerced to a float and can lose information or raise unexpectedly. Validate column types in your source CSV if this matters for your use case.

#### Script

```python
#!/usr/bin/env python3
"""
generate_json_report.py

Generates a structured, hierarchical JSON report — the kind typically
consumed by APIs, dashboards, or other automated systems. Reads from
CSV or uses built-in sample data, and nests records under metadata.

Usage:
    python generate_json_report.py
    python generate_json_report.py --input data.csv --output report.json --title "My Report"

Expected output:
    A .json file with a top-level "report" object containing "metadata"
    (title, generated_at, record_count) and a "data" array of objects,
    one per row, keyed by the CSV column headers. Valid, pretty-printed
    JSON that any API or script can parse with json.load().
"""

import argparse
import csv
import datetime
import json
import os
import sys


def load_sample_data():
    headers = ["host", "cpu_percent", "memory_percent", "uptime_days", "status"]
    rows = [
        ["web-01", 42, 63, 120, "healthy"],
        ["web-02", 88, 77, 45, "degraded"],
        ["db-01", 35, 81, 300, "healthy"],
        ["cache-01", 12, 22, 12, "healthy"],
    ]
    return headers, rows


def load_csv_data(path):
    with open(path, newline="", encoding="utf-8") as f:
        rows = list(csv.reader(f))
    if not rows:
        raise ValueError("CSV file is empty.")
    return rows[0], rows[1:]


def coerce(value):
    """Try to coerce a CSV string into int/float, else leave as string."""
    try:
        if "." in value:
            return float(value)
        return int(value)
    except (ValueError, TypeError):
        return value


def build_report(headers, rows, title):
    generated_at = datetime.datetime.now().isoformat()

    data = []
    for row in rows:
        record = {headers[i]: coerce(row[i]) for i in range(len(headers))}
        data.append(record)

    return {
        "report": {
            "metadata": {
                "title": title,
                "generated_at": generated_at,
                "record_count": len(data),
                "fields": headers,
            },
            "data": data,
        }
    }


def main():
    parser = argparse.ArgumentParser(description="Generate a structured JSON report from CSV data.")
    parser.add_argument("--input", "-i", help="Path to input CSV file. If omitted, sample data is used.")
    parser.add_argument("--output", "-o", default="report.json", help="Path to output JSON file (default: report.json)")
    parser.add_argument("--title", "-t", default="Infrastructure Health Report", help="Report title")
    args = parser.parse_args()

    if args.input:
        if not os.path.isfile(args.input):
            print(f"Error: input file not found: {args.input}", file=sys.stderr)
            sys.exit(1)
        headers, rows = load_csv_data(args.input)
    else:
        headers, rows = load_sample_data()
        print("No --input provided, using built-in sample data.")

    report = build_report(headers, rows, args.title)

    with open(args.output, "w", encoding="utf-8") as f:
        json.dump(report, f, indent=2)

    print(f"JSON report written to: {os.path.abspath(args.output)}")
    print(f"Records: {report['report']['metadata']['record_count']}. Valid for use with json.load() or any API.")


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

`build_report()` zips each row of the CSV against the header list to build one dict per record (with `coerce()` attempting to turn numeric-looking strings into real `int`/`float` types), then nests the full `data` list under a `metadata` block containing the title, ISO-8601 timestamp, record count, and field list — a shape designed to be stable enough that a consumer's parser doesn't need to change even as row content changes.


#### Variations / Flags

- Pipe the output straight into `curl` for an API push: `python generate_json_report.py -i hosts.csv -o /tmp/health.json && curl -X POST -H "Content-Type: application/json" -d @/tmp/health.json https://api.example.com/reports`.
- Add a `--compact` flag that calls `json.dump(report, f)` without `indent=2` when you need minified output for network transfer instead of a pretty-printed file.
- Extend `build_report()`'s metadata block with a `source_file` field (the input CSV's path) if downstream consumers need provenance tracking.

#### Common Pitfalls

- **Type coercion surprises:** as noted above, `coerce()`'s `.`-detection heuristic isn't a real type check — a text column like `"v1.0"` becomes a `ValueError`-caught string (safe), but `"1.0"` in a text field silently becomes the float `1.0` — check output types against your source data if downstream code is type-strict.
- **Large CSVs and memory:** the whole CSV is loaded into memory as a list of rows before conversion — for very large exports (hundreds of thousands of rows), consider streaming with `csv.DictReader` row-by-row instead of building the full list first.
- **Non-UTF-8 source CSVs:** the file is opened with `encoding="utf-8"` explicitly; a CSV saved with a different encoding will raise `UnicodeDecodeError` rather than producing garbled JSON — fail loudly is the safer default here, but be ready for it.

### 20.7 Generate XML Reports

#### Use Case

This script produces a well-formed, indented `<report>` document with a `<metadata>` block and a `<records>` collection, so you can hand data off to that older tooling without writing an XML serializer by hand each time.

- **Libraries used:** `argparse`, `csv`, `datetime`, `os`, `re`, `sys`, `xml.etree.ElementTree`, `xml.dom.minidom` (all stdlib).

#### Cautions

XML element names have strict rules (no spaces, can't start with a digit) — `sanitize_tag()` handles this by replacing invalid characters with underscores and prefixing a digit-leading tag with `field_`, but that means two differently-named CSV columns could theoretically sanitize to the same tag name (e.g., `"Order #"` and `"Order_#"` both become `Order_`), silently colliding in the output. Check your header names for near-duplicates before relying on this in production.

#### Script

```python
#!/usr/bin/env python3
"""
generate_xml_report.py

Generates a well-formed, indented XML report in a format compatible
with legacy enterprise tooling (a root <report> element containing
<metadata> and a <records> collection of <record> elements). Uses
only the Python standard library (xml.etree.ElementTree).

Usage:
    python generate_xml_report.py
    python generate_xml_report.py --input data.csv --output report.xml --title "My Report"

Expected output:
    An .xml file, UTF-8 encoded, with a declaration line, a <report>
    root element, a <metadata> block (title, generated_at, count),
    and a <records> block with one <record> per row, each field as
    a child element named after the CSV column header.
"""

import argparse
import csv
import datetime
import os
import re
import sys
import xml.etree.ElementTree as ET
from xml.dom import minidom


def load_sample_data():
    headers = ["order_id", "customer", "total", "currency", "status"]
    rows = [
        ["ORD-001", "Zainab Traders", "1200.00", "USD", "Shipped"],
        ["ORD-002", "Khan Enterprises", "875.50", "USD", "Pending"],
        ["ORD-003", "Malik & Co", "430.00", "USD", "Delivered"],
    ]
    return headers, rows


def load_csv_data(path):
    with open(path, newline="", encoding="utf-8") as f:
        rows = list(csv.reader(f))
    if not rows:
        raise ValueError("CSV file is empty.")
    return rows[0], rows[1:]


def sanitize_tag(name):
    """XML element names can't contain spaces or start with a digit."""
    tag = re.sub(r"[^0-9a-zA-Z_]", "_", name.strip())
    if not tag or tag[0].isdigit():
        tag = f"field_{tag}"
    return tag


def build_xml(headers, rows, title):
    root = ET.Element("report")

    metadata = ET.SubElement(root, "metadata")
    ET.SubElement(metadata, "title").text = title
    ET.SubElement(metadata, "generated_at").text = datetime.datetime.now().isoformat()
    ET.SubElement(metadata, "record_count").text = str(len(rows))

    tags = [sanitize_tag(h) for h in headers]

    records = ET.SubElement(root, "records")
    for row in rows:
        record = ET.SubElement(records, "record")
        for tag, value in zip(tags, row):
            ET.SubElement(record, tag).text = str(value)

    rough_string = ET.tostring(root, encoding="utf-8")
    reparsed = minidom.parseString(rough_string)
    return reparsed.toprettyxml(indent="  ", encoding="UTF-8")


def main():
    parser = argparse.ArgumentParser(description="Generate a legacy-compatible XML report from CSV data.")
    parser.add_argument("--input", "-i", help="Path to input CSV file. If omitted, sample data is used.")
    parser.add_argument("--output", "-o", default="report.xml", help="Path to output XML file (default: report.xml)")
    parser.add_argument("--title", "-t", default="Order Summary Report", help="Report title")
    args = parser.parse_args()

    if args.input:
        if not os.path.isfile(args.input):
            print(f"Error: input file not found: {args.input}", file=sys.stderr)
            sys.exit(1)
        headers, rows = load_csv_data(args.input)
    else:
        headers, rows = load_sample_data()
        print("No --input provided, using built-in sample data.")

    xml_bytes = build_xml(headers, rows, args.title)

    with open(args.output, "wb") as f:
        f.write(xml_bytes)

    print(f"XML report written to: {os.path.abspath(args.output)}")
    print("Valid, well-formed XML. Open in a browser, text editor, or parse with any XML library.")


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

`build_xml()` builds the tree with `ElementTree` (root `<report>` → `<metadata>` + `<records>` → one `<record>` per row), then re-parses the compact output through `xml.dom.minidom` purely to pretty-print it with two-space indentation — `ElementTree` alone doesn't indent nicely, so this two-step dance is what gives you human-readable XML instead of one long line. Note the file is opened in binary mode (`"wb"`) because `toprettyxml(..., encoding="UTF-8")` already returns encoded bytes, not a string.

```bash
# Short flags
```


#### Variations / Flags

- Add a `--schema` flag that validates the output against an XSD after generation, if the receiving system requires schema-validated XML.
- Add a `--namespace` flag if your target system expects a specific XML namespace declared on the root `<report>` element.
- Validate well-formedness quickly from the command line after generation: `python3 -c "import xml.dom.minidom; xml.dom.minidom.parse('report.xml')"`.

#### Common Pitfalls

- **Column names with special characters:** as noted in Cautions, `sanitize_tag()` can produce colliding tag names for CSV headers that differ only in punctuation — inspect the generated tag names against your header list before shipping to a strict downstream parser.
- **Ampersands and other CSV cell content:** `ElementTree` automatically escapes special XML characters (`&`, `<`, `>`) in element text — you don't need to pre-escape your CSV data, and doing so would double-escape it.
- **Binary write mode confusion:** if you modify the script to write with `"w"` (text mode) instead of `"wb"`, it will fail because `toprettyxml(..., encoding="UTF-8")` returns `bytes`, not `str` — keep the write mode aligned with what `build_xml()` returns.
- **Legacy parsers and XML declaration strictness:** some older enterprise XML parsers are picky about the exact declaration line (`<?xml version="1.0" encoding="UTF-8"?>`) — verify the receiving system's parser accepts `minidom`'s default declaration formatting before relying on this in a production integration.

### 20.8 Generate Charts

`generate_charts.py`

#### Use Case

A table of numbers is precise but slow to read; a chart lets someone spot a trend, an outlier, or a proportion at a glance. This script takes the same kind of CSV data you'd feed into any of the tabular report scripts and instead renders it as three standalone PNG images — bar, line, and pie — ready to embed in a slide deck, a Word document, or an email.

#### Prerequisites

Requires the `matplotlib` package.

```bash
pip install matplotlib
```

- **OS scope:** Linux, macOS, Windows — matplotlib runs everywhere, and the script explicitly sets the `Agg` backend so it also works headlessly on a server with no display attached.
- **Libraries used:** `argparse`, `csv`, `os`, `sys` (stdlib) + `matplotlib` (pip).

#### Cautions

The script assumes one label column and one numeric value column — it has no support for multi-series charts (e.g., comparing two metrics side by side) out of the box. Pie charts in particular get unreadable past roughly 8–10 slices; if your label column has many unique values, the bar or line chart will communicate the data far better than the pie chart will.

#### Script

```python
#!/usr/bin/env python3
"""
generate_charts.py

Generates bar, line, and pie chart images from CSV data using
Matplotlib. Requires 'matplotlib'.

Usage:
    python generate_charts.py
    python generate_charts.py --input data.csv --outdir charts --label-col Category --value-col Value

Expected output:
    Three PNG image files written to the output directory:
    bar_chart.png, line_chart.png, pie_chart.png — each a standalone
    image you can open, embed in documents, or attach to an email.
"""

import argparse
import csv
import os
import sys

try:
    import matplotlib
    matplotlib.use("Agg")  # headless / no display needed
    import matplotlib.pyplot as plt
except ImportError:
    print("Missing dependency 'matplotlib'. Install it with: pip install matplotlib", file=sys.stderr)
    sys.exit(1)


def load_sample_data():
    labels = ["Jan", "Feb", "Mar", "Apr", "May", "Jun"]
    values = [120, 150, 90, 200, 175, 210]
    return labels, values


def load_csv_data(path, label_col, value_col):
    with open(path, newline="", encoding="utf-8") as f:
        reader = csv.DictReader(f)
        rows = list(reader)
    if not rows:
        raise ValueError("CSV file is empty.")

    if label_col not in rows[0] or value_col not in rows[0]:
        raise ValueError(
            f"Columns not found. Available columns: {list(rows[0].keys())}. "
            f"Use --label-col and --value-col to select them."
        )

    labels = [row[label_col] for row in rows]
    values = [float(row[value_col]) for row in rows]
    return labels, values


def make_bar_chart(labels, values, output_path, title):
    plt.figure(figsize=(8, 5))
    bars = plt.bar(labels, values, color="#2563eb")
    plt.title(title)
    plt.ylabel("Value")
    plt.xticks(rotation=30, ha="right")
    for bar in bars:
        height = bar.get_height()
        plt.text(bar.get_x() + bar.get_width() / 2, height, f"{height:g}",
                  ha="center", va="bottom", fontsize=8)
    plt.tight_layout()
    plt.savefig(output_path, dpi=150)
    plt.close()


def make_line_chart(labels, values, output_path, title):
    plt.figure(figsize=(8, 5))
    plt.plot(labels, values, marker="o", color="#16a34a", linewidth=2)
    plt.title(title)
    plt.ylabel("Value")
    plt.xticks(rotation=30, ha="right")
    plt.grid(alpha=0.3)
    plt.tight_layout()
    plt.savefig(output_path, dpi=150)
    plt.close()


def make_pie_chart(labels, values, output_path, title):
    plt.figure(figsize=(7, 7))
    plt.pie(values, labels=labels, autopct="%1.1f%%", startangle=90,
            colors=plt.cm.Pastel1.colors)
    plt.title(title)
    plt.tight_layout()
    plt.savefig(output_path, dpi=150)
    plt.close()


def main():
    parser = argparse.ArgumentParser(description="Generate bar, line, and pie charts from CSV data.")
    parser.add_argument("--input", "-i", help="Path to input CSV file. If omitted, sample data is used.")
    parser.add_argument("--outdir", "-d", default="charts", help="Output directory for chart images (default: charts)")
    parser.add_argument("--label-col", default=None, help="CSV column to use as labels (required if --input is given)")
    parser.add_argument("--value-col", default=None, help="CSV column to use as numeric values (required if --input is given)")
    parser.add_argument("--title", "-t", default="Monthly Values", help="Base chart title")
    args = parser.parse_args()

    if args.input:
        if not os.path.isfile(args.input):
            print(f"Error: input file not found: {args.input}", file=sys.stderr)
            sys.exit(1)
        if not args.label_col or not args.value_col:
            print("Error: --label-col and --value-col are required when using --input.", file=sys.stderr)
            sys.exit(1)
        labels, values = load_csv_data(args.input, args.label_col, args.value_col)
    else:
        labels, values = load_sample_data()
        print("No --input provided, using built-in sample data.")

    os.makedirs(args.outdir, exist_ok=True)

    bar_path = os.path.join(args.outdir, "bar_chart.png")
    line_path = os.path.join(args.outdir, "line_chart.png")
    pie_path = os.path.join(args.outdir, "pie_chart.png")

    make_bar_chart(labels, values, bar_path, f"{args.title} — Bar")
    make_line_chart(labels, values, line_path, f"{args.title} — Line")
    make_pie_chart(labels, values, pie_path, f"{args.title} — Distribution")

    print(f"Charts written to: {os.path.abspath(args.outdir)}")
    print(f"  - {bar_path}\n  - {line_path}\n  - {pie_path}")


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

`matplotlib.use("Agg")` is set immediately after import, before `pyplot` is imported — this forces the non-interactive, file-only rendering backend so the script works on a headless server with no X11/display, which is the common case for a sysadmin running this over SSH. Each chart function creates a fresh `plt.figure()`, draws onto it, saves a PNG with `plt.savefig()`, and calls `plt.close()` to free the figure before moving to the next chart — skipping `plt.close()` in a modified version of this script would leak memory if you ever loop it to generate many chart sets.

```bash
# Point it at your own CSV, choosing which columns to plot
python generate_charts.py --input sales.csv --outdir sales_charts --label-col Month --value-col Revenue --title "2026 Revenue"

# Short flags
python generate_charts.py -i sales.csv -d sales_charts --label-col Month --value-col Revenue -t "2026 Revenue"
```

#### Sample Output

```bash
$ python generate_charts.py
No --input provided, using built-in sample data.
Charts written to: /home/user/reports/charts
  - charts/bar_chart.png
  - charts/line_chart.png
  - charts/pie_chart.png
```

`bar_chart.png` shows six blue bars for Jan–Jun with the value labeled above each bar; `line_chart.png` shows the same values as a green line with circular markers and a light grid; `pie_chart.png` shows six pastel-colored wedges each labeled with month and percentage of the total.

#### Variations / Flags

- Add a `--dpi` flag (defaulting to 150) if you need print-resolution (300 dpi) images for a formal document instead of screen-resolution ones.
- Add a `--format` flag to save as `.svg` instead of `.png` via `plt.savefig(path, format="svg")` when you need vector output that scales cleanly in a slide deck.
- Skip chart types you don't need by commenting out the relevant `make_*_chart()` call — useful when you only want the bar chart on a scheduled job and don't need three files every run.

#### Common Pitfalls

- **Missing `--label-col`/`--value-col` with `--input`:** the script explicitly requires both flags together with `--input` and exits with an error if either is missing — this is intentional (there's no reliable way to guess which two of an arbitrary CSV's columns you want charted).
- **Non-numeric value column:** `float(row[value_col])` will raise `ValueError` if the chosen value column contains any non-numeric cells (including blank cells) — clean or filter the CSV first.
- **Headless server without Agg:** if you remove or comment out the `matplotlib.use("Agg")` line, the script may try to open a GUI window and fail (or hang) on a server with no display — leave that line in place for any non-interactive/cron use.
- **Overwriting previous chart runs:** the output filenames (`bar_chart.png`, etc.) are fixed within `--outdir` — point `--outdir` at a fresh or dated directory per run if you need to retain historical chart snapshots.

### 20.9 Generate Dashboards

#### Use Case

Sometimes a static chart isn't enough — someone needs to filter the data themselves, drilling into one region or one category at a time without asking you to regenerate a new chart for every slice they want to see. This script launches a small local web app (built on Plotly Dash) with a dropdown filter and two live-updating charts, giving people a self-serve way to explore the data during a meeting or an incident review.

#### Prerequisites

Requires `dash`, `plotly`, and `pandas`.

```bash
pip install dash plotly pandas
```

- **OS scope:** Linux, macOS, Windows — Dash's dev server runs anywhere Python runs; on a remote Linux server you'll typically want to bind it to a port you can reach via SSH tunnel or reverse proxy rather than expose it directly.
- **Libraries used:** `argparse`, `csv`, `os`, `sys` (stdlib) + `pandas`, `dash`, `plotly` (pip).

#### Cautions

This runs `app.run(debug=False, port=args.port)`, which starts Dash's built-in development server — it is explicitly **not** meant to face the public internet as-is; there's no authentication, TLS, or production WSGI server involved. For anything beyond a quick internal look, put it behind a reverse proxy with access control, or deploy the underlying Dash app with a production server (e.g., gunicorn) instead of running this script directly.

#### Script

```python
#!/usr/bin/env python3
"""
generate_dashboard.py

Launches an interactive web dashboard (bar chart, line chart, and a
data table with a dropdown filter) using Plotly + Dash. Requires
'dash' and 'plotly'.

Usage:
    python generate_dashboard.py
    python generate_dashboard.py --input data.csv --port 8050

Expected output:
    A local web server starts (default: http://127.0.0.1:8050).
    Open that URL in a browser to see an interactive dashboard:
    a dropdown to filter by category, a bar chart, and a line chart
    that update live as you change the filter. Press Ctrl+C in the
    terminal to stop the server.
"""

import argparse
import csv
import os
import sys

try:
    import pandas as pd
    from dash import Dash, dcc, html, Input, Output
    import plotly.express as px
except ImportError:
    print(
        "Missing dependency. Install requirements with: pip install dash plotly pandas",
        file=sys.stderr,
    )
    sys.exit(1)


def load_sample_dataframe():
    data = {
        "Region": ["North", "South", "East", "West", "North", "South", "East", "West"],
        "Month": ["Jan", "Jan", "Jan", "Jan", "Feb", "Feb", "Feb", "Feb"],
        "Sales": [220, 180, 260, 150, 240, 200, 280, 170],
    }
    return pd.DataFrame(data)


def load_csv_dataframe(path):
    return pd.read_csv(path)


def build_app(df):
    app = Dash(__name__)
    app.title = "Interactive Report Dashboard"

    category_col = df.columns[0]
    categories = ["All"] + sorted(df[category_col].dropna().unique().tolist())

    app.layout = html.Div(
        style={"fontFamily": "Arial, sans-serif", "maxWidth": "1000px", "margin": "0 auto", "padding": "20px"},
        children=[
            html.H2("Interactive Report Dashboard"),
            html.P(f"Filter by {category_col}:"),
            dcc.Dropdown(
                id="category-filter",
                options=[{"label": c, "value": c} for c in categories],
                value="All",
                clearable=False,
                style={"marginBottom": "20px"},
            ),
            dcc.Graph(id="bar-graph"),
            dcc.Graph(id="line-graph"),
        ],
    )

    numeric_cols = df.select_dtypes(include="number").columns.tolist()
    value_col = numeric_cols[0] if numeric_cols else df.columns[-1]

    @app.callback(
        Output("bar-graph", "figure"),
        Output("line-graph", "figure"),
        Input("category-filter", "value"),
    )
    def update_graphs(selected_category):
        filtered = df if selected_category == "All" else df[df[category_col] == selected_category]
        bar_fig = px.bar(filtered, x=category_col, y=value_col, color=category_col,
                          title=f"{value_col} by {category_col}")
        line_fig = px.line(filtered, x=category_col, y=value_col, markers=True,
                            title=f"{value_col} Trend")
        return bar_fig, line_fig

    return app


def main():
    parser = argparse.ArgumentParser(description="Launch an interactive Plotly/Dash dashboard from CSV data.")
    parser.add_argument("--input", "-i", help="Path to input CSV file. If omitted, sample data is used.")
    parser.add_argument("--port", "-p", type=int, default=8050, help="Port to run the dashboard on (default: 8050)")
    args = parser.parse_args()

    if args.input:
        if not os.path.isfile(args.input):
            print(f"Error: input file not found: {args.input}", file=sys.stderr)
            sys.exit(1)
        df = load_csv_dataframe(args.input)
    else:
        df = load_sample_dataframe()
        print("No --input provided, using built-in sample data.")

    app = build_app(df)

    print(f"Starting dashboard at: http://127.0.0.1:{args.port}")
    print("Press CTRL+C to stop the server.")
    app.run(debug=False, port=args.port)


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

`build_app()` assumes your first CSV column is the categorical filter column (used to populate the dropdown) and picks the first numeric column as the value to plot. The `@app.callback` decorator wires the dropdown's `value` as an `Input` to two `Output`s (the bar and line figures) — every time someone changes the dropdown in the browser, Dash calls `update_graphs()` server-side and pushes the newly filtered figures back to the page, all without a full page reload.

```bash

python generate_dashboard.py

# Point it at your own CSV and choose a different port
python generate_dashboard.py --input regional_sales.csv --port 8060

# Short flags
python generate_dashboard.py -i regional_sales.csv -p 8060
```

#### Sample Output

```bash
$ python generate_dashboard.py
No --input provided, using built-in sample data.
Starting dashboard at: http://127.0.0.1:8050
Press CTRL+C to stop the server.
 * Serving Flask app 'generate_dashboard'
 * Debug mode: off
```

Visiting `http://127.0.0.1:8050` in a browser shows "Interactive Report Dashboard" with a "Filter by Region:" dropdown (All/North/South/East/West), a bar chart of Sales by Region, and a line chart of the Sales trend — both updating instantly when a different region is selected.

#### Variations / Flags

- Add a `--host` flag (defaulting to `127.0.0.1`) so the dashboard can be reached from other machines on a trusted internal network — remember the caution above about this dev server having no built-in authentication.
- Run it under a process manager (`systemd`, `supervisor`, or inside `tmux`/`screen`) if you want the dashboard to stay up after your SSH session ends, rather than dying with the terminal.
- Add a second dropdown (e.g., filtering by both Region and Month) by adding another `dcc.Dropdown` and `Input` to the callback signature.

#### Common Pitfalls

- **Port already in use:** if another process already holds the chosen port, `app.run()` raises `OSError: Address already in use` — pick a different `--port` or free the existing one (`lsof -i :8050` on Linux/macOS to find the culprit).
- **Assumed column layout:** the script always treats the *first* CSV column as the filter category and the *first numeric* column as the value — a CSV where the meaningful category isn't in column one will produce a dropdown filtering on the wrong field; reorder columns or adjust `build_app()`.
- **Exposing the dev server:** binding `--host 0.0.0.0` (or otherwise making this reachable outside localhost) without a reverse proxy and authentication in front of it exposes the Dash dev server directly to the network — treat this as an internal/trusted-network tool only.
- **Ctrl+C not stopping cleanly in some terminals:** on Windows terminals in particular, `Ctrl+C` occasionally needs a second press to fully stop the Flask dev server underneath Dash — if the port stays "in use" after stopping, check for a lingering process before restarting.

### 20.10 Generate Dynamic Inline Graphs

#### Use Case

You're SSH'd into a remote box mid-incident and want a quick visual read on a metric — CPU usage over the last hour, request latency across recent samples — without opening a browser, downloading a file, or leaving the terminal at all. This script draws a line or bar graph directly in the terminal using text characters, giving you an instant "does this look like a spike or is it flat" read.

#### Prerequisites

Works with the standard library alone (falls back to a plain-ASCII bar renderer); install `asciichartpy` for a smoother line-plot rendering.

```bash
# Optional but recommended — enables the smooth multi-row line-plot renderer
pip install asciichartpy
```

- **OS scope:** Linux and macOS terminals render ANSI colors natively; on Windows, use Windows Terminal or a modern terminal emulator for `--color` to display correctly (older `cmd.exe` may show raw escape codes instead of color).
- **Libraries used:** `argparse`, `csv`, `os`, `sys` (stdlib, always) + `asciichartpy` (pip, optional — script degrades gracefully without it).

#### Cautions

This script writes no file at all — its only output is what's printed to your terminal. If you need a persisted record of the graph (for a ticket or a report), redirect stdout to a file (`> graph.txt`) or use the `--color` flag only when you're viewing it live, since ANSI escape codes will pollute a plain text file if redirected with color enabled.

#### Script

```python
#!/usr/bin/env python3
"""
generate_inline_graph.py

Renders a dynamic ASCII/ANSI line graph directly in the terminal —
no browser, no file needed. Uses 'asciichartpy' if it is installed
for a smooth multi-row plot; otherwise falls back to a built-in
plain-ASCII bar renderer using only the standard library.

Usage:
    python generate_inline_graph.py
    python generate_inline_graph.py --input data.csv --value-col cpu_percent --color

Expected output:
    A line/bar graph made of text characters printed straight to
    your terminal, with axis labels and (optionally) ANSI colors.
    No files are created; the graph is only for terminal viewing.
"""

import argparse
import csv
import os
import sys

try:
    import asciichartpy
    HAS_ASCIICHART = True
except ImportError:
    HAS_ASCIICHART = False


ANSI_GREEN = "\033[92m"
ANSI_RESET = "\033[0m"


def load_sample_data():
    return [10, 25, 18, 40, 55, 48, 60, 72, 65, 80, 90, 75]


def load_csv_data(path, value_col):
    with open(path, newline="", encoding="utf-8") as f:
        reader = csv.DictReader(f)
        rows = list(reader)
    if not rows:
        raise ValueError("CSV file is empty.")
    if value_col not in rows[0]:
        raise ValueError(f"Column '{value_col}' not found. Available: {list(rows[0].keys())}")
    return [float(row[value_col]) for row in rows]


def render_with_asciichart(values, use_color, height):
    config = {"height": height}
    if use_color:
        config["colors"] = [asciichartpy.green]
    return asciichartpy.plot(values, config)


def render_fallback_bars(values, use_color, height):
    """Plain-ASCII horizontal bar fallback, stdlib only."""
    max_val = max(values) if values else 1
    lines = []
    for i, v in enumerate(values):
        bar_len = int((v / max_val) * height * 4) if max_val else 0
        bar = "#" * max(bar_len, 1)
        if use_color:
            bar = f"{ANSI_GREEN}{bar}{ANSI_RESET}"
        lines.append(f"{i:>3} | {bar} {v:g}")
    return "\n".join(lines)


def main():
    parser = argparse.ArgumentParser(description="Render an ASCII/ANSI graph directly in the terminal.")
    parser.add_argument("--input", "-i", help="Path to input CSV file. If omitted, sample data is used.")
    parser.add_argument("--value-col", default=None, help="CSV column with numeric values (required if --input is given)")
    parser.add_argument("--height", type=int, default=12, help="Graph height in terminal rows (default: 12)")
    parser.add_argument("--color", action="store_true", help="Enable ANSI color output")
    args = parser.parse_args()

    if args.input:
        if not os.path.isfile(args.input):
            print(f"Error: input file not found: {args.input}", file=sys.stderr)
            sys.exit(1)
        if not args.value_col:
            print("Error: --value-col is required when using --input.", file=sys.stderr)
            sys.exit(1)
        values = load_csv_data(args.input, args.value_col)
    else:
        values = load_sample_data()
        print("No --input provided, using built-in sample data.\n")

    print("Terminal Graph Report")
    print("=" * 40)

    if HAS_ASCIICHART:
        print(render_with_asciichart(values, args.color, args.height))
    else:
        print("(Tip: install 'asciichartpy' for a smoother line-plot: pip install asciichartpy)")
        print(render_fallback_bars(values, args.color, args.height))

    print("=" * 40)
    print(f"Data points: {len(values)}  |  Min: {min(values):g}  Max: {max(values):g}  Avg: {sum(values)/len(values):.2f}")


if __name__ == "__main__":
    main()
```

#### How It Works and How to Run

The script tries to `import asciichartpy` at module load time and sets `HAS_ASCIICHART` accordingly; everything downstream branches on that flag. With `asciichartpy` installed, `render_with_asciichart()` produces a proper multi-row line plot with an axis; without it, `render_fallback_bars()` draws one text-based horizontal bar per data point scaled against the maximum value — a cruder but dependency-free substitute that always works.

```bash
# Zero-config run — uses built-in sample data, falls back gracefully if asciichartpy isn't installed
python generate_inline_graph.py

# Point it at your own CSV column, with color and a taller graph
python generate_inline_graph.py --input metrics.csv --value-col cpu_percent --color --height 18

# Short flags (input only has a long form; combine as needed)
python generate_inline_graph.py -i metrics.csv --value-col cpu_percent
```

#### Variations / Flags

- Pipe live data straight in with a shell one-liner, e.g., sampling CPU usage every few seconds into a CSV and re-running the script, for a poor-man's live monitor loop.
- Add a `--width` flag alongside `--height` if you want to control how many data points are shown per screen width on very wide terminals.
- Combine with `watch`: `watch -n 5 python generate_inline_graph.py -i /tmp/live_metrics.csv --value-col latency_ms` for a refreshing terminal view (note `watch` strips ANSI color by default; add `watch -c` to preserve it).

#### Common Pitfalls

- **`--value-col` required with `--input`:** unlike the sample-data path, real CSV input needs an explicit column name — omitting it exits with an error rather than guessing which column to plot.
- **ANSI codes in redirected output:** using `--color` while redirecting to a file (`python generate_inline_graph.py --color > out.txt`) embeds raw escape sequences in the text file, which will show as garbage characters (`\033[92m`) in a plain text editor — drop `--color` for any run whose output you plan to save or forward.
- **Terminal width wrapping:** with many data points and the fallback bar renderer, long bars can wrap past your terminal's width, breaking the one-line-per-point layout — reduce `--height` (which scales bar length) or narrow your data sample.
- **`asciichartpy` not being a stdlib guarantee:** don't assume it's present on a fresh server — the script degrades gracefully, but if you specifically need the smoother line-plot rendering (not just "a graph, any graph"), confirm the `pip install` succeeded rather than assuming the fallback is equivalent.
