## Chapter 23: Certificate & Key Management

Every production system eventually runs into the same wall: something needs to prove its identity, something needs to be encrypted, and something needs to be trusted. Certificates and cryptographic keys are the plumbing behind all three. They sit quietly behind HTTPS padlocks, SSH logins, code-signing pipelines, and VPN tunnels — invisible when everything works, and a 2 a.m. pager alert when they don't.

This chapter is a working toolkit, not a cryptography textbook. Each section pairs a real sysadmin or platform-engineering task with a small, dependency-light Python script built on the `cryptography` library (plus `python-gnupg` and `certbot` where GPG or ACME automation is involved). The scripts are meant to be read, adapted, and dropped into cron jobs, CI pipelines, or incident-response runbooks — not memorized.

The chapter is organized to follow the natural lifecycle of a credential: you create it, you request trust for it, someone signs it, you convert it into whatever format the next tool demands, you inspect it to make sure it is what you think it is, you watch it as it ages, you renew or revoke it, and eventually you retire it. SSH keys and GPG keys get the same treatment, since in practice the same team that manages TLS certificates is usually also managing SSH access and code-signing keys.

A note before you start: every script here defaults to safe, non-destructive behavior — generating new files rather than overwriting production credentials, printing what it *would* do before doing it, and exiting with clear, scriptable exit codes. But private keys are still private keys. Treat every `.key`, `.pem` private key block, `.pfx`, and GPG secret export the same way you would treat a password: encrypted at rest, restricted permissions (`chmod 600`), never committed to source control, and never emailed in plaintext.

#### What you'll need

All scripts target Python 3.8+ and are OS-agnostic unless noted. Two external binaries show up repeatedly: `gpg` (GnuPG, for section 23.12–23.14) and `certbot` (for section 23.9). Install instructions are given per-script, but if you want everything up front:

```bash
# Debian/Ubuntu
sudo apt update && sudo apt install -y gnupg certbot python3-pip

# macOS (Homebrew)
brew install gnupg certbot

# Python dependencies used across this chapter
pip install cryptography python-gnupg requests hvac
```

### 23.1 Generate Self-Signed Certificates

#### Use case

You need a TLS certificate right now for a local dev server, a staging environment, or an internal tool that will never face the public internet — and you don't want to wait on a CA. `self_signed_cert.py` generates an RSA key and a self-signed X.509 certificate in one shot.

#### Prerequisites

Python 3.8+ and the `cryptography` library. No OS-specific binaries required; works on Linux, macOS, and Windows.

```bash
pip install cryptography
```

#### Cautions

Self-signed certificates are not trusted by browsers or OSes by default — expect certificate warnings unless you manually import the cert into a trust store. Never use a self-signed certificate for a public-facing production service.

#### Script

```python
#!/usr/bin/env python3
"""
self_signed_cert.py - Generate a self-signed X.509 certificate and private key.
For internal testing / development only - browsers and OSes will not trust it.

Usage:
    python3 self_signed_cert.py --cn example.local --days 365 --keysize 2048 \
        --out-cert cert.pem --out-key key.pem

Output:
    - PEM encoded private key file
    - PEM encoded self-signed certificate file
"""
import argparse
import datetime
import sys

from cryptography import x509
from cryptography.hazmat.primitives import hashes, serialization
from cryptography.hazmat.primitives.asymmetric import rsa
from cryptography.x509.oid import NameOID


def generate_self_signed_cert(common_name, organization, country, days, key_size,
                               out_cert, out_key, key_password=None):
    print(f"[*] Generating {key_size}-bit RSA private key ...")
    key = rsa.generate_private_key(public_exponent=65537, key_size=key_size)

    subject = issuer = x509.Name([
        x509.NameAttribute(NameOID.COUNTRY_NAME, country),
        x509.NameAttribute(NameOID.ORGANIZATION_NAME, organization),
        x509.NameAttribute(NameOID.COMMON_NAME, common_name),
    ])

    now = datetime.datetime.utcnow()
    cert = (
        x509.CertificateBuilder()
        .subject_name(subject)
        .issuer_name(issuer)
        .public_key(key.public_key())
        .serial_number(x509.random_serial_number())
        .not_valid_before(now)
        .not_valid_after(now + datetime.timedelta(days=days))
        .add_extension(
            x509.SubjectAlternativeName([x509.DNSName(common_name)]),
            critical=False,
        )
        .add_extension(
            x509.BasicConstraints(ca=False, path_length=None), critical=True
        )
        .sign(key, hashes.SHA256())
    )

    encryption = (
        serialization.BestAvailableEncryption(key_password.encode())
        if key_password else serialization.NoEncryption()
    )

    with open(out_key, "wb") as f:
        f.write(key.private_bytes(
            encoding=serialization.Encoding.PEM,
            format=serialization.PrivateFormat.TraditionalOpenSSL,
            encryption_algorithm=encryption,
        ))

    with open(out_cert, "wb") as f:
        f.write(cert.public_bytes(serialization.Encoding.PEM))

    print(f"[+] Private key written to: {out_key}")
    print(f"[+] Certificate written to: {out_cert}")
    print(f"[+] Valid from {now} to {now + datetime.timedelta(days=days)}")


def main():
    parser = argparse.ArgumentParser(description="Generate a self-signed X.509 certificate.")
    parser.add_argument("--cn", required=True, help="Common Name (e.g. example.local)")
    parser.add_argument("--org", default="Internal Testing", help="Organization name")
    parser.add_argument("--country", default="US", help="2-letter country code")
    parser.add_argument("--days", type=int, default=365, help="Validity period in days")
    parser.add_argument("--keysize", type=int, default=2048, choices=[2048, 3072, 4096])
    parser.add_argument("--out-cert", default="cert.pem", help="Output certificate file")
    parser.add_argument("--out-key", default="key.pem", help="Output private key file")
    parser.add_argument("--key-password", default=None, help="Optional password to encrypt private key")
    args = parser.parse_args()

    try:
        generate_self_signed_cert(
            args.cn, args.org, args.country, args.days,
            args.keysize, args.out_cert, args.out_key, args.key_password
        )
    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

The script builds an RSA key pair, constructs an X.509 name used as both subject and issuer (making it "self-signed"), attaches a Subject Alternative Name and `BasicConstraints(ca=False)` extension, and signs the certificate with its own private key using SHA-256.

```bash
# Basic usage with defaults (2048-bit key, 365-day validity)
python3 self_signed_cert.py --cn dev.local.test

# Larger key, longer validity, custom output paths
python3 self_signed_cert.py --cn api.internal.test --org "Platform Team" \
    --country US --days 730 --keysize 4096 \
    --out-cert /etc/ssl/certs/api-internal.pem --out-key /etc/ssl/private/api-internal.key

# Encrypt the private key with a password
python3 self_signed_cert.py --cn secure.local --key-password 'Str0ngP@ss'
```

#### Sample output

```text
$ python3 self_signed_cert.py --cn dev.local.test
[*] Generating 2048-bit RSA private key ...
[+] Private key written to: key.pem
[+] Certificate written to: cert.pem
[+] Valid from 2026-07-18 09:12:03.221114 to 2027-07-18 09:12:03.221114
```

#### Variations / flags

- `--keysize 4096` for stronger keys on internal CAs or long-lived certs.
- `--key-password` to encrypt the private key at rest with a passphrase.
- Wrap the call in a loop to batch-generate certs for a list of hostnames (`--cn ${host}.local`, unique `--out-cert`/`--out-key` per host).
- Pair with `extract_cert_info.py` (23.5) immediately after generation to confirm the fields landed correctly.

#### Common pitfalls

- Writing to `/etc/ssl/private/` without root privileges will fail with a `PermissionError` — run with `sudo` or write locally and move the file with correct ownership afterward.
- Forgetting `chmod 600` on the output key file leaves a private key world-readable; the script does not set permissions for you.
- Browsers will show "Not Secure" / certificate warnings for self-signed certs — this is expected, not a bug. For anything users will hit directly, use 23.9 (Let's Encrypt) or 23.18 (internal PKI) instead.

### 23.2 Create Certificate Signing Requests (CSR)

#### Use case

You're requesting a certificate from a public CA (DigiCert, Sectigo, etc.) or an internal CA, and they need a CSR — a signed document proving you control the private key and stating which domain(s) the certificate should cover.

#### Prerequisites

Python 3.8+ and the `cryptography` library. Cross-platform.

```bash
pip install cryptography
```

#### Cautions

The private key generated alongside the CSR is the one that will ultimately pair with the issued certificate — protect it immediately (`chmod 600`) and never send it to the CA; only the CSR file goes out.

#### Script

```python
#!/usr/bin/env python3
"""
generate_csr.py - Generate a private key and a Certificate Signing Request (CSR)
to submit to a Certificate Authority (public or internal).

Usage:
    python3 generate_csr.py --cn www.example.com --org "Example Inc" \
        --country US --state California --locality "San Francisco" \
        --san www.example.com --san example.com \
        --out-csr request.csr --out-key request.key

Output:
    - PEM private key file
    - PEM CSR file, ready to send to a CA
"""
import argparse
import sys

from cryptography import x509
from cryptography.hazmat.primitives import hashes, serialization
from cryptography.hazmat.primitives.asymmetric import rsa
from cryptography.x509.oid import NameOID


def generate_csr(cn, org, country, state, locality, sans, key_size, out_csr, out_key):
    key = rsa.generate_private_key(public_exponent=65537, key_size=key_size)

    name_attrs = [x509.NameAttribute(NameOID.COMMON_NAME, cn)]
    if org:
        name_attrs.append(x509.NameAttribute(NameOID.ORGANIZATION_NAME, org))
    if country:
        name_attrs.append(x509.NameAttribute(NameOID.COUNTRY_NAME, country))
    if state:
        name_attrs.append(x509.NameAttribute(NameOID.STATE_OR_PROVINCE_NAME, state))
    if locality:
        name_attrs.append(x509.NameAttribute(NameOID.LOCALITY_NAME, locality))

    builder = x509.CertificateSigningRequestBuilder().subject_name(x509.Name(name_attrs))

    san_list = sans or [cn]
    builder = builder.add_extension(
        x509.SubjectAlternativeName([x509.DNSName(d) for d in san_list]),
        critical=False,
    )

    csr = builder.sign(key, hashes.SHA256())

    with open(out_key, "wb") as f:
        f.write(key.private_bytes(
            encoding=serialization.Encoding.PEM,
            format=serialization.PrivateFormat.TraditionalOpenSSL,
            encryption_algorithm=serialization.NoEncryption(),
        ))

    with open(out_csr, "wb") as f:
        f.write(csr.public_bytes(serialization.Encoding.PEM))

    print(f"[+] Private key written to: {out_key}")
    print(f"[+] CSR written to: {out_csr}")
    print("[*] Submit the CSR file to your Certificate Authority.")


def main():
    parser = argparse.ArgumentParser(description="Generate a private key and CSR.")
    parser.add_argument("--cn", required=True)
    parser.add_argument("--org", default=None)
    parser.add_argument("--country", default=None)
    parser.add_argument("--state", default=None)
    parser.add_argument("--locality", default=None)
    parser.add_argument("--san", action="append", help="Additional SAN entries, repeatable")
    parser.add_argument("--keysize", type=int, default=2048, choices=[2048, 3072, 4096])
    parser.add_argument("--out-csr", default="request.csr")
    parser.add_argument("--out-key", default="request.key")
    args = parser.parse_args()

    try:
        generate_csr(args.cn, args.org, args.country, args.state, args.locality,
                     args.san, args.keysize, args.out_csr, args.out_key)
    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

The script builds a distinguished name from the provided fields, attaches a `SubjectAlternativeName` extension covering every `--san` flag (falling back to just the CN if none are given), and self-signs the CSR with the freshly generated key to prove key possession.

```bash
# Minimal CSR, single domain
python3 generate_csr.py --cn app.example.com

# Full org details with multiple SANs (multi-domain / SAN certificate)
python3 generate_csr.py --cn www.example.com --org "Example Inc" \
    --country US --state California --locality "San Francisco" \
    --san www.example.com --san example.com --san api.example.com \
    --out-csr request.csr --out-key request.key

# Stronger key for long-lived internal cert
python3 generate_csr.py --cn vpn.corp.internal --keysize 4096 --out-csr vpn.csr --out-key vpn.key
```

#### Sample output

```text
$ python3 generate_csr.py --cn www.example.com --san www.example.com --san example.com
[+] Private key written to: request.key
[+] CSR written to: request.csr
[*] Submit the CSR file to your Certificate Authority.
```

#### Variations / flags

- Repeat `--san` for as many alternative names as the certificate needs.
- Feed the resulting `request.csr` into `sign_certificate.py` (23.3) if you're using an internal CA instead of a public one.
- Add `--keysize 4096` for CAs or policies that require stronger keys.

#### Common pitfalls

- Forgetting to include the primary hostname in `--san` — modern browsers ignore the CN field and validate against SAN entries only, so a cert without the right SAN will fail hostname validation even if the CN looks correct.
- Losing `request.key` before the CA returns the signed certificate means starting over; keep it until the signed cert is in hand and verified with `openssl x509 -noout -modulus` / `openssl rsa -noout -modulus` matching.
- Some public CAs reject CSRs with 2048-bit keys for EV certs or require ECDSA — check your CA's requirements before generating.

### 23.3 Sign Certificates with a CA

#### Use case

You run an internal Certificate Authority (see 23.18) and need to turn incoming CSRs from services, servers, or teammates into signed, trusted-within-your-org certificates — without manually invoking `openssl ca` each time.

#### Prerequisites

Python 3.8+, `cryptography`, and an existing CA certificate + private key (from `build_internal_pki.py`, 23.18, or any other CA).

```bash
pip install cryptography
```

#### Cautions

This script copies over every extension present on the incoming CSR verbatim before adding `BasicConstraints(ca=False)`. A malicious or malformed CSR requesting CA privileges via a custom extension could otherwise be signed as-is — review CSRs from untrusted submitters before signing, especially their extensions.

#### Script

```python
#!/usr/bin/env python3
"""
sign_certificate.py - Act as an internal Certificate Authority: sign a CSR
with a CA private key/certificate and issue a signed certificate.

Usage:
    python3 sign_certificate.py --csr request.csr --ca-cert ca.pem \
        --ca-key ca.key --days 365 --out signed_cert.pem

Requires: an existing CA certificate & key (see build_internal_pki.py to create one).

Output:
    - PEM certificate signed by the supplied CA
"""
import argparse
import datetime
import sys

from cryptography import x509
from cryptography.hazmat.primitives import hashes, serialization


def load_ca_key(path, password):
    with open(path, "rb") as f:
        return serialization.load_pem_private_key(
            f.read(), password=password.encode() if password else None
        )


def sign_csr(csr_path, ca_cert_path, ca_key_path, ca_key_password, days, out_path):
    with open(csr_path, "rb") as f:
        csr = x509.load_pem_x509_csr(f.read())

    if not csr.is_signature_valid:
        raise ValueError("CSR signature is invalid; refusing to sign.")

    with open(ca_cert_path, "rb") as f:
        ca_cert = x509.load_pem_x509_certificate(f.read())

    ca_key = load_ca_key(ca_key_path, ca_key_password)

    now = datetime.datetime.utcnow()
    builder = (
        x509.CertificateBuilder()
        .subject_name(csr.subject)
        .issuer_name(ca_cert.subject)
        .public_key(csr.public_key())
        .serial_number(x509.random_serial_number())
        .not_valid_before(now)
        .not_valid_after(now + datetime.timedelta(days=days))
    )

    for ext in csr.extensions:
        builder = builder.add_extension(ext.value, critical=ext.critical)

    builder = builder.add_extension(
        x509.BasicConstraints(ca=False, path_length=None), critical=True
    )

    cert = builder.sign(ca_key, hashes.SHA256())

    with open(out_path, "wb") as f:
        f.write(cert.public_bytes(serialization.Encoding.PEM))

    print(f"[+] Signed certificate written to: {out_path}")
    print(f"[+] Subject: {csr.subject}")
    print(f"[+] Issuer: {ca_cert.subject}")
    print(f"[+] Valid until: {now + datetime.timedelta(days=days)}")


def main():
    parser = argparse.ArgumentParser(description="Sign a CSR using an internal CA.")
    parser.add_argument("--csr", required=True, help="Path to CSR file to sign")
    parser.add_argument("--ca-cert", required=True, help="Path to CA certificate")
    parser.add_argument("--ca-key", required=True, help="Path to CA private key")
    parser.add_argument("--ca-key-password", default=None, help="Password for CA key, if encrypted")
    parser.add_argument("--days", type=int, default=365, help="Validity period in days")
    parser.add_argument("--out", default="signed_cert.pem", help="Output signed certificate file")
    args = parser.parse_args()

    try:
        sign_csr(args.csr, args.ca_cert, args.ca_key, args.ca_key_password, args.days, args.out)
    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

The script loads the CSR and CA credentials, verifies the CSR's self-signature is valid (rejecting tampered requests), then builds a new certificate using the CSR's subject and public key, signed by the CA's private key. It copies extensions (like SAN) from the CSR and forces `ca=False` on the output so leaf certs can never masquerade as CAs.

```bash
# Sign a CSR from generate_csr.py against your internal intermediate CA
python3 sign_certificate.py --csr request.csr \
    --ca-cert pki/intermediate_ca.pem --ca-key pki/intermediate_ca.key \
    --days 365 --out signed_cert.pem

# Shorter-lived cert, encrypted CA key
python3 sign_certificate.py --csr web.csr --ca-cert ca.pem --ca-key ca.key \
    --ca-key-password 'CaKeyPass!' --days 90 --out web_signed.pem
```

#### Sample output

```text
$ python3 sign_certificate.py --csr request.csr --ca-cert pki/intermediate_ca.pem --ca-key pki/intermediate_ca.key --out signed_cert.pem
[+] Signed certificate written to: signed_cert.pem
[+] Subject: <Name(CN=www.example.com)>
[+] Issuer: <Name(C=US,O=My Company,CN=My Company Intermediate CA)>
[+] Valid until: 2027-07-18 09:31:47.882201
```

#### Variations / flags

- Shorten `--days` for higher-security environments practicing frequent rotation (e.g., 90-day leaf certs, mirroring public CA trends).
- Concatenate `signed_cert.pem` with `chain.pem` (from 23.18) to give the end service a full chain for TLS termination.
- Wrap in a small internal API/CI step that auto-signs CSRs from a trusted queue, logging every issued serial number for audit/CRL purposes.

#### Common pitfalls

- Signing a CSR whose signature doesn't validate is refused by design — if you hit `CSR signature is invalid`, re-generate the CSR; it may have been corrupted or hand-edited.
- Reusing the Root CA key directly (rather than an Intermediate) to sign leaf certs works technically but violates standard PKI hygiene — always sign leaf certs with an Intermediate CA and keep the Root offline.
- If the CA key is encrypted and `--ca-key-password` is omitted, the script raises a decryption error — password prompts are not interactive here, so the password must be supplied on the command line or read from an env var in a wrapper script.

### 23.4 Convert Certificate Formats (PEM, DER, PFX)

#### Use case

A Windows IIS server wants a `.pfx`, a Java app wants DER, and your CA gave you PEM. `convert_cert_format.py` moves certificates and keys between PEM, DER, and PKCS#12 (PFX) so you're not hand-rolling `openssl` invocations for every platform.

#### Prerequisites

Python 3.8+ and `cryptography` (which includes PKCS#12 support). Cross-platform.

```bash
pip install cryptography
```

#### Cautions

PFX/PKCS#12 files bundle the private key together with the certificate in one file. Always set `--pfx-password` when creating a PFX, and transmit/store the resulting file with the same care as a raw private key — an unencrypted PFX is a private key with extra steps.

#### Script

```python
#!/usr/bin/env python3
"""
convert_cert_format.py - Convert certificates/keys between PEM, DER, and PKCS#12 (PFX) formats.

Usage examples:
    # PEM -> DER
    python3 convert_cert_format.py --in cert.pem --in-form PEM --out cert.der --out-form DER

    # PEM -> PFX (bundling cert + key, optional chain)
    python3 convert_cert_format.py --in cert.pem --in-form PEM --out bundle.pfx --out-form PFX \
        --key key.pem --ca-chain chain.pem --pfx-password mypassword

    # PFX -> PEM
    python3 convert_cert_format.py --in bundle.pfx --in-form PFX --out cert.pem --out-form PEM \
        --pfx-password mypassword --out-key key.pem

Output:
    Certificate/key file(s) in the requested target format.
"""
import argparse
import sys

from cryptography import x509
from cryptography.hazmat.primitives import serialization
from cryptography.hazmat.primitives.serialization import pkcs12


def read_cert_any(path, form, password=None):
    with open(path, "rb") as f:
        data = f.read()
    if form == "PEM":
        return x509.load_pem_x509_certificate(data), None, []
    elif form == "DER":
        return x509.load_der_x509_certificate(data), None, []
    elif form == "PFX":
        key, cert, chain = pkcs12.load_key_and_certificates(
            data, password.encode() if password else None
        )
        return cert, key, chain or []
    raise ValueError(f"Unsupported input format: {form}")


def main():
    parser = argparse.ArgumentParser(description="Convert certificate formats (PEM/DER/PFX).")
    parser.add_argument("--in", dest="infile", required=True)
    parser.add_argument("--in-form", choices=["PEM", "DER", "PFX"], required=True)
    parser.add_argument("--out", required=True)
    parser.add_argument("--out-form", choices=["PEM", "DER", "PFX"], required=True)
    parser.add_argument("--key", help="Private key file (PEM), needed to build a PFX")
    parser.add_argument("--ca-chain", help="PEM file with intermediate/root certs to include in PFX")
    parser.add_argument("--pfx-password", default=None, help="Password for reading/writing PFX")
    parser.add_argument("--out-key", help="Where to write extracted private key (PFX -> PEM/DER)")
    parser.add_argument("--friendly-name", default="cert", help="Friendly name for PFX bundle")
    args = parser.parse_args()

    try:
        cert, key, chain = read_cert_any(args.infile, args.in_form, args.pfx_password)

        if args.out_form == "PEM":
            with open(args.out, "wb") as f:
                f.write(cert.public_bytes(serialization.Encoding.PEM))
            if key and args.out_key:
                with open(args.out_key, "wb") as f:
                    f.write(key.private_bytes(
                        serialization.Encoding.PEM,
                        serialization.PrivateFormat.TraditionalOpenSSL,
                        serialization.NoEncryption(),
                    ))

        elif args.out_form == "DER":
            with open(args.out, "wb") as f:
                f.write(cert.public_bytes(serialization.Encoding.DER))
            if key and args.out_key:
                with open(args.out_key, "wb") as f:
                    f.write(key.private_bytes(
                        serialization.Encoding.DER,
                        serialization.PrivateFormat.PKCS8,
                        serialization.NoEncryption(),
                    ))

        elif args.out_form == "PFX":
            if not args.key:
                raise ValueError("--key is required when converting to PFX")
            with open(args.key, "rb") as f:
                priv_key = serialization.load_pem_private_key(f.read(), password=None)

            extra_certs = []
            if args.ca_chain:
                with open(args.ca_chain, "rb") as f:
                    chain_data = f.read()
                if hasattr(x509, "load_pem_x509_certificates"):
                    extra_certs = x509.load_pem_x509_certificates(chain_data)

            enc = (
                serialization.BestAvailableEncryption(args.pfx_password.encode())
                if args.pfx_password else serialization.NoEncryption()
            )
            pfx_bytes = pkcs12.serialize_key_and_certificates(
                name=args.friendly_name.encode(),
                key=priv_key,
                cert=cert,
                cas=extra_certs or None,
                encryption_algorithm=enc,
            )
            with open(args.out, "wb") as f:
                f.write(pfx_bytes)

        print(f"[+] Converted {args.in_form} -> {args.out_form}: {args.out}")

    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

`read_cert_any()` normalizes PEM, DER, and PFX input into a common `(cert, key, chain)` tuple. The output branch then re-serializes into whichever target format was requested, pulling in `--key` and `--ca-chain` only when building a PFX bundle.

```bash
# PEM certificate -> DER (e.g. for a Java keystore import)
python3 convert_cert_format.py --in cert.pem --in-form PEM --out cert.der --out-form DER

# Bundle cert + key + chain into a password-protected PFX for Windows/IIS
python3 convert_cert_format.py --in cert.pem --in-form PEM --out bundle.pfx --out-form PFX \
    --key key.pem --ca-chain chain.pem --pfx-password 'ExportPass!' --friendly-name "web-server"

# Unpack a PFX back into PEM cert + key
python3 convert_cert_format.py --in bundle.pfx --in-form PFX --out cert.pem --out-form PEM \
    --pfx-password 'ExportPass!' --out-key key.pem
```

#### Sample output

```text
$ python3 convert_cert_format.py --in cert.pem --in-form PEM --out cert.der --out-form DER
[+] Converted PEM -> DER: cert.der

$ python3 convert_cert_format.py --in cert.pem --in-form PEM --out bundle.pfx --out-form PFX --key key.pem --pfx-password ExportPass!
[+] Converted PEM -> PFX: bundle.pfx
```

#### Variations / flags

- Add `--ca-chain` when producing a PFX so the importing system (IIS, Windows cert store) has the full trust chain, not just the leaf.
- Omit `--pfx-password` only for throwaway/dev bundles — an unencrypted PFX still contains the private key.
- Use `--out-key` on PFX→PEM conversions to recover both halves of a keypair from a bundle you only have as PFX.

#### Common pitfalls

- `x509.load_pem_x509_certificates` (plural) requires a reasonably current `cryptography` version; on older installs, `--ca-chain` silently loads zero certs — verify with `openssl pkcs12 -info` afterward.
- PFX passwords are case-sensitive and must match exactly between creation and later `--pfx-password` reads, or you'll get an opaque decryption error.
- DER format cannot represent a private key and certificate in a single file the way PFX can — always request `--out-key` separately for DER private keys.

### 23.5 Extract Certificate Information

#### Use case

Someone hands you a `.pem` or `.der` file and asks "what is this, who issued it, and when does it expire?" `extract_cert_info.py` answers in one command — useful for audits, incident response, or feeding into other automation via `--json`.

#### Prerequisites

Python 3.8+ and `cryptography`. Cross-platform.

```bash
pip install cryptography
```

#### Cautions

This script only reads and prints certificate metadata — it performs no chain validation or revocation checking. A certificate that parses cleanly here may still be expired, self-signed, or untrusted; pair with `check_cert_expiration.py` (23.6) and proper chain validation before relying on it for trust decisions.

#### Script

```python
#!/usr/bin/env python3
"""
extract_cert_info.py - Parse a certificate file and print subject, issuer,
validity dates, serial number, and fingerprints.

Usage:
    python3 extract_cert_info.py --cert cert.pem
    python3 extract_cert_info.py --cert cert.der --form DER
    python3 extract_cert_info.py --cert cert.pem --json

Output:
    Human-readable summary printed to stdout (or JSON with --json).
"""
import argparse
import json
import sys

from cryptography import x509
from cryptography.hazmat.primitives import hashes


def load_cert(path, form):
    with open(path, "rb") as f:
        data = f.read()
    if form == "PEM":
        return x509.load_pem_x509_certificate(data)
    return x509.load_der_x509_certificate(data)


def cert_info(cert):
    not_before = cert.not_valid_before_utc if hasattr(cert, "not_valid_before_utc") else cert.not_valid_before
    not_after = cert.not_valid_after_utc if hasattr(cert, "not_valid_after_utc") else cert.not_valid_after
    return {
        "subject": cert.subject.rfc4514_string(),
        "issuer": cert.issuer.rfc4514_string(),
        "serial_number": str(cert.serial_number),
        "not_valid_before": not_before.isoformat(),
        "not_valid_after": not_after.isoformat(),
        "signature_algorithm": cert.signature_hash_algorithm.name
            if cert.signature_hash_algorithm else "unknown",
        "sha256_fingerprint": cert.fingerprint(hashes.SHA256()).hex(),
        "sha1_fingerprint": cert.fingerprint(hashes.SHA1()).hex(),
    }


def main():
    parser = argparse.ArgumentParser(description="Extract information from a certificate file.")
    parser.add_argument("--cert", required=True)
    parser.add_argument("--form", choices=["PEM", "DER"], default="PEM")
    parser.add_argument("--json", action="store_true", help="Print output as JSON")
    args = parser.parse_args()

    try:
        cert = load_cert(args.cert, args.form)
        info = cert_info(cert)

        if args.json:
            print(json.dumps(info, indent=2))
        else:
            print(f"Subject             : {info['subject']}")
            print(f"Issuer               : {info['issuer']}")
            print(f"Serial Number        : {info['serial_number']}")
            print(f"Valid From           : {info['not_valid_before']}")
            print(f"Valid Until          : {info['not_valid_after']}")
            print(f"Signature Algorithm  : {info['signature_algorithm']}")
            print(f"SHA256 Fingerprint   : {info['sha256_fingerprint']}")
            print(f"SHA1 Fingerprint     : {info['sha1_fingerprint']}")

    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

The script loads the certificate (PEM or DER), then reads the standard X.509 fields off the parsed object — subject/issuer distinguished names, serial number, validity window, signature algorithm, and both SHA-256 and SHA-1 fingerprints (the latter still widely used for identifying certs in legacy tooling and browser UIs).

```bash
# Human-readable summary of a PEM cert
python3 extract_cert_info.py --cert cert.pem

# DER-encoded certificate
python3 extract_cert_info.py --cert cert.der --form DER

# Machine-readable JSON, e.g. for piping into jq or another script
python3 extract_cert_info.py --cert cert.pem --json | jq .sha256_fingerprint
```

#### Sample output

```text
$ python3 extract_cert_info.py --cert cert.pem
Subject             : CN=dev.local.test,O=Internal Testing,C=US
Issuer               : CN=dev.local.test,O=Internal Testing,C=US
Serial Number        : 291847562930481756928374651029384756
Valid From           : 2026-07-18T09:12:03+00:00
Valid Until          : 2027-07-18T09:12:03+00:00
Signature Algorithm  : sha256
SHA256 Fingerprint   : 8f3a1c...e29d
SHA1 Fingerprint     : 4b7e02...aa10
```

#### Variations / flags

- `--json` for feeding into monitoring dashboards, inventory scripts, or CI gates ("fail the build if `sha256_fingerprint` doesn't match the pinned value").
- Loop over a directory of certs (`for f in *.pem; do python3 extract_cert_info.py --cert "$f" --json; done`) to build a quick certificate inventory.
- Combine with `openssl s_client -connect host:443 -showcerts` output (saved to a file) to inspect what a live server is actually presenting.

#### Common pitfalls

- Passing a DER file without `--form DER` raises a parse error — the script does not auto-detect encoding.
- `subject.rfc4514_string()` reverses the field order compared to some OpenSSL output conventions (RFC 4514 vs. traditional `openssl x509 -subject` order) — don't assume identical string output between tools when diffing.
- SHA-1 fingerprints are shown for compatibility/identification only; do not treat SHA-1 as a security guarantee — it's not collision-resistant.

### 23.6 Check Certificate Expiration Dates

#### Use case

You want a one-shot, scriptable answer to "is this certificate about to expire?" — for a cron job, a CI gate, or a quick manual check against a local file or a live host's presented TLS certificate.

#### Prerequisites

Python 3.8+ and `cryptography`. No extra binaries. Works against local files or any TCP host presenting TLS (Linux/macOS/Windows).

```bash
pip install cryptography
```

#### Cautions

The `--host` mode disables hostname verification and certificate validation (`CERT_NONE`) by design, since the goal is to inspect whatever certificate is being served, even an invalid one — do not reuse this connection pattern for anything that needs to trust the server.

#### Script

```python
#!/usr/bin/env python3
"""
check_cert_expiration.py - Check how many days remain until a certificate expires,
and exit with a non-zero code if it is within a warning threshold. Useful for
cron jobs / monitoring pipelines.

Usage:
    python3 check_cert_expiration.py --cert cert.pem --warn-days 30
    python3 check_cert_expiration.py --host example.com --port 443 --warn-days 14

Exit codes:
    0 - certificate is valid and outside the warning window
    1 - certificate expires within the warning window
    2 - certificate has already expired
    3 - error reading/fetching the certificate
"""
import argparse
import datetime
import socket
import ssl
import sys

from cryptography import x509


def get_cert_from_file(path):
    with open(path, "rb") as f:
        data = f.read()
    try:
        return x509.load_pem_x509_certificate(data)
    except ValueError:
        return x509.load_der_x509_certificate(data)


def get_cert_from_host(host, port, timeout=10):
    ctx = ssl.create_default_context()
    ctx.check_hostname = False
    ctx.verify_mode = ssl.CERT_NONE
    with socket.create_connection((host, port), timeout=timeout) as sock:
        with ctx.wrap_socket(sock, server_hostname=host) as ssock:
            der = ssock.getpeercert(binary_form=True)
    return x509.load_der_x509_certificate(der)


def main():
    parser = argparse.ArgumentParser(description="Check certificate expiration.")
    parser.add_argument("--cert", help="Path to a local certificate file")
    parser.add_argument("--host", help="Remote host to check live TLS certificate")
    parser.add_argument("--port", type=int, default=443)
    parser.add_argument("--warn-days", type=int, default=30, help="Warn if expiry is within N days")
    args = parser.parse_args()

    if not args.cert and not args.host:
        print("[!] Provide either --cert or --host", file=sys.stderr)
        sys.exit(3)

    try:
        cert = get_cert_from_file(args.cert) if args.cert else get_cert_from_host(args.host, args.port)
        not_after = cert.not_valid_after_utc if hasattr(cert, "not_valid_after_utc") else cert.not_valid_after
        now = datetime.datetime.now(datetime.timezone.utc) if hasattr(cert, "not_valid_after_utc") \
            else datetime.datetime.utcnow()
        remaining = (not_after - now).days

        target = args.cert if args.cert else f"{args.host}:{args.port}"
        print(f"Target        : {target}")
        print(f"Expires On    : {not_after}")
        print(f"Days Remaining: {remaining}")

        if remaining < 0:
            print("[!] CERTIFICATE HAS EXPIRED")
            sys.exit(2)
        elif remaining <= args.warn_days:
            print(f"[!] WARNING: certificate expires within {args.warn_days} days")
            sys.exit(1)
        else:
            print("[+] Certificate is valid and not near expiry.")
            sys.exit(0)

    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(3)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

The script loads a certificate either from a local file (trying PEM then falling back to DER) or by opening a raw TLS socket to a host:port and pulling the peer certificate without validating it. It then computes days remaining until `not_valid_after` and maps that to one of four exit codes, which is what makes it cron/CI-friendly.

```bash
# Local file, default 30-day warning window
python3 check_cert_expiration.py --cert /etc/ssl/certs/api.pem

# Live host, tighter 14-day window
python3 check_cert_expiration.py --host example.com --port 443 --warn-days 14

# Use the exit code directly in a shell conditional
python3 check_cert_expiration.py --cert cert.pem --warn-days 30 || echo "needs renewal"
```

#### Sample output

```text
$ python3 check_cert_expiration.py --host example.com --port 443 --warn-days 30
Target        : example.com:443
Expires On    : 2026-08-02 00:00:00+00:00
Days Remaining: 15
[!] WARNING: certificate expires within 30 days
```

#### Variations / flags

- Drop into a crontab entry that emails or Slacks on non-zero exit: `0 6 * * * check_cert_expiration.py --host example.com --warn-days 21 || notify-team.sh`.
- Chain multiple invocations across a fleet in a shell loop, collecting exit codes into a summary report.
- Lower `--warn-days` for certs on faster renewal cycles (e.g., 90-day Let's Encrypt certs might use `--warn-days 14`).

#### Common pitfalls

- Non-standard ports need `--port` set explicitly — the default is 443, which will silently connect to the wrong service (or fail to connect) on hosts using another port for TLS.
- Hosts requiring SNI with multiple certificates per IP rely on `server_hostname=host` being correct; a typo in `--host` may fetch the wrong virtual host's certificate.
- Exit code 3 covers both "file not found" and "connection refused" — check stderr for the actual cause rather than assuming expiry-related failure.

### 23.7 Monitor Certificates for Expiry

#### Use case

You have more than one certificate to worry about — a handful of internal services plus a few public-facing hosts — and want a standing process (or scheduled job) that checks all of them and pages your team via webhook or email the moment any of them enters the warning window.

#### Prerequisites

Python 3.8+, `cryptography`, and `requests`. Email notification uses the stdlib `smtplib`, so no extra install is needed for that path. Cross-platform.

```bash
pip install cryptography requests
```

#### Cautions

Like 23.6, `--host` checks skip certificate validation on purpose (to inspect whatever is being served). SMTP credentials passed via `--smtp-password` are visible in shell history and process listings — prefer environment variables or a secrets manager in production, and consider wrapping this script rather than typing the password on the CLI.

#### Script

```python
#!/usr/bin/env python3
"""
monitor_cert_expiry.py - Continuously (or once) scan a list of certificates/hosts
and send a notification (email or webhook) when any of them are within a
warning window of expiring.

Usage:
    # one-off scan of local cert files
    python3 monitor_cert_expiry.py --cert cert1.pem --cert cert2.pem --warn-days 30

    # scan remote hosts on an interval, notify via webhook
    python3 monitor_cert_expiry.py --host example.com:443 --host example.org:443 \
        --warn-days 14 --interval 3600 --webhook https://hooks.example.com/notify

    # notify via email (requires SMTP settings)
    python3 monitor_cert_expiry.py --cert cert.pem --smtp-host smtp.example.com \
        --smtp-user user --smtp-password secret --email-to admin@example.com

Output:
    Console log of each check; outbound notification when a cert is expiring soon.
"""
import argparse
import datetime
import smtplib
import socket
import ssl
import sys
import time
from email.mime.text import MIMEText

import requests
from cryptography import x509


def get_cert_from_file(path):
    with open(path, "rb") as f:
        data = f.read()
    try:
        return x509.load_pem_x509_certificate(data)
    except ValueError:
        return x509.load_der_x509_certificate(data)


def get_cert_from_host(host, port, timeout=10):
    ctx = ssl.create_default_context()
    ctx.check_hostname = False
    ctx.verify_mode = ssl.CERT_NONE
    with socket.create_connection((host, port), timeout=timeout) as sock:
        with ctx.wrap_socket(sock, server_hostname=host) as ssock:
            der = ssock.getpeercert(binary_form=True)
    return x509.load_der_x509_certificate(der)


def days_remaining(cert):
    not_after = cert.not_valid_after_utc if hasattr(cert, "not_valid_after_utc") else cert.not_valid_after
    now = datetime.datetime.now(datetime.timezone.utc) if hasattr(cert, "not_valid_after_utc") \
        else datetime.datetime.utcnow()
    return (not_after - now).days


def send_webhook(url, message):
    try:
        requests.post(url, json={"text": message}, timeout=10)
    except Exception as e:
        print(f"[!] Failed to send webhook: {e}", file=sys.stderr)


def send_email(smtp_host, smtp_port, smtp_user, smtp_password, to_addr, subject, body):
    msg = MIMEText(body)
    msg["Subject"] = subject
    msg["From"] = smtp_user
    msg["To"] = to_addr
    try:
        with smtplib.SMTP(smtp_host, smtp_port, timeout=10) as server:
            server.starttls()
            server.login(smtp_user, smtp_password)
            server.sendmail(smtp_user, [to_addr], msg.as_string())
    except Exception as e:
        print(f"[!] Failed to send email: {e}", file=sys.stderr)


def run_once(targets, warn_days, webhook, email_cfg):
    alerts = []
    for kind, value in targets:
        try:
            if kind == "file":
                cert = get_cert_from_file(value)
                label = value
            else:
                host, port = value
                cert = get_cert_from_host(host, port)
                label = f"{host}:{port}"

            remaining = days_remaining(cert)
            status = "EXPIRED" if remaining < 0 else ("WARNING" if remaining <= warn_days else "OK")
            print(f"[{status}] {label} -> {remaining} days remaining")

            if status != "OK":
                alerts.append(f"{label}: {remaining} days remaining ({status})")

        except Exception as e:
            print(f"[!] Failed to check {value}: {e}", file=sys.stderr)

    if alerts:
        message = "Certificate expiry alert:\n" + "\n".join(alerts)
        if webhook:
            send_webhook(webhook, message)
        if email_cfg:
            send_email(*email_cfg, subject="Certificate Expiry Alert", body=message)


def main():
    parser = argparse.ArgumentParser(description="Monitor certificates for upcoming expiry.")
    parser.add_argument("--cert", action="append", default=[], help="Local cert file, repeatable")
    parser.add_argument("--host", action="append", default=[], help="host:port, repeatable")
    parser.add_argument("--warn-days", type=int, default=30)
    parser.add_argument("--interval", type=int, default=0, help="Seconds between checks; 0 = run once")
    parser.add_argument("--webhook", default=None, help="Webhook URL for notifications")
    parser.add_argument("--smtp-host", default=None)
    parser.add_argument("--smtp-port", type=int, default=587)
    parser.add_argument("--smtp-user", default=None)
    parser.add_argument("--smtp-password", default=None)
    parser.add_argument("--email-to", default=None)
    args = parser.parse_args()

    targets = [("file", c) for c in args.cert]
    for h in args.host:
        host, _, port = h.partition(":")
        targets.append(("host", (host, int(port) if port else 443)))

    if not targets:
        print("[!] Provide at least one --cert or --host", file=sys.stderr)
        sys.exit(1)

    email_cfg = None
    if args.smtp_host and args.smtp_user and args.smtp_password and args.email_to:
        email_cfg = (args.smtp_host, args.smtp_port, args.smtp_user, args.smtp_password, args.email_to)

    if args.interval <= 0:
        run_once(targets, args.warn_days, args.webhook, email_cfg)
    else:
        print(f"[*] Monitoring every {args.interval}s. Press Ctrl+C to stop.")
        try:
            while True:
                run_once(targets, args.warn_days, args.webhook, email_cfg)
                time.sleep(args.interval)
        except KeyboardInterrupt:
            print("\n[*] Monitoring stopped.")


if __name__ == "__main__":
    main()
```

#### How it works and how to run

Targets accumulate from repeated `--cert` and `--host` flags into a unified list. `run_once()` checks each target, classifies it OK/WARNING/EXPIRED, and — only if there's at least one non-OK result — batches everything into a single notification sent to a webhook and/or via SMTP email. With `--interval` set, the whole cycle repeats in a loop until interrupted.

```bash
# One-off scan of two local cert files
python3 monitor_cert_expiry.py --cert cert1.pem --cert cert2.pem --warn-days 30

# Continuous monitoring of two public hosts, hourly, alert via webhook
python3 monitor_cert_expiry.py --host example.com:443 --host example.org:443 \
    --warn-days 14 --interval 3600 --webhook https://hooks.example.com/notify

# Local cert, email alert (SMTP password via env var recommended)
python3 monitor_cert_expiry.py --cert cert.pem --smtp-host smtp.example.com --smtp-port 587 \
    --smtp-user alerts@example.com --smtp-password "$SMTP_PASSWORD" --email-to admin@example.com
```

#### Sample output

```text
$ python3 monitor_cert_expiry.py --host example.com:443 --host example.org:443 --warn-days 14
[OK] example.com:443 -> 210 days remaining
[WARNING] example.org:443 -> 9 days remaining
```

#### Variations / flags

- Set `--interval 86400` and run under `systemd` (as a service) or `supervisord` for always-on monitoring instead of relying on cron.
- Combine `--webhook` and email flags together to notify both a chat channel and an on-call inbox from one run.
- Mix `--cert` and `--host` targets freely in the same invocation to cover both internal file-based certs and public-facing endpoints.

#### Common pitfalls

- Running with `--interval` inside a container without a process supervisor means a crash silently stops monitoring — wrap it in `systemd` `Restart=on-failure` or similar.
- Webhook payload format (`{"text": message}`) matches Slack-style incoming webhooks; other services (PagerDuty, Teams) expect a different JSON schema and will need the `send_webhook()` function adjusted.
- SMTP `starttls()` assumes the server supports STARTTLS on the given port (587 by default) — servers requiring implicit TLS (port 465) need `smtplib.SMTP_SSL` instead.

### 23.8 Automate Certificate Renewal

#### Use case

A certificate is approaching expiry and you want to renew it automatically — either by reusing its existing key to mint a fresh self-signed cert, or by generating a new CSR to resubmit to a CA — but only when it's actually within the renewal window, not every time the job runs.

#### Prerequisites

Python 3.8+ and `cryptography`. Cross-platform.

```bash
pip install cryptography
```

#### Cautions

`renew_csr()` builds the new CSR from the existing certificate's subject only — it does not copy the original certificate's SAN or other extensions. If the cert relies on SAN entries (almost all modern certs do), add SAN handling before relying on this in production, or regenerate the CSR from scratch with `generate_csr.py` (23.2) instead.

#### Script

```python
#!/usr/bin/env python3
"""
renew_certificate.py - Check a certificate's remaining validity and, if it is
within a renewal threshold, generate a fresh self-signed certificate (reusing
the existing private key) or a new CSR for re-submission to a CA.

Usage:
    # Renew a self-signed cert in place if it expires within 30 days
    python3 renew_certificate.py --cert cert.pem --key key.pem --mode self-signed \
        --renew-days 30 --valid-days 365

    # Generate a fresh CSR for CA renewal if within threshold
    python3 renew_certificate.py --cert cert.pem --key key.pem --mode csr \
        --renew-days 30 --out-csr renewed.csr

Output:
    A renewed certificate (or CSR) only if the existing certificate is within
    the renewal window; otherwise reports that renewal is not yet needed.
"""
import argparse
import datetime
import sys

from cryptography import x509
from cryptography.hazmat.primitives import hashes, serialization


def load_cert(path):
    with open(path, "rb") as f:
        return x509.load_pem_x509_certificate(f.read())


def load_key(path, password=None):
    with open(path, "rb") as f:
        return serialization.load_pem_private_key(f.read(), password=password.encode() if password else None)


def days_remaining(cert):
    not_after = cert.not_valid_after_utc if hasattr(cert, "not_valid_after_utc") else cert.not_valid_after
    now = datetime.datetime.now(datetime.timezone.utc) if hasattr(cert, "not_valid_after_utc") \
        else datetime.datetime.utcnow()
    return (not_after - now).days


def renew_self_signed(cert, key, valid_days, out_cert):
    now = datetime.datetime.utcnow()
    new_cert = (
        x509.CertificateBuilder()
        .subject_name(cert.subject)
        .issuer_name(cert.subject)
        .public_key(key.public_key())
        .serial_number(x509.random_serial_number())
        .not_valid_before(now)
        .not_valid_after(now + datetime.timedelta(days=valid_days))
        .add_extension(x509.BasicConstraints(ca=False, path_length=None), critical=True)
        .sign(key, hashes.SHA256())
    )
    with open(out_cert, "wb") as f:
        f.write(new_cert.public_bytes(serialization.Encoding.PEM))
    print(f"[+] Renewed self-signed certificate written to: {out_cert}")
    print(f"[+] New expiry: {now + datetime.timedelta(days=valid_days)}")


def renew_csr(cert, key, out_csr):
    builder = x509.CertificateSigningRequestBuilder().subject_name(cert.subject)
    csr = builder.sign(key, hashes.SHA256())
    with open(out_csr, "wb") as f:
        f.write(csr.public_bytes(serialization.Encoding.PEM))
    print(f"[+] New CSR written to: {out_csr}")
    print("[*] Submit this CSR to your CA to obtain the renewed certificate.")


def main():
    parser = argparse.ArgumentParser(description="Renew a certificate before it expires.")
    parser.add_argument("--cert", required=True, help="Existing certificate file")
    parser.add_argument("--key", required=True, help="Existing private key file")
    parser.add_argument("--key-password", default=None)
    parser.add_argument("--mode", choices=["self-signed", "csr"], required=True)
    parser.add_argument("--renew-days", type=int, default=30, help="Renew if expiry is within N days")
    parser.add_argument("--valid-days", type=int, default=365, help="Validity period for the renewed cert")
    parser.add_argument("--out-cert", default="renewed_cert.pem")
    parser.add_argument("--out-csr", default="renewed.csr")
    parser.add_argument("--force", action="store_true", help="Renew regardless of remaining validity")
    args = parser.parse_args()

    try:
        cert = load_cert(args.cert)
        key = load_key(args.key, args.key_password)
        remaining = days_remaining(cert)
        print(f"[*] Current certificate expires in {remaining} days.")

        if remaining > args.renew_days and not args.force:
            print("[*] Renewal not needed yet. Use --force to renew anyway.")
            return

        if args.mode == "self-signed":
            renew_self_signed(cert, key, args.valid_days, args.out_cert)
        else:
            renew_csr(cert, key, args.out_csr)

    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

The script computes days remaining on the existing cert. If that's still outside `--renew-days` and `--force` wasn't passed, it exits without changing anything. Otherwise, in `self-signed` mode it mints a brand-new self-signed cert reusing the same key (same subject, new serial, new validity window); in `csr` mode it builds a fresh CSR from the existing subject for resubmission to a CA.

```bash
# Check-and-renew a self-signed cert in place, only if within 30 days of expiry
python3 renew_certificate.py --cert cert.pem --key key.pem --mode self-signed \
    --renew-days 30 --valid-days 365 --out-cert cert.pem

# Generate a renewal CSR for CA resubmission
python3 renew_certificate.py --cert cert.pem --key key.pem --mode csr \
    --renew-days 30 --out-csr renewed.csr

# Force renewal regardless of current expiry (e.g. testing the pipeline)
python3 renew_certificate.py --cert cert.pem --key key.pem --mode self-signed --force
```

```text
[+] Renewed self-signed certificate written to: renewed_cert.pem
[+] New expiry: 2027-07-18 09:52:11.004821

[*] Renewal not needed yet. Use --force to renew anyway.
```

#### Variations / flags

- Schedule with cron for automatic self-signed renewal on internal dev/test certs: `0 3 * * * renew_certificate.py --cert /etc/ssl/dev.pem --key /etc/ssl/dev.key --mode self-signed --renew-days 14 --out-cert /etc/ssl/dev.pem`.
- Use `--mode csr` in a pipeline that emails or uploads the CSR to your CA's API automatically, then feeds the returned cert back into place.
- `--force` is handy for dry-testing the renewal path in staging without waiting for a real expiry window.

#### Common pitfalls

- Overwriting `--out-cert` to the same path as `--cert` works but leaves no rollback copy — back up the old cert first in production automation.
- `renew_csr()` does not preserve SAN entries from the original cert; a CA may reject or mis-issue the renewal if SANs are required and missing. Prefer `generate_csr.py` with explicit `--san` flags for anything with multiple hostnames.
- If the private key is encrypted and `--key-password` isn't supplied, loading fails with a decryption error rather than prompting interactively.

### 23.9 Manage Let's Encrypt Certificates

#### Use case

You need a certificate that browsers actually trust out of the box, for free, for a public-facing service — the standard modern answer is Let's Encrypt via the ACME protocol. `letsencrypt_manager.py` is a thin wrapper that orchestrates the `certbot` CLI for obtaining and renewing certificates without you having to remember its flags.

#### Prerequisites

Python 3.8+ (stdlib only — no `cryptography` dependency here) plus `certbot` installed and on `PATH`. Standalone mode needs port 80 free and root/administrator privileges to bind it.

```bash
# Ubuntu/Debian
sudo apt update && sudo apt install -y certbot

# macOS (Homebrew)
brew install certbot

# Verify installation
certbot --version
```

#### Cautions

The `standalone` method temporarily binds port 80, which will fail (and may briefly disrupt traffic) if a web server is already listening there — use `webroot` mode instead for hosts already running nginx/Apache. Let's Encrypt enforces rate limits (roughly 5 duplicate certificates per exact domain set per week); always test with `--dry-run` first.

#### Script

```python
#!/usr/bin/env python3
"""
letsencrypt_manager.py - Thin automation wrapper around the `certbot` CLI to
obtain and renew free, trusted TLS certificates via the ACME protocol.

IMPORTANT: This script does not implement the ACME protocol itself. It
requires certbot to be installed separately (https://certbot.eff.org) and
simply orchestrates it. It must be run with sufficient privileges to bind
port 80/443 (standalone mode) or to write to your webroot.

Usage:
    # Obtain a new certificate using the standalone plugin (port 80 must be free)
    python3 letsencrypt_manager.py obtain --domain example.com --email admin@example.com \
        --method standalone

    # Obtain using webroot plugin
    python3 letsencrypt_manager.py obtain --domain example.com --email admin@example.com \
        --method webroot --webroot-path /var/www/html

    # Renew all certificates due for renewal
    python3 letsencrypt_manager.py renew

    # Dry run (test without hitting real rate limits)
    python3 letsencrypt_manager.py obtain --domain example.com --email admin@example.com \
        --method standalone --dry-run

Output:
    Certificates are stored by certbot under /etc/letsencrypt/live/<domain>/
    This script prints certbot's output and the resulting exit code.
"""
import argparse
import shutil
import subprocess
import sys


def ensure_certbot_installed():
    if shutil.which("certbot") is None:
        print(
            "[!] certbot is not installed or not on PATH.\n"
            "    Install it first, e.g.:\n"
            "      Ubuntu/Debian : sudo apt install certbot\n"
            "      macOS (brew)  : brew install certbot\n"
            "    See https://certbot.eff.org for OS-specific instructions.",
            file=sys.stderr,
        )
        sys.exit(1)


def obtain(args):
    cmd = ["certbot", "certonly", "--non-interactive", "--agree-tos",
           "--email", args.email, "-d", args.domain]

    if args.method == "standalone":
        cmd += ["--standalone"]
    elif args.method == "webroot":
        if not args.webroot_path:
            print("[!] --webroot-path is required for the webroot method", file=sys.stderr)
            sys.exit(1)
        cmd += ["--webroot", "-w", args.webroot_path]
    elif args.method == "dns":
        print("[*] DNS-01 challenges require a provider-specific certbot plugin "
              "(e.g. certbot-dns-cloudflare). Install it and re-run certbot manually "
              "or extend this script with the relevant --dns-* flags.")
        sys.exit(1)

    if args.dry_run:
        cmd += ["--dry-run"]

    print(f"[*] Running: {' '.join(cmd)}")
    result = subprocess.run(cmd)
    sys.exit(result.returncode)


def renew(args):
    cmd = ["certbot", "renew", "--non-interactive"]
    if args.dry_run:
        cmd += ["--dry-run"]
    print(f"[*] Running: {' '.join(cmd)}")
    result = subprocess.run(cmd)
    sys.exit(result.returncode)


def main():
    ensure_certbot_installed()

    parser = argparse.ArgumentParser(description="Automate Let's Encrypt certificates via certbot.")
    sub = parser.add_subparsers(dest="action", required=True)

    p_obtain = sub.add_parser("obtain", help="Obtain a new certificate")
    p_obtain.add_argument("--domain", required=True)
    p_obtain.add_argument("--email", required=True)
    p_obtain.add_argument("--method", choices=["standalone", "webroot", "dns"], default="standalone")
    p_obtain.add_argument("--webroot-path", default=None)
    p_obtain.add_argument("--dry-run", action="store_true")
    p_obtain.set_defaults(func=obtain)

    p_renew = sub.add_parser("renew", help="Renew all due certificates")
    p_renew.add_argument("--dry-run", action="store_true")
    p_renew.set_defaults(func=renew)

    args = parser.parse_args()
    args.func(args)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

The script checks that `certbot` is on `PATH`, then builds and shells out to the appropriate `certbot certonly` or `certbot renew` command, forwarding its exit code back to the caller. `standalone` runs certbot's own temporary web server to answer the HTTP-01 challenge; `webroot` drops challenge files into an existing web server's document root instead.

```bash
# Obtain a new cert with certbot's built-in standalone server (port 80 must be free)
sudo python3 letsencrypt_manager.py obtain --domain example.com --email admin@example.com --method standalone

# Obtain a cert while nginx/Apache is already serving on port 80
sudo python3 letsencrypt_manager.py obtain --domain example.com --email admin@example.com \
    --method webroot --webroot-path /var/www/html

# Renew everything due for renewal (typically run twice daily via cron/systemd timer)
sudo python3 letsencrypt_manager.py renew

# Dry run first, to avoid burning rate limits
sudo python3 letsencrypt_manager.py obtain --domain example.com --email admin@example.com \
    --method standalone --dry-run
```

#### Sample output

```text
$ sudo python3 letsencrypt_manager.py obtain --domain example.com --email admin@example.com --method standalone --dry-run
[*] Running: certbot certonly --non-interactive --agree-tos --email admin@example.com -d example.com --standalone --dry-run
Simulating a certificate request for example.com
The dry run was successful.

$ sudo python3 letsencrypt_manager.py renew
[*] Running: certbot renew --non-interactive
Cert not yet due for renewal
```

#### Variations / flags

- Add a systemd timer or cron entry running `renew` twice daily — this mirrors certbot's own recommended cadence and is idempotent (it only renews certs that are actually due).
- For DNS-01 challenges (needed for wildcard certs, or hosts with no public HTTP access), install a provider-specific certbot plugin (e.g. `certbot-dns-route53`, `certbot-dns-cloudflare`) and extend the `dns` branch with the relevant `--dns-*-credentials` flags.
- Combine with `check_cert_expiration.py` (23.6) post-renewal to confirm the new certificate actually landed with the expected expiry.

#### Common pitfalls

- Running `obtain --method standalone` without `sudo`/administrator rights fails to bind port 80 — certbot needs elevated privileges for standalone and most webroot setups.
- Forgetting `--dry-run` while iterating on a broken config can burn into Let's Encrypt's real rate limits, locking you out of issuing for that domain set for up to a week.
- `certbot renew` renews everything certbot manages on the host, not just one domain — there's no per-domain selective renewal flag in this wrapper; use certbot's own `--cert-name` directly if you need that.

### 23.10 Generate SSH Key Pairs

#### Use case

A new server, a new CI runner, or a new teammate needs an SSH key pair. `generate_ssh_keypair.py` creates RSA, ECDSA, or Ed25519 keys in standard OpenSSH format, ready to drop into `authorized_keys` or an SSH agent.

#### Prerequisites

Python 3.8+ and `cryptography`. Cross-platform (Linux/macOS/Windows).

```bash
pip install cryptography
```

#### Cautions

Ed25519 is the modern recommendation for new keys (smaller, fast, resistant to several classes of implementation bugs) unless a specific system requires RSA/ECDSA compatibility. Always protect the private key file's permissions — an SSH client will refuse to use a private key that's readable by other users on Linux/macOS.

#### Script

```python
#!/usr/bin/env python3
"""
generate_ssh_keypair.py - Generate RSA, ECDSA, or Ed25519 SSH key pairs.

Usage:
    python3 generate_ssh_keypair.py --type ed25519 --out-private id_ed25519 \
        --out-public id_ed25519.pub --comment "user@host"

    python3 generate_ssh_keypair.py --type rsa --bits 4096 --out-private id_rsa \
        --out-public id_rsa.pub --passphrase mypassphrase

Output:
    - OpenSSH-formatted private key file (chmod 600 recommended)
    - OpenSSH public key file (id_*.pub), ready for authorized_keys
"""
import argparse
import os
import sys

from cryptography.hazmat.primitives import serialization
from cryptography.hazmat.primitives.asymmetric import ed25519, ec, rsa


def generate_key(key_type, bits, curve):
    if key_type == "rsa":
        return rsa.generate_private_key(public_exponent=65537, key_size=bits)
    elif key_type == "ecdsa":
        curve_map = {"256": ec.SECP256R1(), "384": ec.SECP384R1(), "521": ec.SECP521R1()}
        return ec.generate_private_key(curve_map[curve])
    elif key_type == "ed25519":
        return ed25519.Ed25519PrivateKey.generate()
    raise ValueError(f"Unsupported key type: {key_type}")


def main():
    parser = argparse.ArgumentParser(description="Generate an SSH key pair.")
    parser.add_argument("--type", choices=["rsa", "ecdsa", "ed25519"], default="ed25519")
    parser.add_argument("--bits", type=int, default=4096, help="Key size for RSA (2048/3072/4096)")
    parser.add_argument("--curve", choices=["256", "384", "521"], default="256", help="Curve for ECDSA")
    parser.add_argument("--comment", default="", help="Comment appended to the public key")
    parser.add_argument("--passphrase", default=None, help="Optional passphrase to encrypt the private key")
    parser.add_argument("--out-private", default="id_key", help="Output private key path")
    parser.add_argument("--out-public", default="id_key.pub", help="Output public key path")
    args = parser.parse_args()

    try:
        key = generate_key(args.type, args.bits, args.curve)

        encryption = (
            serialization.BestAvailableEncryption(args.passphrase.encode())
            if args.passphrase else serialization.NoEncryption()
        )

        private_bytes = key.private_bytes(
            encoding=serialization.Encoding.PEM,
            format=serialization.PrivateFormat.OpenSSH,
            encryption_algorithm=encryption,
        )
        public_bytes = key.public_key().public_bytes(
            encoding=serialization.Encoding.OpenSSH,
            format=serialization.PublicFormat.OpenSSH,
        )

        with open(args.out_private, "wb") as f:
            f.write(private_bytes)
        os.chmod(args.out_private, 0o600)

        with open(args.out_public, "wb") as f:
            f.write(public_bytes)
            if args.comment:
                f.write(f" {args.comment}".encode())
            f.write(b"\n")

        print(f"[+] Private key written to: {args.out_private} (permissions set to 600)")
        print(f"[+] Public key written to: {args.out_public}")

    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

`generate_key()` dispatches to the right `cryptography` primitive based on `--type`. The resulting key pair is serialized in OpenSSH format — the private key optionally encrypted with a passphrase, the public key written with an appended comment, matching the format `ssh-keygen` produces.

```bash
# Default: Ed25519 key pair
python3 generate_ssh_keypair.py --out-private id_ed25519 --out-public id_ed25519.pub --comment "alice@laptop"

# RSA-4096 with a passphrase
python3 generate_ssh_keypair.py --type rsa --bits 4096 --passphrase 'Str0ngP@ss' \
    --out-private id_rsa --out-public id_rsa.pub

# ECDSA on the P-384 curve
python3 generate_ssh_keypair.py --type ecdsa --curve 384 --out-private id_ecdsa --out-public id_ecdsa.pub
```

#### Sample output

```text
$ python3 generate_ssh_keypair.py --out-private id_ed25519 --out-public id_ed25519.pub --comment "alice@laptop"
[+] Private key written to: id_ed25519 (permissions set to 600)
[+] Public key written to: id_ed25519.pub

$ cat id_ed25519.pub
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIH3k... alice@laptop
```

#### Variations / flags

- Append the resulting `.pub` file to a remote host's `~/.ssh/authorized_keys` to grant access without a password.
- Use `--passphrase` for any key that will live on a laptop or shared workstation; leave it unset for automation keys stored in a secrets manager with its own access controls.
- Feed the private key into `convert_ssh_key_format.py` (23.11) if a downstream tool needs PEM/PKCS8 or PuTTY's PPK format instead.

#### Common pitfalls

- On Windows, `os.chmod(..., 0o600)` has limited effect (NTFS permissions work differently) — use `icacls` or the OpenSSH-for-Windows equivalent to restrict access instead.
- RSA keys below 2048 bits are rejected by most modern SSH servers by policy; this script defaults to 4096 for RSA specifically because 2048 is increasingly considered a minimum, not a target.
- Forgetting `--comment` still produces a valid key, just without the trailing identifier — useful for automation, less useful for humans auditing `authorized_keys` later.

### 23.11 Convert SSH Key Formats

#### Use case

You generated a key in OpenSSH format but a legacy tool wants PEM/PKCS8, or a Windows user needs a `.ppk` file for PuTTY. `convert_ssh_key_format.py` handles OpenSSH ↔ PEM natively, and shells out to `puttygen` for PPK.

#### Prerequisites

Python 3.8+ and `cryptography` for OpenSSH/PEM conversion. `puttygen` is required only for PPK output.

```bash
pip install cryptography

# puttygen, only needed for --out-form PPK
# Ubuntu/Debian
sudo apt install -y putty-tools
# macOS (Homebrew)
brew install putty
# Windows: install PuTTY (puttygen.exe ships with it) and ensure it's on PATH
```

#### Cautions

Converting a key does not change its underlying security properties — a weak key stays weak in any format. Encrypted output (`--out-passphrase`) is only as strong as the passphrase chosen; an unencrypted private key in any format is equally sensitive regardless of file extension.

#### Script

```python
#!/usr/bin/env python3
"""
convert_ssh_key_format.py - Convert SSH private keys between OpenSSH and PEM
(PKCS8) formats. PPK conversion (PuTTY format) requires the external
`puttygen` tool; this script will call it via subprocess if available.

Usage:
    # OpenSSH -> PEM
    python3 convert_ssh_key_format.py --in id_ed25519 --in-form OpenSSH \
        --out id_ed25519.pem --out-form PEM

    # PEM -> OpenSSH
    python3 convert_ssh_key_format.py --in key.pem --in-form PEM \
        --out id_key --out-form OpenSSH

    # OpenSSH -> PPK (requires puttygen installed)
    python3 convert_ssh_key_format.py --in id_rsa --in-form OpenSSH \
        --out id_rsa.ppk --out-form PPK

Output:
    Converted private key file in the requested format.
"""
import argparse
import shutil
import subprocess
import sys

from cryptography.hazmat.primitives import serialization


def load_key(path, passphrase):
    with open(path, "rb") as f:
        data = f.read()
    if data.strip().startswith(b"-----BEGIN OPENSSH"):
        return serialization.load_ssh_private_key(
            data, password=passphrase.encode() if passphrase else None
        )
    return serialization.load_pem_private_key(
        data, password=passphrase.encode() if passphrase else None
    )


def main():
    parser = argparse.ArgumentParser(description="Convert SSH key formats (OpenSSH / PEM / PPK).")
    parser.add_argument("--in", dest="infile", required=True)
    parser.add_argument("--in-form", choices=["OpenSSH", "PEM"], required=True)
    parser.add_argument("--out", required=True)
    parser.add_argument("--out-form", choices=["OpenSSH", "PEM", "PPK"], required=True)
    parser.add_argument("--passphrase", default=None, help="Passphrase for reading an encrypted key")
    parser.add_argument("--out-passphrase", default=None, help="Passphrase to encrypt the output key")
    args = parser.parse_args()

    try:
        if args.out_form == "PPK":
            if shutil.which("puttygen") is None:
                print(
                    "[!] PPK conversion requires `puttygen` to be installed and on PATH.\n"
                    "    Ubuntu/Debian: sudo apt install putty-tools\n"
                    "    macOS (brew) : brew install putty",
                    file=sys.stderr,
                )
                sys.exit(1)
            cmd = ["puttygen", args.infile, "-o", args.out]
            print(f"[*] Running: {' '.join(cmd)}")
            result = subprocess.run(cmd)
            sys.exit(result.returncode)

        key = load_key(args.infile, args.passphrase)
        encryption = (
            serialization.BestAvailableEncryption(args.out_passphrase.encode())
            if args.out_passphrase else serialization.NoEncryption()
        )

        if args.out_form == "OpenSSH":
            out_bytes = key.private_bytes(
                encoding=serialization.Encoding.PEM,
                format=serialization.PrivateFormat.OpenSSH,
                encryption_algorithm=encryption,
            )
        else:  # PEM
            out_bytes = key.private_bytes(
                encoding=serialization.Encoding.PEM,
                format=serialization.PrivateFormat.PKCS8,
                encryption_algorithm=encryption,
            )

        with open(args.out, "wb") as f:
            f.write(out_bytes)

        print(f"[+] Converted {args.in_form} -> {args.out_form}: {args.out}")

    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

`load_key()` sniffs the file header to decide whether to parse it as OpenSSH or PEM/PKCS8, decrypting with `--passphrase` if provided. PPK output bypasses Python entirely and shells out to `puttygen`, since PPK is a PuTTY-specific format not supported by `cryptography`.

```bash
# OpenSSH private key -> PEM/PKCS8 (e.g. for a Java or legacy tool)
python3 convert_ssh_key_format.py --in id_ed25519 --in-form OpenSSH --out id_ed25519.pem --out-form PEM

# PEM -> OpenSSH, re-encrypting the output with a new passphrase
python3 convert_ssh_key_format.py --in key.pem --in-form PEM --out id_key --out-form OpenSSH \
    --out-passphrase 'NewPass123'

# OpenSSH -> PuTTY PPK for a Windows user
python3 convert_ssh_key_format.py --in id_rsa --in-form OpenSSH --out id_rsa.ppk --out-form PPK
```

#### Sample output

```text
$ python3 convert_ssh_key_format.py --in id_ed25519 --in-form OpenSSH --out id_ed25519.pem --out-form PEM
[+] Converted OpenSSH -> PEM: id_ed25519.pem

$ python3 convert_ssh_key_format.py --in id_rsa --in-form OpenSSH --out id_rsa.ppk --out-form PPK
[*] Running: puttygen id_rsa -o id_rsa.ppk
```

#### Variations / flags

- Use `--passphrase` when the source key is encrypted, and `--out-passphrase` to encrypt the converted output — the two are independent.
- For batch conversions across a directory of keys, loop the script over each file and derive `--out` from the input filename.
- If `puttygen` supports it on your platform, you can also convert PPK back to OpenSSH directly with `puttygen key.ppk -O private-openssh -o id_key` — that path isn't wrapped here but is a natural extension.

#### Common pitfalls

- The `--in-form PPK` case isn't implemented for *reading* PPK files in this script — only PPK as an *output* format via `puttygen`; converting PPK to OpenSSH requires calling `puttygen` directly.
- `puttygen` not being on `PATH` produces a clear early error, but only when targeting `--out-form PPK` — verify the binary is installed before scripting this into automation.
- Ed25519 keys are supported by modern `puttygen`/PuTTY builds but not by very old versions — check your PuTTY version if PPK conversion of an Ed25519 key fails.

### 23.12 Manage GPG Keys

#### Use case

You need a GPG identity for signing releases, encrypting sensitive files, or verifying software provenance — and want to generate, list, export, import, or revoke keys without memorizing raw `gpg` flags.

#### Prerequisites

Python 3.8+, the `python-gnupg` package, and GnuPG itself installed and on `PATH`.

```bash
# GnuPG binary
# Ubuntu/Debian
sudo apt install -y gnupg
# macOS (Homebrew)
brew install gnupg
# Windows: install Gpg4win (https://gpg4win.org)

# Python wrapper
pip install python-gnupg
```

#### Cautions

Key generation with `RSA`/3072+ bits can take noticeably longer on systems with low entropy (common on fresh VMs/containers) — install `rng-tools` or `haveged` on Linux servers if generation hangs. Exporting a secret key (`--secret`) puts your private key material into a plaintext-adjacent `.asc` file; treat it exactly like a raw private key.

#### Script

```python
#!/usr/bin/env python3
"""
manage_gpg_keys.py - Generate, import, export, list, and revoke GPG keys
using the `gpg` executable (via python-gnupg).

Requires: GnuPG installed on the system (gpg binary on PATH).

Usage:
    python3 manage_gpg_keys.py generate --name "Alice" --email alice@example.com \
        --passphrase mypassphrase

    python3 manage_gpg_keys.py list
    python3 manage_gpg_keys.py export --fingerprint ABCD1234 --out alice_pub.asc
    python3 manage_gpg_keys.py export --fingerprint ABCD1234 --secret --out alice_priv.asc --passphrase mypassphrase
    python3 manage_gpg_keys.py import --file alice_pub.asc
    python3 manage_gpg_keys.py revoke --fingerprint ABCD1234 --out revoke.asc

Output:
    Console confirmation of the action; exported keys/revocation certificates
    are written to the requested file paths.
"""
import argparse
import sys

import gnupg


def get_gpg(home_dir=None):
    return gnupg.GPG(gnupghome=home_dir) if home_dir else gnupg.GPG()


def cmd_generate(args, gpg):
    input_data = gpg.gen_key_input(
        name_real=args.name,
        name_email=args.email,
        passphrase=args.passphrase,
        key_type="RSA",
        key_length=args.bits,
    )
    key = gpg.gen_key(input_data)
    if not key.fingerprint:
        print("[!] Key generation failed.", file=sys.stderr)
        sys.exit(1)
    print(f"[+] Generated key with fingerprint: {key.fingerprint}")


def cmd_list(args, gpg):
    keys = gpg.list_keys(secret=args.secret)
    for k in keys:
        uids = ", ".join(k.get("uids", []))
        print(f"{k['fingerprint']}  {uids}")


def cmd_export(args, gpg):
    if args.secret:
        data = gpg.export_keys(args.fingerprint, secret=True, passphrase=args.passphrase)
    else:
        data = gpg.export_keys(args.fingerprint)
    if not data:
        print("[!] Export produced no data - check the fingerprint/passphrase.", file=sys.stderr)
        sys.exit(1)
    with open(args.out, "w") as f:
        f.write(data)
    print(f"[+] Key exported to: {args.out}")


def cmd_import(args, gpg):
    with open(args.file, "r") as f:
        data = f.read()
    result = gpg.import_keys(data)
    print(f"[+] Imported {result.count} key(s). Fingerprints: {result.fingerprints}")


def cmd_revoke(args, gpg):
    result = gpg.gen_revoke(args.fingerprint)
    with open(args.out, "w") as f:
        f.write(str(result))
    print(f"[+] Revocation certificate written to: {args.out}")
    print("[*] Import this certificate into keyrings/keyservers to revoke the key.")


def main():
    parser = argparse.ArgumentParser(description="Manage GPG keys.")
    parser.add_argument("--gnupghome", default=None, help="Custom GNUPGHOME directory")
    sub = parser.add_subparsers(dest="action", required=True)

    p_gen = sub.add_parser("generate")
    p_gen.add_argument("--name", required=True)
    p_gen.add_argument("--email", required=True)
    p_gen.add_argument("--passphrase", required=True)
    p_gen.add_argument("--bits", type=int, default=3072)
    p_gen.set_defaults(func=cmd_generate)

    p_list = sub.add_parser("list")
    p_list.add_argument("--secret", action="store_true", help="List secret keys instead of public")
    p_list.set_defaults(func=cmd_list)

    p_exp = sub.add_parser("export")
    p_exp.add_argument("--fingerprint", required=True)
    p_exp.add_argument("--secret", action="store_true")
    p_exp.add_argument("--passphrase", default=None)
    p_exp.add_argument("--out", required=True)
    p_exp.set_defaults(func=cmd_export)

    p_imp = sub.add_parser("import")
    p_imp.add_argument("--file", required=True)
    p_imp.set_defaults(func=cmd_import)

    p_rev = sub.add_parser("revoke")
    p_rev.add_argument("--fingerprint", required=True)
    p_rev.add_argument("--out", default="revoke.asc")
    p_rev.set_defaults(func=cmd_revoke)

    args = parser.parse_args()
    gpg = get_gpg(args.gnupghome)

    try:
        args.func(args, gpg)
    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

Each subcommand maps to a `python-gnupg` call against the local `gpg` keyring (or a custom one via `--gnupghome`, handy for CI or per-project isolated keyrings). `generate` builds a batch key-generation script under the hood; `export`/`import` move ASCII-armored key blocks; `revoke` produces a standard revocation certificate you can later publish.

```bash

# Export the public key for sharing

# Export the secret key (handle like a private key!)
python3 manage_gpg_keys.py export --fingerprint ABCD1234EF567890ABCD1234EF567890ABCD1234 \
    --secret --passphrase 'KeyPass!23' --out alice_priv.asc

# Import someone else's public key
python3 manage_gpg_keys.py import --file bob_pub.asc

# Pre-generate a revocation certificate (do this right after creating the key!)
```


#### Variations / flags

- `--gnupghome /path/to/keyring` isolates a keyring per project or CI job instead of touching the user's default `~/.gnupg`.
- Generate a revocation certificate (`revoke`) immediately after `generate` and store it somewhere safe offline — it's the only way to invalidate the key later if the passphrase or key material is lost.
- Combine `export` (public) with `distribute_public_keys.py` (23.19) to push the new key to a keyserver right after generation.

#### Common pitfalls

- `gen_key` can silently hang on systems with insufficient entropy (containers, minimal VMs) — install `rng-tools`/`haveged` or use `--bits 2048` temporarily for testing.
- `list_keys(secret=True)` only shows keys whose private material is present in that keyring — don't confuse "not listed" with "doesn't exist," it may simply live in a different `GNUPGHOME`.
- Revocation via `gen_revoke` only creates the certificate locally; it does not automatically propagate to keyservers — you still need to import it into your keyring and publish it (23.19) for others to see the key as revoked.

### 23.13 Sign Files with GPG

#### Use case

You're publishing a release artifact, a script, or a report and want recipients to be able to verify it came from you and wasn't tampered with in transit. `sign_file_gpg.py` produces a detached signature (a separate `.sig` file) or a clearsigned document (`.asc`, content + signature combined).

#### Prerequisites

Python 3.8+, `python-gnupg`, GnuPG installed, and a secret key already present in your keyring (see 23.12 to generate one).

```bash
pip install python-gnupg
# GnuPG binary — see 23.12 for OS-specific install commands
```

#### Cautions

Passing `--passphrase` on the command line exposes it in shell history and process listings on multi-user systems — prefer piping it from an environment variable or a secrets manager in scripted/CI use. Detached signatures (`.sig`) must always travel alongside the original file; a `.sig` file alone verifies nothing.

#### Script

```python
#!/usr/bin/env python3
"""
sign_file_gpg.py - Digitally sign a file using a GPG key (detached or clearsign).

Requires: GnuPG installed on the system and a secret key already available
in your keyring (see manage_gpg_keys.py to generate one).

Usage:
    python3 sign_file_gpg.py --file report.txt --fingerprint ABCD1234 \
        --passphrase mypassphrase --detach --out report.txt.sig

    python3 sign_file_gpg.py --file report.txt --fingerprint ABCD1234 \
        --passphrase mypassphrase --clearsign --out report.txt.asc

Output:
    A detached signature (.sig) or clear-signed file (.asc).
"""
import argparse
import sys

import gnupg


def main():
    parser = argparse.ArgumentParser(description="Sign a file with GPG.")
    parser.add_argument("--file", required=True, help="File to sign")
    parser.add_argument("--fingerprint", required=True, help="Signing key fingerprint or key ID")
    parser.add_argument("--passphrase", required=True)
    parser.add_argument("--gnupghome", default=None)
    parser.add_argument("--out", required=True)
    mode = parser.add_mutually_exclusive_group()
    mode.add_argument("--detach", action="store_true", help="Create a detached signature (default)")
    mode.add_argument("--clearsign", action="store_true", help="Create a clear-signed document")
    args = parser.parse_args()

    gpg = gnupg.GPG(gnupghome=args.gnupghome) if args.gnupghome else gnupg.GPG()

    try:
        with open(args.file, "rb") as f:
            if args.clearsign:
                signed = gpg.sign_file(
                    f, keyid=args.fingerprint, passphrase=args.passphrase, clearsign=True
                )
            else:
                signed = gpg.sign_file(
                    f, keyid=args.fingerprint, passphrase=args.passphrase, detach=True
                )

        if not signed.data:
            print("[!] Signing failed. Check the fingerprint/passphrase and that the key is present.",
                  file=sys.stderr)
            sys.exit(1)

        with open(args.out, "wb") as f:
            f.write(signed.data)

        print(f"[+] Signature written to: {args.out}")

    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

The script opens the target file in binary mode and calls `gpg.sign_file()`, telling GnuPG whether to produce a detached signature (default, or explicit `--detach`) or a clearsigned document (`--clearsign`, which embeds the original text alongside the signature — only sensible for text files).

```bash
# Detached signature — ships as report.txt + report.txt.sig
python3 sign_file_gpg.py --file report.txt --fingerprint ABCD1234EF567890ABCD1234EF567890ABCD1234 \
    --passphrase "$GPG_PASSPHRASE" --detach --out report.txt.sig

# Clearsigned text document — content and signature in one .asc file
python3 sign_file_gpg.py --file notice.txt --fingerprint ABCD1234EF567890ABCD1234EF567890ABCD1234 \
    --passphrase "$GPG_PASSPHRASE" --clearsign --out notice.txt.asc
```

#### Sample output

```text
$ python3 sign_file_gpg.py --file report.txt --fingerprint ABCD1234EF567890ABCD1234EF567890ABCD1234 --passphrase "$GPG_PASSPHRASE" --detach --out report.txt.sig
[+] Signature written to: report.txt.sig
```

#### Variations / flags

- Use `--detach` (the more common choice) for binary release artifacts (tarballs, installers) — distribute the original file and `.sig` together.
- Use `--clearsign` for plain-text announcements, changelogs, or emails where embedding the signature inline is more convenient than a separate file.
- Batch-sign a release directory: `for f in dist/*; do python3 sign_file_gpg.py --file "$f" --fingerprint "$FPR" --passphrase "$GPG_PASSPHRASE" --out "$f.sig"; done`.

#### Common pitfalls

- If the signing key requires a passphrase and none is supplied (or it's wrong), `gpg.sign_file()` returns empty `signed.data` rather than raising — the script catches this and reports a generic failure, so double-check the fingerprint and passphrase first.
- GnuPG's `gpg-agent` may cache or prompt for passphrases outside this script's control on some systems, causing unexpected interactive prompts in "non-interactive" automation — configure `pinentry-mode loopback` in `gpg.conf` for fully unattended signing.
- Signing with an expired or revoked key succeeds locally but produces a signature verifiers will reject — check key validity with `manage_gpg_keys.py list` before relying on automated release signing.

### 23.14 Verify GPG Signatures

#### Use case

You downloaded a release artifact and its `.sig` file, or received a clearsigned document, and need to confirm it's authentic and untampered before trusting it — the essential counterpart to 23.13.

#### Prerequisites

Python 3.8+, `python-gnupg`, GnuPG installed, and the signer's public key already imported into your keyring (see 23.12's `import` command, or 23.19 to fetch one from a keyserver).

```bash
pip install python-gnupg
# GnuPG binary — see 23.12 for OS-specific install commands
```

#### Cautions

Verification only proves the signature is cryptographically valid for a key in your keyring — it does not by itself prove that key belongs to who you think it does. Check the key's trust level and, ideally, verify the fingerprint through an independent channel (the project's website, a separate communication) before trusting a new signer's key.

#### Script

```python
#!/usr/bin/env python3
"""
verify_gpg_signature.py - Verify a GPG signature (detached or clearsigned)
against the original file to confirm authenticity and integrity.

Usage:
    # Detached signature
    python3 verify_gpg_signature.py --file report.txt --sig report.txt.sig

    # Clearsigned file (signature + content combined)
    python3 verify_gpg_signature.py --clearsigned report.txt.asc

Output:
    Prints whether the signature is valid, the signer's key ID, and trust level.
    Exit code 0 = valid signature, 1 = invalid/unverifiable.
"""
import argparse
import sys

import gnupg


def main():
    parser = argparse.ArgumentParser(description="Verify a GPG signature.")
    parser.add_argument("--file", help="Original file (required for detached signatures)")
    parser.add_argument("--sig", help="Detached signature file")
    parser.add_argument("--clearsigned", help="Clear-signed file containing both content and signature")
    parser.add_argument("--gnupghome", default=None)
    args = parser.parse_args()

    if not args.clearsigned and not (args.file and args.sig):
        print("[!] Provide either --clearsigned FILE, or both --file and --sig", file=sys.stderr)
        sys.exit(1)

    gpg = gnupg.GPG(gnupghome=args.gnupghome) if args.gnupghome else gnupg.GPG()

    try:
        if args.clearsigned:
            with open(args.clearsigned, "rb") as f:
                verified = gpg.verify_file(f)
        else:
            with open(args.sig, "rb") as f:
                verified = gpg.verify_file(f, args.file)

        if verified.valid:
            print("[+] Signature is VALID")
            print(f"    Signed by  : {verified.username}")
            print(f"    Key ID     : {verified.key_id}")
            print(f"    Fingerprint: {verified.fingerprint}")
            print(f"    Trust level: {verified.trust_text}")
            sys.exit(0)
        else:
            print("[!] Signature is INVALID or could not be verified.", file=sys.stderr)
            print(f"    Status: {verified.status}", file=sys.stderr)
            sys.exit(1)

    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

For detached signatures, the script opens the `.sig` file and passes the original `--file` path alongside it to `gpg.verify_file()`. For clearsigned documents, it verifies the combined file directly. Either way, GnuPG checks the signature against keys present in the keyring and reports validity, signer identity, and trust level.

```bash
# Verify a detached signature against the original file
python3 verify_gpg_signature.py --file report.txt --sig report.txt.sig

# Verify a clearsigned document
python3 verify_gpg_signature.py --clearsigned notice.txt.asc

# Use the exit code in a release pipeline gate
python3 verify_gpg_signature.py --file app.tar.gz --sig app.tar.gz.sig || { echo "signature check failed"; exit 1; }
```

#### Sample output

```text
$ python3 verify_gpg_signature.py --file report.txt --sig report.txt.sig
[+] Signature is VALID
    Signed by  : Alice Example <alice@example.com>
    Key ID     : EF567890ABCD1234
    Fingerprint: ABCD1234EF567890ABCD1234EF567890ABCD1234
    Trust level: TRUST_ULTIMATE
```

#### Variations / flags

- Wire this into CI/CD as a release gate: fail the pipeline (non-zero exit) if a downloaded dependency or artifact's signature doesn't verify.
- `--gnupghome` lets you verify against an isolated keyring containing only trusted publisher keys, rather than a user's full personal keyring.
- Combine with `distribute_public_keys.py` (23.19) in an onboarding script that fetches a project's signing key from a keyserver before the first verification.

#### Common pitfalls

- "Signature is INVALID" is also what you'll see if the signer's public key simply isn't in your keyring yet — import it first (23.12 `import`, or fetch via keyserver) before assuming tampering.
- Trust level (`TRUST_UNKNOWN`, `TRUST_NEVER`, etc.) is separate from signature validity — a signature can be cryptographically valid while the key itself is untrusted in your web of trust; read both fields, not just "VALID".
- Clearsigned files must not be edited (even whitespace changes) after signing, or verification will fail — always treat the `.asc` file as the canonical, unmodifiable artifact.

### 23.15 Rotate Encryption Keys

#### Use case

Your security policy requires periodic rotation of the symmetric key protecting data at rest. `rotate_encryption_keys.py` generates a new Fernet (AES-based) key, re-encrypts a directory of existing encrypted files under it, and archives the retired key so old backups remain recoverable.

#### Prerequisites

Python 3.8+ and `cryptography`. Cross-platform.

```bash
pip install cryptography
```

#### Cautions

Fernet requires the *entire* file to fit in memory for decryption/re-encryption — this script is not suitable for very large files or high-throughput bulk rotation without modification. Files that fail to decrypt with the current key are skipped (not deleted), but always take a backup of `--data-dir` before running `rotate` in production.

#### Script

```python
#!/usr/bin/env python3
"""
rotate_encryption_keys.py - Rotate a symmetric encryption key (Fernet/AES),
re-encrypting existing data with a freshly generated key and archiving the
old key for a defined grace period. Useful for periodic key-rotation policies
on encrypted data at rest.

Usage:
    # Generate the very first key
    python3 rotate_encryption_keys.py init --keystore keys/

    # Rotate: generates a new key, re-encrypts files, archives the old key
    python3 rotate_encryption_keys.py rotate --keystore keys/ --data-dir encrypted_data/

Output:
    - keys/current.key                   the active key
    - keys/archive/<timestamp>.key        previous key(s), retained for recovery
    - Files in --data-dir re-encrypted under the new key
"""
import argparse
import datetime
import os
import sys

from cryptography.fernet import Fernet


def init_keystore(keystore_dir):
    os.makedirs(keystore_dir, exist_ok=True)
    os.makedirs(os.path.join(keystore_dir, "archive"), exist_ok=True)
    key_path = os.path.join(keystore_dir, "current.key")
    if os.path.exists(key_path):
        print(f"[!] {key_path} already exists. Refusing to overwrite.", file=sys.stderr)
        sys.exit(1)
    key = Fernet.generate_key()
    with open(key_path, "wb") as f:
        f.write(key)
    os.chmod(key_path, 0o600)
    print(f"[+] New key store initialized: {key_path}")


def rotate(keystore_dir, data_dir):
    current_path = os.path.join(keystore_dir, "current.key")
    if not os.path.exists(current_path):
        print("[!] No current key found. Run `init` first.", file=sys.stderr)
        sys.exit(1)

    with open(current_path, "rb") as f:
        old_key = f.read()
    old_fernet = Fernet(old_key)

    new_key = Fernet.generate_key()
    new_fernet = Fernet(new_key)

    # Re-encrypt data files under the new key, if a data directory was given
    if data_dir and os.path.isdir(data_dir):
        for name in os.listdir(data_dir):
            path = os.path.join(data_dir, name)
            if not os.path.isfile(path):
                continue
            with open(path, "rb") as f:
                token = f.read()
            try:
                plaintext = old_fernet.decrypt(token)
            except Exception:
                print(f"[!] Skipping {name}: could not decrypt with current key.", file=sys.stderr)
                continue
            new_token = new_fernet.encrypt(plaintext)
            with open(path, "wb") as f:
                f.write(new_token)
            print(f"[+] Re-encrypted: {path}")

    # Archive the old key, then promote the new key
    timestamp = datetime.datetime.utcnow().strftime("%Y%m%dT%H%M%SZ")
    archive_path = os.path.join(keystore_dir, "archive", f"{timestamp}.key")
    with open(archive_path, "wb") as f:
        f.write(old_key)
    os.chmod(archive_path, 0o600)

    with open(current_path, "wb") as f:
        f.write(new_key)
    os.chmod(current_path, 0o600)

    print(f"[+] Old key archived to: {archive_path}")
    print(f"[+] New active key written to: {current_path}")


def main():
    parser = argparse.ArgumentParser(description="Rotate symmetric encryption keys.")
    sub = parser.add_subparsers(dest="action", required=True)

    p_init = sub.add_parser("init", help="Initialize a new key store")
    p_init.add_argument("--keystore", required=True)

    p_rot = sub.add_parser("rotate", help="Rotate the active key")
    p_rot.add_argument("--keystore", required=True)
    p_rot.add_argument("--data-dir", default=None, help="Directory of Fernet-encrypted files to re-encrypt")

    args = parser.parse_args()

    try:
        if args.action == "init":
            init_keystore(args.keystore)
        else:
            rotate(args.keystore, args.data_dir)
    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

`init` creates a fresh `current.key` plus an `archive/` subdirectory. `rotate` loads the current key, generates a new one, walks `--data-dir` decrypting each file with the old key and re-encrypting with the new one in place, then archives the old key under a UTC timestamp before promoting the new key to `current.key`.

```bash
# First-time setup
python3 rotate_encryption_keys.py init --keystore keys/

# Rotate the key and re-encrypt everything under encrypted_data/
python3 rotate_encryption_keys.py rotate --keystore keys/ --data-dir encrypted_data/

# Rotate the key without touching any data (e.g. no files encrypted yet)
python3 rotate_encryption_keys.py rotate --keystore keys/
```

#### Sample output

```text
$ python3 rotate_encryption_keys.py init --keystore keys/
[+] New key store initialized: keys/current.key

$ python3 rotate_encryption_keys.py rotate --keystore keys/ --data-dir encrypted_data/
[+] Re-encrypted: encrypted_data/customers.enc
[+] Re-encrypted: encrypted_data/invoices.enc
[+] Old key archived to: keys/archive/20260718T095213Z.key
[+] New active key written to: keys/current.key
```

#### Variations / flags

- Schedule `rotate` via cron/systemd timer on whatever cadence your data-protection policy requires (monthly, quarterly).
- Retain archived keys per your compliance/retention policy — old backups encrypted under a retired key can only be restored using the matching archived key.
- For large datasets, adapt the re-encryption loop to stream files in chunks rather than reading them fully into memory (Fernet's single-token model needs modification for true streaming; consider `cryptography`'s lower-level AEAD primitives for very large files).

#### Common pitfalls

- Running `rotate` against a `--data-dir` containing files not encrypted with the *current* key (e.g., already rotated once, or plaintext) causes those files to be skipped with a warning — verify the directory's contents match expectations first.
- Losing an archived key means permanently losing access to any data still encrypted under it — back up the `archive/` directory with the same rigor as the active key.
- This script re-encrypts files in place with no atomic write/rollback — a crash mid-rotation could leave some files re-encrypted and others not; consider writing to temp files and renaming for production-grade durability.

### 23.16 Secure Private Key Storage

#### Use case

You have private keys (SSH, TLS, or otherwise) that need to live somewhere safer than a loose file on disk — either a password-encrypted local keystore for small teams/single machines, or HashiCorp Vault for centralized, access-controlled secret storage.

#### Prerequisites

Python 3.8+ and `cryptography` for the local backend. `hvac` and a running Vault server for the vault backend.

```bash
pip install cryptography

# Only needed for the vault backend
pip install hvac
```

#### Cautions

Read the script's own warning carefully: the `SALT` constant is hardcoded for demonstration purposes. In production, generate a random per-keystore salt, store it alongside (not inside) the encrypted file, and load it at runtime — a shared, hardcoded salt weakens the password-based key derivation against precomputed attacks. Also, `--password` on the command line is visible in shell history; prefer prompting or environment variables in real deployments.

#### Script

```python
#!/usr/bin/env python3
"""
secure_key_storage.py - Store and retrieve private keys securely, either in a
local encrypted keystore (password-derived Fernet key) or in a HashiCorp
Vault instance (if the `hvac` library and a running Vault server are
available).

Usage:
    # Local encrypted keystore
    python3 secure_key_storage.py store --backend local --keystore store.enc \
        --key-file id_rsa --password mypassword --name my-server-key

    python3 secure_key_storage.py retrieve --backend local --keystore store.enc \
        --password mypassword --name my-server-key --out id_rsa_recovered

    # HashiCorp Vault (requires VAULT_ADDR / VAULT_TOKEN env vars or --vault-* flags)
    python3 secure_key_storage.py store --backend vault --name my-server-key \
        --key-file id_rsa --vault-addr https://vault.internal:8200 --vault-token s.xxxx

    python3 secure_key_storage.py retrieve --backend vault --name my-server-key \
        --vault-addr https://vault.internal:8200 --vault-token s.xxxx --out id_rsa_recovered

Output:
    - local backend: a single encrypted file (store.enc) holding one or more named keys
    - vault backend: secrets written under the `secret/` KV path in Vault

NOTE: The static salt below is for demonstration only. In production, generate
a random per-keystore salt, store it alongside (not inside) the encrypted
file, and load it at runtime instead of hardcoding it in source.
"""
import argparse
import base64
import json
import os
import sys

from cryptography.fernet import Fernet
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC

SALT = b"cert-key-mgmt-static-salt-change-me"
ITERATIONS = 390000


def derive_fernet_key(password):
    kdf = PBKDF2HMAC(algorithm=hashes.SHA256(), length=32, salt=SALT, iterations=ITERATIONS)
    return base64.urlsafe_b64encode(kdf.derive(password.encode()))


def load_store(path, password):
    fkey = derive_fernet_key(password)
    fernet = Fernet(fkey)
    if not os.path.exists(path):
        return fernet, {}
    with open(path, "rb") as f:
        token = f.read()
    data = json.loads(fernet.decrypt(token).decode())
    return fernet, data


def save_store(path, fernet, data):
    token = fernet.encrypt(json.dumps(data).encode())
    with open(path, "wb") as f:
        f.write(token)
    os.chmod(path, 0o600)


def store_local(args):
    with open(args.key_file, "r") as f:
        key_contents = f.read()
    fernet, data = load_store(args.keystore, args.password)
    data[args.name] = key_contents
    save_store(args.keystore, fernet, data)
    print(f"[+] Stored '{args.name}' in {args.keystore}")


def retrieve_local(args):
    fernet, data = load_store(args.keystore, args.password)
    if args.name not in data:
        print(f"[!] No entry named '{args.name}' found.", file=sys.stderr)
        sys.exit(1)
    with open(args.out, "w") as f:
        f.write(data[args.name])
    os.chmod(args.out, 0o600)
    print(f"[+] Retrieved '{args.name}' -> {args.out}")


def get_vault_client(args):
    import hvac
    addr = args.vault_addr or os.environ.get("VAULT_ADDR")
    token = args.vault_token or os.environ.get("VAULT_TOKEN")
    if not addr or not token:
        print("[!] Vault address/token required (via flags or VAULT_ADDR/VAULT_TOKEN).", file=sys.stderr)
        sys.exit(1)
    client = hvac.Client(url=addr, token=token)
    if not client.is_authenticated():
        print("[!] Vault authentication failed.", file=sys.stderr)
        sys.exit(1)
    return client


def store_vault(args):
    with open(args.key_file, "r") as f:
        key_contents = f.read()
    client = get_vault_client(args)
    client.secrets.kv.v2.create_or_update_secret(
        path=args.name, secret={"private_key": key_contents}
    )
    print(f"[+] Stored '{args.name}' in Vault under secret/{args.name}")


def retrieve_vault(args):
    client = get_vault_client(args)
    secret = client.secrets.kv.v2.read_secret_version(path=args.name)
    key_contents = secret["data"]["data"]["private_key"]
    with open(args.out, "w") as f:
        f.write(key_contents)
    os.chmod(args.out, 0o600)
    print(f"[+] Retrieved '{args.name}' from Vault -> {args.out}")


def main():
    parser = argparse.ArgumentParser(description="Securely store/retrieve private keys.")
    sub = parser.add_subparsers(dest="action", required=True)

    for action in ("store", "retrieve"):
        p = sub.add_parser(action)
        p.add_argument("--backend", choices=["local", "vault"], required=True)
        p.add_argument("--name", required=True, help="Logical name/path for the key")
        p.add_argument("--keystore", default="store.enc", help="Local keystore file (local backend)")
        p.add_argument("--password", default=None, help="Password for local keystore encryption")
        p.add_argument("--key-file", default=None, help="Private key file to store (store action)")
        p.add_argument("--out", default=None, help="Output path to write retrieved key (retrieve action)")
        p.add_argument("--vault-addr", default=None)
        p.add_argument("--vault-token", default=None)

    args = parser.parse_args()

    try:
        if args.backend == "local" and not args.password:
            print("[!] --password is required for the local backend.", file=sys.stderr)
            sys.exit(1)

        if args.action == "store":
            if not args.key_file:
                print("[!] --key-file is required for store.", file=sys.stderr)
                sys.exit(1)
            store_local(args) if args.backend == "local" else store_vault(args)
        else:
            if not args.out:
                print("[!] --out is required for retrieve.", file=sys.stderr)
                sys.exit(1)
            retrieve_local(args) if args.backend == "local" else retrieve_vault(args)

    except ImportError:
        print("[!] The 'hvac' package is required for the vault backend. "
              "Install it with: pip install hvac", file=sys.stderr)
        sys.exit(1)
    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

The local backend derives a Fernet key from your password via PBKDF2-HMAC-SHA256 (390,000 iterations), then encrypts a JSON dictionary of `{name: key_contents}` entries into a single `store.enc` file. The vault backend instead authenticates to a running Vault server and reads/writes secrets under its KV v2 engine, with Vault itself handling encryption at rest.

```bash
# Store an SSH private key in a local encrypted keystore
python3 secure_key_storage.py store --backend local --keystore store.enc \
    --key-file id_rsa --password "$KEYSTORE_PASSWORD" --name my-server-key

# Retrieve it later
python3 secure_key_storage.py retrieve --backend local --keystore store.enc \
    --password "$KEYSTORE_PASSWORD" --name my-server-key --out id_rsa_recovered

# Store in HashiCorp Vault instead
python3 secure_key_storage.py store --backend vault --name my-server-key \
    --key-file id_rsa --vault-addr https://vault.internal:8200 --vault-token "$VAULT_TOKEN"

# Retrieve from Vault
python3 secure_key_storage.py retrieve --backend vault --name my-server-key \
    --vault-addr https://vault.internal:8200 --vault-token "$VAULT_TOKEN" --out id_rsa_recovered
```

#### Sample output

```text
$ python3 secure_key_storage.py store --backend local --keystore store.enc --key-file id_rsa --password "$KEYSTORE_PASSWORD" --name my-server-key
[+] Stored 'my-server-key' in store.enc

$ python3 secure_key_storage.py retrieve --backend local --keystore store.enc --password "$KEYSTORE_PASSWORD" --name my-server-key --out id_rsa_recovered
```

#### Variations / flags

- For team use, prefer the `vault` backend — it gives per-user access control, audit logging, and secret versioning that a shared local `store.enc` file cannot.
- Store multiple keys under one local keystore file by calling `store` repeatedly with different `--name` values; they accumulate in the same encrypted JSON blob.
- Set `VAULT_ADDR`/`VAULT_TOKEN` as environment variables instead of CLI flags to avoid leaking the token into shell history.

#### Common pitfalls

- **The hardcoded `SALT` is a real production risk** — before using the local backend for anything sensitive, change it to a randomly generated, per-keystore salt stored separately from the encrypted file (e.g., in a sidecar file or a KMS), as the script's own docstring warns.
- Wrong password on `retrieve` fails at `fernet.decrypt()` with an `InvalidToken` exception, not a friendly "wrong password" message — the generic error handler will still report it, just without that specific framing.
- The Vault backend assumes a KV v2 secrets engine is mounted at the default path; if your Vault deployment uses a custom mount point or KV v1, the `client.secrets.kv.v2` calls need adjusting accordingly.

### 23.17 Revoke Compromised Certificates

#### Use case

A private key has been compromised, an employee has left, or a certificate was mis-issued — you need to formally revoke it so relying parties checking your internal CA's CRL will reject it, even though it hasn't expired yet.

#### Prerequisites

Python 3.8+, `cryptography`, and your internal CA's certificate + private key (from `build_internal_pki.py`, 23.18).

```bash
pip install cryptography
```

#### Cautions

Revocation via CRL only protects relying parties that actually fetch and check the CRL — if your services don't perform CRL (or OCSP) checking, revoking a certificate here does not stop it from being accepted elsewhere. Make sure `--next-update-days` is short enough that a compromised cert doesn't stay "live" in stale caches longer than your risk tolerance allows.

#### Script

```python
#!/usr/bin/env python3
"""
revoke_certificate.py - Maintain a Certificate Revocation List (CRL) for an
internal CA: add a certificate's serial number to the revoked set and issue
an updated, signed CRL.

Usage:
    python3 revoke_certificate.py --ca-cert ca.pem --ca-key ca.key \
        --revoke-cert compromised_cert.pem --crl-in crl.pem --crl-out crl.pem \
        --reason keyCompromise --next-update-days 7

    # First CRL for a CA (no existing --crl-in)
    python3 revoke_certificate.py --ca-cert ca.pem --ca-key ca.key \
        --revoke-serial 123456789 --crl-out crl.pem

Output:
    An updated PEM-encoded CRL listing all previously revoked certificates
    plus the newly revoked one.
"""
import argparse
import datetime
import sys

from cryptography import x509
from cryptography.hazmat.primitives import hashes, serialization

REASON_MAP = {
    "unspecified": x509.ReasonFlags.unspecified,
    "keyCompromise": x509.ReasonFlags.key_compromise,
    "caCompromise": x509.ReasonFlags.ca_compromise,
    "affiliationChanged": x509.ReasonFlags.affiliation_changed,
    "superseded": x509.ReasonFlags.superseded,
    "cessationOfOperation": x509.ReasonFlags.cessation_of_operation,
    "certificateHold": x509.ReasonFlags.certificate_hold,
}


def main():
    parser = argparse.ArgumentParser(description="Revoke a certificate and issue an updated CRL.")
    parser.add_argument("--ca-cert", required=True)
    parser.add_argument("--ca-key", required=True)
    parser.add_argument("--ca-key-password", default=None)
    parser.add_argument("--revoke-cert", default=None, help="Certificate file to revoke")
    parser.add_argument("--revoke-serial", type=int, default=None, help="Serial number to revoke directly")
    parser.add_argument("--reason", choices=list(REASON_MAP.keys()), default="unspecified")
    parser.add_argument("--crl-in", default=None, help="Existing CRL to extend (optional)")
    parser.add_argument("--crl-out", required=True)
    parser.add_argument("--next-update-days", type=int, default=7)
    args = parser.parse_args()

    if not args.revoke_cert and not args.revoke_serial:
        print("[!] Provide either --revoke-cert or --revoke-serial", file=sys.stderr)
        sys.exit(1)

    try:
        with open(args.ca_cert, "rb") as f:
            ca_cert = x509.load_pem_x509_certificate(f.read())
        with open(args.ca_key, "rb") as f:
            ca_key = serialization.load_pem_private_key(
                f.read(), password=args.ca_key_password.encode() if args.ca_key_password else None
            )

        if args.revoke_cert:
            with open(args.revoke_cert, "rb") as f:
                target_cert = x509.load_pem_x509_certificate(f.read())
            serial = target_cert.serial_number
        else:
            serial = args.revoke_serial

        now = datetime.datetime.utcnow()
        builder = x509.CertificateRevocationListBuilder().issuer_name(ca_cert.subject)
        builder = builder.last_update(now).next_update(now + datetime.timedelta(days=args.next_update_days))

        # Re-add existing revoked entries
        if args.crl_in:
            with open(args.crl_in, "rb") as f:
                existing_crl = x509.load_pem_x509_crl(f.read())
            for entry in existing_crl:
                builder = builder.add_revoked_certificate(entry)

        revoked = (
            x509.RevokedCertificateBuilder()
            .serial_number(serial)
            .revocation_date(now)
            .add_extension(
                x509.CRLReason(REASON_MAP[args.reason]), critical=False
            )
            .build()
        )
        builder = builder.add_revoked_certificate(revoked)

        crl = builder.sign(private_key=ca_key, algorithm=hashes.SHA256())

        with open(args.crl_out, "wb") as f:
            f.write(crl.public_bytes(serialization.Encoding.PEM))

        print(f"[+] Certificate with serial {serial} added to CRL")
        print(f"[+] Updated CRL written to: {args.crl_out}")
        print(f"[+] Next update due: {now + datetime.timedelta(days=args.next_update_days)}")

    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

You identify the certificate to revoke either by file (`--revoke-cert`, its serial number is extracted automatically) or directly by `--revoke-serial`. If `--crl-in` is given, every existing revoked entry is copied forward into the new CRL builder so history isn't lost; the new entry is then added with its revocation reason, and the whole list is re-signed by the CA key.

```bash
# Revoke a certificate, extending an existing CRL
python3 revoke_certificate.py --ca-cert pki/intermediate_ca.pem --ca-key pki/intermediate_ca.key \
    --revoke-cert compromised_cert.pem --crl-in crl.pem --crl-out crl.pem \
    --reason keyCompromise --next-update-days 7

# First-ever CRL for this CA, revoking by serial number directly
python3 revoke_certificate.py --ca-cert pki/intermediate_ca.pem --ca-key pki/intermediate_ca.key \
    --revoke-serial 291847562930481756928374651029384756 --crl-out crl.pem
```

#### Sample output

```text
$ python3 revoke_certificate.py --ca-cert pki/intermediate_ca.pem --ca-key pki/intermediate_ca.key --revoke-cert compromised_cert.pem --crl-out crl.pem --reason keyCompromise
[+] Certificate with serial 291847562930481756928374651029384756 added to CRL
[+] Updated CRL written to: crl.pem
[+] Next update due: 2026-07-25 09:58:44.117532
```

#### Variations / flags

- Choose `--reason` deliberately — `keyCompromise` and `caCompromise` should trigger the most urgent downstream response, while `superseded` and `cessationOfOperation` are routine housekeeping.
- Publish `crl.pem` to a well-known, publicly reachable URL referenced by your certificates' CRL Distribution Point extension so relying parties can actually fetch it.
- Shorten `--next-update-days` (e.g., to 1) immediately after a high-severity `keyCompromise` event, then lengthen it back to your normal cadence once the incident settles.

#### Common pitfalls

- Omitting `--crl-in` on every run silently drops all previously revoked certificates from the new CRL — always pass the prior CRL back in as `--crl-in` unless this is genuinely the first revocation ever issued.
- A CRL with a `next_update` in the past is considered stale by strict validators; make sure you regenerate and republish before that deadline, ideally via a scheduled job.
- This script builds CRLs but does not publish them anywhere — distributing the updated `crl.pem` to wherever your CRL Distribution Point URL serves it from is a separate manual or scripted step.

### 23.18 Build Internal PKI Infrastructure

#### Use case

You're standing up internal TLS for a Kubernetes cluster, an internal API mesh, or corporate infrastructure that shouldn't rely on public CAs. `build_internal_pki.py` bootstraps a standard two-tier PKI — a Root CA and an Intermediate CA signed by it — ready to issue leaf certificates via `sign_certificate.py` (23.3).

#### Prerequisites

Python 3.8+ and `cryptography`. Cross-platform.

```bash
pip install cryptography
```

#### Cautions

The Root CA private key is the single most sensitive artifact this chapter produces — anyone holding it can mint trust for your entire internal PKI. Generate it on an air-gapped or otherwise tightly controlled machine if possible, move `root_ca.key` to offline/cold storage immediately after building the intermediate, and use only the Intermediate CA key for day-to-day signing (23.3).

#### Script

```python
#!/usr/bin/env python3
"""
build_internal_pki.py - Build a two-tier internal PKI: a self-signed Root CA
and an Intermediate CA signed by the Root, ready to issue leaf certificates
for enterprise/internal infrastructure (use sign_certificate.py with the
intermediate to issue end-entity certs).

Usage:
    python3 build_internal_pki.py --org "My Company" --country US \
        --root-cn "My Company Root CA" --intermediate-cn "My Company Intermediate CA" \
        --root-days 7300 --intermediate-days 3650 --out-dir pki/

Output (under --out-dir):
    root_ca.key, root_ca.pem                 - Root CA private key + self-signed cert
    intermediate_ca.key, intermediate_ca.pem  - Intermediate CA key + cert (signed by root)
    chain.pem                                 - Intermediate + Root chain, for distribution
"""
import argparse
import datetime
import os
import sys

from cryptography import x509
from cryptography.hazmat.primitives import hashes, serialization
from cryptography.hazmat.primitives.asymmetric import rsa
from cryptography.x509.oid import NameOID


def build_name(cn, org, country):
    return x509.Name([
        x509.NameAttribute(NameOID.COUNTRY_NAME, country),
        x509.NameAttribute(NameOID.ORGANIZATION_NAME, org),
        x509.NameAttribute(NameOID.COMMON_NAME, cn),
    ])


def make_ca_cert(subject_name, issuer_name, public_key, signing_key, days, path_length):
    now = datetime.datetime.utcnow()
    builder = (
        x509.CertificateBuilder()
        .subject_name(subject_name)
        .issuer_name(issuer_name)
        .public_key(public_key)
        .serial_number(x509.random_serial_number())
        .not_valid_before(now)
        .not_valid_after(now + datetime.timedelta(days=days))
        .add_extension(x509.BasicConstraints(ca=True, path_length=path_length), critical=True)
        .add_extension(
            x509.KeyUsage(
                digital_signature=True, content_commitment=False, key_encipherment=False,
                data_encipherment=False, key_agreement=False, key_cert_sign=True,
                crl_sign=True, encipher_only=False, decipher_only=False,
            ),
            critical=True,
        )
        .add_extension(
            x509.SubjectKeyIdentifier.from_public_key(public_key), critical=False
        )
    )
    return builder.sign(signing_key, hashes.SHA256())


def main():
    parser = argparse.ArgumentParser(description="Build a two-tier internal PKI (Root + Intermediate CA).")
    parser.add_argument("--org", default="Internal Org")
    parser.add_argument("--country", default="US")
    parser.add_argument("--root-cn", default="Internal Root CA")
    parser.add_argument("--intermediate-cn", default="Internal Intermediate CA")
    parser.add_argument("--root-days", type=int, default=7300, help="Root CA validity (default ~20y)")
    parser.add_argument("--intermediate-days", type=int, default=3650, help="Intermediate validity (~10y)")
    parser.add_argument("--keysize", type=int, default=4096, choices=[2048, 3072, 4096])
    parser.add_argument("--out-dir", default="pki")
    args = parser.parse_args()

    try:
        os.makedirs(args.out_dir, exist_ok=True)

        print("[*] Generating Root CA key and self-signed certificate ...")
        root_key = rsa.generate_private_key(public_exponent=65537, key_size=args.keysize)
        root_name = build_name(args.root_cn, args.org, args.country)
        root_cert = make_ca_cert(root_name, root_name, root_key.public_key(), root_key,
                                  args.root_days, path_length=1)

        print("[*] Generating Intermediate CA key and certificate signed by Root ...")
        inter_key = rsa.generate_private_key(public_exponent=65537, key_size=args.keysize)
        inter_name = build_name(args.intermediate_cn, args.org, args.country)
        inter_cert = make_ca_cert(inter_name, root_name, inter_key.public_key(), root_key,
                                   args.intermediate_days, path_length=0)

        root_key_path = os.path.join(args.out_dir, "root_ca.key")
        root_cert_path = os.path.join(args.out_dir, "root_ca.pem")
        inter_key_path = os.path.join(args.out_dir, "intermediate_ca.key")
        inter_cert_path = os.path.join(args.out_dir, "intermediate_ca.pem")
        chain_path = os.path.join(args.out_dir, "chain.pem")

        with open(root_key_path, "wb") as f:
            f.write(root_key.private_bytes(
                serialization.Encoding.PEM, serialization.PrivateFormat.TraditionalOpenSSL,
                serialization.NoEncryption(),
            ))
        with open(root_cert_path, "wb") as f:
            f.write(root_cert.public_bytes(serialization.Encoding.PEM))
        with open(inter_key_path, "wb") as f:
            f.write(inter_key.private_bytes(
                serialization.Encoding.PEM, serialization.PrivateFormat.TraditionalOpenSSL,
                serialization.NoEncryption(),
            ))
        with open(inter_cert_path, "wb") as f:
            f.write(inter_cert.public_bytes(serialization.Encoding.PEM))
        with open(chain_path, "wb") as f:
            f.write(inter_cert.public_bytes(serialization.Encoding.PEM))
            f.write(root_cert.public_bytes(serialization.Encoding.PEM))

        for p in (root_key_path, inter_key_path):
            os.chmod(p, 0o600)

        print(f"[+] Root CA key/cert     : {root_key_path}, {root_cert_path}")
        print(f"[+] Intermediate key/cert: {inter_key_path}, {inter_cert_path}")
        print(f"[+] Full chain           : {chain_path}")
        print("[*] Use sign_certificate.py with intermediate_ca.pem/.key to issue leaf certificates.")
        print("[!] Protect root_ca.key offline; it should rarely, if ever, be used directly.")

    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

The script generates a Root CA key and self-signs it (`path_length=1`, meaning it can sign at most one level of subordinate CA below it), then generates an Intermediate CA key and has the Root sign it (`path_length=0`, meaning the Intermediate can only sign leaf/end-entity certs, not further CAs). Both CA certs get `KeyUsage` extensions restricting them to certificate/CRL signing. A `chain.pem` combining Intermediate + Root is written for distributing to clients that need the full trust path.

```bash
# Build a PKI with sensible defaults (20-year root, 10-year intermediate, 4096-bit keys)
python3 build_internal_pki.py --org "My Company" --country US \
    --root-cn "My Company Root CA" --intermediate-cn "My Company Intermediate CA" \
    --out-dir pki/

# Shorter-lived hierarchy for a lab/test environment
python3 build_internal_pki.py --org "Test Lab" --root-days 730 --intermediate-days 365 \
    --keysize 2048 --out-dir test_pki/
```

#### Sample output

```text
$ python3 build_internal_pki.py --org "My Company" --country US --out-dir pki/
[*] Generating Root CA key and self-signed certificate ...
[*] Generating Intermediate CA key and certificate signed by Root ...
[+] Root CA key/cert     : pki/root_ca.key, pki/root_ca.pem
[+] Intermediate key/cert: pki/intermediate_ca.key, pki/intermediate_ca.pem
[+] Full chain           : pki/chain.pem
[*] Use sign_certificate.py with intermediate_ca.pem/.key to issue leaf certificates.
[!] Protect root_ca.key offline; it should rarely, if ever, be used directly.
```

#### Variations / flags

- Distribute `root_ca.pem` (public cert only, never the key) to every internal host/OS trust store so your Intermediate-signed leaf certs are trusted org-wide.
- After building, move `root_ca.key` to offline storage (e.g., an encrypted USB drive in a safe, or an HSM) and delete it from the machine that ran this script.
- Use `--keysize 2048` only for throwaway lab PKIs; production internal CAs should use 4096-bit RSA (or migrate this script to ECDSA for smaller, faster keys with equivalent security).

#### Common pitfalls

- Treating the Intermediate CA key with the same casual handling as a leaf cert's key is a common mistake — it can sign trusted certificates for your entire org and deserves strong access controls (restricted filesystem permissions at minimum, ideally an HSM or Vault-backed signing service).
- Setting `--root-days` shorter than `--intermediate-days` produces a hierarchy where the Intermediate outlives the Root, which is invalid — keep the Root's validity comfortably longer than the Intermediate's.
- This script has no built-in CRL or OCSP endpoint configuration — pair it with `revoke_certificate.py` (23.17) and publish the resulting CRL somewhere your infrastructure actually checks, or revocation will be silently ineffective.

### 23.19 Distribute Public Keys Securely

#### Use case

You've generated an SSH key, a GPG key, or a certificate, and now people or systems need the *public* half in order to trust or communicate with you. `distribute_public_keys.py` covers three common distribution paths: a shared trusted directory, an internal HTTP endpoint, or a public GPG keyserver.

#### Prerequisites

Python 3.8+, `requests` for the HTTP mode, and `python-gnupg` + GnuPG for the keyserver mode. The local mode needs only the standard library.

```bash
pip install requests python-gnupg
# GnuPG binary required for gpg-keyserver mode — see 23.12 for OS-specific install commands
```

#### Cautions

Public keys are, by definition, not secret — but *which* key you trust absolutely matters. Always verify a fingerprint out-of-band (a phone call, a separate signed channel, an in-person exchange) before trusting a key you didn't generate yourself, regardless of which distribution method delivered it.

#### Script

```python
#!/usr/bin/env python3
"""
distribute_public_keys.py - Publish a public key (SSH, GPG, or certificate)
to a shared distribution point: a local trusted directory, an HTTP(S)
endpoint, or (for GPG keys) a public keyserver.

Usage:
    # Copy a public key into a shared/trusted directory
    python3 distribute_public_keys.py local --file id_ed25519.pub \
        --dest-dir /shared/trusted_keys/

    # Upload a public key to an internal HTTP endpoint
    python3 distribute_public_keys.py http --file cert.pem \
        --url https://keys.internal.example.com/upload --name web-server-cert

    # Publish a GPG public key to a public keyserver
    python3 distribute_public_keys.py gpg-keyserver --fingerprint ABCD1234EF \
        --keyserver hkps://keys.openpgp.org

Output:
    Confirmation message describing where the key was published.
"""
import argparse
import hashlib
import os
import shutil
import sys

import requests


def dist_local(args):
    os.makedirs(args.dest_dir, exist_ok=True)
    dest_path = os.path.join(args.dest_dir, os.path.basename(args.file))
    shutil.copy2(args.file, dest_path)
    with open(args.file, "rb") as f:
        digest = hashlib.sha256(f.read()).hexdigest()
    print(f"[+] Key copied to: {dest_path}")
    print(f"[+] SHA256 (verify out-of-band before trusting): {digest}")


def dist_http(args):
    with open(args.file, "rb") as f:
        files = {"file": (args.name or os.path.basename(args.file), f)}
        response = requests.post(args.url, files=files, timeout=30)
    response.raise_for_status()
    print(f"[+] Uploaded to {args.url} (HTTP {response.status_code})")


def dist_gpg_keyserver(args):
    import gnupg
    gpg = gnupg.GPG()
    result = gpg.send_keys(args.keyserver, args.fingerprint)
    print(f"[+] Sent key {args.fingerprint} to {args.keyserver}")
    print(f"[*] Result: {getattr(result, 'stderr', result)}")


def main():
    parser = argparse.ArgumentParser(description="Distribute public keys/certificates securely.")
    sub = parser.add_subparsers(dest="mode", required=True)

    p_local = sub.add_parser("local", help="Copy to a local/shared trusted directory")
    p_local.add_argument("--file", required=True)
    p_local.add_argument("--dest-dir", required=True)
    p_local.set_defaults(func=dist_local)

    p_http = sub.add_parser("http", help="Upload to an HTTP(S) endpoint")
    p_http.add_argument("--file", required=True)
    p_http.add_argument("--url", required=True)
    p_http.add_argument("--name", default=None)
    p_http.set_defaults(func=dist_http)

    p_gpg = sub.add_parser("gpg-keyserver", help="Publish a GPG key to a public keyserver")
    p_gpg.add_argument("--fingerprint", required=True)
    p_gpg.add_argument("--keyserver", default="hkps://keys.openpgp.org")
    p_gpg.set_defaults(func=dist_gpg_keyserver)

    args = parser.parse_args()

    try:
        args.func(args)
    except Exception as e:
        print(f"[!] Error: {e}", file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How it works and how to run

Three subcommands, three distribution channels. `local` copies the key file into a shared directory and prints a SHA-256 hash so recipients can verify integrity out-of-band. `http` POSTs the file to an internal endpoint (e.g., an internal key-directory service). `gpg-keyserver` uses the local `gpg` keyring to push a public key by fingerprint to a standard HKP/HKPS keyserver.

```bash
# Publish an SSH public key to a shared trusted-keys directory
python3 distribute_public_keys.py local --file id_ed25519.pub --dest-dir /shared/trusted_keys/

# Upload a certificate to an internal key-management HTTP endpoint
python3 distribute_public_keys.py http --file cert.pem \
    --url https://keys.internal.example.com/upload --name web-server-cert

# Publish a GPG public key to keys.openpgp.org
python3 distribute_public_keys.py gpg-keyserver --fingerprint ABCD1234EF567890ABCD1234EF567890ABCD1234 \
    --keyserver hkps://keys.openpgp.org
```

#### Variations / flags

- Print and share the SHA-256 hash from `local` mode over a separate channel (chat, ticket comment) so recipients have an independent way to confirm they copied the right file.
- Use `--name` in `http` mode to control how the key is labeled/filed on the receiving service, independent of the local filename.
- `keys.openpgp.org` requires the key's email identity to be verified via a confirmation email before it's searchable by email address — the raw key is published immediately, but discoverability by UID may lag until verification completes.

#### Common pitfalls

- `dist_local` uses `shutil.copy2`, which preserves file metadata but will silently overwrite an existing file of the same name in `--dest-dir` — check for naming collisions if multiple people might publish keys with the same filename.
- `dist_http` assumes the endpoint accepts a `multipart/form-data` upload under the field name `file`; a mismatched API contract will surface as an HTTP error from `response.raise_for_status()` — confirm your internal endpoint's expected schema first.
- Sending a *secret* key fingerprint to `gpg-keyserver` by mistake would attempt to publish only the public portion (GPG keyservers reject secret key material by protocol), but always double-check you're referencing the intended key before running this against a public keyserver — entries on public keyservers are effectively permanent and cannot be fully deleted.
