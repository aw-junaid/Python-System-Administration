## Chapter 10: Email Automation

Email is still the backbone of how systems talk to humans. A cron job that finishes a backup, a monitoring script that spots a disk filling up, a marketing list that needs a personalized note, a support inbox that needs triage — all of it eventually comes down to the same handful of operations: connect to a mail server, send something, or read something back. Python's standard library has quietly supported all of this for over two decades through `smtplib`, `imaplib`, and the `email` package, with no third-party dependencies required.

This chapter is a practical toolkit for automating both ends of email: sending it and receiving it. Every script here is designed to be safe to experiment with. None of them require you to have working credentials to see what they do — each one ships with a demo or dry-run mode that shows exactly what would happen, so you can read the output, understand the behavior, and only then point it at a real mailbox.

We'll build up from the simplest possible case — sending one plain-text email — through HTML formatting, attachments, reading a mailbox, parsing raw messages, sending to a whole list at once, firing off automated alerts from other scripts, and finally rendering templates safely before anything gets sent. By the end of the chapter you'll have a small, composable library of email utilities you can drop into almost any automation project.

A sysadmin uses these scripts constantly and often without thinking about it: a deployment pipeline that emails the team when a release goes out, a health-check script that pages someone when a service goes down, a weekly report that goes to a distribution list, or a one-off script to notify twenty customers that their accounts were migrated. None of these need a full email marketing platform or a heavyweight framework — they need ten to fifty lines of Python and a working SMTP or IMAP connection.

#### A note on credentials and safety

Every script in this chapter reads its mail server credentials from environment variables, never from hard-coded strings in the script itself. This is not a stylistic preference — committing a real email password into a script, a git repository, or a shared file is one of the most common ways credentials leak. If a script here is missing its environment variables, it does not fail loudly or crash; it falls back to a dry-run or demo mode so you always have a safe path to understanding the tool before using it for real.

Most modern providers (Gmail, Outlook, Yahoo, and others) also require an **app-specific password** rather than your normal login password once two-factor authentication is enabled. If `smtplib` or `imaplib` reports an authentication failure, that is almost always the first thing to check.

### 10.1 Send Emails

#### Use case

You need a script to send exactly one email — a status update, a manual alert, a one-off message to a colleague — without pulling in a third-party mail library or setting up a full mail client. This is the building block every other script in this chapter extends.

#### Prerequisites

- Python 3.6+ (uses only the standard library: `smtplib`, `email.mime.text`, `os`, `sys`)
- No `pip install` required
- Works identically on Linux, macOS, and Windows
- An SMTP-capable email account (Gmail, Outlook, a company mail server, etc.) with an app-specific password if 2FA is enabled

```bash
# No installation needed - smtplib and email are part of the Python standard library.
# Just confirm your Python version:
python3 --version
```

Set these environment variables before running for real:

```bash
export SMTP_HOST="smtp.gmail.com"
export SMTP_PORT="587"
export SMTP_USER="you@example.com"
export SMTP_PASSWORD="your-app-specific-password"
```

#### Cautions

Never hard-code your real email password inside the script — always use environment variables. Sending real email to a real recipient has real consequences, so double-check the `--to` address before running outside dry-run mode, especially inside loops.

#### Script

```python
#!/usr/bin/env python3
"""
send_email.py

Purpose:
    Send a plain text email via SMTP using Python's built-in smtplib
    and email libraries.
"""

import os
import smtplib
import sys
from email.mime.text import MIMEText


def send_plain_email(smtp_host, smtp_port, smtp_user, smtp_password,
                      to_addr, subject, body):
    msg = MIMEText(body, "plain")
    msg["From"] = smtp_user
    msg["To"] = to_addr
    msg["Subject"] = subject

    print(f"Connecting to {smtp_host}:{smtp_port} as {smtp_user}...")
    with smtplib.SMTP(smtp_host, smtp_port) as server:
        server.starttls()  # Use smtplib.SMTP_SSL(host, 465) instead if your provider needs implicit SSL
        server.login(smtp_user, smtp_password)
        server.sendmail(smtp_user, [to_addr], msg.as_string())

    print(f"Email sent successfully to {to_addr}.")


def dry_run(to_addr, subject, body, from_addr="you@example.com"):
    print("DRY RUN (no SMTP_HOST configured or no --to given) - email not sent.")
    print("---- Email that would be sent ----")
    print(f"From: {from_addr}")
    print(f"To: {to_addr}")
    print(f"Subject: {subject}")
    print(body)
    print("-----------------------------------")


def parse_args():
    args = sys.argv[1:]
    to_addr, subject, body = None, None, None
    i = 0
    while i < len(args):
        if args[i] == "--to" and i + 1 < len(args):
            to_addr = args[i + 1]; i += 2
        elif args[i] == "--subject" and i + 1 < len(args):
            subject = args[i + 1]; i += 2
        elif args[i] == "--body" and i + 1 < len(args):
            body = args[i + 1]; i += 2
        else:
            i += 1
    return to_addr, subject, body


def main():
    to_addr, subject, body = parse_args()
    to_addr = to_addr or "recipient@example.com"
    subject = subject or "Hello"
    body = body or "This is a test email."

    smtp_host = os.environ.get("SMTP_HOST")
    smtp_port = os.environ.get("SMTP_PORT")
    smtp_user = os.environ.get("SMTP_USER")
    smtp_password = os.environ.get("SMTP_PASSWORD")

    if smtp_host and smtp_port and smtp_user and smtp_password:
        try:
            send_plain_email(smtp_host, int(smtp_port), smtp_user, smtp_password,
                              to_addr, subject, body)
        except smtplib.SMTPAuthenticationError:
            print("Error: SMTP authentication failed. Check SMTP_USER/SMTP_PASSWORD "
                  "(you may need an app-specific password).")
        except (smtplib.SMTPException, OSError) as e:
            print(f"Error sending email: {e}")
    else:
        dry_run(to_addr, subject, body, from_addr=smtp_user or "you@example.com")


if __name__ == "__main__":
    main()
```

#### How it works

The script builds a `MIMEText` object with `From`, `To`, and `Subject` headers, opens an SMTP connection with `smtplib.SMTP`, upgrades it to an encrypted connection with `server.starttls()`, logs in with `server.login()`, and hands the message off with `server.sendmail()`. If any of the four required `SMTP_*` environment variables are missing, it skips the network entirely and prints a preview instead.

Run it with no arguments to see the safe dry-run demo:

```bash
python3 send_email.py
```

Run it for real once your environment variables are set:

```bash
python3 send_email.py --to recipient@example.com --subject "Hello" --body "This is a test email."
```

#### Sample output

Dry-run (no env vars set):

```

---- Email that would be sent ----
From: you@example.com
To: recipient@example.com
Subject: Hello
This is a test email.
-----------------------------------
```

```

```

#### Variations / flags

- `--to`, `--subject`, `--body` can each be supplied independently; any omitted flag falls back to a sensible default.
- Swap `server.starttls()` for `smtplib.SMTP_SSL(host, 465)` if your provider requires implicit SSL instead of STARTTLS on 587.
- Wrap the `send_plain_email()` call in a retry loop with `time.sleep()` between attempts if you're calling this from an unreliable network.
- Schedule it with cron for recurring one-line reminders: `0 9 * * 1 SMTP_HOST=... python3 /path/send_email.py --to team@example.com --subject "Monday standup"`.

#### Common pitfalls

Authentication errors are almost always caused by using your normal account password instead of an app-specific password when 2FA is enabled. A `ConnectionRefusedError` or timeout usually means the wrong port (587 vs 465) or a firewall blocking outbound SMTP. Some ISPs and cloud providers (notably many VPS hosts) block outbound port 25 and sometimes 587 by default to fight spam — check your provider's networking rules if connections hang rather than fail immediately.


### 10.2 Send HTML Emails

#### Use case

This is standard practice for any automated email a human is meant to actually read rather than just scan in a terminal.

- Python 3.6+ (standard library only: `smtplib`, `email.mime.multipart`, `email.mime.text`)
- No `pip install` required
- Works identically on Linux, macOS, and Windows
- Same SMTP credentials as 10.1

#### Script

```python
#!/usr/bin/env python3
"""
send_html_email.py

Purpose:
    Send an HTML-formatted email via SMTP, with a plain text fallback
    for email clients that don't render HTML.
"""

import os
import smtplib
import sys
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText

DEMO_HTML = """\
<html>
  <body>
    <h1>Hello!</h1>
    <p>This is a <strong>demo HTML email</strong> sent by send_html_email.py.</p>
  </body>
</html>
"""
DEMO_PLAIN_FALLBACK = "Please view this email in an HTML-capable client."


def send_html_email(smtp_host, smtp_port, smtp_user, smtp_password,
                     to_addr, subject, html_body, plain_fallback):
    msg = MIMEMultipart("alternative")
    msg["From"] = smtp_user
    msg["To"] = to_addr
    msg["Subject"] = subject

    msg.attach(MIMEText(plain_fallback, "plain"))
    msg.attach(MIMEText(html_body, "html"))

    print(f"Connecting to {smtp_host}:{smtp_port} as {smtp_user}...")
    with smtplib.SMTP(smtp_host, smtp_port) as server:
        server.starttls()
        server.login(smtp_user, smtp_password)
        server.sendmail(smtp_user, [to_addr], msg.as_string())

    print(f"HTML email sent successfully to {to_addr}.")


def dry_run(to_addr, subject, html_body, plain_fallback, from_addr="you@example.com"):
    print("DRY RUN (no SMTP_HOST configured or no --to given) - email not sent.")
    print("---- Email that would be sent ----")
    print(f"From: {from_addr}")
    print(f"To: {to_addr}")
    print(f"Subject: {subject}")
    print("--- Plain text part ---")
    print(plain_fallback)
    print("--- HTML part ---")
    print(html_body)
    print("-----------------------------------")


def parse_args():
    args = sys.argv[1:]
    to_addr, subject = None, None
    i = 0
    while i < len(args):
        if args[i] == "--to" and i + 1 < len(args):
            to_addr = args[i + 1]; i += 2
        elif args[i] == "--subject" and i + 1 < len(args):
            subject = args[i + 1]; i += 2
        else:
            i += 1
    return to_addr, subject


def main():
    to_addr, subject = parse_args()
    to_addr = to_addr or "recipient@example.com"
    subject = subject or "Newsletter"

    smtp_host = os.environ.get("SMTP_HOST")
    smtp_port = os.environ.get("SMTP_PORT")
    smtp_user = os.environ.get("SMTP_USER")
    smtp_password = os.environ.get("SMTP_PASSWORD")

    if smtp_host and smtp_port and smtp_user and smtp_password:
        try:
            send_html_email(smtp_host, int(smtp_port), smtp_user, smtp_password,
                             to_addr, subject, DEMO_HTML, DEMO_PLAIN_FALLBACK)
        except smtplib.SMTPAuthenticationError:
            print("Error: SMTP authentication failed. Check SMTP_USER/SMTP_PASSWORD.")
        except (smtplib.SMTPException, OSError) as e:
            print(f"Error sending email: {e}")
    else:
        dry_run(to_addr, subject, DEMO_HTML, DEMO_PLAIN_FALLBACK,
                 from_addr=smtp_user or "you@example.com")


if __name__ == "__main__":
    main()
```

#### How it works

`MIMEMultipart("alternative")` builds a message with two parts — plain text and HTML — attached in that order. Email clients that understand HTML render the last (HTML) part; clients that don't fall back to the plain-text part automatically. This is the standard MIME pattern for HTML email and is what every major mail client expects.

```bash

```

```bash

```

```
To: recipient@example.com
Subject: Newsletter
--- Plain text part ---
Please view this email in an HTML-capable client.
--- HTML part ---
<html>
  <body>
```

#### Variations / flags

- Replace `DEMO_HTML` with a Jinja2-rendered template if your HTML needs to be dynamic per recipient (combine with the 10.8 template-rendering approach).
- Read the HTML body from a `.html` file on disk instead of a hard-coded string for longer newsletters.
- Add a `--html-file` flag that loads external HTML so designers can edit templates without touching Python.

#### Common pitfalls

Forgetting the plain-text fallback (or attaching parts in the wrong order) causes some clients to show raw HTML tags instead of rendering them — always attach plain text first, HTML second, inside a `multipart/alternative` container. Inline CSS is far more reliably supported across email clients than `<style>` blocks or external stylesheets; complex CSS layouts frequently break in Outlook specifically.

### 10.3 Send Attachments

#### Use case

You need a script to email a generated report, log file, invoice, or export to someone automatically — for example, a nightly job that emails a CSV export to finance, or a monitoring script that attaches a diagnostic log when it alerts on-call. This extends the plain-text sender in 10.1 with MIME attachment handling.

#### Prerequisites

- Python 3.6+ (standard library only: `smtplib`, `email.mime.base`, `email.mime.multipart`, `email.mime.text`, `email.encoders`)
- No `pip install` required
- Works identically on Linux, macOS, and Windows
- Same SMTP credentials as 10.1

```bash
python3 --version
```

```bash
export SMTP_HOST="smtp.gmail.com"
export SMTP_PORT="587"
export SMTP_USER="you@example.com"
export SMTP_PASSWORD="your-app-specific-password"
```

#### Cautions

Most providers cap total message size (body + attachments) around 20–25 MB; oversized files are rejected by the SMTP server with an error, not silently truncated. Be careful attaching files that contain personal or sensitive data — verify the recipient address before sending, especially in scripted or bulk contexts.

#### Script

```python
#!/usr/bin/env python3
"""
send_email_with_attachment.py

Purpose:
    Send an email with one or more file attachments via SMTP.
"""

import os
import smtplib
import sys
from email.mime.base import MIMEBase
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText
from email import encoders

DEMO_ATTACHMENT = "demo_attachment.txt"
DEMO_ATTACHMENT_CONTENT = "This is a demo attachment created by send_email_with_attachment.py.\n"


def build_message(smtp_user, to_addr, subject, body, attachment_paths):
    msg = MIMEMultipart()
    msg["From"] = smtp_user
    msg["To"] = to_addr
    msg["Subject"] = subject
    msg.attach(MIMEText(body, "plain"))

    for path in attachment_paths:
        if not os.path.isfile(path):
            print(f"Warning: attachment not found, skipping: {path}")
            continue
        with open(path, "rb") as f:
            part = MIMEBase("application", "octet-stream")
            part.set_payload(f.read())
        encoders.encode_base64(part)
        filename = os.path.basename(path)
        part.add_header("Content-Disposition", f"attachment; filename={filename}")
        msg.attach(part)
        size_kb = os.path.getsize(path) / 1024
        print(f"Attached: {filename} ({size_kb:.1f} KB)")

    return msg


def send_with_attachments(smtp_host, smtp_port, smtp_user, smtp_password,
                           to_addr, subject, body, attachment_paths):
    msg = build_message(smtp_user, to_addr, subject, body, attachment_paths)

    print(f"Connecting to {smtp_host}:{smtp_port} as {smtp_user}...")
    with smtplib.SMTP(smtp_host, smtp_port) as server:
        server.starttls()
        server.login(smtp_user, smtp_password)
        server.sendmail(smtp_user, [to_addr], msg.as_string())

    print(f"Email with attachment(s) sent successfully to {to_addr}.")


def dry_run(to_addr, subject, body, attachment_paths, from_addr="you@example.com"):
    print("DRY RUN (no SMTP_HOST configured or no --to given) - email not sent.")
    print("---- Email that would be sent ----")
    print(f"From: {from_addr}")
    print(f"To: {to_addr}")
    print(f"Subject: {subject}")
    print(body)
    print(f"Attachments: {[os.path.basename(p) for p in attachment_paths]}")
    print("-----------------------------------")


def parse_args():
    args = sys.argv[1:]
    to_addr, subject, body = None, None, None
    attachments = []
    i = 0
    while i < len(args):
        if args[i] == "--to" and i + 1 < len(args):
            to_addr = args[i + 1]; i += 2
        elif args[i] == "--subject" and i + 1 < len(args):
            subject = args[i + 1]; i += 2
        elif args[i] == "--body" and i + 1 < len(args):
            body = args[i + 1]; i += 2
        elif args[i] == "--attach" and i + 1 < len(args):
            attachments.append(args[i + 1]); i += 2
        else:
            i += 1
    return to_addr, subject, body, attachments


def main():
    to_addr, subject, body, attachments = parse_args()
    to_addr = to_addr or "recipient@example.com"
    subject = subject or "Report"
    body = body or "See attached."

    if not attachments:
        if not os.path.isfile(DEMO_ATTACHMENT):
            with open(DEMO_ATTACHMENT, "w", encoding="utf-8") as f:
                f.write(DEMO_ATTACHMENT_CONTENT)
        attachments = [DEMO_ATTACHMENT]
        print("No --attach given, using a demo attachment file.\n")

    smtp_host = os.environ.get("SMTP_HOST")
    smtp_port = os.environ.get("SMTP_PORT")
    smtp_user = os.environ.get("SMTP_USER")
    smtp_password = os.environ.get("SMTP_PASSWORD")

    if smtp_host and smtp_port and smtp_user and smtp_password:
        try:
            send_with_attachments(smtp_host, int(smtp_port), smtp_user, smtp_password,
                                   to_addr, subject, body, attachments)
        except smtplib.SMTPAuthenticationError:
            print("Error: SMTP authentication failed. Check SMTP_USER/SMTP_PASSWORD.")
        except (smtplib.SMTPException, OSError) as e:
            print(f"Error sending email: {e}")
    else:
        dry_run(to_addr, subject, body, attachments, from_addr=smtp_user or "you@example.com")


if __name__ == "__main__":
    main()
```

#### How it works

The message is built as `MIMEMultipart()` (the default `"mixed"` subtype, appropriate for body + binary attachments). The plain-text body is attached first, then each file is read as raw bytes, wrapped in a `MIMEBase("application", "octet-stream")` part, base64-encoded with `encoders.encode_base64()`, and given a `Content-Disposition: attachment; filename=...` header so mail clients offer it as a downloadable file rather than trying to render it inline. Missing files are skipped with a warning rather than aborting the whole send.

```bash
python3 send_email_with_attachment.py
```

```bash
python3 send_email_with_attachment.py --to recipient@example.com \
    --subject "Report" --body "See attached." --attach report.pdf
```

Pass `--attach` multiple times to attach several files:

```bash
python3 send_email_with_attachment.py --to recipient@example.com \
    --attach report.pdf --attach summary.csv
```

#### Sample output

Dry-run (demo attachment, no env vars set):

```

DRY RUN (no SMTP_HOST configured or no --to given) - email not sent.
---- Email that would be sent ----
From: you@example.com
To: recipient@example.com
Subject: Report
See attached.
Attachments: ['demo_attachment.txt']

```


#### Variations / flags

- Pass `--attach` as many times as needed; each valid path is added as a separate attachment.
- For large files, consider uploading to cloud storage and emailing a link instead — this script reads each attachment entirely into memory, which doesn't scale to very large files.
- Add a `--max-size-mb` guard that checks `os.path.getsize()` before attaching, so oversized files fail fast locally instead of being rejected by the SMTP server after upload.

#### Common pitfalls

A missing or misspelled attachment path is skipped silently (with only a printed warning) rather than raising an error — watch the "Attached:" lines in the output to confirm every file you expected actually made it in. Very large attachments can cause the SMTP server to reject the message entirely or time out mid-upload; check your provider's size limit before assuming a failure is a bug. Binary files that aren't correctly base64-encoded (e.g. if you bypass `build_message` and write your own logic) will arrive corrupted — always use `encoders.encode_base64()` for non-text attachments.

### 10.4 Receive Emails

#### Use case

You need a script to check a mailbox for new messages as part of an automated workflow — for example, polling a support inbox every few minutes, checking whether a confirmation email has arrived, or building a simple audit of what landed in an inbox overnight. This is a read-only building block; combine it with `parse_email.py` (10.5) once you need full message bodies.

#### Prerequisites

- Python 3.6+ (standard library only: `imaplib`, `email`, `os`, `sys`)
- No `pip install` required
- Works identically on Linux, macOS, and Windows
- An IMAP-enabled email account; Gmail and Outlook require IMAP access to be explicitly enabled in account settings, plus an app-specific password if 2FA is on

```bash
python3 --version
```

```bash
export IMAP_HOST="imap.gmail.com"
export IMAP_USER="you@example.com"
export IMAP_PASSWORD="your-app-specific-password"
```

#### Cautions

This script is strictly read-only — it never deletes, moves, or marks messages as read. Never hard-code your real IMAP password inside the script; always use environment variables.

#### Script

```python
#!/usr/bin/env python3
"""
receive_emails.py

Purpose:
    Connect to an IMAP mailbox and list recent emails (sender, subject,
    date), optionally limited to unread messages.
"""

import email
import imaplib
import os
import sys


def list_emails(imap_host, imap_user, imap_password, limit=5, unread_only=False):
    print(f"Connecting to {imap_host} as {imap_user}...")
    conn = imaplib.IMAP4_SSL(imap_host)
    conn.login(imap_user, imap_password)
    conn.select("INBOX")

    criterion = "UNSEEN" if unread_only else "ALL"
    status, data = conn.search(None, criterion)
    if status != "OK":
        print("Error: could not search mailbox.")
        conn.logout()
        return

    message_ids = data[0].split()
    message_ids = message_ids[-limit:]  # most recent N
    message_ids.reverse()

    print(f"Found {len(message_ids)} message(s) in INBOX (showing most recent first):\n")

    for i, msg_id in enumerate(message_ids, start=1):
        status, msg_data = conn.fetch(msg_id, "(RFC822)")
        if status != "OK":
            continue
        raw_email = msg_data[0][1]
        msg = email.message_from_bytes(raw_email)
        print(f"{i}. From: {msg.get('From')}")
        print(f"   Subject: {msg.get('Subject')}")
        print(f"   Date: {msg.get('Date')}\n")

    conn.logout()


def demo_mode(limit=2):
    print("DEMO MODE (no IMAP_HOST configured) - showing sample data.\n")
    samples = [
        {"from": "sender@example.com", "subject": "Demo: Meeting tomorrow"},
        {"from": "newsletter@example.com", "subject": "Demo: Weekly digest"},
    ]
    for i, sample in enumerate(samples[:limit], start=1):
        print(f"{i}. From: {sample['from']}")
        print(f"   Subject: {sample['subject']}")
        print(f"   Date: (demo data, not a real email)\n")


def parse_args():
    args = sys.argv[1:]
    limit = 5
    unread_only = "--unread-only" in args
    if "--limit" in args:
        idx = args.index("--limit")
        if idx + 1 < len(args):
            limit = int(args[idx + 1])
    return limit, unread_only


def main():
    limit, unread_only = parse_args()

    imap_host = os.environ.get("IMAP_HOST")
    imap_user = os.environ.get("IMAP_USER")
    imap_password = os.environ.get("IMAP_PASSWORD")

    if imap_host and imap_user and imap_password:
        try:
            list_emails(imap_host, imap_user, imap_password, limit, unread_only)
        except imaplib.IMAP4.error as e:
            print(f"Error: IMAP login/search failed - {e}")
        except (OSError, ConnectionError) as e:
            print(f"Error connecting to IMAP server: {e}")
    else:
        demo_mode(limit)


if __name__ == "__main__":
    main()
```

#### How it works

`imaplib.IMAP4_SSL` opens an encrypted connection on port 993 (the IMAP-over-SSL standard). After `login()` and `select("INBOX")`, `conn.search(None, criterion)` returns matching message IDs — `"ALL"` or `"UNSEEN"` depending on the `--unread-only` flag. Only the most recent `--limit` IDs are kept and reversed so newest-first is printed. Each message is fetched in full (`RFC822`) purely to read its headers via `email.message_from_bytes()`; nothing is written back to the server, so unread flags are untouched.

### 10.5 Parse Email Contents

This complements `receive_emails.py` (10.4), which lists messages but doesn't dig into their contents.

#### Prerequisites

- Python 3.6+ (standard library only: `email`, `os`, `sys`)
- No `pip install` required

- No mail server connection needed — this script only reads a `.eml` file already saved to disk

#### Cautions

This script only parses a `.eml` file already on disk; it does not connect to any mail server. HTML-only emails print their raw HTML as the body — this script does not strip tags for you. By design it does not extract or save attachment *contents*, only filenames, to avoid writing arbitrary files to disk without you explicitly asking for it.

#### Script

```python
#!/usr/bin/env python3
"""
parse_email.py

Purpose:
    Parse a raw email (.eml file, or the latest message fetched live
    via IMAP) and extract structured info: sender, subject, date, body
    text, and a list of attachment filenames.
"""

import email
import os
import sys
from email import policy
from email.parser import BytesParser

DEMO_FILE = "demo_message.eml"
DEMO_CONTENT = (
    "From: sender@example.com\n"
    "To: recipient@example.com\n"
    "Subject: Demo message\n"
    "Date: Thu, 17 Jul 2026 09:00:00 +0000\n"
    "Content-Type: text/plain; charset=UTF-8\n"
    "\n"
    "Hello, this is a demo email body.\n"
)


def parse_eml(path: str) -> None:
    if not os.path.isfile(path):
        print(f"Error: file not found: {path}")
        return

    with open(path, "rb") as f:
        msg = BytesParser(policy=policy.default).parse(f)

    print(f"From:    {msg.get('From')}")
    print(f"To:      {msg.get('To')}")
    print(f"Subject: {msg.get('Subject')}")
    print(f"Date:    {msg.get('Date')}")
    print()

    body_part = msg.get_body(preferencelist=("plain", "html"))
    if body_part:
        content_type = body_part.get_content_type()
        print(f"--- Body ({content_type}) ---")
        print(body_part.get_content())
    else:
        print("--- Body ---")
        print("(no readable body part found)")

    attachments = [
        part.get_filename()
        for part in msg.iter_attachments()
        if part.get_filename()
    ]
    print("\n--- Attachments ---")
    if attachments:
        for name in attachments:
            print(f"  {name}")
    else:
        print("(none found)")


def main():
    if len(sys.argv) > 1:
        path = sys.argv[1]
    else:
        print("No file path given, creating and parsing a demo .eml file.\n")
        if not os.path.isfile(DEMO_FILE):
            with open(DEMO_FILE, "w", encoding="utf-8") as f:
                f.write(DEMO_CONTENT)
        path = DEMO_FILE
    parse_eml(path)


if __name__ == "__main__":
    main()
```

#### How it works

`BytesParser(policy=policy.default).parse(f)` reads the raw `.eml` bytes and builds a modern `EmailMessage` object. Header fields (`From`, `To`, `Subject`, `Date`) are read directly via `msg.get(...)`. `msg.get_body(preferencelist=("plain", "html"))` walks the MIME tree and returns the best available body part, preferring plain text over HTML. `msg.iter_attachments()` walks the same tree looking for parts explicitly marked as attachments and lists their filenames only — file contents are never written to disk.



```
To:      recipient@example.com
Subject: Demo message
Date:    Thu, 17 Jul 2026 09:00:00 +0000

--- Body (text/plain) ---
Hello, this is a demo email body.

--- Attachments ---
(none found)
```

#### Variations / flags

- Pass any `.eml` file path as the sole positional argument — most mail clients (Outlook, Apple Mail, Thunderbird) support "Save As .eml" / "Export" for a single message.
- To actually extract attachment *contents*, extend the `iter_attachments()` loop to call `part.get_payload(decode=True)` and write the bytes to a controlled output directory — do this deliberately, since automatically writing arbitrary email attachments to disk is a common malware vector.
- Combine with `receive_emails.py`: fetch a message's raw bytes over IMAP (`RFC822`), write them to a temporary `.eml` file, then parse that file with this script.
- Add HTML-to-text conversion (e.g. with `html.parser` or a library like `beautifulsoup4`) if you need readable plain text from HTML-only emails.

#### Common pitfalls

Multipart/alternative messages (plain + HTML versions of the same content) are common — `get_body(preferencelist=("plain", "html"))` picks plain text first by default, so if you specifically need the HTML version, reorder the preference list. Malformed or non-standard `.eml` files (common with exports from some webmail clients) can cause `BytesParser` to raise or return unexpected results — wrap parsing in a `try/except` for untrusted input. Because this script deliberately does not save attachment contents, don't assume attachments are "handled" just because their filenames are listed — you must add explicit extraction logic if you need the files themselves.

#### Script

```python
#!/usr/bin/env python3
"""
bulk_email_sender.py

Purpose:
    Send a personalized email to many recipients at once, reading
    recipient data (email + name, or any other placeholders) from a
    CSV file and substituting them into a message template.
"""

import csv
import os
import smtplib
import sys
import time
from email.mime.text import MIMEText

SEND_DELAY_SECONDS = 1  # polite delay between sends to avoid rate limits

DEMO_CSV = "demo_recipients.csv"
DEMO_CSV_CONTENT = "email,name\nalice@example.com,Alice\nbob@example.com,Bob\n"


def read_recipients(csv_path: str) -> list:
    with open(csv_path, "r", encoding="utf-8", newline="") as f:
        reader = csv.DictReader(f)
        return list(reader)


def render_template(template: str, row: dict) -> str:
    try:
        return template.format(**row)
    except KeyError as e:
        # Missing placeholder column in CSV - leave template as-is for that field
        return template


def send_bulk(smtp_host, smtp_port, smtp_user, smtp_password,
              recipients, subject_template, body_template):
    print(f"Connecting to {smtp_host}:{smtp_port} as {smtp_user}...")
    succeeded, failed = 0, 0

    with smtplib.SMTP(smtp_host, smtp_port) as server:
        server.starttls()
        server.login(smtp_user, smtp_password)

        for i, row in enumerate(recipients, start=1):
            to_addr = row.get("email")
            if not to_addr:
                print(f"Skipping row {i}: no 'email' column value.")
                failed += 1
                continue

            subject = render_template(subject_template, row)
            body = render_template(body_template, row)

            msg = MIMEText(body, "plain")
            msg["From"] = smtp_user
            msg["To"] = to_addr
            msg["Subject"] = subject

            try:
                server.sendmail(smtp_user, [to_addr], msg.as_string())
                print(f"Sent to {to_addr} ({i}/{len(recipients)})")
                succeeded += 1
            except smtplib.SMTPException as e:
                print(f"Failed to send to {to_addr}: {e}")
                failed += 1

            time.sleep(SEND_DELAY_SECONDS)

    print(f"Bulk send complete: {succeeded} succeeded, {failed} failed.")


def dry_run(recipients, subject_template, body_template):
    print("DRY RUN (no SMTP_HOST configured or no --csv given) - no emails sent.")
    for row in recipients:
        to_addr = row.get("email", "unknown@example.com")
        subject = render_template(subject_template, row)
        body = render_template(body_template, row)
        print(f"---- Preview for {to_addr} ----")
        print(f"Subject: {subject}")
        print(body)
        print("----------------------------------------")


def parse_args():
    args = sys.argv[1:]
    csv_path, subject, body = None, None, None
    i = 0
    while i < len(args):
        if args[i] == "--csv" and i + 1 < len(args):
            csv_path = args[i + 1]; i += 2
        elif args[i] == "--subject" and i + 1 < len(args):
            subject = args[i + 1]; i += 2
        elif args[i] == "--body" and i + 1 < len(args):
            body = args[i + 1]; i += 2
        else:
            i += 1
    return csv_path, subject, body


def main():
    csv_path, subject_template, body_template = parse_args()
    subject_template = subject_template or "Hello {name}"
    body_template = body_template or "Hi {name}, this is a bulk email."

    if not csv_path:
        print("No --csv given, creating a demo recipients file.\n")
        if not os.path.isfile(DEMO_CSV):
            with open(DEMO_CSV, "w", encoding="utf-8") as f:
                f.write(DEMO_CSV_CONTENT)
        csv_path = DEMO_CSV

    if not os.path.isfile(csv_path):
        print(f"Error: CSV file not found: {csv_path}")
        return

    recipients = read_recipients(csv_path)

    smtp_host = os.environ.get("SMTP_HOST")
    smtp_port = os.environ.get("SMTP_PORT")
    smtp_user = os.environ.get("SMTP_USER")
    smtp_password = os.environ.get("SMTP_PASSWORD")

    if smtp_host and smtp_port and smtp_user and smtp_password:
        try:
            send_bulk(smtp_host, int(smtp_port), smtp_user, smtp_password,
                      recipients, subject_template, body_template)
        except smtplib.SMTPAuthenticationError:
            print("Error: SMTP authentication failed. Check SMTP_USER/SMTP_PASSWORD.")
        except (smtplib.SMTPException, OSError) as e:
            print(f"Error during bulk send: {e}")
    else:
        dry_run(recipients, subject_template, body_template)


if __name__ == "__main__":
    main()
```

#### How it works

`csv.DictReader` reads each row of the CSV into a dictionary keyed by column header, so a row like `email,name` becomes `{"email": "alice@example.com", "name": "Alice"}`. `render_template()` calls Python's `str.format(**row)`, substituting `{name}` and any other placeholder with matching CSV columns. A single SMTP connection is opened once and reused for every recipient (rather than reconnecting per email), and `time.sleep(SEND_DELAY_SECONDS)` adds a polite one-second pause between sends to reduce the chance of tripping provider rate limits.

```python
#!/usr/bin/env python3
"""
email_notifier.py

Purpose:
    A small reusable notification helper for automation scripts: send
    yourself (or a team) an email alert when something happens - e.g.
    a backup finishes, a job fails, disk space is low, etc.
"""

import os
import smtplib
import sys
from email.mime.text import MIMEText


def send_notification(smtp_host, smtp_port, smtp_user, smtp_password,
                       to_addr, event, status, message):
    status_label = status.upper()
    subject = f"[{status_label}] {event}"

    msg = MIMEText(message, "plain")
    msg["From"] = smtp_user
    msg["To"] = to_addr
    msg["Subject"] = subject

    print(f"Connecting to {smtp_host}:{smtp_port} as {smtp_user}...")
    with smtplib.SMTP(smtp_host, smtp_port) as server:
        server.starttls()
        server.login(smtp_user, smtp_password)
        server.sendmail(smtp_user, [to_addr], msg.as_string())

    print(f"Notification email sent to {to_addr}. Subject: {subject}")


def dry_run(to_addr, event, status, message):
    subject = f"[{status.upper()}] {event}"
    print("DRY RUN (no SMTP_HOST configured or no --to given) - notification not sent.")
    print("---- Notification that would be sent ----")
    print(f"To: {to_addr}")
    print(f"Subject: {subject}")
    print(message)
    print("------------------------------------------")


def parse_args():
    args = sys.argv[1:]
    to_addr, event, status, message = None, None, None, None
    i = 0
    while i < len(args):
        if args[i] == "--to" and i + 1 < len(args):
            to_addr = args[i + 1]; i += 2
        elif args[i] == "--event" and i + 1 < len(args):
            event = args[i + 1]; i += 2
        elif args[i] == "--status" and i + 1 < len(args):
            status = args[i + 1]; i += 2
        elif args[i] == "--message" and i + 1 < len(args):
            message = args[i + 1]; i += 2
        else:
            i += 1
    return to_addr, event, status, message


def main():
    to_addr, event, status, message = parse_args()
    to_addr = to_addr or "you@example.com"
    event = event or "Backup Job"
    status = status or "success"
    message = message or "Backup completed in 4m12s."

    smtp_host = os.environ.get("SMTP_HOST")
    smtp_port = os.environ.get("SMTP_PORT")
    smtp_user = os.environ.get("SMTP_USER")
    smtp_password = os.environ.get("SMTP_PASSWORD")

    if smtp_host and smtp_port and smtp_user and smtp_password:
        try:
            send_notification(smtp_host, int(smtp_port), smtp_user, smtp_password,
                               to_addr, event, status, message)
        except smtplib.SMTPAuthenticationError:
            print("Error: SMTP authentication failed. Check SMTP_USER/SMTP_PASSWORD.")
        except (smtplib.SMTPException, OSError) as e:
            print(f"Error sending notification: {e}")
    else:
        dry_run(to_addr, event, status, message)


if __name__ == "__main__":
    main()
```

### 10.8 Email Template Rendering

#### Use case

Before sending a bulk or personalized email for real, you want to see exactly what will land in each recipient's inbox — catching typo'd placeholders, missing CSV columns, or formatting mistakes without risking an accidental send. This is the safety-net step that belongs between "I wrote a template" and "I ran `bulk_email_sender.py` for real."

#### Prerequisites

- Python 3.6+ (standard library only: `csv`, `json`, `os`, `sys`)
- No `pip install` required
- Works identically on Linux, macOS, and Windows
- No SMTP or IMAP credentials needed — this script does no network I/O at all

```bash
python3 --version
```

No environment variables are required for this script.

#### Cautions

This script uses Python's `str.format()` under the hood: literal curly braces in your template that are *not* meant as placeholders (e.g. `{` or `}` characters in JSON examples inside an email body) must be escaped as `{{` and `}}` or rendering will fail. A row missing a required placeholder column raises a clear, per-row error and is skipped rather than silently sending a half-rendered message — always check the "skipped" summary at the end of the output.

#### Script

```python
#!/usr/bin/env python3
"""
render_email_template.py

Purpose:
    Render an email subject/body template file against one or more
    data rows (from a CSV or a single --data JSON object), substituting
    placeholders like {name} or {company} with real values. Useful for
    previewing exactly what bulk_email_sender.py or a notification
    script would send, without touching SMTP at all.
"""

import csv
import json
import os
import sys

DEMO_TEMPLATE = "Hi {name}, your invoice #{invoice_id} for {company} is ready."
DEMO_ROWS = [
    {"name": "Alice", "invoice_id": "1001", "company": "Acme Corp"},
    {"name": "Bob", "invoice_id": "1002", "company": "Acme Corp"},
]


def load_template(path: str) -> str:
    with open(path, "r", encoding="utf-8") as f:
        return f.read().strip()


def load_csv_rows(path: str) -> list:
    with open(path, "r", encoding="utf-8", newline="") as f:
        reader = csv.DictReader(f)
        return list(reader)


def render_row(template: str, row: dict) -> str:
    # Raises KeyError if a placeholder in the template has no matching
    # column in the row - caller decides whether to skip or fill in.
    return template.format(**row)


def render_all(template: str, rows: list) -> None:
    rendered_count, skipped_count = 0, 0

    for i, row in enumerate(rows, start=1):
        print(f"---- Rendered for row {i} ----")
        try:
            print(render_row(template, row))
            rendered_count += 1
        except KeyError as e:
            print(f"Skipped: missing placeholder value for {e} in row {row}")
            skipped_count += 1
        print("----------------------------------------")

    print(f"\nDone: {rendered_count} rendered, {skipped_count} skipped.")


def parse_args():
    args = sys.argv[1:]
    template_path, csv_path, data_json = None, None, None
    i = 0
    while i < len(args):
        if args[i] == "--template" and i + 1 < len(args):
            template_path = args[i + 1]; i += 2
        elif args[i] == "--csv" and i + 1 < len(args):
            csv_path = args[i + 1]; i += 2
        elif args[i] == "--data" and i + 1 < len(args):
            data_json = args[i + 1]; i += 2
        else:
            i += 1
    return template_path, csv_path, data_json


def main():
    template_path, csv_path, data_json = parse_args()

    if not template_path:
        print("DEMO MODE (no --template given) - rendering built-in sample.\n")
        render_all(DEMO_TEMPLATE, DEMO_ROWS)
        return

    if not os.path.isfile(template_path):
        print(f"Error: template file not found: {template_path}")
        return
    template = load_template(template_path)

    if csv_path:
        if not os.path.isfile(csv_path):
            print(f"Error: CSV file not found: {csv_path}")
            return
        rows = load_csv_rows(csv_path)
    elif data_json:
        try:
            rows = [json.loads(data_json)]
        except json.JSONDecodeError as e:
            print(f"Error: --data is not valid JSON: {e}")
            return
    else:
        print("No --csv or --data given, rendering template against built-in demo data.\n")
        rows = DEMO_ROWS

    render_all(template, rows)


if __name__ == "__main__":
    main()
```

#### How it works

`load_template()` reads a plain-text template file containing `{placeholder}` markers. Data comes from one of three sources, checked in order: a CSV file (`--csv`, one rendered preview per row via `csv.DictReader`), a single JSON object (`--data`, one rendered preview), or — if neither is given — a small built-in demo template and demo rows. `render_row()` calls `template.format(**row)`; if the template references a placeholder that isn't a key in that row's dictionary, `str.format()` raises `KeyError`, which is caught per-row so one bad row doesn't stop the whole preview. A final summary line reports how many rows rendered cleanly versus were skipped.

#### Variations / flags

- `--template` points at any plain-text file containing `{placeholder}` markers for the subject or body.
- `--csv` renders the template once per row, exactly matching how `bulk_email_sender.py` would process the same file.
- `--data` accepts a single JSON object for quick one-off previews without needing a CSV file at all.
- Pipe the output of this script into a file (`> preview.txt`) to review a large batch offline before running the real sender.
- Extend `render_all()` to also validate subject-line length or check for unescaped `{`/`}` characters before declaring a template "ready."

#### Common pitfalls

Literal curly braces anywhere in the template (common if the email body includes example JSON, code, or set notation) must be doubled — `{{` and `}}` — or `str.format()` will raise a confusing `KeyError` or `IndexError` that isn't actually about a missing column. Column name mismatches between the template and the CSV header (`{Name}` vs a `name` column) are case-sensitive and will be reported as a skipped row, not auto-corrected. Because this script never sends anything, it's easy to forget to re-run it after editing a template — always re-preview immediately before running the real `bulk_email_sender.py` send.
