## Chapter 15: System Information

Every piece of automation eventually needs to answer one basic question: *what am I actually running on?* Before you can right-size a deployment, diagnose a slow server, license software correctly, or write a monitoring agent, you need a reliable way to pull hardware and OS facts out of the machine in front of you — programmatically, repeatably, and without opening a GUI.

This chapter is a toolbox of thirteen small, single-purpose Python scripts that each answer one system-information question: how much CPU, RAM, and disk is available; what network interfaces exist; what software and environment variables are configured; what the BIOS and motherboard report; which OS and kernel build is running; how long the box has been up; who is logged in; and what GPU hardware is present.

Every script in this chapter follows the same shape you've already seen in the networking chapter: a documented CLI built with `argparse`, sensible default output for humans, optional flags for scripting/JSON output where it matters, and clear, non-zero exit codes on failure so these scripts can be dropped straight into monitoring dashboards, cron jobs, or CI health checks. Most rely on the excellent third-party `psutil` library, which gives one consistent API across Linux, Windows, and macOS for CPU, memory, disk, network, boot time, and user data. A handful of topics — BIOS, motherboard, and installed software — have no reliable OS-agnostic API at all, so those scripts detect the platform and shell out to the right native tool (`dmidecode`/`system_profiler`/PowerShell) instead of pretending a single library can do it all.

Treat this chapter as a parts bin: each script is independent, has no dependency on the others, and can be copied into a larger asset-inventory or monitoring tool as-is.

### 15.1 Get CPU Information

#### Use Case

Find and report CPU details for a machine — core counts, clock speed, model, and live per-core load — before scaling a service, sizing a container's `cpu` limit, or diagnosing why a box "feels slow."

#### Prerequisites

- Python 3.7+
- Third-party library: `psutil` (not in the standard library)
- OS scope: Linux, Windows, macOS (single script, no branching needed)

```bash
pip install psutil
```

#### Cautions

`psutil.cpu_percent()` needs a short sampling interval to be accurate — calling it with no interval on the very first call returns `0.0` or a meaningless value, so the script deliberately blocks for `--interval` seconds. On some virtualized/cloud hosts, `cpu_freq()` may return `None` for max/min frequency; the script handles this gracefully instead of crashing.

#### Script

```python
#!/usr/bin/env python3
"""
get_cpu_info.py

Purpose:
    Report CPU hardware details and live utilization: logical/physical
    core counts, current/min/max clock speed, per-core usage percentage,
    and the CPU model string, using the cross-platform `psutil` library.

Usage:
    python get_cpu_info.py
    python get_cpu_info.py --interval 2
    python get_cpu_info.py --per-core
    python get_cpu_info.py --json

Expected output:
    - CPU model, physical core count, logical core count.
    - Current/min/max clock frequency in MHz (when available).
    - Overall CPU utilization percentage sampled over --interval seconds.
    - With --per-core: utilization percentage for every logical core.
    - With --json: the same data as a single JSON object (for scripting).
"""

import argparse
import json
import platform
import sys

try:
    import psutil
except ImportError:
    print("[ERROR] Missing dependency 'psutil'. Install it with: pip install psutil", file=sys.stderr)
    sys.exit(1)


def get_cpu_model() -> str:
    system = platform.system().lower()
    if system == "windows":
        return platform.processor() or "Unknown"
    if system == "darwin":
        try:
            import subprocess
            out = subprocess.run(["sysctl", "-n", "machdep.cpu.brand_string"],
                                  capture_output=True, text=True, timeout=5)
            return out.stdout.strip() or "Unknown"
        except Exception:
            return platform.processor() or "Unknown"
    # Linux
    try:
        with open("/proc/cpuinfo") as f:
            for line in f:
                if line.lower().startswith("model name"):
                    return line.split(":", 1)[1].strip()
    except OSError:
        pass
    return platform.processor() or "Unknown"


def collect_cpu_info(interval: float, per_core: bool) -> dict:
    freq = psutil.cpu_freq()
    per_core_usage = psutil.cpu_percent(interval=interval, percpu=True)
    overall_usage = sum(per_core_usage) / len(per_core_usage) if per_core_usage else 0.0

    info = {
        "model": get_cpu_model(),
        "physical_cores": psutil.cpu_count(logical=False),
        "logical_cores": psutil.cpu_count(logical=True),
        "current_freq_mhz": round(freq.current, 1) if freq else None,
        "min_freq_mhz": round(freq.min, 1) if freq and freq.min else None,
        "max_freq_mhz": round(freq.max, 1) if freq and freq.max else None,
        "overall_usage_percent": round(overall_usage, 1),
    }
    if per_core:
        info["per_core_usage_percent"] = [round(p, 1) for p in per_core_usage]
    return info


def main():
    parser = argparse.ArgumentParser(description="Report CPU hardware info and live utilization.")
    parser.add_argument("--interval", "-i", type=float, default=1.0,
                         help="Sampling interval in seconds for usage percentage (default: 1.0).")
    parser.add_argument("--per-core", action="store_true", help="Show utilization for each logical core.")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    args = parser.parse_args()

    info = collect_cpu_info(args.interval, args.per_core)

    if args.json:
        print(json.dumps(info, indent=2))
        return

    print("=== CPU Information ===")
    print(f"Model            : {info['model']}")
    print(f"Physical cores   : {info['physical_cores']}")
    print(f"Logical cores    : {info['logical_cores']}")
    print(f"Current frequency: {info['current_freq_mhz']} MHz" if info['current_freq_mhz'] else "Current frequency: n/a")
    print(f"Min / Max freq   : {info['min_freq_mhz']} / {info['max_freq_mhz']} MHz")
    print(f"Overall usage    : {info['overall_usage_percent']}% (sampled over {args.interval}s)")

    if args.per_core:
        print("\nPer-core usage:")
        for idx, pct in enumerate(info["per_core_usage_percent"]):
            print(f"  Core {idx}: {pct}%")


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

`psutil.cpu_count()` reports physical vs. logical cores (logical includes hyperthreads); `psutil.cpu_freq()` reads the current/min/max clock from the OS; `psutil.cpu_percent(interval=..., percpu=True)` blocks for `interval` seconds, sampling CPU busy-time before and after to compute a percentage per logical core. The CPU model string isn't exposed by `psutil`, so the script reads it from `/proc/cpuinfo` on Linux, `sysctl` on macOS, and `platform.processor()` on Windows.

```bash
# Basic report, 1-second sample
python get_cpu_info.py

# Longer sample for a steadier reading
python get_cpu_info.py --interval 3

# Show usage for every logical core (good for spotting one hot core)
python get_cpu_info.py --per-core

# Machine-readable output for a monitoring pipeline
python get_cpu_info.py --json
```

#### Variations / Flags

- Add a `--watch N` flag that loops `collect_cpu_info` every `N` seconds and clears the screen, turning this into a mini `top`.
- Add a `--threshold` flag that exits non-zero if `overall_usage_percent` exceeds a value, for use as a Nagios/cron alert check.
- Pipe `--json` output into `jq` or a log shipper for time-series CPU monitoring.

#### Common Pitfalls

- Calling `cpu_percent()` twice in quick succession without an interval between calls returns `0.0` for the second call — `psutil` measures the delta since the *last* call.
- `cpu_freq()` can return `None` (not an object) inside some containers/VMs with restricted access to hardware counters — the script checks for this before formatting.
- On Linux, `/proc/cpuinfo` model name parsing assumes an x86-style `model name` field; on ARM boards it may be missing, so the script falls back to `platform.processor()` (which is often just `""` — a known upstream Python limitation on Linux).

### 15.2 Get Memory Information

#### Use Case

Check how much RAM and swap a machine has, how much is actually free, and how much is being consumed — the first stop when triaging an out-of-memory kill, a slow server, or setting realistic memory limits (`--memory` in Docker, requests/limits in Kubernetes).

#### Prerequisites

- Python 3.7+
- Third-party library: `psutil`
- OS scope: Linux, Windows, macOS

```bash
pip install psutil
```

#### Cautions

"Available" memory (what the OS considers usable by new processes) is not the same as "free" memory — on Linux especially, a lot of RAM is held by the page cache and is reclaimable, so "free" alone dramatically understates what's actually available. This script reports both and labels them clearly to avoid the classic "why does `free` say almost no RAM is left" confusion.

#### Script

```python
#!/usr/bin/env python3
"""
get_memory_info.py

Purpose:
    Report physical RAM and swap usage using the cross-platform
    `psutil` library: total, used, available, free, and percent-used
    for both RAM and swap.

Usage:
    python get_memory_info.py
    python get_memory_info.py --json
    python get_memory_info.py --watch 5

Expected output:
    - Total/used/available/free RAM in human-readable units and percent used.
    - Total/used/free swap and percent used.
    - With --json: the same data as a JSON object.
    - With --watch N: refreshes every N seconds until Ctrl+C.
"""

import argparse
import json
import sys
import time

try:
    import psutil
except ImportError:
    print("[ERROR] Missing dependency 'psutil'. Install it with: pip install psutil", file=sys.stderr)
    sys.exit(1)


def human_size(num_bytes: int) -> str:
    size = float(num_bytes)
    for unit in ["B", "KB", "MB", "GB", "TB"]:
        if size < 1024:
            return f"{size:.2f} {unit}"
        size /= 1024
    return f"{size:.2f} PB"


def collect_memory_info() -> dict:
    vm = psutil.virtual_memory()
    sm = psutil.swap_memory()
    return {
        "ram": {
            "total": vm.total, "used": vm.used, "available": vm.available,
            "free": vm.free, "percent_used": vm.percent,
        },
        "swap": {
            "total": sm.total, "used": sm.used, "free": sm.free,
            "percent_used": sm.percent,
        },
    }


def print_report(info: dict):
    ram, swap = info["ram"], info["swap"]
    print("=== Memory Information ===")
    print(f"RAM total     : {human_size(ram['total'])}")
    print(f"RAM used      : {human_size(ram['used'])} ({ram['percent_used']}%)")
    print(f"RAM available : {human_size(ram['available'])}  (usable by new processes)")
    print(f"RAM free      : {human_size(ram['free'])}  (untouched, excludes reclaimable cache)")
    print()
    print(f"Swap total    : {human_size(swap['total'])}")
    print(f"Swap used     : {human_size(swap['used'])} ({swap['percent_used']}%)")
    print(f"Swap free     : {human_size(swap['free'])}")


def main():
    parser = argparse.ArgumentParser(description="Report RAM and swap usage.")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    parser.add_argument("--watch", "-w", type=int, default=None,
                         help="Refresh every N seconds until interrupted with Ctrl+C.")
    args = parser.parse_args()

    def run_once():
        info = collect_memory_info()
        if args.json:
            print(json.dumps(info, indent=2))
        else:
            print_report(info)

    if args.watch:
        try:
            while True:
                run_once()
                print(f"\n(refreshing every {args.watch}s, Ctrl+C to stop)")
                time.sleep(args.watch)
                print("\033c", end="")  # clear screen between refreshes
        except KeyboardInterrupt:
            print("\nStopped.")
    else:
        run_once()


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

`psutil.virtual_memory()` reads the OS's memory accounting (via `/proc/meminfo` on Linux, `GlobalMemoryStatusEx` on Windows, `vm_stat`/Mach APIs on macOS) and returns total/used/available/free figures already normalized across platforms. `psutil.swap_memory()` does the same for swap/page file.

```bash
# One-shot human-readable report
python get_memory_info.py

# Machine-readable output for a metrics pipeline
python get_memory_info.py --json

# Live-refreshing view every 5 seconds (like a minimal 'free -s 5')
python get_memory_info.py --watch 5
```

#### Sample Output

```bash
$ python get_memory_info.py
=== Memory Information ===
RAM total     : 15.62 GB
RAM used      : 8.94 GB (61.3%)
RAM available : 6.05 GB  (usable by new processes)
RAM free      : 1.21 GB  (untouched, excludes reclaimable cache)

Swap total    : 2.00 GB
Swap used     : 0.31 GB (15.5%)
Swap free     : 1.69 GB
```

#### Variations / Flags

- Add a `--threshold 90` flag that exits code `1` if `percent_used` for RAM exceeds 90%, for use in a cron-based alert.
- Add a `--csv` flag that appends a timestamped row to a log file, turning this into a lightweight memory-history logger.
- Combine `--watch` with `--json` and pipe through `jq` for a live dashboard feed.

#### Common Pitfalls

- Comparing this script's "free" number to what a container's memory limit reports can be misleading — cgroup-limited environments (Docker/Kubernetes) report host-level memory here unless `psutil` is specifically cgroup-aware, which it is not by default.
- On systems with no swap/page file configured, `swap_memory()` correctly returns zeros rather than erroring — don't assume `total: 0` is a bug.
- `--watch` clears the terminal with an ANSI escape code (`\033c`), which may render oddly in some CI log viewers that don't interpret ANSI codes — omit `--watch` when running inside CI.

### 15.3 Get Disk Information

#### Use Case

Enumerate every mounted disk/partition on a machine and report total, used, and free space per volume — the standard first check for a "disk full" alert, before provisioning storage, or as part of a periodic capacity report across a fleet.

#### Prerequisites

- Python 3.7+
- Third-party library: `psutil`
- OS scope: Linux, Windows, macOS

```bash
pip install psutil
```

#### Cautions

Network-mounted filesystems (NFS/CIFS shares) and removable media that are unmounted mid-scan can cause `disk_usage()` to raise or hang briefly — the script catches `OSError` per-partition so one bad mount doesn't kill the whole report. Pseudo-filesystems (`tmpfs`, `devtmpfs`, `overlay` on containers, etc.) are included by default; use `--physical-only` to hide them if you only care about real disks.

#### Script

```python
#!/usr/bin/env python3
"""
get_disk_info.py

Purpose:
    Enumerate mounted disk partitions/drives and report capacity, usage,
    and filesystem type for each, using the cross-platform `psutil`
    library.

Usage:
    python get_disk_info.py
    python get_disk_info.py --physical-only
    python get_disk_info.py --json
    python get_disk_info.py --min-free-percent 10

Expected output:
    - One row per mounted volume: device, mountpoint, filesystem type,
      total/used/free space, and percent used.
    - A summary line count of volumes reported.
    - With --min-free-percent: only volumes at or below that free-space
      threshold are shown, and exit code is 1 if any qualify (useful
      for low-disk-space alerting).
"""

import argparse
import json
import sys

try:
    import psutil
except ImportError:
    print("[ERROR] Missing dependency 'psutil'. Install it with: pip install psutil", file=sys.stderr)
    sys.exit(1)


def human_size(num_bytes: int) -> str:
    size = float(num_bytes)
    for unit in ["B", "KB", "MB", "GB", "TB"]:
        if size < 1024:
            return f"{size:.2f} {unit}"
        size /= 1024
    return f"{size:.2f} PB"


def collect_disk_info(physical_only: bool) -> list:
    partitions = psutil.disk_partitions(all=not physical_only)
    results = []
    for part in partitions:
        try:
            usage = psutil.disk_usage(part.mountpoint)
        except OSError:
            continue  # unreadable/unmounted mid-scan, skip gracefully
        results.append({
            "device": part.device,
            "mountpoint": part.mountpoint,
            "fstype": part.fstype,
            "total": usage.total,
            "used": usage.used,
            "free": usage.free,
            "percent_used": usage.percent,
        })
    return results


def main():
    parser = argparse.ArgumentParser(description="Report disk partition capacity and usage.")
    parser.add_argument("--physical-only", action="store_true",
                         help="Hide pseudo-filesystems (tmpfs, overlay, etc.); show only real physical devices.")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    parser.add_argument("--min-free-percent", type=float, default=None,
                         help="Only show/alert on volumes whose free space percent is at or below this value.")
    args = parser.parse_args()

    disks = collect_disk_info(args.physical_only)

    low_space = []
    if args.min_free_percent is not None:
        filtered = []
        for d in disks:
            free_percent = 100 - d["percent_used"]
            if free_percent <= args.min_free_percent:
                filtered.append(d)
                low_space.append(d)
        disks = filtered

    if args.json:
        print(json.dumps(disks, indent=2))
    else:
        print("=== Disk Information ===")
        if not disks:
            print("(no volumes matched)")
        for d in disks:
            print(f"\nDevice      : {d['device']}")
            print(f"Mountpoint  : {d['mountpoint']}")
            print(f"Filesystem  : {d['fstype']}")
            print(f"Total       : {human_size(d['total'])}")
            print(f"Used        : {human_size(d['used'])} ({d['percent_used']}%)")
            print(f"Free        : {human_size(d['free'])}")
        print(f"\n=== Summary: {len(disks)} volume(s) reported ===")

    if args.min_free_percent is not None:
        sys.exit(1 if low_space else 0)


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

`psutil.disk_partitions()` lists mounted filesystems as reported by the OS (`/proc/mounts` on Linux, `GetLogicalDrives`/WMI on Windows, `getfsstat` on macOS); passing `all=False` filters out pseudo/duplicate filesystems. For each mountpoint, `psutil.disk_usage()` calls the platform's `statvfs`/`GetDiskFreeSpaceEx` equivalent to get total/used/free bytes.

```bash
# Full report, including pseudo-filesystems
python get_disk_info.py

# Only real physical volumes (hides tmpfs/overlay/etc.)
python get_disk_info.py --physical-only

# Machine-readable output
python get_disk_info.py --json

# Alert mode: only show volumes with 10% or less free space, exit 1 if any found
python get_disk_info.py --physical-only --min-free-percent 10
```

```bash

Mountpoint  : /
Filesystem  : ext4
Total       : 465.66 GB
Used        : 210.11 GB (47.9%)
Free        : 231.87 GB

Mountpoint  : /mnt/data
```

#### Variations / Flags

- Add a `--sort-by used|free|percent` flag to rank volumes by fullest-first, handy on hosts with many mounts.
- Combine `--min-free-percent` with cron and an email/Slack webhook for automated low-disk alerts.
- Add a `--mountpoint /path` filter to check a single volume, e.g. inside a deployment pre-flight check.

#### Common Pitfalls

- On Windows, unmapped/empty CD-ROM or card-reader drive letters can raise `OSError` from `disk_usage()` — already handled here by skipping rather than crashing, but worth knowing if output looks shorter than expected.
- Container environments (Docker) often show the *host's* root filesystem usage for `/`, not a container-specific quota, unless the container has its own dedicated mount.
- `--min-free-percent` compares against `100 - percent_used`, not raw free bytes — a huge multi-terabyte volume at "95% used" may still have hundreds of GB free in absolute terms.

### 15.4 Get Network Interfaces

#### Use Case

List every network interface on a machine — Ethernet, Wi-Fi, loopback, virtual/tunnel adapters — along with its IP/MAC addresses and link state, before troubleshooting connectivity or building a hardware/network inventory.

#### Prerequisites

- Python 3.7+
- Third-party library: `psutil`
- OS scope: Linux, Windows, macOS

```bash
pip install psutil
```

#### Cautions

Interface names differ wildly across platforms (`eth0`/`enp3s0` on Linux, `Ethernet`/`Wi-Fi` on Windows, `en0` on macOS) — don't hardcode a name in downstream automation; filter by state (`is_up`) or address family instead. Speed and duplex are only reported by the OS for interfaces that are currently up; a down interface will show `0`/`unknown`, which is expected, not a bug.

#### Script

```python
#!/usr/bin/env python3
"""
get_network_interfaces.py

Purpose:
    Enumerate all network interfaces on the machine with their IPv4/
    IPv6/MAC addresses and link state (up/down, speed, MTU), using the
    cross-platform `psutil` library.

Usage:
    python get_network_interfaces.py
    python get_network_interfaces.py --up-only
    python get_network_interfaces.py --json

Expected output:
    - One block per interface: name, up/down state, speed (Mbps), MTU,
      and every address assigned to it (IPv4, IPv6, MAC) with netmask.
    - With --up-only: interfaces that are currently down are omitted.
    - With --json: the same data as a JSON object.
"""

import argparse
import json
import socket
import sys

try:
    import psutil
except ImportError:
    print("[ERROR] Missing dependency 'psutil'. Install it with: pip install psutil", file=sys.stderr)
    sys.exit(1)

FAMILY_NAMES = {
    socket.AF_INET: "IPv4",
    socket.AF_INET6: "IPv6",
}
try:
    FAMILY_NAMES[psutil.AF_LINK] = "MAC"
except AttributeError:
    pass


def collect_interfaces(up_only: bool) -> dict:
    addrs = psutil.net_if_addrs()
    stats = psutil.net_if_stats()

    interfaces = {}
    for name, addr_list in addrs.items():
        stat = stats.get(name)
        is_up = stat.isup if stat else False
        if up_only and not is_up:
            continue

        entry = {
            "is_up": is_up,
            "speed_mbps": stat.speed if stat else None,
            "mtu": stat.mtu if stat else None,
            "addresses": [],
        }
        for addr in addr_list:
            family = FAMILY_NAMES.get(addr.family, str(addr.family))
            entry["addresses"].append({
                "family": family,
                "address": addr.address,
                "netmask": addr.netmask,
                "broadcast": addr.broadcast,
            })
        interfaces[name] = entry
    return interfaces


def main():
    parser = argparse.ArgumentParser(description="Enumerate network interfaces and their addresses/link state.")
    parser.add_argument("--up-only", action="store_true", help="Only show interfaces that are currently up.")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    args = parser.parse_args()

    interfaces = collect_interfaces(args.up_only)

    if args.json:
        print(json.dumps(interfaces, indent=2))
        return

    print("=== Network Interfaces ===")
    for name, info in interfaces.items():
        print(f"\n{name}")
        print(f"  State : {'UP' if info['is_up'] else 'DOWN'}")
        print(f"  Speed : {info['speed_mbps']} Mbps" if info['speed_mbps'] else "  Speed : n/a")
        print(f"  MTU   : {info['mtu']}")
        for addr in info["addresses"]:
            netmask = f"  netmask={addr['netmask']}" if addr["netmask"] else ""
            print(f"  [{addr['family']}] {addr['address']}{netmask}")

    print(f"\n=== Summary: {len(interfaces)} interface(s) reported ===")


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

`psutil.net_if_addrs()` returns every address (IPv4, IPv6, and link-layer/MAC) bound to each interface name, sourced from `getifaddrs()` on Linux/macOS and the Windows IP Helper API on Windows. `psutil.net_if_stats()` separately reports link state, negotiated speed, duplex, and MTU. The script merges both by interface name into one report.

```bash
# Full report of every interface, up or down
python get_network_interfaces.py

# Only interfaces that are currently active
python get_network_interfaces.py --up-only

# Machine-readable output for an inventory pipeline
python get_network_interfaces.py --json
```

### 15.5 Get Installed Software

#### Use Case

Enumerate installed applications/packages on a machine for a software audit, license-compliance check, vulnerability scan against known package versions, or to detect unauthorized/unexpected installs on a fleet.

#### Prerequisites

- Python 3.7+ (standard library `subprocess`, `winreg` only — no pip packages required)
- OS scope: platform-specific logic per OS
  - Linux: `dpkg` (Debian/Ubuntu) or `rpm` (RHEL/Fedora/CentOS), whichever is present
  - Windows: reads the `Uninstall` registry keys via the built-in `winreg` module
  - macOS: reads `/Applications` and, if present, Homebrew (`brew list`)

```bash
# No extra packages needed — pure standard library.
# On Linux, dpkg or rpm must already be installed (they are, on virtually all distros).
# On macOS, Homebrew is optional; the script still lists /Applications without it.
```

#### Cautions

This is a read-only inventory script — it never installs, removes, or modifies anything. Package lists can be large (thousands of entries on a mature Linux server); use `--filter` to narrow results. Windows registry reads only cover software that registered an uninstall entry — some portable/manually-copied apps won't appear.

#### Script

```python
#!/usr/bin/env python3
"""
get_installed_software.py

Purpose:
    Enumerate installed applications/packages using the native
    inventory mechanism for the detected OS: dpkg/rpm on Linux, the
    Windows Uninstall registry keys on Windows, and /Applications plus
    Homebrew on macOS.

Usage:
    python get_installed_software.py
    python get_installed_software.py --filter nginx
    python get_installed_software.py --json

Expected output:
    - One row per installed package/app: name and version (where the
      platform exposes a version string).
    - A summary count of total packages/apps found.
    - With --filter: only entries whose name contains the given
      substring (case-insensitive) are shown.
"""

import argparse
import json
import platform
import shutil
import subprocess
import sys


def get_linux_software() -> list:
    software = []
    if shutil.which("dpkg"):
        try:
            out = subprocess.run(["dpkg-query", "-W", "-f=${Package}\t${Version}\n"],
                                  capture_output=True, text=True, timeout=30)
            for line in out.stdout.strip().splitlines():
                if "\t" in line:
                    name, version = line.split("\t", 1)
                    software.append({"name": name, "version": version})
            return software
        except Exception:
            pass
    if shutil.which("rpm"):
        try:
            out = subprocess.run(["rpm", "-qa", "--qf=%{NAME}\t%{VERSION}-%{RELEASE}\n"],
                                  capture_output=True, text=True, timeout=30)
            for line in out.stdout.strip().splitlines():
                if "\t" in line:
                    name, version = line.split("\t", 1)
                    software.append({"name": name, "version": version})
        except Exception:
            pass
    return software


def get_windows_software() -> list:
    try:
        import winreg
    except ImportError:
        return []

    software = []
    uninstall_keys = [
        (winreg.HKEY_LOCAL_MACHINE, r"SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall"),
        (winreg.HKEY_LOCAL_MACHINE, r"SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\Uninstall"),
        (winreg.HKEY_CURRENT_USER, r"SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall"),
    ]
    for hive, path in uninstall_keys:
        try:
            key = winreg.OpenKey(hive, path)
        except FileNotFoundError:
            continue
        for i in range(winreg.QueryInfoKey(key)[0]):
            try:
                subkey_name = winreg.EnumKey(key, i)
                subkey = winreg.OpenKey(key, subkey_name)
                name = winreg.QueryValueEx(subkey, "DisplayName")[0]
                try:
                    version = winreg.QueryValueEx(subkey, "DisplayVersion")[0]
                except FileNotFoundError:
                    version = "unknown"
                software.append({"name": name, "version": version})
            except (FileNotFoundError, OSError):
                continue
    return software


def get_macos_software() -> list:
    software = []
    import os
    apps_dir = "/Applications"
    if os.path.isdir(apps_dir):
        for entry in sorted(os.listdir(apps_dir)):
            if entry.endswith(".app"):
                software.append({"name": entry.replace(".app", ""), "version": "unknown"})
    if shutil.which("brew"):
        try:
            out = subprocess.run(["brew", "list", "--versions"], capture_output=True, text=True, timeout=30)
            for line in out.stdout.strip().splitlines():
                parts = line.split()
                if len(parts) >= 2:
                    software.append({"name": parts[0], "version": parts[-1]})
        except Exception:
            pass
    return software


def main():
    parser = argparse.ArgumentParser(description="Enumerate installed software/packages for the current OS.")
    parser.add_argument("--filter", "-f", default=None,
                         help="Only show entries whose name contains this substring (case-insensitive).")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    args = parser.parse_args()

    system = platform.system().lower()
    if system == "linux":
        software = get_linux_software()
    elif system == "windows":
        software = get_windows_software()
    elif system == "darwin":
        software = get_macos_software()
    else:
        print(f"[ERROR] Unsupported platform: {system}", file=sys.stderr)
        sys.exit(1)

    if not software:
        print("[WARN] No software inventory tool found or list is empty "
              "(dpkg/rpm on Linux, registry on Windows, /Applications+brew on macOS).", file=sys.stderr)

    if args.filter:
        needle = args.filter.lower()
        software = [s for s in software if needle in s["name"].lower()]

    software.sort(key=lambda s: s["name"].lower())

    if args.json:
        print(json.dumps(software, indent=2))
        return

    print(f"=== Installed Software ({system}) ===")
    for s in software:
        print(f"{s['name']:<50}{s['version']}")
    print(f"\n=== Summary: {len(software)} package(s)/app(s) found ===")


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

On Linux the script prefers `dpkg-query` (Debian/Ubuntu) and falls back to `rpm -qa` (RHEL/Fedora) — both query the local package database directly, no network access needed. On Windows it walks the standard and WOW6432Node `Uninstall` registry hives via `winreg`, which is exactly what Control Panel's "Programs and Features" reads from. On macOS it lists bundle names in `/Applications` and, if Homebrew is installed, merges in `brew list --versions`.

```bash

# Only entries matching a name substring
python get_installed_software.py --filter python

# Machine-readable output for a compliance database

```

```bash
nginx-common                                     1.18.0-6.1ubuntu14.4
nginx-core                                       1.18.0-6.1ubuntu14.4

=== Summary: 2 package(s)/app(s) found ===
```

### 15.6 Get Environment Variables

#### Use Case

Dump the current process's environment variables to debug "works on my machine" issues, verify that a deployment/CI pipeline actually injected the configuration you expect (`DATABASE_URL`, `PATH`, feature flags), or check for a missing variable before a service fails to start.

#### Prerequisites

- Python 3.7+ (standard library `os` module only)
- OS scope: Linux, Windows, macOS (identical behavior everywhere — environment variables are a universal OS concept)

```bash
# No installation needed — pure standard library.
```

#### Cautions

Environment variables commonly hold secrets (API keys, database passwords, tokens). This script masks values for any variable name that looks secret-related by default (`--show-secrets` disables masking) — be careful before pasting unmasked output into a ticket, chat channel, or log aggregator.

#### Script

```python
#!/usr/bin/env python3
"""
get_environment_variables.py

Purpose:
    Report the current process's environment variables, with optional
    name filtering and automatic masking of values whose variable name
    looks secret-related (password, token, key, secret, etc.).

Usage:
    python get_environment_variables.py
    python get_environment_variables.py --filter PATH
    python get_environment_variables.py --show-secrets
    python get_environment_variables.py --json

Expected output:
    - Every environment variable name and value, one per line, sorted
      alphabetically.
    - Variables that look secret-related are shown as "****" unless
      --show-secrets is passed.
    - With --filter: only variable names containing the given substring
      (case-insensitive) are shown.
    - With --json: the same data as a JSON object.
"""

import argparse
import json
import os

SECRET_HINTS = ("password", "secret", "token", "key", "credential", "auth", "pwd", "apikey")


def looks_secret(name: str) -> bool:
    lowered = name.lower()
    return any(hint in lowered for hint in SECRET_HINTS)


def collect_environment(show_secrets: bool, name_filter: str) -> dict:
    env = dict(os.environ)
    if name_filter:
        needle = name_filter.lower()
        env = {k: v for k, v in env.items() if needle in k.lower()}

    if not show_secrets:
        env = {k: ("****" if looks_secret(k) else v) for k, v in env.items()}

    return dict(sorted(env.items()))


def main():
    parser = argparse.ArgumentParser(description="Report current process environment variables.")
    parser.add_argument("--filter", "-f", default=None,
                         help="Only show variables whose name contains this substring (case-insensitive).")
    parser.add_argument("--show-secrets", action="store_true",
                         help="Show real values for variables that look secret-related (password/token/key/etc.). Use with care.")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    args = parser.parse_args()

    env = collect_environment(args.show_secrets, args.filter)

    if args.json:
        print(json.dumps(env, indent=2))
        return

    print("=== Environment Variables ===")
    if not env:
        print("(no variables matched)")
    for name, value in env.items():
        print(f"{name}={value}")
    print(f"\n=== Summary: {len(env)} variable(s) reported ===")
    if not args.show_secrets:
        print("(secret-looking values masked as **** — use --show-secrets to reveal)")


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

`os.environ` is a dict-like snapshot of the current process's environment, populated by the OS at process start from whatever the parent shell/service manager passed in. The script copies it, optionally filters by substring, and masks any variable whose *name* matches a list of common secret-related keywords before printing — this is a heuristic, not a guarantee, so treat it as a safety net rather than a compliance control.

```bash
# Full environment dump, secrets masked
python get_environment_variables.py

# Only variables with "PATH" in the name
python get_environment_variables.py --filter PATH

# Reveal real secret values too (be careful where this output goes)
python get_environment_variables.py --show-secrets

# Machine-readable output for a config-verification step in CI
python get_environment_variables.py --json
```

#### Sample Output

```bash
$ python get_environment_variables.py --filter DATABASE
=== Environment Variables ===
DATABASE_PASSWORD=****
DATABASE_URL=postgres://app_user@db.internal:5432/app_prod

=== Summary: 2 variable(s) reported ===
(secret-looking values masked as **** — use --show-secrets to reveal)
```

#### Variations / Flags

- Add a `--diff expected.json` flag that compares against an expected-config snapshot and reports missing/mismatched variables — useful as a deployment pre-flight check.
- Add a `--export-shell` flag that prints output as `export NAME=VALUE` lines, ready to `source` into another shell session.
- Run with `--json --filter FEATURE_` to quickly check which feature flags are active on a given host.

#### Common Pitfalls

- This script only sees variables set for *its own* process — if you're debugging a variable a service should have but doesn't, run this script as (or inside) that exact service's process/container, not just as your interactive shell, since the two environments can differ.
- The secret-masking list is a best-effort heuristic based on variable *names*; a poorly-named secret variable (e.g. `DB_CONN`) won't be masked, and a non-secret variable that happens to contain "key" (e.g. `KEYBOARD_LAYOUT`) will be masked unnecessarily — review before trusting it as a hard security boundary.
- On Windows, environment variable names are case-insensitive at the OS level but Python's `os.environ` preserves whatever case was set — `--filter` here is case-insensitive to compensate, but exact-name lookups elsewhere in your code should account for this.

### 15.7 Get BIOS Information

#### Use Case

Report BIOS/firmware vendor, version, and release date — needed when auditing a fleet for a known firmware vulnerability, confirming a BIOS update was actually applied after a maintenance window, or filling in a hardware asset-management record.

#### Prerequisites

- Python 3.7+ (standard library `subprocess`, `platform` only)
- OS scope: platform-specific, shells out to the native tool
  - Linux: `dmidecode` (usually requires `sudo`/root to read full DMI tables)
  - Windows: PowerShell's `Get-CimInstance Win32_BIOS` (no admin required)
  - macOS: `system_profiler SPHardwareDataType` (firmware info is limited on Apple Silicon; reports what's available)

```bash
# Linux — dmidecode is usually preinstalled; if not:
sudo apt install dmidecode        # Debian/Ubuntu
sudo dnf install dmidecode        # Fedora/RHEL

# Windows and macOS — no installation needed, uses built-in tools.
```

#### Cautions

On Linux, `dmidecode` needs to read `/dev/mem`, which requires root — running this script unprivileged on Linux will fail or return incomplete data; run with `sudo`. This script only *reads* firmware information; it never flashes or modifies BIOS/EFI settings.

#### Script

```python
#!/usr/bin/env python3
"""
get_bios_info.py

Purpose:
    Report BIOS/firmware vendor, version, and release date, using the
    native tool for the detected OS: dmidecode on Linux, PowerShell's
    Win32_BIOS class on Windows, and system_profiler on macOS.

Usage:
    sudo python3 get_bios_info.py      # Linux (root required for dmidecode)
    python get_bios_info.py            # Windows / macOS
    python get_bios_info.py --json

Expected output:
    - BIOS/firmware vendor, version string, and release date.
    - On Linux, also BIOS ROM size when available from dmidecode.
    - Clear error message if the required native tool isn't available
      or the script isn't run with sufficient privileges.
"""

import argparse
import json
import platform
import re
import shutil
import subprocess
import sys


def get_linux_bios() -> dict:
    if not shutil.which("dmidecode"):
        return {"error": "dmidecode not found. Install it with: sudo apt install dmidecode"}
    try:
        out = subprocess.run(["dmidecode", "-t", "bios"], capture_output=True, text=True, timeout=10)
    except Exception as e:
        return {"error": str(e)}

    if out.returncode != 0 or "Permission denied" in out.stderr:
        return {"error": "Could not read BIOS table. Try running with sudo/root."}

    text = out.stdout
    return {
        "vendor": _extract(text, "Vendor"),
        "version": _extract(text, "Version"),
        "release_date": _extract(text, "Release Date"),
        "rom_size": _extract(text, "ROM Size"),
    }


def get_windows_bios() -> dict:
    ps_cmd = ("Get-CimInstance -ClassName Win32_BIOS | "
              "Select-Object Manufacturer,SMBIOSBIOSVersion,ReleaseDate | ConvertTo-Json")
    try:
        out = subprocess.run(["powershell", "-NoProfile", "-Command", ps_cmd],
                              capture_output=True, text=True, timeout=15)
    except Exception as e:
        return {"error": str(e)}
    try:
        data = json.loads(out.stdout)
    except json.JSONDecodeError:
        return {"error": "Could not parse PowerShell output."}
    return {
        "vendor": data.get("Manufacturer", "Unknown"),
        "version": data.get("SMBIOSBIOSVersion", "Unknown"),
        "release_date": data.get("ReleaseDate", "Unknown"),
    }


def get_macos_bios() -> dict:
    try:
        out = subprocess.run(["system_profiler", "SPHardwareDataType"],
                              capture_output=True, text=True, timeout=15)
    except Exception as e:
        return {"error": str(e)}
    text = out.stdout
    return {
        "vendor": "Apple Inc.",
        "version": _extract(text, "Boot ROM Version") or _extract(text, "System Firmware Version"),
        "release_date": "n/a (not exposed by system_profiler)",
    }


def _extract(text: str, label: str) -> str:
    match = re.search(rf"{label}:\s*(.+)", text)
    return match.group(1).strip() if match else "Not found"


def main():
    parser = argparse.ArgumentParser(description="Report BIOS/firmware vendor, version, and release date.")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    args = parser.parse_args()

    system = platform.system().lower()
    if system == "linux":
        info = get_linux_bios()
    elif system == "windows":
        info = get_windows_bios()
    elif system == "darwin":
        info = get_macos_bios()
    else:
        print(f"[ERROR] Unsupported platform: {system}", file=sys.stderr)
        sys.exit(1)

    if "error" in info:
        print(f"[ERROR] {info['error']}", file=sys.stderr)
        sys.exit(1)

    if args.json:
        print(json.dumps(info, indent=2))
        return

    print("=== BIOS / Firmware Information ===")
    for key, value in info.items():
        print(f"{key.replace('_', ' ').title():<15}: {value}")


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

The script detects the OS with `platform.system()` and shells out to whichever tool that platform's own DMI/hardware-report system exposes: `dmidecode -t bios` parses the SMBIOS BIOS table on Linux, `Get-CimInstance Win32_BIOS` queries WMI on Windows, and `system_profiler SPHardwareDataType` reads Apple's hardware descriptor on macOS. Output text is parsed with small regexes into a common `{vendor, version, release_date}` shape.

```bash

# macOS

```

```bash
Version        : F.64
Release Date   : 03/14/2024
```

#### Variations / Flags

- Add a `--check-min-version X` flag that exits non-zero if the reported version string is older than a known-good baseline, for use in a firmware-compliance scan.

### 15.9 Get Operating System Version

#### Use Case

Report OS name, version, build number, and CPU architecture — the first fact needed when filing a bug report, checking package/software compatibility, or branching automation logic based on OS release (e.g., "only run this on Ubuntu 22.04+").

#### Prerequisites

- Python 3.7+ (standard library `platform` module only)
- OS scope: Linux, Windows, macOS

```bash
# No installation needed — pure standard library.
```

#### Cautions

Linux "version" reporting varies a lot by distribution — this script reads `/etc/os-release` (the modern standard, present on virtually every current distro) rather than relying solely on `platform.dist()`-style guessing, which was removed from Python 3.8+ anyway. If `/etc/os-release` is missing (rare, very old/minimal distros), the script falls back to `platform.release()`.

#### Script

```python
#!/usr/bin/env python3
"""
get_os_version.py

Purpose:
    Report the operating system name, version, build number, and CPU
    architecture, using the standard-library `platform` module and,
    on Linux, /etc/os-release for distro-accurate naming.

Usage:
    python get_os_version.py
    python get_os_version.py --json

Expected output:
    - OS name (e.g. "Ubuntu", "Windows", "macOS"), version string,
      build number (where exposed), and CPU architecture (e.g. x86_64,
      arm64).
"""

import argparse
import json
import platform
import re


def get_linux_os() -> dict:
    info = {"name": "Linux", "version": platform.release(), "build": "n/a"}
    try:
        with open("/etc/os-release") as f:
            data = {}
            for line in f:
                if "=" in line:
                    key, _, value = line.strip().partition("=")
                    data[key] = value.strip('"')
        info["name"] = data.get("PRETTY_NAME", data.get("NAME", "Linux"))
        info["version"] = data.get("VERSION_ID", platform.release())
    except OSError:
        pass
    return info


def get_windows_os() -> dict:
    version = platform.win32_ver()  # (release, version, csd, ptype)
    build_match = re.search(r"\d+$", platform.version())
    return {
        "name": f"Windows {version[0]}",
        "version": platform.version(),
        "build": build_match.group(0) if build_match else "unknown",
    }


def get_macos_os() -> dict:
    mac_ver = platform.mac_ver()[0]
    return {"name": "macOS", "version": mac_ver, "build": "n/a (see sw_vers -buildVersion)"}


def collect_os_info() -> dict:
    system = platform.system().lower()
    if system == "linux":
        info = get_linux_os()
    elif system == "windows":
        info = get_windows_os()
    elif system == "darwin":
        info = get_macos_os()
    else:
        info = {"name": platform.system(), "version": platform.release(), "build": "unknown"}

    info["architecture"] = platform.machine()
    info["python_version"] = platform.python_version()
    return info


def main():
    parser = argparse.ArgumentParser(description="Report OS name, version, build, and architecture.")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    args = parser.parse_args()

    info = collect_os_info()

    if args.json:
        print(json.dumps(info, indent=2))
        return

    print("=== Operating System Information ===")
    print(f"Name         : {info['name']}")
    print(f"Version      : {info['version']}")
    print(f"Build        : {info['build']}")
    print(f"Architecture : {info['architecture']}")
    print(f"Python       : {info['python_version']}  (version running this script, not the OS)")


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

On Linux, the script parses `/etc/os-release` (a simple `KEY=value` file every systemd-based distro ships) for the human-friendly `PRETTY_NAME` and `VERSION_ID`. On Windows, `platform.win32_ver()` and `platform.version()` report the release name and full build string from the Windows API. On macOS, `platform.mac_ver()` reads the product version directly from the OS. `platform.machine()` gives the CPU architecture on all three.

```bash
# Human-readable OS report
python get_os_version.py

# Machine-readable output for a CI compatibility check
python get_os_version.py --json
```

#### Sample Output

```bash
$ python get_os_version.py
=== Operating System Information ===
Name         : Ubuntu 22.04.4 LTS
Version      : 22.04
Build        : n/a
Architecture : x86_64
Python       : 3.11.6  (version running this script, not the OS)
```

#### Variations / Flags

- Add a `--min-version 20.04` flag (with distro-aware version comparison) to gate a deployment script on a minimum supported OS version.
- Combine with `get_kernel_version.py` (15.10) for a complete "OS + kernel" fingerprint in a single support-ticket template.
- Add a `--compact` flag that prints a single one-line summary (`Ubuntu 22.04 x86_64`) for embedding in log headers.

#### Common Pitfalls

- `platform.release()` on Linux returns the *kernel* release string (e.g. `5.15.0-91-generic`), not the distro version — this is a very common source of confusion, which is exactly why this script prefers `/etc/os-release` for the human-facing "version" instead.
- Windows build numbers reported by `platform.version()` can look identical between different feature updates unless you check the trailing revision digits carefully — for exact build/UBR matching, query the registry (`CurrentBuildNumber`/`UBR`) instead.
- On macOS, `platform.mac_ver()` can return an empty string when running inside certain sandboxed/containerized Python builds — if you get blank output, cross-check with `sw_vers` directly via `subprocess`.

### 15.10 Get Kernel Version

#### Use Case

Report the exact kernel build string (Linux kernel version, Windows NT kernel build, or Darwin/XNU kernel version on macOS) — needed when a bug fix or feature depends on a minimum kernel version, such as a specific cgroups v2 behavior, a syscall, or a filesystem feature.

#### Prerequisites

- Python 3.7+ (standard library `platform` module only)
- OS scope: Linux, Windows, macOS

```bash
# No installation needed — pure standard library.
```

#### Cautions

"Kernel version" and "OS version" (see 15.9) are frequently confused but are not the same number — two different distro releases can ship the same kernel, and the same distro release can be running very different kernels after patching. Don't use this script's output as a stand-in for OS version, or vice versa.

#### Script

```python
#!/usr/bin/env python3
"""
get_kernel_version.py

Purpose:
    Report the kernel release/build string for the current machine:
    the Linux kernel version, the Windows NT kernel build, or the
    Darwin (XNU) kernel version on macOS.

Usage:
    python get_kernel_version.py
    python get_kernel_version.py --json

Expected output:
    - The full kernel version/release string as reported by the OS.
    - The kernel's compile date/details, when the platform exposes them
      (Linux `uname -v` field).
"""

import argparse
import json
import platform


def collect_kernel_info() -> dict:
    system = platform.system().lower()
    uname = platform.uname()

    info = {
        "system": platform.system(),
        "kernel_release": uname.release,
        "kernel_version_detail": uname.version,
    }

    if system == "windows":
        info["notes"] = "Windows reports its NT kernel build number here, not a separate 'Linux-style' kernel version."
    elif system == "darwin":
        info["notes"] = "macOS runs the Darwin (XNU) kernel; 'kernel_release' is the Darwin version, not the macOS product version."
    else:
        info["notes"] = ""

    return info


def main():
    parser = argparse.ArgumentParser(description="Report the kernel release/build version.")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    args = parser.parse_args()

    info = collect_kernel_info()

    if args.json:
        print(json.dumps(info, indent=2))
        return

    print("=== Kernel Information ===")
    print(f"System                : {info['system']}")
    print(f"Kernel release        : {info['kernel_release']}")
    print(f"Kernel version detail : {info['kernel_version_detail']}")
    if info["notes"]:
        print(f"\nNote: {info['notes']}")


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

`platform.uname()` wraps the POSIX `uname()` system call on Linux/macOS and its Windows-API equivalent, returning a named tuple that includes `release` (the kernel version string) and `version` (extra build detail — compile date/flags on Linux, empty or minimal on Windows). The script simply surfaces these two fields plus a short platform-specific note to prevent the common Linux-kernel-vs-OS-version mixup.

```bash

python get_kernel_version.py

# Machine-readable output for a compatibility gate in CI
python get_kernel_version.py --json
```

#### Sample Output

```bash
Kernel version detail : #101-Ubuntu SMP Tue Nov 14 13:30:08 UTC 2023

Note:
```

#### Variations / Flags

- Add a `--min-kernel 5.10` flag with numeric version comparison to gate a script that depends on a specific kernel feature.
- Combine with `get_os_version.py` (15.9) into a single "OS + kernel" fingerprint block for support tickets.
- Add a `--compact` flag for a one-line summary suitable for log file headers.

#### Common Pitfalls

- Confusing kernel release with OS/distro version is the single most common mistake here — `5.15.0-91-generic` tells you nothing about which Ubuntu release is installed on its own; pair it with `get_os_version.py` for the full picture.
- On Windows, there is no separate "kernel version" the way Linux has one — this script reports the NT kernel build baked into `platform.uname().version`, which is effectively the same number as the OS build reported in 15.9.
- Custom/patched kernels (common on embedded devices, some cloud providers, and security-hardened distros) may include vendor-specific suffixes in the release string that don't match upstream kernel.org version numbers — don't assume the string is directly comparable across environments without checking the vendor's own versioning notes.

### 15.11 Get Boot Time

#### Use Case

Report when a machine last booted and its resulting uptime — the go-to check right after an unexpected reboot (was it a crash? a patch cycle? someone's fat-fingered `reboot`?), and a simple health signal for monitoring dashboards that want to flag hosts with suspiciously short uptime.

#### Prerequisites

- Python 3.7+
- Third-party library: `psutil`
- OS scope: Linux, Windows, macOS

```bash
pip install psutil
```

#### Cautions

Boot time can shift slightly on systems without a reliable real-time clock sync at boot (some embedded/edge devices) — if the reported boot time looks wrong by exactly the system's timezone offset, check whether the script's local-time conversion matches your expectations; the script clearly labels whether it's showing local or UTC time.

#### Script

```python
#!/usr/bin/env python3
"""
get_boot_time.py

Purpose:
    Report when the machine last booted and the resulting uptime,
    using the cross-platform `psutil` library.

Usage:
    python get_boot_time.py
    python get_boot_time.py --utc
    python get_boot_time.py --json
    python get_boot_time.py --max-uptime-days 90

Expected output:
    - Boot timestamp (local time by default, or UTC with --utc).
    - Uptime broken down into days, hours, minutes.
    - With --max-uptime-days: exits code 1 if uptime exceeds the given
      number of days (useful for flagging hosts overdue for a reboot
      after patching).
"""

import argparse
import json
import sys
from datetime import datetime, timezone

try:
    import psutil
except ImportError:
    print("[ERROR] Missing dependency 'psutil'. Install it with: pip install psutil", file=sys.stderr)
    sys.exit(1)


def collect_boot_info(use_utc: bool) -> dict:
    boot_timestamp = psutil.boot_time()
    now = datetime.now(timezone.utc).timestamp()
    uptime_seconds = now - boot_timestamp

    if use_utc:
        boot_dt = datetime.fromtimestamp(boot_timestamp, tz=timezone.utc)
    else:
        boot_dt = datetime.fromtimestamp(boot_timestamp)

    days, remainder = divmod(int(uptime_seconds), 86400)
    hours, remainder = divmod(remainder, 3600)
    minutes, _ = divmod(remainder, 60)

    return {
        "boot_time": boot_dt.isoformat(),
        "timezone": "UTC" if use_utc else "local",
        "uptime_days": days,
        "uptime_hours": hours,
        "uptime_minutes": minutes,
        "uptime_total_seconds": int(uptime_seconds),
    }


def main():
    parser = argparse.ArgumentParser(description="Report system boot time and uptime.")
    parser.add_argument("--utc", action="store_true", help="Show boot time in UTC instead of local time.")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    parser.add_argument("--max-uptime-days", type=int, default=None,
                         help="Exit code 1 if uptime exceeds this many days (e.g. to flag hosts overdue for a reboot).")
    args = parser.parse_args()

    info = collect_boot_info(args.utc)

    if args.json:
        print(json.dumps(info, indent=2))
    else:
        print("=== Boot Time / Uptime ===")
        print(f"Booted at ({info['timezone']}): {info['boot_time']}")
        print(f"Uptime               : {info['uptime_days']}d {info['uptime_hours']}h {info['uptime_minutes']}m")

    if args.max_uptime_days is not None:
        if info["uptime_days"] > args.max_uptime_days:
            print(f"\n[ALERT] Uptime exceeds {args.max_uptime_days} day(s) — host may be overdue for a reboot/patch cycle.",
                  file=sys.stderr)
            sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

`psutil.boot_time()` returns the system boot time as a Unix timestamp, sourced from `/proc/stat`'s `btime` field on Linux, `GetTickCount64`-derived calculation on Windows, and `sysctl kern.boottime` on macOS. The script converts that to a readable timestamp and subtracts it from "now" to compute uptime, broken into days/hours/minutes.

```bash
# Human-readable boot time and uptime, local time
python get_boot_time.py

# Show boot time in UTC (useful when comparing logs across timezones)
python get_boot_time.py --utc

# Machine-readable output for a monitoring pipeline
python get_boot_time.py --json

# Alert mode: flag hosts that haven't rebooted in 90+ days (e.g. missed patch reboot)
python get_boot_time.py --max-uptime-days 90
```

#### Sample Output

```bash
$ python get_boot_time.py
=== Boot Time / Uptime ===
Booted at (local): 2026-06-02T08:14:31
Uptime               : 45d 21h 37m
```

#### Variations / Flags

- Add a `--min-uptime-minutes 5` flag (inverse alerting) to detect hosts that rebooted *unexpectedly* very recently, useful right after a suspected crash.
- Combine `--max-uptime-days` with cron across a fleet (via `ssh_automation.py`, Chapter 14) to build a "hosts overdue for patching reboot" report.
- Add a `--history` flag backed by a small local log file that records boot times over time, to spot a host that's crash-looping (rebooting far more often than expected).

#### Common Pitfalls

- On systems where the hardware clock and OS clock disagree (common on misconfigured VMs or dual-boot machines), boot time can appear slightly in the future or with an odd offset — this reflects a real clock-sync issue on the host, not a bug in the script.
- Sleep/hibernate on laptops is *not* the same as a reboot — waking from sleep does not reset `psutil.boot_time()`, so "uptime" here can be much longer than how long the laptop has actually been "in active use," which is expected behavior.
- `--max-uptime-days` alerts are only as good as your actual patch/reboot cadence — set the threshold to match your organization's real maintenance-window policy, not an arbitrary number.

### 15.12 Get Logged-in Users

#### Use Case

List everyone currently logged into a machine, when their session started, and (where available) which terminal or remote host they connected from — a quick security audit check, and the standard "is it safe to reboot this shared server right now" sanity check before doing maintenance.

#### Prerequisites

- Python 3.7+
- Third-party library: `psutil`
- OS scope: Linux, Windows, macOS (Windows reporting is more limited — see Common Pitfalls)

```bash
pip install psutil
```

#### Cautions

This reports interactive login sessions as tracked by the OS's own session accounting (`utmp` on Linux/macOS) — it does not see application-level logins (e.g., users logged into a web app running on the box). Treat "no users listed" as "no OS-level interactive sessions," not "nobody is using this server's services."

#### Script

```python
#!/usr/bin/env python3
"""
get_logged_in_users.py

Purpose:
    List currently logged-in users, their session start time, and the
    terminal/host they connected from, using the cross-platform
    `psutil` library.

Usage:
    python get_logged_in_users.py
    python get_logged_in_users.py --json
    python get_logged_in_users.py --user alice

Expected output:
    - One row per active session: username, terminal/tty (or session
      name on Windows), source host (for remote logins), and login
      time.
    - A summary count of active sessions and distinct users.
    - With --user: only sessions for the given username are shown, and
      exit code is 1 if that user has no active session (useful before
      safely rebooting a box someone might be using).
"""

import argparse
import json
import sys
from datetime import datetime

try:
    import psutil
except ImportError:
    print("[ERROR] Missing dependency 'psutil'. Install it with: pip install psutil", file=sys.stderr)
    sys.exit(1)


def collect_sessions() -> list:
    sessions = []
    for user in psutil.users():
        sessions.append({
            "username": user.name,
            "terminal": user.terminal or "n/a",
            "host": user.host or "local",
            "login_time": datetime.fromtimestamp(user.started).isoformat(),
        })
    return sessions


def main():
    parser = argparse.ArgumentParser(description="List currently logged-in users and their sessions.")
    parser.add_argument("--user", "-u", default=None, help="Only show sessions for this username.")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    args = parser.parse_args()

    sessions = collect_sessions()

    if args.user:
        sessions = [s for s in sessions if s["username"] == args.user]

    if args.json:
        print(json.dumps(sessions, indent=2))
    else:
        print("=== Logged-in Users ===")
        if not sessions:
            print("(no active sessions found)")
        for s in sessions:
            print(f"\nUser       : {s['username']}")
            print(f"Terminal   : {s['terminal']}")
            print(f"From host  : {s['host']}")
            print(f"Login time : {s['login_time']}")

        distinct_users = {s["username"] for s in sessions}
        print(f"\n=== Summary: {len(sessions)} session(s), {len(distinct_users)} distinct user(s) ===")

    if args.user:
        sys.exit(0 if sessions else 1)


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

`psutil.users()` reads the OS's active-session accounting: `utmp`/`who`-equivalent data on Linux and macOS (via the same records the `who` command reads), and the Windows session/logon API on Windows. Each entry includes the username, terminal/session identifier, originating host (populated for remote/SSH logins, empty for local console logins), and a Unix timestamp for when the session started.

```bash

python get_logged_in_users.py

# Machine-readable output for an audit log
python get_logged_in_users.py --json

# Check whether a specific user is logged in before rebooting
python get_logged_in_users.py --user alice
echo $?   # 0 = alice has an active session, 1 = she doesn't
```

#### Sample Output

```bash
$ python get_logged_in_users.py

=== Logged-in Users ===

User       : alice
Terminal   : pts/0
From host  : 203.0.113.44
Login time : 2026-07-18T08:12:03

User       : bob
Terminal   : tty1
From host  : local
Login time : 2026-07-17T22:40:11

=== Summary: 2 session(s), 2 distinct user(s) ===
```

#### Variations / Flags

- Add a `--idle-minutes` flag (Linux/macOS only, parsed from `who -u`) to also show how long each session has been idle, useful for identifying truly abandoned sessions before a forced logout.
- Wrap `--user` checks in a pre-reboot script so scheduled maintenance automatically aborts (or warns) if a specific admin account is actively connected.
- Combine with `ssh_automation.py` (Chapter 14) to sweep an entire fleet and flag any host with an unexpected/unauthorized logged-in account.

#### Common Pitfalls

- On Windows, `psutil.users()` reports far less detail than on Linux/macOS — remote/host information is frequently empty, and RDP sessions may not always be fully represented; for a thorough Windows session audit, cross-check with `query user` via `subprocess` as well.
- A user with multiple open terminals/SSH sessions appears as multiple rows (one per session), which is why the script reports both a session count and a distinct-user count — don't conflate the two when writing alerting logic.
- Container environments typically report zero or minimal logged-in users regardless of how many people are actively `docker exec`-ing into them, since that doesn't go through traditional OS login-session accounting.

### 15.13 Get GPU Information

#### Use Case

Report installed GPU(s), driver version, VRAM capacity, and current utilization/temperature — critical before scheduling an ML training job, confirming a GPU-passthrough VM/container actually sees the card, or diagnosing a render-farm/gaming-server node that's underperforming.

#### Prerequisites

- Python 3.7+
- Third-party library: `GPUtil` (a thin wrapper around `nvidia-smi`) for NVIDIA GPUs; falls back to parsing `nvidia-smi` output directly if `GPUtil` isn't installed
- OS scope: Linux, Windows (NVIDIA GPUs, via `nvidia-smi`, which ships with the NVIDIA driver); best-effort detection only for AMD/Intel/Apple GPUs, which don't expose an equivalent standard CLI

```bash
pip install GPUtil

# nvidia-smi itself is installed alongside the NVIDIA driver, not via pip --
# verify it's on PATH:
nvidia-smi --version
```

#### Cautions

This script is NVIDIA-first because `nvidia-smi` is the only widely available, script-friendly, cross-platform GPU query tool. On machines with AMD or Intel GPUs only, it reports "no NVIDIA GPU detected" rather than fabricating data — extend it with `rocm-smi` (AMD) or `intel_gpu_top` (Intel) if your fleet needs that coverage. Do not attempt to change GPU clock speeds or power limits with this script — it is read-only by design.

#### Script

```python
#!/usr/bin/env python3
"""
get_gpu_info.py

Purpose:
    Report installed NVIDIA GPU(s): name, driver version, total/used/
    free VRAM, current utilization percentage, and temperature. Uses
    `GPUtil` when available, and falls back to parsing `nvidia-smi`
    directly (via its --query-gpu CSV mode) if GPUtil isn't installed.

Usage:
    python get_gpu_info.py
    python get_gpu_info.py --json
    python get_gpu_info.py --max-temp 85

Expected output:
    - Per GPU: name, driver version, total/used/free VRAM, utilization
      percent, and temperature in Celsius.
    - A clear message (not an error) if no NVIDIA GPU is detected.
    - With --max-temp: exits code 1 if any GPU exceeds the given
      temperature, useful for a thermal-alerting cron job.
"""

import argparse
import csv
import io
import json
import shutil
import subprocess
import sys


def get_gpus_via_nvidia_smi() -> list:
    if not shutil.which("nvidia-smi"):
        return []
    fields = "name,driver_version,memory.total,memory.used,memory.free,utilization.gpu,temperature.gpu"
    try:
        out = subprocess.run(
            ["nvidia-smi", f"--query-gpu={fields}", "--format=csv,noheader,nounits"],
            capture_output=True, text=True, timeout=10,
        )
    except Exception:
        return []
    if out.returncode != 0:
        return []

    gpus = []
    reader = csv.reader(io.StringIO(out.stdout))
    for idx, row in enumerate(reader):
        if len(row) < 7:
            continue
        name, driver, mem_total, mem_used, mem_free, util, temp = [v.strip() for v in row]
        gpus.append({
            "index": idx,
            "name": name,
            "driver_version": driver,
            "vram_total_mb": int(mem_total),
            "vram_used_mb": int(mem_used),
            "vram_free_mb": int(mem_free),
            "utilization_percent": int(util),
            "temperature_c": int(temp),
        })
    return gpus


def get_gpus_via_gputil() -> list:
    try:
        import GPUtil
    except ImportError:
        return None  # signal "library not available", caller falls back

    gpus = []
    for gpu in GPUtil.getGPUs():
        gpus.append({
            "index": gpu.id,
            "name": gpu.name,
            "driver_version": gpu.driver,
            "vram_total_mb": int(gpu.memoryTotal),
            "vram_used_mb": int(gpu.memoryUsed),
            "vram_free_mb": int(gpu.memoryFree),
            "utilization_percent": round(gpu.load * 100, 1),
            "temperature_c": gpu.temperature,
        })
    return gpus


def collect_gpu_info() -> list:
    gpus = get_gpus_via_gputil()
    if gpus is None:  # GPUtil not installed, fall back to raw nvidia-smi parsing
        gpus = get_gpus_via_nvidia_smi()
    return gpus


def main():
    parser = argparse.ArgumentParser(description="Report NVIDIA GPU hardware info and live utilization.")
    parser.add_argument("--json", action="store_true", help="Output as JSON instead of human-readable text.")
    parser.add_argument("--max-temp", type=int, default=None,
                         help="Exit code 1 if any GPU's temperature exceeds this value in Celsius.")
    args = parser.parse_args()

    gpus = collect_gpu_info()

    if not gpus:
        msg = "No NVIDIA GPU detected (or nvidia-smi/GPUtil unavailable on this machine)."
        if args.json:
            print(json.dumps({"gpus": [], "message": msg}, indent=2))
        else:
            print(f"[INFO] {msg}")
        return

    if args.json:
        print(json.dumps(gpus, indent=2))
    else:
        print("=== GPU Information ===")
        for gpu in gpus:
            print(f"\nGPU {gpu['index']}: {gpu['name']}")
            print(f"  Driver version : {gpu['driver_version']}")
            print(f"  VRAM total     : {gpu['vram_total_mb']} MB")
            print(f"  VRAM used      : {gpu['vram_used_mb']} MB")
            print(f"  VRAM free      : {gpu['vram_free_mb']} MB")
            print(f"  Utilization    : {gpu['utilization_percent']}%")
            print(f"  Temperature    : {gpu['temperature_c']} C")
        print(f"\n=== Summary: {len(gpus)} GPU(s) reported ===")

    if args.max_temp is not None:
        overheating = [g for g in gpus if g["temperature_c"] > args.max_temp]
        if overheating:
            names = ", ".join(f"GPU {g['index']} ({g['temperature_c']}C)" for g in overheating)
            print(f"\n[ALERT] Temperature threshold ({args.max_temp}C) exceeded: {names}", file=sys.stderr)
            sys.exit(1)


if __name__ == "__main__":
    main()
```

#### How It Works & How to Run

`GPUtil` (when installed) is itself a thin Python wrapper that shells out to `nvidia-smi` and parses its XML output into convenient objects; the script uses it when available for slightly cleaner code, and falls back to calling `nvidia-smi --query-gpu=... --format=csv` directly and parsing the CSV itself if `GPUtil` isn't installed — so the script works whether or not the extra pip package is present, as long as the NVIDIA driver itself is installed.

```bash
# Human-readable GPU report

# Machine-readable output for a monitoring pipeline

# Alert mode: exit 1 if any GPU is running hotter than 85C
```


#### Variations / Flags

- Add a `--watch N` loop (same pattern as 15.1/15.2) to build a lightweight live GPU monitor for a training job.
- Add AMD support by shelling out to `rocm-smi --showuse --showtemp --json` and merging results into the same output shape.
- Combine `--max-temp` with cron on a render farm to catch a failing GPU fan before it causes a hardware fault.

#### Common Pitfalls

- On a machine with no NVIDIA GPU (integrated graphics only, or AMD/Intel discrete GPU), this script correctly and quietly reports "no GPU detected" — that's expected behavior, not a bug, since `nvidia-smi` genuinely has nothing to query.
- Inside a Docker container, `nvidia-smi` only works if the container was started with GPU passthrough enabled (e.g. `--gpus all` with the NVIDIA Container Toolkit installed on the host) — a container without that flag will report no GPU even on a host that has one.
- `GPUtil`'s `load` attribute is a fraction (0–1), not a percentage — the script multiplies by 100 before display; if you extend this script, double-check which convention any new library uses before mixing values from different sources.
